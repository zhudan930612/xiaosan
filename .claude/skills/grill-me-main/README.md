# 🔥 Grill Me

> 无情追问，深度审查，暴露假设，达成共识

## 简介

**Grill Me** 是一个用于压力测试和深度审查的 AI 技能。它会以"魔鬼代言人"的方式，对你的计划、文档、内容或设计进行无情的追问和挑战，帮助你发现隐藏的漏洞、未验证的假设和未解决的依赖关系。

## ✨ 功能特性

- **🔍 无情追问** — 对方案的每个细节进行深度质询，不放过任何模糊地带
- **🌳 决策树遍历** — 逐一分支 walkthrough，确保所有决策路径都被审视
- **🔗 依赖解析** — 理清决策之间的依赖关系，识别关键阻塞点
- **📚 智能探索** — 优先查阅代码库和文档，减少重复提问
- **🤝 共识达成** — 持续追问直到双方对材料有共同理解

## 🚀 使用方法

### 触发方式

说出以下任意一种表达即可激活：

| 中文 | English |
|------|---------|
| "拷打这个方案" | "grill me" |
| "挑战这个设计" | "grill this plan" |
| "拷打这篇文档" | "stress-test this" |
| "挑战这个内容" | "interview me about this" |

### 示例场景

```
用户: 拷打这个数据治理方案
AI: [开始高强度追问...]

用户: grill me on this PRD
AI: [开始深度质询...]

用户: 挑战这个交互设计
AI: [开始压力测试...]
```

## 🎯 适用场景

| 场景 | 说明 |
|------|------|
| **方案评审** | 产品方案、技术方案、业务方案的压力测试 |
| **文档审查** | PRD、技术文档、设计文档的深度质询 |
| **设计挑战** | 交互设计、视觉设计、系统设计的漏洞挖掘 |
| **内容审核** | 产品文案、说明文档、对外材料的准确性检查 |
| **决策验证** | 重要决策的前提假设和依赖关系梳理 |

## ⚙️ 工作原理

1. **材料分析** — 首先理解你提供的计划/文档/设计
2. **问题生成** — 基于决策树生成追问问题
3. **智能探索** — 如需信息，优先查阅现有材料而非直接提问
4. **深度追问** — 对每个决策点进行多轮质询
5. **依赖梳理** — 识别并解决决策之间的依赖关系
6. **共识确认** — 确保所有问题都得到满意答复

## 📝 输出示例

一次典型的 Grill Me 会话可能包含：

```
Q1: 这个方案的核心假设是什么？有数据支撑吗？
Q2: 如果 A 环节失败，B 环节的备选方案是什么？
Q3: 这个依赖关系是否必须在前期解决？
Q4: 用户群体的这个定义有调研依据吗？
...
```

## 🔧 配置

该技能无需额外配置，直接通过自然语言触发即可。

## 💡 最佳实践

1. **准备材料** — 提前准备好要审查的文档或方案
2. **开放心态** — 准备好接受挑战，这是为了发现盲点
3. **持续迭代** — 根据追问结果完善方案后，可以再次拷打
4. **记录问题** — 将发现的问题和假设记录下来，作为改进清单

---

> 💬 **提示**: Grill Me 的目标是帮助你发现盲点，而不是否定你的工作。保持开放心态，你会得到更有价值的反馈。
