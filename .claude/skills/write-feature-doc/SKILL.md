# 编写功能描述文档

## 触发条件

用户说：编写功能文档、记录功能、文档化功能、写功能描述。

## 执行流程

### 步骤 1：加载上下文包（包 A）

自动读取以下参考文档：
- `01_产品主文档/*产品上下文.md`
- `01_产品主文档/*产品规划.md`
- `02_已上线功能描述/*模块.md`（对应模块现有描述，如有）

向用户报告已读取的参考文档。

### 步骤 2：自主产出文档初稿

基于参考文档和标准模板，自主完成文档初稿。
- 信息充分时直接产出
- 信息不足时标注 `⚠️ 待确认`，不中断执行

### 步骤 3：自动执行自检清单

对照 `00_项目说明/AI自检清单.md` 中的【清单二：功能描述文档自检】逐条检查。
- 可直接修正的问题 → 自动修正
- 需用户补充的问题 → 归入"待确认事项"

### 步骤 4：交付

输出完整文档，汇报自检结果和待确认项，等待用户审阅。

## 功能描述文档约束

1. **只记录已上线功能**：不记录规划中的功能
2. **严格基于截图**：所有描述必须基于截图或实际产品
3. **不写完整 PRD**：记录 AI 设计功能时真正需要参考的内容
4. **保持结构一致性**：遵循相同模板结构

## 问答式备选

若用户未提供截图或信息严重不足，退回问答式逐步收集。
