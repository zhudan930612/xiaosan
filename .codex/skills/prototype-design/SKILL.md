# 原型设计技能

## 触发词与模式识别

根据用户输入自动识别工作模式：

| 模式 | 触发词 | 特点 |
|------|--------|------|
| **快速模式** | "帮我做原型"、"生成原型"、"做原型"、"原型设计"、"prototype" | 直接生成，不过度询问 |
| **高级模式** | "高级原型"、"可编辑原型"、"原型编辑器"、"/prototype-advanced" | 完整编辑器功能 |
| **引导模式** | "怎么做原型"、"详细原型设计"、"引导我做原型"、"/prototype-guided" | 逐步引导，适合复杂页面 |

**模式判断优先级：** 引导模式 > 高级模式 > 快速模式。

---

## 工作流程

### 步骤1: 检查标准文档

使用 Glob 搜索 `*原型设计标准*.md` 或 `*原型标准*.md`：

- **如果找到**：读取并应用其中的设计规范
- **如果没有**：询问用户是否需要创建标准文档

### 步骤2: 识别原型类型

- **移动端 App**：关键词如"app"、"移动端"、"手机"
- **管理后台**：关键词如"后台"、"管理系统"、"PC端"
- **小程序**：关键词如"小程序"、"微信"
- **其他**：询问用户

### 步骤3: 收集原型信息

**快速模式**：页面名称、主要功能、是否有参考截图

**高级/引导模式**：页面名称、用途、用户角色、页面类型、功能模块

### 步骤4: 确认输出类型

**必须询问用户**：

> 请选择原型类型：
> 1. **设计协作版** - 页面可编辑 + 功能说明可编辑
> 2. **演示版** - 文案可编辑 + 功能说明可编辑
> 3. **交互演示版** - 页面不可编辑 + 有交互无标注
> 4. **交付版** - 页面不可编辑 + 有交互 + 标注只读

### 步骤5: 页面枚举（避免遗漏）

生成原型前，系统性枚举所有页面：

**主流程页面**：入口页、核心操作页、结果页、详情/查看页

**状态页面**（每个列表/详情必须有）：有数据、空状态、加载中、加载失败、权限不足

**边界情况**：表单提交（未填写/提交中/成功/失败）、搜索/筛选、分页加载

**弹窗/浮层**：确认弹窗、选择弹窗、提示弹窗、底部浮层、全屏弹窗

输出页面清单供用户确认，**必须等待确认后再生成原型**。

### 步骤6: 生成原型

输出格式：HTML 文件，文件名 `[功能名]原型设计.html`

需要详细代码模板时，读取 `references/design-system.md` 和 `references/advanced-editor.md`。

## 平台差异化布局规范

### 移动端 App / 小程序

- 多页面并排展示（每个页面 375px 宽度）
- 标注放置在页面**右侧**，每个页面独立标注面板
- 代码结构：`.page-wrapper` flex 横向排列

### 管理后台 / PC Web / 大屏

- 单页面全宽展示，页面切换标签/导航
- 使用**抽屉式标注**，固定在页面右侧，默认收起
- 代码结构：`.web-wrapper` + `.page-nav` + `.main-content` + `.annotation-drawer`

### 标注内容规范

**标注性质**：需求逻辑规则说明，**不是交互说明**

**标注内容应包含**：
- 数据规则（计算口径、数据来源、更新频率）
- 业务规则（状态流转、权限判断、异常处理）
- 需求说明（功能点描述、字段规则、校验逻辑）

**标注样式**：
- 背景：`#FFFBE6`，边框：`1px dashed #FFC53D`
- 必须使用 flex 横向排列，禁止 absolute 定位
- 可编辑模式下使用 `contenteditable="true"`

---

## HTML原型规范（强制）

1. **高度自适应**：`.phone-frame` 和 `.phone-screen` 必须使用 `height: auto`
2. **禁止占位元素**：不要使用 `<div style="height: 100px;"></div>` 撑开高度
3. **状态栏与导航栏**：纯色背景，禁止渐变；状态栏仅显示时间
4. **最小页面高度**：内容区 `min-height: 667px`
5. **定位约束**：底部按钮使用相对定位或 flex，禁止使用 `position: fixed`
6. **弹窗默认隐藏**：`display: none`，通过 JS 控制显示/隐藏
7. **内容完整**：每个页面必须有实际内容，禁止空白页面
8. **图片占位**：使用内联 SVG 或 CSS 占位，禁止使用外部图片链接

---

## 可编辑元素特殊处理

- **纯色按钮可编辑**：`[contenteditable="true"]:focus` 时保持纯色背景 + 白色内阴影
- **按钮样式**：统一圆角 `10px`，高度 `48px`，纯色背景，禁止渐变和胶囊形
- **输入框提示文字**：禁止使用 `placeholder`，使用可编辑提示元素覆盖

## 引导模式工作流

采用交互式流程收集：
1. 确认原型类型
2. 收集页面信息（逐项询问）
3. 收集功能模块（逐个确认）
4. 确认样式风格（主色调、圆角、按钮样式）
5. 生成原型（实时预览，可迭代调整）

## 输出前检查清单

- [ ] 所有页面高度均为自适应，无固定高度
- [ ] 内容区设置了最小高度（min-height: 667px）
- [ ] 无占位元素，无刘海装饰元素
- [ ] 状态栏与导航栏使用纯色背景（禁止渐变）
- [ ] 页面背景使用纯白 `#FFFFFF`
- [ ] 按钮使用纯色背景，圆角 `10px`（禁止渐变、禁止胶囊形）
- [ ] 卡片圆角统一为 `12px`
- [ ] 使用 SVG 线性图标代替 emoji（描边 1.5px）
- [ ] Tab 栏使用毛玻璃效果 `backdrop-filter: blur(20px)`
- [ ] 字体层级正确（标题 17px/28px、正文 15px、辅助 13px）
- [ ] 留白充足（页面边距 24px、卡片内边距 20px）
- [ ] 图片占位使用 SVG 图标，禁止文字占位
- [ ] 所有页面都有实际内容，无空白页面
- [ ] 标注使用 flex 布局，不使用 absolute 定位
- [ ] 高级模式：调节手柄、撤销/重做功能可用

## 注意事项

1. **先确认原型类型**：若用户未指定，不得自行默认
2. **遵循标准文档**：如果有标准文档，严格遵循其规范
3. **宁可标注"待补充"，也不要猜测**
4. **保持文档可维护性**：使用清晰的章节结构
5. **编辑模式默认关闭**：用户需要点击"进入编辑"才能编辑（高级模式）
