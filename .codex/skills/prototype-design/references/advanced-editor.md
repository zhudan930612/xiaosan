# 高级编辑功能参考（高级模式专属）

当用户选择高级模式时，需要以下完整编辑器功能。

## 1. 全局工具栏（精简）

**工具栏只保留三个按钮**：
- **进入编辑** —— 切换编辑/预览模式
- **调节模式** —— 开启/关闭卡片大小调节
- **保存** —— 导出干净 HTML

**注意**：「添加内容」按钮**不在**全局工具栏，而是在每个卡片内部。

## 2. 卡片编辑功能

每个可编辑卡片必须包含工具栏：

| 按钮 | 功能 | 说明 |
|------|------|------|
| ⋮⋮ | 拖拽排序 | 拖拽调整卡片顺序 |
| ➕ | 添加内容 | 在卡片内添加表单元素 |
| 📋 | 复制卡片 | 复制当前卡片及其内容 |
| 🗑️ | 删除卡片 | 删除当前卡片（确认后） |

## 3. 卡片内容添加功能

**功能入口**：每个卡片内部的「➕」按钮

**添加菜单**（点击弹出）：

| 图标 | 类型 | 生成的 HTML |
|------|------|-------------|
| 输入框图标 | 单行文本输入 | `<input type="text" class="form-input">` |
| 标签图标 | 说明/提示文字 | `<div class="form-label">标签</div>` |
| 文本图标 | 多行输入 | `<textarea class="form-textarea"></textarea>` |
| 单选图标 | 选项组 | `<div class="radio-group">...</div>` |
| 复选图标 | 勾选项 | `<div class="checkbox-group">...</div>` |
| 上传图标 | 图片/文件上传 | `<div class="upload-box">...</div>` |
| 分隔线图标 | 内容分隔 | `<hr class="divider">` |

**重要规则**：
1. **添加的元素必须可删除** —— 每个添加的元素旁边要有删除按钮
2. 添加后元素插入到当前卡片内部末尾
3. 页面高度自适应调整（卡片撑开 → 页面撑开）

## 4. 卡片大小调节功能（调节模式）

**功能入口**：全局工具栏的「调节模式」按钮 ↔️

**交互逻辑**：
1. 点击「调节模式」进入调节状态
2. 所有卡片右下角显示绿色拖拽手柄（resize handle）
3. 拖拽手柄可改变卡片宽高
4. 再次点击「调节模式」退出调节状态，手柄隐藏

```css
/* 调节手柄样式 */
.resize-handle {
    position: absolute;
    right: 0;
    bottom: 0;
    width: 20px;
    height: 20px;
    background: #00B578;
    cursor: nwse-resize;
    border-radius: 8px 0 8px 0;
    display: none;
    align-items: center;
    justify-content: center;
    color: #fff;
    font-size: 12px;
}

.card.resizable .resize-handle {
    display: flex;
}

.card.resizing {
    outline: 2px solid #00B578;
    user-select: none;
}
```

## 5. 撤销/重做功能（Ctrl+Z / Ctrl+Y）

**快捷键定义**：
- **Ctrl+Z** —— 撤销上一步操作
- **Ctrl+Y** 或 **Ctrl+Shift+Z** —— 重做

**需要记录的操作类型**：

| 操作类型 | 记录内容 | 撤销动作 |
|----------|----------|----------|
| **文本编辑** | 元素选择器、编辑前文本、编辑后文本 | 恢复为编辑前文本 |
| **添加卡片** | 新卡片的 HTML、插入位置 | 删除该卡片 |
| **删除卡片** | 被删卡片的 HTML、原位置 | 在原地恢复卡片 |
| **复制卡片** | 复制生成的卡片 ID | 删除复制的卡片 |
| **拖拽排序** | 卡片 ID、原位置索引、新位置索引 | 移回原位置 |
| **调节大小** | 卡片 ID、原宽高、新宽高 | 恢复原宽高 |
| **添加元素** | 卡片 ID、新元素的 HTML | 删除该元素 |
| **删除元素** | 元素选择器、父卡片、元素 HTML | 恢复该元素 |

**实现要点**：
- 使用 HistoryManager 类管理历史记录栈（max 50 条）
- 文本编辑使用 `blur` 事件记录最终修改，避免记录每次按键
- 撤销/重做后显示临时 Toast 提示
