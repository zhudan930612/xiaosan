## 产品定位

**系统名称**：上海小型工程安全纳管系统

**核心定位**：构建"政府引导、社会参与、市场运作"的小型工程安全治理平台，通过数字化手段实现精准防控，最大限度降低安全风险。

**治理模式**：
- **政府引导**：出台政策与标准，明确各方责任，全局统筹与调度，建设数字化监管平台
- **社会参与**：引入物业企业、保险机构、专业技术服务机构、高校/科研机构/行业专家等社会力量参与安全治理
- **市场运作**：运用保险机制，实现风险共担，激发社会力量参与安全治理

**目标用户**：
- **政府端**：街镇管理人员、区建设行政管理部门、网格员/安全协管员
- **业主端**：业主/建设单位、施工单位
- **社会力量**：物业服务企业、保险机构、第三方专业机构

**核心场景**：
- **政府监管**：街镇对辖区内小型工程进行备案登记、日常巡查、安全检查、执法处理的全流程管理
- **社会参与**：物业企业现场管理、保险机构事故预防服务、专业机构技术支持
- **风险共担**：通过保险机制实现风险转移和事故预防

## 商业模式

**商业模式定位**：
- **公司定位**：华浙智擎定位为风险管理领域的科技公司
- **核心业务**：为保险公司提供事故预防服务
- **核心客户**：保险公司（而非政府）
- **业务聚焦**：围绕风险管理专业和技术应用（如AI技术），提供专业技术、开发工具、制定标准和体系

**收费模式**：
- **政府端**：纳管平台和政府巡检平台免费搭建，只收取运维费，硬件由政府自行采购
- **保险端**：通过保险业务收费
  - **保险佣金**：帮助保险公司销售保险产品，收取保费佣金（各级公司不一致，基本5%起）
    - 方式一：通过宣发物料线下收单
    - 方式二：自行开展保险业务
    - 方式三：通过「铃铛保」保险商城（暂定）线上销售商业补充险
  - **事故预防服务费**（核心收费）：承接安责险事故预防服务，收取保费的21%（保险公司法定支出，用于事故预防服务）

**安责险经营现状与改革方向**（引自 2026-05-09 经营决策会议）：
- **现行制度**：500元/1000元两档强制投保，保险公司每单净亏损933元，财政兜底救助险2023年支出210万元
- **改革方案**：推动「两档变多档」，按工程风险等级实施差异化多档定价，匹配成本与风险，从根本上解决亏损
- **商业补充险**：铃铛保保险商城面向用户销售第三者责任险等商业补充险，预估保费1000元/单，综合投保率（初期）20%

**核心价值主张**：
- **对政府**：免费提供信息化基础设施，满足市级政策要求，实现小型工程全流程信息化监管
- **对保险公司**：提供专业的事故预防服务，降低理赔风险，提升风险管理能力
- **对社会**：引导社会力量参与安全治理，推动从"事后处置"向"事前预防"转型

**战略原则**：政府引导、社会参与、市场运作

**核心竞争力与商业壁垒**：
- **壁垒本质**：不是简单的收费模式，而是通过**事故预防服务质量**持续获得保险公司业务
- **价值闭环**：
  - 对政府：维持政府形象 + 满足内部考核要求
  - 对保险：降低赔付风险 + 获得营收增长
- **竞争力构建路径**：
  - **技术能力**（短期竞争力）：利用AI隐患识别、摄像头监测等技术手段提升事故预防服务质量
  - **数据积累**（长期壁垒）：通过巡查上报、监管整改积累业务数据，构建风险预测模型，动态判断事故发生的风险概率，实现更早介入处理
  - **时间壁垒**：数据积累和模型训练需要时间，新入行的公司无法快速跟进

**风险规避策略**：
- 暂时不开展政府端的巡检巡查业务，避免承担安全生产连带责任
- 人力相关业务（巡检）尽量外包
- 后续考虑成立民非组织或借助第三方机构（如南方应急管理研究院）搭建政府平台

**政府端免费策略背景**：
- **当前阶段（青浦）**：业务从0-1开展的代价，之前没有做过相关业务产品，只能承诺免费开发平台换取进入此业务的机会；早期凭借商务关系和对标竞品来设计产品功能，满足政策要求
- **未来复制阶段**：在其他地区推广时可根据实际情况灵活调整——可以免费、可以收费、也可以不做政府端平台，直接与保险公司合作

**对功能设计的影响**：
- 政府端平台当前阶段免费提供，功能设计以满足政策合规要求为主，不过度投入
- 保险端功能是核心盈利点，需重点打磨事故预防服务功能（风险评估、安全培训、隐患排查等）
- **铃铛保保险商城**是市场化转型的核心载体，需独立设计保险销售流程（产品展示、保费计算、在线支付、保单管理），与事故预防服务系统形成"销售+服务"闭环
- 备案流程中的"小IDI"入口需考虑合规性，明确导流形式和展示规则
- 数据统计功能需同时服务政府监管和保险风险管理两个场景

**青浦区行政区划**
- 下辖11个街镇：夏阳街道、盈浦街道、香花桥街道、朱家角镇、练塘镇、金泽镇、赵巷镇、徐泾镇、华新镇、重固镇、白鹤镇

**监管要求**
- 小型工程必须办理备案信息登记,未备案不得施工（《青浦区小型工程管理办法》第10条）
- 高风险小型工程实施重点监管，高风险判定标准由区建管委牵头制定（《青浦区小型工程管理办法》第7条）
- 街镇需建立监管台账,全面掌握辖区小型工程基本情况（《青浦区小型工程管理办法》第37条、第38条）
- 备案受理时限：1个工作日（《青浦区小型工程管理办法》第14条）
- 工程完工后3个工作日内申报销项,实现闭环管理（《青浦区小型工程管理办法》第15条）
- 依托信息系统全面实施信息化管理,推进数据共享（《青浦区小型工程管理办法》第9条）
- 除住宅室内装修和零星施工作业以外的小型工程，施工单位必须在开工前投保安责险，保单须覆盖完整施工周期；鼓励住宅室内装修、零星施工作业投保安责险或其他商业保险（《青浦区小型工程管理办法》第8条）

**住宅室内装修工程特殊管理要求**（依据《青浦区住宅小区室内装饰装修工程安全生产实施细则》）
- 事前备案：装修人（业主或经业主授权的使用人）在开工前通过小型工程管理平台提交备案申请；材料包括业主身份证/授权书、不动产权证、装修施工合同、施工图纸等；镇街审核通过后生成电子备案凭证；**物业必须核验有效备案凭证才可签订装修管理协议和办理临时出入证**，无凭证的装饰装修企业不得进入小区
- 事中管理：居委会和物业落实每日现场巡查；发现违规应在24小时内向属地镇街报告；镇街对已备案项目开展日常安全检查
- 事后销项：工程竣工后，装修人通过平台提交竣工销项申请，镇街审核确认完成闭环；物业和居委会协助提醒督促装修人及时办理
- 职责特点：装修人、装饰装修企业、物业服务企业、居委会四方共同参与，镇街负责平台审核和属地监管，**物业不得擅自收取装修管理保证金**

**青浦区具体实施要求**（依据青浦区建管委2024年12月会议汇报）
- 建立区级联席会议机制：区主要领导挂帅,住建、规划等部门参与,定期研究解决重大问题,建立联合执法机制
- 完善网格化"两级"管理：建立健全网格化管理体系,明确网格长、街镇主要领导责任分工,实行"一工程一档案",全面记录工程基本信息、监管情况、隐患整改等
- 严格执法处罚标准：对未备案工程依法处罚,对拒不整改的从重处罚,通过媒体、网站等渠道公开违法违规行为和处罚结果,形成震慑效应
- 信息化平台全流程管理：平台需具备备案登记、监管巡查、执法处罚、数据统计分析等功能,实现全流程、全要素管理

**青浦区除险保安“三张清单”机制对小散业务的影响**（依据《关于落实除险保安“三张清单”工作方案》及《系统三张清单.xls》）
- 区级监管正在推进以“底数清单、隐患清单、整改清单”为核心的动态治理机制，小型工程相关监管设计需兼容该治理口径，但当前项目仍聚焦小型工程场景，不直接承接其他12个重点领域的全量业务
- 小型工程相关数据原则上需支持“点位建档、隐患录入、整改闭环、责任到人”的闭环表达，避免后续与区级统一监管平台的数据口径不一致
- 底数管理强调“必填项 + 领域自定义项”的结构：对小型工程场景，至少需稳定沉淀点位名称、所属街镇、地址、等级、企业安全负责人、联系电话等通用底数要素
- 隐患管理强调“发现即录入、整改即更新”：巡查发现的问题除隐患内容本身外，还需逐步支持隐患分类、所属场景、整改时限、整改方式、销号状态等监管字段
- 整改管理除整改结果外，还强调责任清单：隐患需明确责任分级（区级、街镇级、村居级）和责任人，便于跨层级督办和后续考核
- 区级方案要求底数动态更新、隐患闭环、过程留痕、考核通报和追责问责，因此小型工程平台后续的数据统计不应只展示数量，还需支持闭环率、逾期率、责任层级分布等过程指标

**安责险事故预防服务要求**（依据《安全生产责任保险实施办法》应急〔2025〕27号）
- 保险机构应按不高于安责险实际收取保费的21%投入事故预防服务费用,制定专项预算,据实支出,不得挤占、挪用
- 事故预防服务费用应专门用于被保险人的事故预防及相关技术支持工作,包括：安全生产宣传教育培训、安全风险辨识评估、生产安全重大事故隐患排查、应急预案编制和演练等
- 保险机构每年至少为大中型被保险人提供1次安全风险辨识评估或重大事故隐患排查服务
- 保险机构应建立事故预防服务档案,确保服务过程可追溯,留存期限不少于5年

## 用户角色

| 角色 | 描述 | 法定职责 | 核心权限范围 | 用户特征 |
|------|------|----------|-------------|----------|
| 街镇管理人员 | 负责辖区小型工程属地监管 | 依法负责辖区小型工程安全生产管理,组织日常巡查和安全检查 | 备案受理、安全检查、执法处理、完工销项 | 技术水平中等,PC端办公为主,需处理大量备案和巡查工作 |
| 区建设行政管理部门 | 协调指导街镇监管工作 | 协调指导街镇具体实施小型工程安全监管,开展督导检查 | 业务培训、督导检查、联合检查、抽查、执法处理 | 技术水平较高,PC端办公,关注数据统计和上报 |
| 业主/建设单位 | 小型工程的申报方 | 办理备案信息登记,对备案信息真实性负责,落实安全措施 | 备案申报、完工销项 | 技术水平参差不齐,移动端和PC端均有使用,希望备案流程简单快捷 |
| 施工单位 | 承接小型工程的施工方 | 对工程质量、安全生产、文明施工承担主体责任,建立项目负责人为第一责任人的安全生产责任制 | 施工管理、安全生产、质量控制、文明施工 | 需具有相应营业执照,高风险工程需具有施工资质和安全生产许可证 |
| 物业服务企业 | 物业管理区域内的现场管理方 | 对物业管理区域内小型工程承担巡查、告知、制止、报告等现场管理责任 | 进场手续办理、日常巡查、安全检查、违规制止、信息报告 | 熟悉物业管理区域情况,需要简单易用的巡查和报告工具 |
| 安全协管员/网格员 | 协助街镇开展日常巡查 | 开展日常网格化巡查,发现未备案工程和安全隐患 | 巡查登记、隐患上报 | 技术水平较低,移动端现场使用,需要简单易用的巡查工具 |
| 保险机构 | 提供事故预防服务和保险保障 | 通过应保联动机制提供风险评估、安全培训、隐患排查等预防服务,承担事故保险赔偿 | 保险信息登记、预防服务记录、理赔处理 | 专业服务机构,PC端和移动端均使用,需要与监管数据联动 |
| 第三方专业机构 | 受政府或保险公司委托提供专业服务，存在两种场景：①政府委托：承接日常巡查工作（替代或补充安全协管员/网格员）；②保险公司委托：承接事故预防服务外包（风险评估、安全培训、隐患排查等） | 按委托方要求开展专业安全巡查、风险评估、隐患排查、安全培训等服务，对服务质量和结果负责 | 专业巡查、风险评估、隐患排查、服务记录上传 | 专业技术人员,移动端现场作业为主,需要专业的检查工具和标准；两种场景下的操作权限和工作流程有所不同 |

| 小型工程（青浦区）| 青浦区管理范围，包含三类：①按规定可不申请施工许可证的建设工程（含房建、交通、水务、园林绿化、修缮等12类，投资额100万元以下）；②住宅室内装饰装修工程；③零星施工作业（空调安装、外墙清洗、广告牌安装等9类）（《青浦区小型工程管理办法》第2-5条） | - |
| 高风险小型工程 | 符合建设行政管理部门制定的高风险判定标准的小型工程,需重点监管。⚠️ 具体判定标准待区建管委制定后补充 | 与"普通小型工程"区别：高风险工程需在普通备案材料基础上额外提交施工方案、施工资质证书等,且至少开展一次安全检查 |
| 备案信息登记 | 业主/建设单位在工程开工前通过系统申报工程信息并提交相关材料的过程 | 与"施工许可证"区别：备案是告知性程序,施工许可证是审批程序 |
| 完工销项 | 工程完工后申报人通过系统申报完工,街镇核实确认的闭环管理环节 | 与"竣工验收"区别：销项是备案管理的闭环确认,竣工验收是工程质量验收 |
| 街镇 | 街道办事处、乡镇政府的简称,小型工程属地监管的责任主体 | 与"区建设行政管理部门"区别：街镇负责具体实施监管,区建设部门负责协调指导 |
| 网格化巡查 | 街镇组织安全协管员、网格员等对辖区进行分片管理、定期巡查的工作方式 | 与"安全检查"区别：巡查是日常发现机制,安全检查是针对在建工程的专项检查 |
| 小IDI | 备案流程中安责险投保完成后展示的补充险推荐入口，类比车险附加险推广模式，引导用户进入铃铛保保险商城选购商业补充险 | 非具体保险产品，是导流入口/广告位 |
| 铃铛保（暂定） | 公司独立保险销售平台的暂定名称，面向施工方/业主提供商业补充险选购服务，与事故预防服务系统协同形成"保险销售+风控服务"商业闭环 | 非保险公司，是保险销售平台/保险商城 |

### 工程分类枚举

> 依据《青浦区小型工程管理办法》第2-5条，青浦区小型工程包含以下三大类。

#### 不申请办理施工许可证的建设工程（11类）

1. 工程投资额在100万元（不含）以下的新建、改建、扩建房屋建筑（及其附属设施）工程
2. 工程投资额在100万元（不含）以下的交通（及其附属设施）工程
3. 工程投资额在100万元（不含）以下的新（改）建供水、排水项目，以及不纳入区水务工程质量安全监督管理的河道、泵（闸）、农污、厂站等设施维修类水务工程
4. 工程投资额在100万元（不含）以下的园林绿化工程
5. 工程投资额在100万元（不含）以下的既有建筑修缮工程
6. 工程投资额在100万元（不含）以下并且建筑面积在300平方米（不含）以下的公共建筑、商铺、办公楼、厂房等各类非住宅既有建筑装饰装修工程（含原装饰装修内容拆除），对不涉及用途转换的商务楼宇简易装修工程，其建筑面积可放宽至500平方米以下
7. 工程投资额在100万元以下的既有建筑加装电梯工程
8. 已批准的建设工程基地范围内的临时设施施工及已取得《临时建设工程规划许可证》的临时建设工程
9. 依法获得批准建设的农民自建或翻建低层住宅
10. 农业基础性、生产性以及附属性设施等建设工程
11. 农村地区基础设施、公共服务场所、绿化美化美丽庭院（小三园）、标识标牌等综合性建设工程

#### 住宅室内装饰装修工程（1类）

1. 住宅竣工验收合格后，业主或者住宅使用人对住宅室内进行装饰装修的建筑活动

#### 零星施工作业（9类）

1. 空调、太阳能、雨棚、防盗网、管道、工业环保设备设施等非主体工程配套设备设施的安装、维护、拆除作业
2. 建筑外墙清洗、修补、屋面检修等各类建筑外墙零星高处作业
3. 依法取得批准或完成备案手续的户外小型广告及门楣招牌的安装、维护、拆除作业
4. 新能源汽车、电动自行车充电设施的安装、维护、拆除作业
5. 临时搭建舞台、看台、展台、布景等搭设、拆除作业
6. 锅炉、压力容器（含气瓶）、压力管道、电梯、起重机械等特种设备的安装、改造、修理
7. 道路、住宅区、公共场所绿化枝叶修剪
8. 供水、排水、通信、电力、燃气等管线设施维护、维修工程
9. 其他存在坍塌、物体打击、高处坠落、消防安全、机械伤害等事故风险的零星施工作业活动

## 模块清单

> 描述产品各模块的业务定位、场景、角色和主要流程，帮助 AI 理解业务逻辑。
> 各模块的具体功能点、字段设计、实现状态和已知问题请查看 [已上线功能描述目录](../02_已上线功能描述)。

### 备案管理
**定位**：小型工程开工前的信息登记和审核入口，是所有后续监管的起点
**核心场景**：业主/建设单位在工程开工前，通过系统申报工程信息并提交材料，街镇审核后出具回执，工程正式纳入监管
**使用角色**：业主/建设单位（申报）、街镇工作人员（审核）
**主要流程**：
```
业主提交备案申报（填写基础信息、工程信息、安全信息、匹配保单）
→ 系统校验工程类型（不属于小型工程范围则无法提交）
→ 街镇1个工作日内审核材料
  ├─ 符合要求 → 出具办理回执（含"一码安"二维码）→ 工程进入日常巡查
  └─ 材料不全 → 一次性告知修改意见 → 业主补充后重新提交
```
**"一码安"二维码功能**：
- 每个备案工程生成唯一二维码，打印在办理回执上，张贴于施工现场
- 扫码对象：区/街镇/社区/物业巡查人员、保险事故预防服务人员
- 扫码后展示内容：
  - 工程基本信息（工程类型、地址、工期）
  - 政府端巡查台账（巡查次数、历史记录）
  - 保险端服务台账（已完成服务、隐患排查、安全教育）
- 核心价值：实现政府端巡查与保险端事故预防服务的深度融合，政府可查看保险服务进度，保险可同步服务记录
**与其他模块的关系**：
- → 日常巡查：备案通过后，工程自动进入监管范围，监管人员可查看工程信息开展巡查检查
- → 完工销项：备案工程完工后需申报销项，形成从备案到销项的闭环
- → 事故预防服务系统：二维码连接政府端和保险端，实现数据双向同步

### 日常巡查
**定位**：工程施工期间的日常监管，通过巡查和检查发现安全隐患并督促整改，整改不执行的移交执法部门
**核心场景**：街镇安全协管员/网格员对辖区内在建工程进行日常巡查，发现隐患或未备案工程及时处置；对高风险工程开展专项安全检查；对拒不整改的违规行为移交执法部门处理
⚠️ 完整隐患清单（标准化巡查检查项）待制定后补充
**使用角色**：安全协管员/网格员（巡查）、街镇监管人员（检查、调度、违规处置）、第三方专业机构（受政府委托巡查）、物业服务企业（物业管理区域内巡查、违规制止和上报）、施工单位（接收整改通知、提交整改记录）
**主要流程**：
```
日常网格化巡查
  ├─ 发现已备案工程违规 → 登记违规（勾选标准化隐患清单）→ 系统自动变更工程状态
  │     ├─ 不需要限时整改（轻微违规）→ 提交记录即完成，工程状态不变
  │     ├─ 需要现场整改 → 工程状态变为"现场整改" → 施工方提交整改记录 → 审核通过→恢复"正常施工"
  │     ├─ 需要停工整改（较大隐患/严重违规）→ 工程状态变为"停工整改" → 施工方提交整改记录 → 审核通过→恢复"正常施工"
  │     └─ 拒不整改 → 监管人员移交区建设/规划资源/城管执法等部门
  ├─ 发现未备案工程 → 督促办理备案 → 跟踪备案进度
  │     ├─ 备案完成 → 纳入正常监管
  │     └─ 拒不办理 → 移交执法部门
  └─ 发现违法违规行为 → 登记违规 → 督促整改或直接移交执法部门

物业管理区域内巡查（物业服务企业）
  业主申请进场施工 → 物业告知备案要求并办理进场手续
  → 施工期间物业日常巡查
    ├─ 发现未备案 → 记录并督促办理备案 → 上报街镇
    ├─ 发现安全隐患 → 口头劝阻/书面整改通知 → 拒不整改则上报街镇
    └─ 正常施工 → 提供便利服务（用水用电协调、垃圾堆放点指定等）
```
**三方巡查模型**：
同一工程由街道、社区、物业三类组织各自独立开展巡查，频次要求不同，互不替代：

| 组织 | 高风险工程 | 非高风险·住宅装修 | 非高风险·其他工程 |
|------|-----------|-----------------|-----------------|
| 街道 | 每10天至少1次 | 每30天至少1次 | 每21天至少1次 |
| 社区 | 每14天至少1次 | 每21天至少1次 | 每14天至少1次 |
| 物业 | 每2天至少1次 | 每7天至少1次 | 每3天至少1次 |

**违规分级处置规则**：
- **较大隐患**：存在重大安全风险，处置方式强制锁定为"停工整改"，工程状态变为停工整改
- **一般隐患**：由巡查员判断，可选"现场整改"（工程状态变为现场整改）或"不需要限时整改"（工程状态不变）
- 同一次巡查存在多条违规时，工程状态取最严级别（有一条较大隐患即停工整改）
- 整改审核由**同组织内部**完成（街道巡查员发起的违规→街道审核员审核整改）

**与其他模块的关系**：
- ← 备案管理：监管对象来源于备案工程，工程类型和风险等级决定监管力度和巡查频次
- ← 应保联动：第三方机构的事故预防服务数据同步到监管平台，补充监管信息
- → 信用记分：违规行为和处罚结果作为"青安分"记分依据
- → 完工销项：工程状态为"正常施工"且无未完成整改时方可申请核销

### 完工销项
**定位**：工程完工后的闭环确认，确保每个备案工程都有明确的结束节点
**核心场景**：工程完工后，申报人在3个工作日内申报销项，街镇核实确认，实现备案→施工→完工的全生命周期闭环
**使用角色**：业主/建设单位（申报完工）、街镇工作人员（核实确认）
**主要流程**：
```
工程实际完工 → 申报人3个工作日内申报销项 → 街镇核实确认 → 工程归档
                                              ↑
              街镇发现工程已完工但未申报 → 联系申报人核实并办理销项
```
**与其他模块的关系**：
- ← 备案管理：销项对象是备案工程，完成销项后工程档案归档
- ← 日常巡查：完工前的监管记录纳入工程档案

### 动态风险量化
**定位**：引入人机料法环多维数据，从"经验式监管"到"数据化预警"，为青浦城市安全治理改革提供技术支撑
**核心场景**：基于事故致因分析理论，采集施工过程中的人员、设备、材料、方法、环境等多维数据，通过风险量化模型计算工程动态风险值，实现风险预警和差异化监管
**使用角色**：区级管理人员（风险态势分析）、街镇管理人员（风险预警响应）、第三方专业机构（风险评估）
**风险量化维度**：
- **技术风险**：未遵循技术交底方案、脚手架搭设不规范等
- **设备材料风险**：超载调运、材料性能不符合设计规范等
- **环境风险**：临边洞口防护措施未到位、气候条件、照明情况等
- **管理风险**：施工技术文件资料不完整、安全检查及安全监督不到位等
- **人为因素**：施工人员安全意识、施工人员职业资格等
**主要流程**：
```
采集多维数据（人机料法环）→ 风险量化模型计算 → 生成动态风险值
  ├─ 低风险 → 常规监管频次
  ├─ 中风险 → 增加巡查频次 + 风险提示
  └─ 高风险 → 预警通知 + 强制检查 + 专家介入
```
**与其他模块的关系**：
- ← 备案管理：工程基础信息作为风险评估基础数据
- ← 日常巡查：巡查发现的隐患和违规行为作为风险量化输入
- → 日常巡查：风险值驱动差异化监管策略（高风险工程增加检查频次）
- ← 事故预防服务系统：第三方机构的风险评估报告纳入风险量化模型

### AI智能分拨
**定位**：适配青浦区各部门职责划分，支撑一网协同跨部门、跨层级快速处置，避免推诿扯皮，提升监管效率
**核心场景**：基于AI算法，根据问题类型、隐患等级、地点、部门职责等维度，自动将任务分拨至相关责任部门，实现精准派单和闭环处置
**使用角色**：区级管理人员（分拨规则配置）、街镇监管人员（接收派单、处置反馈）、相关职能部门（接收跨部门派单）
**三大应用场景**：
- **处理群众举报与投诉**：市民反映装修现场私拉电线或作业无防护，AI依据隐患类型和地点，自动分拨至相关部门
- **日常巡查隐患闭环**：发现隐患并上传平台后，AI依据隐患等级和整改时限，自动派单督促施工方整改；若逾期未改，则自动升级分拨至相关部门
- **应对突发与应急事件**：发生重大风险预警时，AI依据预案自动分拨并联动相关部门
**主要流程**：
```
问题/隐患/事件上报 → AI智能分析（类型、等级、地点、职责）→ 自动分拨派单
  ├─ 责任部门接收派单 → 现场处置 → 上传处置结果和照片 → 结案
  ├─ 逾期未处置 → 自动升级 → 上级部门介入
  └─ 跨部门协同 → 多部门联动处置 → 汇总结果 → 结案
```
**与其他模块的关系**：
- ← 日常巡查：巡查发现的隐患作为分拨来源
- ← 动态风险量化：高风险预警触发自动分拨
- → 信用记分：处置结果影响施工单位信用记分

### 信用记分管理
**定位**：通过"青安分"信用体系和量化评价指标体系，对施工单位、监理单位、代建单位实施差异化监管，建立安全监管白名单，助力失信行为联合惩戒
**核心场景**：施工单位在施工过程中的违规行为被记分，累计记分达到不同阈值触发不同监管措施；同时通过量化评价体系对各参建单位进行综合评分，表现良好的单位享受激励性监管（减少检查频次等），失信单位纳入联合惩戒
⚠️ 具体记分规则（违规行为分值、阈值线、修复条件）待制定后补充
**使用角色**：街镇监管人员（记分操作）、区级管理人员（规则管理、统计分析）、施工单位/监理单位/代建单位（查询自身信用和评分）
**量化评价指标体系**：
- **合规要求**：备案材料完整性、资质证书有效性、保险投保情况
- **隐患整改**：隐患按期整改率、平均整改周期、整改质量
- **过程安全管理**：安全检查配合度、标准动作执行情况、安全培训参与度
- **协同配合**：与监管部门协同配合情况、信息报送及时性
- **过往评价/表现**：历史项目表现、事故发生情况、违规记录
**主要流程**：
```
制定信用记分规则和量化评价指标 → 施工单位/监理单位/代建单位纳入信用管理
→ 违规行为发生 → 现场记分
  ├─ 累计达到警示线 → 约谈
  ├─ 累计达到重点监管线 → 加大检查频次
  ├─ 累计达到暂停线 → 暂停承接新工程
  └─ 施工单位申请信用修复 → 审核通过后恢复记分
→ 定期量化评价（月度/季度）→ 生成综合得分
  ├─ 高分单位 → 纳入安全监管白名单 → 享受激励性监管
  └─ 低分单位 → 纳入失信行为联合惩戒体系 → 限制承接新项目
→ 信用信息公开（社会公示、单位可自助查询）
```
**与其他模块的关系**：
- ← 日常巡查：巡查/检查中发现的违规行为和执法处罚结果作为记分依据
- → 数据统计：信用记分数据纳入统计分析和考核

## 设计原则

> 全产品通用的不可违反规则，所有功能设计必须遵守。

- 所有操作必须留痕：备案登记、审核、检查、执法、销项等关键操作必须记录操作人、操作时间、操作内容
- 高风险工程必须标识：系统自动识别高风险工程并在列表、详情页明显标识,提醒街镇重点监管
- 备案材料完整性校验：高风险工程必须提交施工组织方案、保险保单、资质证书等材料,系统需强制校验
- 时限提醒机制：街镇受理审核1个工作日、完工销项3个工作日等时限需系统自动提醒和预警
- 数据不可篡改：已出具办理回执的备案信息、已完成的检查记录、已处理的执法案件不允许修改核心字段
- 移动端优先：网格巡查、现场检查等场景必须支持移动端操作,支持现场拍照上传
- 数据共享与上报：系统数据需支持向区级、市级平台共享和上报
- 三张清单闭环兼容：涉及巡查整改的数据结构需兼容“底数-隐患-整改-责任”链路，避免后续接入区级统一监管平台时重复改模
- 违法违规数据支撑：为政府提供违法违规行为和处罚结果的数据报表,支持政府通过媒体、网站等渠道公开曝光
- 考核机制支持：系统需记录人员操作情况、系统使用情况等数据,支持将系统使用纳入年度考核
- 档案完整性：实行"一工程一档案",完整记录工程全生命周期信息,包括备案、监管、执法、销项等全流程数据
- 社会力量参与：系统需支持保险机构、第三方专业机构、物业服务企业等社会力量的接入,实现政府监管与市场化服务的协同
- 应保联动机制：系统需支持保险信息登记、事故预防服务记录、理赔信息管理,推动安全治理从"事后处置"向"事前预防"转型
- 信用记分管理：系统需支持"青安分"信用记分制度,实时记录、动态更新、社会公开,支持差异化监管和信用修复
- 物业企业管理：系统需支持物业服务企业的巡查、告知、制止、报告等现场管理职责,提供便捷的巡查和报告工具
- 强制保险要求：除住宅室内装修和零星施工作业外,小型工程施工单位必须投保安全生产责任保险,系统需强制校验保单信息

## 功能规划原则

> 指导功能优先级判断和投入策略的战略原则。

### 优先级判断标准

功能优先级按以下顺序判断：

1. **政策合规** > 用户痛点 > 商业价值 > 技术创新
   - 政策强制要求的功能必须优先实现
   - 满足政策合规后，优先解决用户高频痛点
   - 在满足前两者的基础上，考虑商业价值（保险端收费功能）
   - 技术创新作为加分项，但不作为优先级主要依据

2. **政策强制要求功能**（必须实现）
   - 备案登记功能（市级政策要求）
   - 备案信息上报市级平台（数据对接要求）
   - 工程档案管理（一工程一档案）
   - 安全生产责任保险要求（应保尽保）

3. **用户高频需求功能**（优先实现）
   - 备案申报流程优化（减少填写时间、提升通过率）
   - 审核效率提升（AI审核、批量审核）
   - 移动端便捷操作（巡查上报、回执查看）

4. **商业价值驱动功能**（重点打磨）
   - 保险端事故预防服务功能（核心盈利点）
   - 风险评估和预警功能（提升保险公司价值感知）
   - 数据分析和报表功能（服务政府和保险双方）

5. **技术创新功能**（适度投入）
   - AI隐患识别
   - 视频监控智能分析
   - 风险预测模型

### 功能投入策略

- **政府端平台**：当前阶段免费提供，功能设计以满足政策合规要求为主，不过度投入
- **保险端功能**：核心盈利点，需重点打磨事故预防服务功能
- **数据统计功能**：需同时服务政府监管和保险风险管理两个场景
