## 功能实现现状

### 备案管理

> **模块定义**：详见 [产品上下文 - 备案管理](./小散业务产品上下文.md#备案管理)
> **详细功能描述**：见 [备案管理模块.md](../02_已上线功能描述/备案管理模块.md)

**模块定位**（引自产品上下文）：
小型工程开工前的信息登记和审核入口，是所有后续监管的起点。业主/建设单位在工程开工前，通过系统申报工程信息并提交材料，街镇审核后出具回执，工程正式纳入监管。

**核心流程**：
```
查看列表 → 在线申报 → AI审核 → 人工审核 → 生成回执 → 档案管理
```

#### 已上线子功能

**备案列表查询** ✅ 已上线
- 功能说明：用户查看自己提交的备案记录，了解审核进度和状态
- 核心能力：按工程类型Tab切换（小型工程、住宅装修、零星作业），按审核状态筛选（待填报、待审核、已驳回、已通过），按工程名称或地址搜索，备案列表按时间倒序显示，点击备案卡片进入备案详情页面，支持按时间/状态等字段排序，支持导出备案列表数据为Excel
- 能力边界：用户只能查看自己提交的项目，待填报状态的项目可以继续编辑，已驳回的项目可以修改后重新提交，高级搜索功能（按时间范围、投资额范围等多维度筛选）待规划

**在线申报** ✅ 已上线
- 功能说明：施工方/业主通过线上填写备案信息，提交备案申请
- 核心能力：支持3类工程类型（小型工程、住宅装修、零星作业），分步填写基础信息/工程信息/安全信息，保单匹配（上传保单），OCR识别证件信息，地图选点/楼栋选择，草稿保存
- 能力边界：三类工程在基础信息填写环节有差异，施工作业风险类别共17种，特种作业证要求因工种而异
- ⚠️ 待优化：上海会议确认了一批字段调整，需在4月1日正式上线前完成更新（字段内容待补充）

**施工单位管理** ✅ 已上线
- 功能说明：管理施工单位信息，支持选择已有施工单位或新增施工单位
- 核心能力：施工单位选择器（搜索施工单位名称），新增施工单位（填写营业执照、资质证明、安全生产许可证、法人身份证等信息），新增施工单位需经过人工审核，审核通过后可在选择器中搜索到
- 能力边界：新增施工单位仅支持人工审核，不经过AI审核环节

**施工工人管理** ✅ 已上线
- 功能说明：管理施工工人信息，支持选择已有工人或新增工人
- 核心能力：施工工人选择器（搜索姓名或身份证号，显示历史搜索），新增施工工人（填写工人身份证、工种、特种作业证），支持一个工人添加多个工种，特种作业证要求因工种而异（电工、电焊工、高空作业工必须上传）
- 能力边界：工人信息与备案工程关联，可在多个备案中复用同一工人信息

**保单匹配** 🚧 部分上线
- 功能说明：需投保安责险的工程，在备案时完成保单匹配
- 核心能力（已上线）：上传保单（支持PDF/图片格式，OCR识别保单关键字段，可手动修改识别结果）
- 设计中：线上投保功能（见[功能设计文档](../03_需求设计文件/线上投保功能设计.md)）
  - 3种技术方案：外链跳转 / H5内嵌 / 平台统一API
  - 待与保险公司洽谈确定方案后启动开发
- 设计中：保单匹配可配置化（见[功能设计文档](../03_需求设计文件/保单匹配配置功能设计.md)）
  - 支持按街道×工程类型配置：关保险/开保险非强制/开保险强制
- 能力边界：当前仅支持上传保单方式，线上投保暂不可用；配置功能开发中

**人工审核** ✅ 已上线
- 功能说明：街镇审核人员对备案申请进行人工审核，确保材料完整性和合规性
- 核心能力：街镇人员查看备案列表（筛选待审核/已审核等状态），人工审核页面（支持点击任意字段进行驳回），驳回原因输入（限200字），审核通过后生成备案回执，修改驳回事项后重新提交
- 能力边界：人工审核由街道级审核人员进行，系统上不做时限限制，审核人员可同时驳回多个字段

**备案回执生成** ✅ 已上线
- 功能说明：审核通过后自动生成备案回执，作为施工许可凭证
- 核心能力：自动生成"青浦区小型工程施工告示牌"，回执包含工程基本信息、参建单位信息、监管部门信息，生成二维码（扫码可查看备案信息及巡查信息），支持查看回执和下载保存到相册，支持短信/邮件推送回执，移动端直接查看和保存回执
- 能力边界：回执为图片格式，内容由备案信息自动填充生成，监管部门根据所属街镇从后台配置中自动获取

**工程档案管理** ✅ 已上线
- 功能说明："一工程一档案"，完整记录工程全生命周期信息
- 核心能力：后台管理系统功能，记录工程从备案到完工的全生命周期信息，包含备案信息、日常巡查记录、完工销项记录、应保联动记录（保险服务记录）
- 能力边界：属于后台管理系统模块，需与日常巡查模块、完工销项模块、应保联动接口配合使用

#### 待规划子功能

**警示教育视频播放** 📐 设计中
- 功能说明：用户选择备案类型后，进入填写资料前，强制展示警示教育介绍页面，提升安全意识
- 核心能力：
  - 触发时机：点击备案类型后跳转介绍页面（而非直接进入填写页面）
  - 视频置顶自动播放，不强制要求看完
  - 页面下方展示结构化文案说明，采用"核心要点摘要 + 分段展开说明 + 完整文案收起"方式，支持滑动查看
  - 滑动到底部后显示"我已知晓"按钮，点击后进入资料填写页面
  - 支持"近30天内不再显示"选项，勾选后30天内跳过该页面直接进入填写页面
- 能力边界：每种备案类型（小型工程/住宅装修/零星作业）可配置独立的视频和说明文案；30天免显示状态存储在本地设备，换设备需重新确认
- 设计文档：[安全警示教育功能设计.md](../03_需求设计文件/安全警示教育功能设计.md)

**物业进场备案核验** ❌ 待规划
- 功能说明：在住宅装修等物业管理场景下，物业核验有效备案凭证后，方可办理装修管理协议和临时出入证等进场手续
- 核心能力：备案凭证真伪核验、进场手续办理记录、无有效备案凭证拦截、未备案工程上报街镇
- 能力边界：仅适用于物业管理区域内的住宅装修场景；不替代街镇备案审核，不生成新的备案结论

**“一码安”二维码增强** ❌ 待规划
- 功能说明：在现有备案回执二维码基础上，增强现场扫码能力，作为政府巡查与事故预防服务的统一入口
- 核心能力：展示工程基本信息、政府巡查台账、保险服务台账，支持政府端扫码巡查、保险端扫码记录服务过程
- 能力边界：当前已上线能力以“扫码查看备案信息及巡查信息”为主；身份核验、打卡等更深能力待单独确认后规划

**AI审核** ⏸ 暂缓上线
- 功能说明：对提交的备案信息进行自动化一致性校验，初步筛选不符合要求的申请
- 核心能力：多维度一致性校验（证件信息与填写信息匹配、材料完整性、逻辑合理性、字段关联关系），审核配置（是否启用AI审核、AI审核通过后是否需要人工审核），驳回事项处理（显示驳回字段、原填写信息、驳回原因），支持提交人工审核或修改驳回事项，审批记录查看
- 能力边界：AI审核侧重一致性校验，不替代人工审核的专业判断，OCR识别在文件上传时触发，不属于AI审核环节
- 暂缓原因：技术流程已跑通，但训练数据量不足，识别准确率未达到生产标准，暂不对外开放，待数据积累后再评估上线时间

### 日常巡查

> **模块定义**：详见 [产品上下文 - 日常巡查](./小散业务产品上下文.md#日常巡查)
> **详细功能设计**：见 [安全巡检功能设计.md](../03_需求设计文件/安全巡检功能设计.md)（✅ 已人工确认，覆盖施工方和巡查方全流程）

**模块定位**（引自产品上下文）：
工程施工期间的日常监管，通过巡查和检查发现安全隐患并督促整改，形成巡查、整改、审核的闭环管理。

**核心流程**：
```
日常网格化巡查 → 发现违规/隐患 → 登记并下发整改通知 → 施工方提交整改记录 → 审核通过恢复正常施工
```

#### 已开发子功能（待上线）

**安全巡检首页（巡查概览）** 🚧 已开发
- 功能说明：巡查人员工作台，汇总显示待巡查提醒、工程状态分布、本月巡查统计和最近巡查记录
- 核心能力：多身份切换（同一账号可绑定街道/社区/物业多个角色）、待巡查工程提醒（超出频次要求）、工程状态汇总（正常施工/现场整改/停工整改）、个人巡查统计（本月/累计）
- 能力边界：切换身份后所有数据刷新为所选组织视角；数据统计范围限定为当前身份所属组织

**工程列表与工程详情** 🚧 已开发
- 功能说明：查看辖区所有在建工程，支持多维筛选和搜索，点击进入工程详情
- 核心能力：工程类型Tab筛选（小型工程/住宅装修/零星作业）、工程状态Tab筛选（正常施工/现场整改/停工整改/已核销）、三方巡查统计栏（街道/社区/物业各自巡查次数/隐患数/未巡查时长横向对比）、高级筛选面板（工程规模/风险类型/施工单位等）
- 能力边界：三方巡查统计栏不区分当前登录身份，始终展示三类组织横向对比

**巡查发起与记录** 🚧 已开发
- 功能说明：巡查人员在工程详情页发起巡查，使用标准化隐患清单现场核查，提交巡查记录
- 核心能力：发起巡查（日期/现场照片/隐患清单勾选）、违规登记（隐患等级映射处置方式：较大隐患强制停工整改，**一般隐患→现场整改**）、记录隐患问题分类和所属场景、标记是否重大事故隐患、巡查记录查看
- 能力边界：已核销工程不可发起新巡查；较大隐患处置方式系统自动锁定为"停工整改"，不可修改

**违规登记与整改跟踪** 🚧 已开发
- 功能说明：巡查登记违规后，工程状态自动变更，系统记录待整改事项并支持监管人员跟踪整改进度
- 核心能力：违规分级（较大隐患→停工整改，**一般隐患→现场整改**）、计划整改时限登记、整改方式记录（即知即改/限期整改等）、责任分级和责任人登记、工程状态自动流转（有未完成整改则显示整改中，整改通过后恢复正常施工）、整改跟踪入口（首页快捷功能区，红点提醒）
- 能力边界：同一工程若同时存在停工整改和现场整改，以停工整改优先；工程状态由系统自动维护，非人工设置

**施工方整改提交** 🚧 已开发
- 功能说明：施工方收到整改通知后，提交整改记录（照片+说明），**由巡查员审核**

### 完工销项

> **模块定义**：详见 [产品上下文 - 完工销项](./小散业务产品上下文.md#完工销项)
> **详细功能设计**：见 [完工核销功能设计.md](../03_需求设计文件/完工核销功能设计.md)（AI草稿，覆盖施工方申请、监管方审核、巡检端直接核销全流程）

**模块定位**（引自产品上下文）：
工程完工后的闭环确认，确保每个备案工程都有明确的结束节点。工程完工后，申报人在3个工作日内申报销项，街镇核实确认，实现备案→施工→完工的全生命周期闭环。

**核心流程**：
```
工程实际完工 → 申报人3个工作日内申报销项 → 街镇核实确认 → 工程归档
```

**施工方完工核销申请** 🚧 已开发
- 功能说明：施工方/业主在工程完工后，通过移动端提交完工核销申请，上传现场照片和完工日期
- 核心能力：核销申请入口（工程状态判断）、核销申请页面（完工日期/现场照片/备注）、申请提交后工程状态变为"核销待审核"

- 核心能力：核销待审核列表、核销审核详情（查看照片/审核决定/驳回原因）、审核通过→工程变为"已核销"、驳回→工程恢复申请前状态
- 能力边界：核销通过后工程为终态，不可撤销；已核销工程不可发起新巡查，但可继续提交整改记录并审核待审核中的整改记录

**巡检端直接核销** 🚧 已开发
- 功能说明：街道和社区巡检人员发现工程已完工时，可在巡检端直接发起核销
- 核心能力：直接核销入口（工程详情页）、核销资料填写（是否完工/现场图片/补充说明）、核销记录标注为"巡检端直接核销"
- 能力边界：仅街道/社区巡检角色可用；已存在待审核核销申请时不重复发起直接核销；核销后不可新增巡查；整改中工程也可直接核销

**核销记录与工程归档** 🚧 已开发
- 功能说明：核销完成后自动生成核销记录，工程档案完整归档，支持查询历史核销记录
- 核心能力：核销记录生成（含核销方式/时间/操作人）、工程档案归档、已核销工程查询
- 能力边界：归档工程仍可查询，但不可再发起巡查；核销后15天内可继续提交核销前遗留整改记录

### AI智能分拨

> **模块定义**：详见 [产品上下文 - AI智能分拨](./小散业务产品上下文.md#ai智能分拨)

**模块定位**（引自产品上下文）：
适配青浦区各部门职责划分，支撑一网协同跨部门、跨层级快速处置，避免推诿扯皮，提升监管效率。基于AI算法，根据问题类型、隐患等级、地点、部门职责等维度，自动将任务分拨至相关责任部门，实现精准派单和闭环处置。

**核心流程**：
```
任务来源（群众举报/巡查隐患/风险预警） → AI分析（问题类型、隐患等级、地点、部门职责）
  → 自动分拨至责任部门 → 部门处置 → 反馈结果 → 闭环确认
  ├─ 逾期未处置 → 自动升级 → 上级部门介入
  └─ 跨部门协同 → 多部门联动处置 → 汇总结果 → 结案
```

**三大应用场景**：
- 处理群众举报与投诉：市民反映装修现场私拉电线或作业无防护，AI依据隐患类型和地点，自动分拨至相关部门
- 日常巡查隐患闭环：发现隐患并上传平台后，AI依据隐患等级和整改时限，自动派单督促施工方整改；若逾期未改，则自动升级分拨至相关部门
- 应对突发与应急事件：发生重大风险预警时，AI依据预案自动分拨并联动相关部门

### 信用记分管理

> **模块定义**：详见 [产品上下文 - 信用记分管理](./小散业务产品上下文.md#信用记分管理)

**模块定位**（引自产品上下文）：
通过"青安分"信用体系和量化评价指标体系，对施工单位、监理单位、代建单位实施差异化监管，建立安全监管白名单，助力失信行为联合惩戒。施工单位在施工过程中的违规行为被记分，累计记分达到不同阈值触发不同监管措施；同时通过量化评价体系对各参建单位进行综合评分，表现良好的单位享受激励性监管（减少检查频次等），失信单位纳入联合惩戒。

**核心流程**：
```
违规行为发生 → 监管人员记分 → 累计记分计算 → 触发监管措施
  ├─ 低风险（高分）→ 激励性监管（减少检查频次、优先办理等）
  ├─ 中风险（中分）→ 常规监管
  └─ 高风险（低分）→ 强化监管（增加检查频次、约谈警告、联合惩戒等）
同时：量化评价指标体系 → 综合评分 → 白名单/黑名单 → 差异化监管
```

**量化评价指标体系**：
- 合规要求：备案材料完整性、资质证书有效性、保险投保情况
- 隐患整改：隐患按期整改率、平均整改周期、整改质量
- 过程安全管理：安全检查配合度、标准动作执行情况、安全培训参与度
- 事故记录：事故发生次数、事故等级、事故责任认定

#### 待规划子功能

> 本模块属于政策合规功能，需在日常巡查模块稳定后考虑投入。当前尚未开始规划，待记分规则和量化评价指标体系明确后，再进行子功能拆分。

#### 待解决问题

- 具体记分规则（违规行为分值、阈值线、修复条件）待制定
- 量化评价指标的权重和计算公式待确定
- 信用修复的条件和流程待明确
- 白名单/黑名单的认定标准待确认
- 与其他部门的联合惩戒机制待对接
- 信用信息的社会公开范围和方式待明确

## 技术架构现状

### 前端技术栈
- 框架：Vue 3
- UI组件库：Element Plus
- 移动端：H5（Vue 3 + Vant）
- 状态管理：Pinia
- 路由：Vue Router

### 后端技术栈
- 框架：Spring Boot 2.7
- 数据库：MySQL 8.0
- 缓存：Redis
- 认证：对接区级统一认证系统（OAuth 2.0）

### 部署架构
- 部署方式：私有化部署在青浦区数据局服务器上
- 网络环境：可外网访问
- 容器化：Docker + Docker Compose
- 认证依赖：必须对接区级统一认证系统，不可独立实现用户登录

### 性能约束
- 单个街镇年均备案量约500-1000条
- 区级年均备案量约1-2万条
- 列表页需支持分页加载，单页最多显示50条

### 数据安全要求
- 涉及房屋权属、身份证号等敏感信息，需加密存储和传输
- 部署在青浦区数据局服务器上，可外网访问

### 已对接系统
- ✅ 区级统一认证系统（用户登录和权限管理）

### 待对接系统

- ❌ 保险机构系统（应保联动）
- ❌ 第三方事故预防服务系统（服务数据同步/纳入工程档案）
