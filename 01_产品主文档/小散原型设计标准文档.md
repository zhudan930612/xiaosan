# 原型设计标准文档

> 小型工程安全纳管系统 UI 设计规范

---

## 1. 设计原则

- **iOS原生风格**：毛玻璃、大圆角、线性图标
- **清晰层级**：字体大小、字重、色彩区分信息层级
- **呼吸感留白**：20px页面边距，16px卡片间距
- **精致细节**：12px卡片圆角，10px按钮圆角，**头像圆形**
- **线性图标**：使用SVG线性图标（stroke-width: 1.5-2px）
  - **禁止使用emoji**作为图标
  - **禁止使用中文文字**（如[首页][我的]）作为图标占位符
  - **必须使用内联SVG**，确保视觉一致性和可替换性

**应用场景**：
- 备案申请（小程序端）
- 巡查检查（APP端）
- 管理后台（Web端）：企业中心、运营管理、系统管理

```

│ 标签1 │ 标签2 │ 标签3 │ 标签4  ← Tab栏   │
```

#### 状态栏（Status Bar）
- **高度**：34px
- **背景**：线性渐变 `linear-gradient(90deg, #35ad73, #58b8d7)`（绿色到青色）
- **内容**：时间（左侧）、状态图标（右侧：信号、WiFi、电池）
- **文字颜色**：白色
- **字体**：12px，字重800

**状态图标样式**：
- 信号图标：4条竖线，高度递增（8px、11px、14px、17px），圆角
- WiFi图标：三层弧线 + 中心圆点
- 电池图标：圆角矩形边框 + 内部填充

#### 导航栏（Header）
- **背景**：线性渐变 `linear-gradient(90deg, #35ad73, #58b8d7)`（与状态栏连续）
- **内边距**：10px 18px 14px
- **高度**：自适应（包含padding）
- **布局**：三栏网格 `44px 1fr 88px`

**元素规范**：
- **返回按钮**：
  - 位置：左侧
  - 尺寸：34px × 34px
  - 图标：左向箭头 `‹`，32px，字重300，白色
  
- **页面标题**：
  - 位置：居中
  - 字号：18px
  - 字重：800
  - 颜色：白色
  - 对齐：居中

- **右侧菜单按钮**：
  - 位置：右侧
  - 尺寸：78px × 36px
  - 样式：圆角胶囊形（border-radius: 999px）

  - 内容：三个圆点 + 分隔线 + 圆环图标

#### Tab栏（Tabs）
- **背景**：`#f1f3f1`（浅灰色）
- **圆角**：顶部圆角 30px（`border-radius: 30px 30px 0 0`）

- **高度**：自适应（padding: 18px 4px 16px）
- **阴影**：内阴影 `inset 0 1px 0 rgba(255,255,255,0.32)`

**Tab项规范**：

### 4.10 SVG线性图标规范

**图标风格**：
- 描边宽度：1.5-2px
- 线端样式：round（圆角线端）
- 拐角样式：round（圆角拐角）
- 填充：none（纯描边，无填充）
- 颜色：根据场景使用品牌绿或系统灰

**常用图标尺寸**：
| 场景 | 尺寸 | 描边宽度 |
|-----|------|---------|
| 导航栏返回 | 24px × 24px | 2px |

| 按钮内图标 | 20px × 20px | 1.5px |
| Tab栏图标 | 28px × 28px | 2px |
| 头像/用户 | 圆形 40px | - |

**图标占位符禁止规则**（强制）：
| 禁止类型 | 错误示例 | 正确做法 |
|---------|---------|---------|
| 中文文字占位 | [首页] [我的] [设置] | 使用对应SVG图标 |
| Emoji表情 | ✅ ⚠️ 📊 🔔 | 使用SVG状态图标 |
| 纯文字标签 | AI问答 数据报表 | 图标+文字组合 |

**图标占位符正确做法**：
- 所有功能图标必须使用内联SVG <svg> 标签
- 状态图标（成功/警告/错误）使用带颜色的SVG
- 导航图标使用线性描边SVG（stroke-width: 1.5-2px）
- 图标尺寸遵循规范：Tab栏28px，列表20px，按钮20px

## 7. 原型输出规范

### 7.1 标注规范（强制）

**标注样式**：
- 背景：`#FFFBE6`（浅黄色）
- 边框：1px dashed `#FFC53D`
- 圆角：4px
- 标题：📌 需求说明，14px 加粗
- 正文：13px 常规

**标注内容性质**：**需求逻辑规则说明**，不是交互说明

**标注内容应包含**：
| 类型 | 说明 | 示例 |
|------|------|------|
| **数据规则** | 计算口径、数据来源、更新频率 | 巡查覆盖率 = 已巡查工程数 / 总工程数 |
| **业务规则** | 状态流转、权限判断、异常处理 | 隐患整改期限：一般3天，较大7天 |
| **字段规则** | 字段定义、校验逻辑、枚举值 | 风险等级：高风险/一般风险/低风险 |
| **需求说明** | 功能点描述、逻辑说明 | 本周指自然周（周一至周日） |

**标注内容不应包含**：
- ❌ 不写交互操作（点击哪里、怎么跳转）
- ❌ 不写页面流程（进入哪个页面）
- ❌ 不写显而易见的操作说明

**标注内容示例**：
```markdown
📌 巡查覆盖率计算规则
- 分子：本周有巡查记录的在建工程数
- 分母：当前在建工程总数（状态=正常施工|现场整改|停工整改）
- 计算公式：分子 / 分母 × 100%
- 数据来源：巡查记录表、工程状态表
- 更新频率：每日凌晨 2:00 自动更新
- 口径说明：暂按"有巡查即覆盖"，后续按频次要求

📌 隐患等级判定规则
- 较大隐患：需停工整改的隐患（如高处坠落风险）
- 一般隐患：现场整改或不限时整改的隐患
- 判定依据：标准化隐患清单 V2.0

📌 整改状态流转
- 待整改 → 整改中（提交整改记录）
- 整改中 → 已整改（审核通过）
- 整改中 → 待整改（审核不通过，退回）
```

**大屏/管理后台标注位置（强制抽屉式）**：
- 使用抽屉式标注，默认收起
- 固定在页面右侧，垂直居中
- 点击"标注"按钮展开/收起

### 7.2 HTML原型规范

**高度自适应**（强制）：
- `.phone-frame` 使用 `height: auto`，禁止固定高度
- `.phone-screen` 使用 `height: auto`
- 禁止 `<div style="height: 100px;">` 占位元素

**定位约束**（强制）：
- 底部按钮使用相对定位或 flex 布局
- 禁止使用 `position: fixed`

**弹窗默认隐藏**（强制）：
- 默认 `display: none`
- 通过 JS 控制显示/隐藏

**可编辑原则**：
- 文案使用 `contenteditable="true"`
- hover 显示绿色边框，focus 显示绿色描边

**移动端原型布局标准（强制）**：
- 整体布局：左侧手机原型区 + 右侧需求说明区域，左右并排
- 主容器 `.main-container` 使用 `display: flex; flex-direction: row; flex-wrap: nowrap;`
- 主容器设置 `min-width: fit-content; margin: 0 auto;`，禁止自动换行且整体居中
- body 设置 `overflow-x: auto;`，当视口不够宽时支持水平滚动
- 左侧 `.prototype-area` 固定宽度（手机框架宽度），`flex-shrink: 0`
- 右侧 `.annotation-area` 弹性宽度，`min-width: 400px; max-width: 600px`
- 两区域间距：`gap: 24px`

**页面导航标准（强制）**：
- 使用浮动导航栏，默认隐藏，不遮挡原型内容
- 快捷键 `F8` 控制导航显示/隐藏
- 导航包含：页面指示器 + 各页面快捷跳转按钮
- 导航位置固定在底部居中

**需求说明区域标准（强制）**：
- 每个页面必须有独立的需求说明区域，跟随页面切换同步显示/隐藏
- 说明区域样式：背景 `#FFFBE6`，边框 `2px dashed #FFC53D`，圆角 `12px`
- 说明区域必须系统性覆盖5类规则：
  1. 页面状态规则
  2. 数据规则
  3. 字段规则
  4. 交互分支规则
  5. 边界情况规则
- 每个页面切换时，右侧说明区域同步切换对应内容
- 标题格式：`页面N：页面名称 — 需求说明`
