## 一、模块概述

### 1.1 业务场景

智慧大屏是青浦区小型工程安全纳管系统的数据可视化展示平台，以青浦区地图为展示底座，将各类业务数据在地图上汇总展示，为管理决策和考核提供直观的数据支撑。

### 1.2 核心价值

| 用户角色 | 核心价值 |
|---------|---------|
| 区级管理人员 | 全局态势掌控，一句话获取全区工程分布、监管动态、风险态势 |
| 街镇管理人员 | 辖区数据查看，快速掌握本辖区备案、巡查、隐患情况 |

### 1.3 主要功能

| 功能类别 | 功能名称 | 状态 |
|---------|---------|------|
| 数据统揽 | 实时统揽 | ✅ 已上线 |
| 数据统揽 | 备案统揽 | ✅ 已上线 |
| 数据统揽 | 巡查统揽 | ⏳ 待上线（显示"即将上线"） |
| 数据统揽 | 组织统揽 | ⏳ 待上线（显示"即将上线"） |
| 地图展示 | 工程点位分布 | ✅ 已上线 |
| 地图展示 | 街镇排名展示 | ✅ 已上线 |
| 明细查询 | 备案申请记录 | ✅ 已上线 |
| 明细查询 | 备案详情查看 | ✅ 已上线 |
| 明细查询 | 工程列表弹窗 | ✅ 已上线 |

### 1.4 适用范围

- **使用端**：PC端大屏展示
- **使用场景**：区级/街镇监管中心大屏展示、领导视察、日常监管

### 1.5 受众角色

| 角色 | 使用场景 | 数据权限 |
|------|---------|---------|
| 区级管理人员 | 查看全区态势、各街镇对比、决策支持 | 全区所有数据 |
| 街镇管理人员 | 查看本辖区数据、发现问题、跟踪整改 | 管辖区域内数据（根据管辖区域和可查看工程类型限制） |

### 1.6 业务价值

- **全局可视**：一屏掌握全区小型工程分布和监管态势
- **数据下钻**：从宏观统计到微观明细的层层穿透
- **辅助决策**：通过数据对比和趋势分析，支撑管理决策
- **考核支撑**：提供年度考核所需的数据统计和上报

## 二、基础信息

### 2.1 模块入口

**入口路径**：监管PC端 → 智慧大屏

**页面布局**：
- 顶部：系统标题"青浦区小型工程与零星作业智慧监管平台"、日期时间、当前用户信息
- 左上角：功能切换菜单（全部工程、小型工程、零星作业、住宅装修）
- 右上角：街道筛选下拉框
- 中间主体：青浦区地图底座 + 各类数据卡片和图表

### 2.2 工程类型筛选

**触发操作**：点击左上角下拉菜单

**可选类型**：
- 全部工程
- 小型工程
- 零星作业
- 住宅装修

**交互规则**：
- 切换工程类型后，所有数据卡片和图表自动刷新，展示对应类型的数据
- 默认展示"全部工程"

**业务规则**：
1. 工程类型筛选是全局筛选器，影响当前页面所有数据展示
2. 筛选条件在页面跳转后保持，直到用户手动切换或清除
3. 如果用户无某工程类型权限，该类型选项不可见或置灰
4. 切换类型后，地图视角自动调整到该类型工程分布最密集的区域

### 2.3 街道筛选

**触发操作**：点击右上角"全部街道"下拉框

**可选范围**：
- 全部街道
- 夏阳街道
- 盈浦街道
- 香花桥街道
- 朱家角镇
- 练塘镇
- 金泽镇
- 赵巷镇
- 徐泾镇
- 华新镇
- 重固镇
- 白鹤镇

**交互规则**：
- 区级管理人员：可选择全部街道或任意单个街道
- 街镇管理人员：根据管辖区域限制，只能查看管辖区域内的街道
- 切换街道后，地图聚焦到对应区域，数据自动刷新

**业务规则**：
1. 街道筛选与工程类型筛选是组合关系，数据需同时满足两个筛选条件
2. 选择"全部街道"时，展示用户权限范围内所有街道的数据汇总
3. 选择单条街道时，如该街道不在用户管辖范围内，提示"无权限查看"
4. 街道筛选状态在用户会话期间保持，关闭页面后重置

## 三、详细功能描述

### 3.1 主控制台（全部工程视角）

**页面名称**："青浦区小型工程与零星作业智慧监管平台"主控制台

**入口路径**：智慧大屏首页

**页面功能**：

#### 3.1.1 四大统揽卡片

**位置**：页面顶部，横向排列4个卡片

**卡片内容**：

| 卡片名称 | 显示内容 | 状态 |
|---------|---------|------|
| **实时统揽** | 总备案数量：167项<br>总巡查次数：（即将上线） | ✅ 已上线 |
| **备案统揽** | 本周新增备案：33项<br>待审核备案：69项 | ✅ 已上线 |
| **巡查统揽** | 本周巡查覆盖率：（即将上线）<br>本周隐患整改率：（即将上线） | ⏳ 待上线 |
| **组织统揽** | 施工单位：15家<br>施工人数：0人 | ✅ 部分上线（施工单位已显示，施工人数待上线） |

**交互规则**：
- 点击卡片可进入对应统揽详情页面
- 显示"刷新数据"按钮支持手动刷新

**业务规则**：
1. 统揽卡片数据计算口径：
   - 总备案数量：已通过审核的工程累计数（历史全部）
   - 本周新增备案：本周内首次提交审核的工程数（含待审核、已通过、已驳回）
   - 待审核备案：当前状态为"待审核"的工程数
   - 施工单位：有工程备案通过的唯一施工单位数
2. 数据刷新频率：
   - 关键数据（待审核数）实时刷新
   - 统计类数据每5分钟自动刷新
3. 待上线功能显示"即将上线"占位符，不参与数据计算

#### 3.1.2 工程数量图表

**位置**：左中区域

**图表类型**：柱状图

**显示维度**：

- 点击柱状图可下钻查看对应街道的工程列表

**业务规则**：
1. 图表数据计算：统计各街道当前筛选条件下的工程数量
2. 数据汇总逻辑：如一个工程涉及多种类型（如同时是小型工程和住宅装修），在各类型中分别计数
3. 柱状图Y轴最大值根据数据自动调整，最小刻度为1
4. 数据为0的街道仍显示在X轴上，柱状高度为0

### 3.2 实时统揽页面

**页面名称**："实时统揽"页面

**入口路径**：主控制台 → 点击"实时统揽"卡片 或 点击顶部"实时统揽"Tab

**页面功能**：

#### 3.2.1 时间维度切换

**位置**：页面顶部Tab栏

**可选维度**：
- 今日（当前选中）
- 本周
- 近七日
- 本月

**交互规则**：
- 切换时间维度后，下方所有数据自动刷新
- 默认显示"今日"数据

**业务规则**：
1. 时间维度定义：
   - 今日：当日0:00至当前时间
   - 本周：本周一0:00至当前时间
   - 近七日：前7天（含今天）
   - 本月：本月1日0:00至当前时间
2. 数据计算基准：以备案提交时间或审核时间为准（需确认具体口径）
3. 切换时间维度后，所有关联图表和表格联动更新
4. 用户离开页面后重新进入，默认恢复"今日"维度

#### 3.2.2 核心指标卡片

**位置**：顶部Tab下方，横向排列5个指标卡片

**指标内容**：

| 指标名称 | 数值 | 环比变化 |
|---------|------|---------|
| 新增备案数量 | 8 | 较上周增长 ↑33.33% |
| 高风险工程数量 | 0 | 较上周增长 ↑0% |
| 在建工程数量 | 8 | 较上周增长 ↑25% |
| 今日巡查次数 | 0 | 较上周增长 ↑0% |
| 发现隐患次数 | 0 | 较上周增长 ↑0% |

**业务规则**：
1. 核心指标定义：
   - 新增备案数量：当前时间维度内新提交审核的备案数
   - 高风险工程数量：当前时间维度内新增的高风险工程数
   - 在建工程数量：截至当前时间在库的在建工程数
   - 今日巡查次数：当前时间维度内完成的巡查次数
   - 发现隐患次数：当前时间维度内巡查发现的隐患数
2. 环比计算：与上一周期对比（如"今日"对比"昨日"，"本周"对比"上周"）
3. 颜色标识规则：增长（正值）绿色↑，下降（负值）红色↓，持平灰色-
4. 数值为0时显示"0"，不显示空或--

## 五、业务规则详解

### 5.1 指标计算口径

#### 5.1.1 环比变化计算

环比变化公式：
```
环比变化率 = (本期数值 - 上期数值) / 上期数值 × 100%
```

计算规则：
1. 本期为今日，则上期为上一天（昨日）
2. 本期为本周，则上周为上一个自然周（周一到周日）
3. 本期为本月，则上月为上一个自然月
4. 上期数值为0时，环比显示"--"（避免除以0）
5. 增长时显示绿色↑，下降时显示红色↓

#### 5.1.2 工程状态统计口径

备案状态定义：
- **待填报**：已创建备案草稿但尚未提交审核的工程
- **待审核**：已提交审核但尚未完成审核的工程
- **已通过**：审核通过且已出具备案回执的工程
- **已驳回**：审核不通过被退回的工程（可重新提交）

统计规则：
1. 各状态数量统计的是该状态下当前仍在库的工程数
2. 已驳回的工程如果重新提交并通过，从已驳回移出，计入已通过
3. 一个工程在生命周期内可能经历多个状态，但同一时刻只归属一个状态

⚠️ 数据异常说明：截图中待填报112+待审核69+已通过167+已驳回94=442，远大于总备案167，说明：
- 待填报、待审核可能包含非当前周期的历史数据
- 或各状态统计口径不一致（如已通过只统计当前周期，待填报统计全部草稿）
- 需确认各状态的统计时间范围是否一致

#### 5.1.3 备案数量统计口径

总备案数量：
- 统计范围：已通过审核且已出具备案回执的工程
- 去重规则：同一工程多次备案（驳回后重新提交）按最新一次有效备案计算
- 时间范围：历史累计（不限时间）

本月备案数量：
- 统计范围：本月内通过审核的工程
- 计算时间：以审核通过时间为准，不是提交时间

本周新增备案：
- 统计范围：本周内新提交审核的工程（含待审核、已通过、已驳回）
- 计算时间：以首次提交审核时间为准

#### 5.1.4 工程类型占比计算

环形图占比公式：
```
某类型占比 = 该类型工程数 / 工程总数 × 100%
```

⚠️ 数据异常说明：截图中小型工程86项占比100%，零星作业44项占比51%，住宅装修37项占比43%，三者之和超过100%，说明：
- 存在工程同时归属多个类型（如一个工程既是小型工程又涉及动火作业）
- 或占比计算的分母不是工程总数，而是其他统计口径
- 需确认占比的计算逻辑

#### 5.1.5 驳回原因占比计算

驳回原因占比公式：
```
某原因占比 = 该原因驳回次数 / 总驳回次数 × 100%
```

⚠️ 数据异常说明：截图中"其他"占比1334.38%，明显异常，可能原因：
- 一个工程可能因多个原因被驳回，导致各原因次数之和大于总驳回次数
- 或"其他"包含了未归类原因的汇总，但计算方式有误
- 需确认驳回原因统计是按工程数统计还是按驳回次数统计

### 5.2 数据定义

#### 5.2.1 工程类型定义

小型工程：符合《青浦区小型工程管理办法》规定的11类工程，投资额100万元以下

住宅装修：住宅竣工验收合格后，业主或住宅使用人对住宅室内进行装饰装修的建筑活动

零星作业：9类非主体工程配套作业，包括空调安装、外墙清洗、广告牌安装等

工程类型判定规则：
1. 一个工程可能同时属于多个类型（如小型工程中的装修工程）
2. 类型判定以备案时选择的工程类型为准
3. 类型变更需重新备案

#### 5.2.2 高风险工程定义

高风险工程判定标准（依据产品上下文）：
- 符合建设行政管理部门制定的高风险判定标准
- 需在普通备案材料基础上额外提交施工方案、施工资质证书等
- 系统自动标识并在列表、详情页明显标识

⚠️ 待确认：具体高风险判定标准需区建管委制定后补充

#### 5.2.3 在建工程定义

在建工程判定规则：
- 已备案通过（状态为"已通过"）
- 尚未完成完工销项
- 工程状态不是"停工整改"或"已核销"

#### 5.2.4 隐患整改率计算

隐患整改率公式：
```
隐患整改率 = 已整改隐患数 / 发现隐患总数 × 100%
```

计算规则：
1. 统计时间范围：当前周期内发现的隐患
2. 已整改定义：已提交整改记录且审核通过
3. 逾期隐患：超过整改期限仍未整改的，计入未整改

### 5.3 权限判断规则

#### 5.3.1 数据可见性判断

数据可见性判断逻辑：
```
IF 用户角色 = "区级管理人员" THEN
    可见全部数据
ELSE IF 用户角色 = "街镇管理人员" THEN
    IF 工程所属街道 IN 用户管辖街道列表 THEN
        IF 工程类型 IN 用户可查看工程类型列表 THEN
            可见该工程数据
        ELSE
            不可见
        END IF
    ELSE
        不可见
    END IF
END IF
```

#### 5.3.2 工程类型权限细化

小型工程权限可细化到具体工程规模：
- 如用户只配置了"住宅装修"权限，则只能看到住宅装修类型的工程
- 如用户配置了"小型工程-建筑装饰装修工程"权限，则只能看到该细分类型

零星作业权限可细化到具体作业类型：
- 如用户配置了"零星作业-动火作业"权限，则只能看到涉及动火作业的工程

权限组合规则：
- 多选为"或"关系，只要满足任一配置即可见
- 未配置任何类型则默认全部可见（需确认具体产品逻辑）
