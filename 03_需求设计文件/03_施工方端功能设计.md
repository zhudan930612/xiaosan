## 整体页面结构

```
底部导航（2个Tab）
├─ 项目中心（首页）
│     ├─ 顶部 Header（绿色渐变）
│     ├─ 快捷功能区（4个图标）
│     └─ 在建工程列表
│           ├─ 搜索栏 + 工程状态筛选
│           ├─ 工程类型 Tab（小型工程 / 住宅装修 / 零星作业）
│           └─ 工程卡片列表（含停工整改 / 现场整改 / 正常施工 / 已核销）
└─ 个人中心
```

> **设计决策**：备案状态（待填报/待审核/已驳回）通过"工程备案"快捷入口进入独立列表查看，不与施工状态混合展示。在建工程列表只展示备案已通过的工程及其施工状态。

## 一、首页（项目中心）

### 1.1 顶部 Header

**背景色**：绿色渐变（与整体主色一致）

| 元素 | 位置 | 说明 |
|------|------|------|
| 招呼语 | 左侧 | 格式："你好，[姓名]师傅 👋"；姓名取自随申办账号实名 |
| 安全宣传语 | 右侧 | 横向滚动文字（跑马灯效果），如"先备案，再开工，安全施工有始终" |

**宣传语规则**：
- 文字内容由后台配置，支持多条轮播
- 默认滚动速度适中，不可手动停止

---

### 1.2 快捷功能区

横向一行排列，4个图标入口：

| 图标 | 入口名称 | 角标显示规则 | 角标数量口径 |
|------|---------|------------|------------|
| 📝 | 工程备案 | 红点 + 数字 | 当前用户名下，备案状态为`已驳回`的工程数 |
| ⚠️ | 隐患整改 | 红点 + 数字 | 当前用户名下，整改状态为`待整改`或`已驳回`的违规条目所在工程数（去重） |
| 📢 | 完工核销 | 红点 + 数字 | 当前用户名下，工程状态为`正常施工`且满足核销条件（如工期已过）的工程数 |
| 🔔 | 安全投诉 | 不显示角标 | — |

**角标数量为 0 时**：不显示角标（不显示"0"）

**点击跳转**：
- 工程备案 → 备案列表页（展示备案流程中的工程，含待填报/待审核/已驳回/已通过）
- 隐患整改 → 整改任务列表（展示有待处理整改的工程，默认选中"待整改"Tab）
- 完工核销 → 核销申请列表（展示可申请核销的工程）
- 安全投诉 → 安全投诉列表（历史投诉记录 + 新增投诉入口）

### 1.4 工程卡片

每张卡片代表一个工程，卡片整体可点击，进入该工程的详情页。

#### 卡片左侧竖线颜色（反映当前工程状态）

| 工程状态 | 竖线颜色 |
|---------|---------|
| 停工整改 | 红色 |
| 现场整改 | 橙色 |
| 正常施工 | 绿色 |
| 已核销 | 灰色 |

#### 卡片内容结构

```
┌───────────────────────────────────────────┐
│ [工程名称]                   [工程状态标签] │
│ 📍 [工程地址]                              │
│  📅 [施工日期]     
🕐 [整改提示]        │
│                                           │
│ [标签1] [标签2] ...    [操作按钮 or 辅助信息]│
└───────────────────────────────────────────┘
```

#### 各字段说明

**工程名称**：加粗，最多显示1行，超出省略

**工程状态标签**（右上角）：

| 状态 | 标签样式 |
|------|---------|
| 停工整改 | 红色描边 + 红色文字 |
| 现场整改 | 橙色描边 + 橙色文字 |
| 正常施工 | 绿色描边 + 绿色文字 |
| 已核销 | 灰色填充 + 灰色文字 |

**第二行信息（根据工程状态显示不同内容）**：

整改已逾期，需立即处理---当该项目有限时整改未按期提交并审核通过时显示
整改期限：3月15日（剩2天）---当该项目有限时整改未提交并审核通过时显示，优先显示距离整改时间最短的那条
当前有3项隐患，需立即处理--无限时整改、无逾期时只显示当前未整改的隐患数量

> **注意**：停工整改无截止日期，不显示倒计时；现场整改（限时整改）有截止日期，超时工程状态自动升级为停工整改（被控停）。

**特殊属性标签行**（第三行，左侧）：

| 标签 | 触发条件 | 样式 |
|------|---------|------|
| 高风险 | 备案信息中工程被标记为高风险 | 红色背景 + 红色文字 |
| 夜间施工 | 备案信息中包含夜间施工 | 紫色背景 + 紫色文字 |
| 周末施工 | 备案信息中包含周末施工 | 蓝色描边 + 蓝色文字 |
| 一般项目 | 非高风险 | 灰色背景 + 灰色文字 |

**辅助信息（右侧，仅正常施工/已核销状态显示）**：
- 格式："上次巡查：N天前"
- 若从未巡查，显示："暂无巡查记录"
- 灰色小字

**操作按钮（右下角，仅停工整改/现场整改状态显示）**：

| 工程状态 | 按钮文字 | 按钮样式 | 点击行为 |
|---------|---------|---------|---------|
| 停工整改 | 去整改 | 蓝色圆角填充按钮 | 进入该工程待整改的整改提交页 |
| 现场整改 | 去整改 | 蓝色圆角填充按钮 | 进入该工程待整改的整改提交页 |
| 正常施工 | 不显示按钮 | — | — |
| 已核销 | 不显示按钮 | — | — |

#### 卡片排序规则

同一类型Tab内，卡片按以下优先级排序：
1.停工整改（已逾期）
2. 停工整改（未逾期）
3. 现场整改（限时整改）
4. 现场整改（不限时整改）
5. 正常施工
6. 已核销

## 二、违规整改类型说明

> 理解三种违规类型是设计工程详情页和整改流程的基础。

| 违规类型 | 工程状态变化 | 有无截止日期 | 不整改后果 | 整改完成后 |
|---------|------------|------------|----------|----------|
| **停工整改** | → 停工整改 | ❌ 无截止日期 | 工程一直停工，不能复工 | 审核通过 → 工程恢复正常施工 |
| **现场整改（限时）** | → 现场整改 | ✅ 有，巡查员设定 | 逾期自动升级为停工整改（被控停） | 审核通过 → 工程恢复正常施工 |
| **不限时整改** | 不变（仍正常施工） | ❌ 无截止日期 | 不整改则不加分，工程不受影响 | 审核通过 → 施工方积分+加 |

**关键规则**：
- 停工整改 = 严重违规，工程强制停工，**整改完成审核通过**是唯一复工条件，无时间限制
- 现场整改 = 限时整改，**逾期未整改自动被控停**（工程状态变为停工整改）
- 不限时整改 = 需要整改，但无时间压力；**整改后提交审核可获加分**，是主动整改的激励机制

## 三、工程详情页（施工方视角）

### 3.1 页面结构与状态自适应

工程详情页内容根据**当前工程状态**动态调整布局优先级，核心原则：**有紧急任务时，任务优先于信息**。

---

### 3.2 状态 A：停工整改

```
┌──────────────────────────────────┐
│ 🔴 停工整改中                     │  ← 顶部状态横幅（红色背景）
│ 完成整改并通过审核后方可复工        │
│ 工程名称 · 地址                   │
└──────────────────────────────────┘

【待处理整改】（展开，卡片列表）
  ┌────────────────────────────────┐
  │ ① 较大隐患：临边防护缺失        │  ← 红色卡片头
  │ 处置：停工整改                  │
  │ 整改要求：安装临时防护栏，高度≥1.2m│
  │ 巡查照片：[可查看]              │
  └────────────────────────────────┘
  ┌────────────────────────────────┐
  │ ② 较大隐患：电工未持证上岗      │
  │ 处置：停工整改                  │
  │ 整改要求：立即停止电工作业...    │
  │ 巡查照片：[可查看]              │
  └────────────────────────────────┘

【工程信息】（折叠，默认收起）
  工程地址 · 工程类型 · 监管机构
  查看完整备案 →

【巡查记录】（折叠，近3条，可展开全部）
  2026-03-10  发现违规3条·停工整改
  2026-02-28  未发现违规·正常

底部固定按钮：[ 提交整改记录（2条）→ ]
```

---

### 3.3 状态 B：现场整改（限时整改）

```
┌──────────────────────────────────┐
│ 🟠 现场整改中                     │  ← 橙色背景
│ 整改期限：3月15日（剩 2 天）       │  ← 有倒计时
│ ⚠️ 逾期将被控停，请尽快处理        │
└──────────────────────────────────┘

【待处理整改】
  ① 一般隐患：安全帽佩戴不规范
    处置：现场整改
    期限：3月15日（剩 2 天）

【工程信息】（折叠）
【巡查记录】（折叠）

底部固定按钮：[ 提交整改记录 → ]
```

**逾期时**（期限已过）：
- 横幅变红，文字改为"整改已逾期 · 工程即将被控停"
- 倒计时改为"已逾期 N 天"

---

### 3.4 状态 C：正常施工（含不限时整改）

正常施工工程，若有**不限时整改**待处理，在工程详情页展示加分提示区块：

```
┌──────────────────────────────────┐
│ ✅ 正常施工中                     │  ← 绿色背景
│ 工程名称 · 地址 · 工期             │
└──────────────────────────────────┘

【可加分整改项】（黄色提示卡）        ← 若有不限时整改
  💡 有 1 项整改可提交，审核通过后可获加分
  ① 一般隐患：安全帽佩戴不规范（已当场提醒）
    [查看详情并提交整改]  ← 可选操作，非强制

【工程信息】（展开显示）
  工程地址 · 工程类型 · 监管机构
  查看完整备案 →

【巡查记录】（近3条）

底部按钮（满足条件时显示）：[ 申请完工核销 ]
```

若**无不限时整改**，则不显示"可加分整改项"区块。

---

### 3.5 状态 D：已核销

```
┌──────────────────────────────────┐
│ ⬜ 已核销                         │  ← 灰色背景
│ 工程已完工归档，只读              │
└──────────────────────────────────┘

【工程信息】（展开显示）
【巡查记录】（全部历史，只读）
【整改记录】（历史，只读）

无操作按钮
```

---

### 3.6 整改内容展示规则

**待处理整改卡片内**必须展示以下信息，方便施工方去现场处理：

| 字段 | 说明 |
|------|------|
| 隐患等级 | 较大隐患 / 一般隐患 |
| 处置方式 | 停工整改 / 现场整改 / 不限时整改 |
| 违规描述 | 巡查员填写的具体违规说明 |
| 整改要求 | 巡查员填写的整改要求说明 |
| 整改期限 | 仅现场整改显示；格式"X月X日（剩N天）"或"已逾期N天" |
| 巡查现场照片 | 可点击查看大图，不可修改 |
