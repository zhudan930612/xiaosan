> **状态**: 🤖 AI草稿，待人工确认

> **策略**: 最小可行方案，只保留过期强制修改

## 一、核心需求（安测要求）

1. 密码有效期180天
2. 过期后强制修改
3. 新密码不能与原密码相同
4. 密码需符合复杂度要求

---

## 二、数据库变更

### 2.1 用户表新增字段

```sql
ALTER TABLE sys_user ADD COLUMN password_modify_time DATETIME COMMENT '密码最后修改时间';
```

**初始化**：现有用户设置为当前时间（`UPDATE sys_user SET password_modify_time = NOW();`）

---

## 三、接口变更

### 3.1 登录接口增强

**响应增加**：
```json
{
  "code": 200,
  "data": {
    "token": "xxx",
    "passwordExpired": true  // true=已过期，需强制修改
  }
}
```

**判断逻辑**：
```java
// 180天有效期
boolean isExpired = daysBetween(passwordModifyTime, now) > 180;
```

### 3.2 修改密码接口

**请求**：
```json
{
  "oldPassword": "原密码",
  "newPassword": "新密码"
}
```

**校验规则**：
1. 原密码正确
2. 新密码 ≠ 原密码
3. 新密码复杂度（8-20位，至少3种字符类型）

**成功后**：更新 `password_modify_time = NOW()`

---

## 四、前端页面

### 4.1 强制修改密码页

**路径**：`/reset-password`

**布局**：
- 顶部警告：⚠️ 您的密码已过期（超过180天），请修改密码
- 表单：
  - 原密码（必填）
  - 新密码（必填，下方显示复杂度提示）
  - 确认新密码（必填）
- 按钮：确认修改

**交互**：
- 密码过期状态下，任何页面访问都重定向到此页
- 修改成功后跳转首页

### 4.2 登录后路由控制

```javascript
if (passwordExpired) {
  window.location.href = '/reset-password';
} else {
  window.location.href = '/dashboard';
}
```

## 七、测试用例
