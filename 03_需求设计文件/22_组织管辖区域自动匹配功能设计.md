# 监管组织管辖区域自动匹配功能设计

> 🤖 AI草稿，待人工确认

---

## 一、背景与问题

### 1.1 现状说明

管理后台的监管组织数据从青浦区数据局同步获取。同步内容包含机构编码和监管组织名称，但**不包含管辖区域信息**（基于截图）。

监管组织名称遵循以下命名规则（基于用户提供信息）：

- **区级别**：区建管委-科室名称，如"区建管委-安监科"
- **街道级别**：xx街道-科室名称，如"盈浦街道-城运中心"
- **居委会级别**：xx街道-居委会/村委会-科室名称，如"盈浦街道-居委会-城运中心"

### 1.2 问题描述

系统需要为每个监管组织配置管辖区域（级联选择：上海市/市辖区/青浦区/街道/居委会），但数据局同步数据缺少该字段。

### 1.3 解决思路

监管组织名称中已隐含管辖区域信息（街道名称、区名称），可通过解析监管组织名称自动推断管辖区域，对无法自动解析的则管辖区域留空，**后续需要添加权限时再手动修正**。

---

## 二、方案概述

### 2.1 核心机制

采用"自动解析 + 映射复用"两层机制：

```
数据局同步监管组织数据
    ↓
【第一层】自动解析引擎
    - 按规则从监管组织名称提取管辖区域关键词
    - 匹配到标准行政区划树
    - 解析成功 → 自动填充管辖区域
    - 解析失败 → 管辖区域留空，不影响数据入库
    ↓
【第二层】映射关系复用
    - 已修正的监管组织名称→管辖区域映射写入映射表
    - 后续同步时优先查映射表，命中则直接填充
    - 未命中再走自动解析引擎
```

### 2.2 功能范围

| 功能模块 | 说明 |
|---------|------|
| 自动解析引擎 | 后台服务，同步后自动执行，无需人工触发 |
| 映射关系管理 | 管理后台页面，查看和维护历史映射关系 |
| 组织编辑修正 | 在监管单位编辑页面支持手动修正管辖区域 |

## 三、业务流程

### 3.1 主流程：数据同步与自动解析

```
[数据局] 推送监管组织同步数据
    ↓
[系统] 接收同步数据，触发自动解析引擎
    ↓
[系统] 逐条解析监管组织名称，推断管辖区域
    ├─ 映射表已存在记录 → 直接采用映射结果（状态：已映射）
    ├─ 映射表无记录，规则匹配成功 → 自动填充管辖区域（状态：自动匹配）
    └─ 映射表无记录，规则匹配失败 → 管辖区域留空（状态：未匹配）
    ↓
[系统] 生成同步批次记录，记录每条监管组织的匹配状态
    ↓
[系统] 同步完成，监管组织正常入库
```

### 3.2 子流程：映射关系复用

```
[数据局] 再次推送同步数据（含新增监管组织）
    ↓
[系统] 逐条处理
    ├─ 监管组织名称在映射表中有记录 → 直接采用历史映射结果
    │     → 状态：已映射
    └─ 监管组织名称在映射表中无记录 → 走自动解析引擎
          → 状态：自动匹配 或 未匹配
```

### 3.3 子流程：后续权限配置时修正

```
[运营人员] 为某监管组织配置权限/用户/审核流程
    ↓
[系统] 校验该监管组织管辖区域是否为空
    ├─ 不为空 → 正常继续权限配置
    └─ 为空 → 提示"该监管组织管辖区域未配置，请先配置管辖区域"
        ↓
[运营人员] 进入监管单位编辑页，选择管辖区域
    ↓
[系统] 保存管辖区域，同时写入映射表
```

## 四、功能点描述

### 4.1 自动解析引擎

**功能说明**：后台服务，在接收数据局监管组织同步数据后自动执行，无需人工触发。

**解析规则**（基于用户提供信息）：

| 监管组织级别 | 名称特征 | 解析规则 | 示例 |
|---------|---------|---------|------|
| 区级别 | 以"区"开头 | 管辖区域 = 青浦区 | 区建管委-安监科 → 青浦区 |
| 虚拟街镇 | 含"西虹桥公司" | 管辖区域 = 西虹桥商务区 | 西虹桥公司-安监部 → 西虹桥商务区 |
| 虚拟街镇 | 含"青浦工业园区/青浦工业区" | 管辖区域 = 青浦工业园区 | 青浦工业园区-安环部 → 青浦工业园区 |
| 居委会级别 | 含"居委会"或"村委会" | 管辖区域 = 街道/镇-居委会/村委会（居委会节点精确匹配，不匹配则降级到街道/镇） | 赵巷镇-居委会-赵巷居委会 → 赵巷镇-赵巷居委会 |
| 街道级别 | 含"xx街道"或"xx镇"（不含居委会/村委会） | 管辖区域 = 对应街道/镇 | 赵巷镇-城市运行管理中心 → 赵巷镇 |

**解析优先级**（从高到低，命中即返回）：
1. 先查映射表（监管组织名称精确匹配或关键词匹配）
2. 映射表无记录时，按以下优先级匹配：
   - ① 虚拟街镇（西虹桥公司 / 青浦工业园区）
   - ② 区级别（以"区"开头）
   - ③ 居委会级别（含"居委会"或"村委会"）
   - ④ 街道级别（含"xx街道"或"xx镇"）

**输出结果**：
- 匹配成功：输出管辖区域（行政区划树节点），自动填充
- 匹配失败：管辖区域留空（null），记录失败原因供后续排查

### 4.2 映射关系管理页

**页面位置**：管理后台 → 企业中心 → 监管组织映射管理

**页面功能**：

1. **映射关系列表**
   - 表格列：序号、监管组织名称关键词、管辖区域、匹配来源（自动/手动）、创建时间、操作
   - 搜索：按关键词、管辖区域筛选
   - 操作：编辑、删除

2. **新增映射**
   - 监管组织名称关键词输入框（支持模糊匹配）
   - 管辖区域选择器（级联）
   - 保存后新增映射记录

3. **编辑映射**
   - 修改管辖区域
   - 修改关键词

4. **删除映射**
   - 支持单条删除和批量删除
   - 删除后不影响已入库的监管组织数据，但后续同步时不再使用该映射

### 4.3 监管单位编辑页修正

**功能说明**：在现有监管单位编辑/新增弹窗中，支持手动修正管辖区域。

**触发场景**：
- 数据同步后管辖区域为空的监管组织，后续需要配置权限时手动修正
- 自动匹配结果有误的监管组织，运营人员手动修正

**交互**：
- 监管单位编辑弹窗中，管辖区域字段为级联选择器（上海市→市辖区→青浦区→街道/镇→居委会）
- 选择管辖区域后保存，同时将该监管组织名称→管辖区域的映射关系写入映射表

### 4.4 权限配置时的管辖区域校验

**功能说明**：在需要用到管辖区域的业务场景中，校验监管组织是否已配置管辖区域。

**校验场景**：
- 为监管组织添加用户时
- 配置AI审核流程时
- 其他需要按管辖区域过滤数据的场景

**校验逻辑**：
  如监管组织的管辖区域为空，则提示"该监管组织管辖区域未配置，请先配置管辖区域"
  点击提示中的链接跳转至监管单位编辑页

### 4.5 数据同步接口适配

**功能说明**：在现有数据局同步接口中增加自动解析触发逻辑。

**触发时机**：每次接收数据局监管组织同步数据后，异步执行自动解析。

**处理逻辑**：
- 同步数据逐条处理，不阻塞同步接口响应
- 解析结果直接写入监管组织表的管辖区域字段
- 解析失败的管辖区域字段留空，记录失败原因到同步批次表

## 五、业务规则与字段规则

### 5.1 自动解析规则（给开发的实现口径）

解析引擎的输入为监管组织名称字符串，输出为管辖区域编码或空值。

解析步骤如下：

第一步：查映射表
  1. 用监管组织名称精确匹配映射表中的关键词字段
  2. 如命中，直接返回映射表中记录的管辖区域编码，匹配来源标记为"已映射"
  3. 如未命中，进入第二步

第二步：按优先级解析（命中即返回，不再往下匹配）

  规则1：虚拟街镇识别
    条件1：监管组织名称包含"西虹桥公司"
    输出：管辖区域 = 西虹桥商务区
    匹配来源标记为"自动匹配"

    条件2：监管组织名称包含"青浦工业园区"或"青浦工业区"
    输出：管辖区域 = 青浦工业园区
    匹配来源标记为"自动匹配"

  规则2：区级别监管组织识别
    条件：监管组织名称以"区"开头
    输出：管辖区域 = 青浦区（编码：310118）
    匹配来源标记为"自动匹配"

  规则3：居委会级别监管组织识别（优先于街道级别）
    条件：监管组织名称包含"居委会"或"村委会"
    处理逻辑：
      第一步：提取"居委会"或"村委会"前面的街道/镇名称部分
        如提取成功且该街道/镇在标准列表中，进入第二步
        如提取失败，进入未匹配处理
      第二步：提取"居委会"或"村委会"后面的具体名称（如"赵巷居委会"、"和睦村"）
        在该街道/镇下查找该名称对应的居委会/村委会节点
        如精确匹配成功，管辖区域 = 街道/镇-居委会/村委会
        如精确匹配失败，管辖区域 = 街道/镇（降级处理），居委会节点留空
    匹配来源标记为"自动匹配"

  规则4：街道级别监管组织识别
    条件：监管组织名称包含青浦区下辖的街道或镇名称
    青浦区下辖11个街镇列表：夏阳街道、盈浦街道、香花桥街道、朱家角镇、练塘镇、金泽镇、赵巷镇、徐泾镇、华新镇、重固镇、白鹤镇
    输出：管辖区域 = 命中的街道/镇名称
    匹配来源标记为"自动匹配"

第三步：未匹配处理
  以下情况管辖区域留空（null）：
  - 监管组织名称不包含任何已知的街道/镇名称、虚拟街镇关键词，且不以"区"开头
  - 监管组织名称同时包含多个不同的街道/镇名称（无法判断归属）
  - 监管组织名称包含的街道/镇名称不在标准列表中
  - 监管组织名称为空或格式异常

  未匹配时记录原因到同步批次明细表，供后续排查参考。
  未匹配不影响监管组织数据正常入库。

### 5.2 映射表数据结构

映射表用于存储已确认的监管组织名称与管辖区域的对应关系。

表名：org_area_mapping

字段说明：
  id：主键，自增
  keyword：监管组织名称关键词，varchar(100)，用于匹配监管组织名称
  area_code：管辖区域编码，varchar(20)，关联行政区划表
  area_name：管辖区域名称，varchar(100)，冗余存储便于展示
  match_type：匹配类型，varchar(20)，枚举值：exact（精确匹配）/fuzzy（模糊匹配）
  source：来源，varchar(20)，枚举值：auto（自动匹配后确认）/manual（手动创建）
  created_at：创建时间，datetime
  updated_at：更新时间，datetime
  created_by：创建人ID，bigint

keyword的设计规则：
  - 对于虚拟街镇，keyword存储"西虹桥公司"或"青浦工业园区"或"青浦工业区"
  - 对于区级别监管组织，keyword存储"区建管委"
  - 对于街道级别监管组织，keyword存储"xx街道"或"xx镇"
  - 对于居委会级别监管组织，keyword存储"xx街道-居委会"或"xx街道-村委会"
  - keyword支持模糊匹配，即监管组织名称包含keyword即可命中

### 5.3 同步批次记录数据结构

表名：org_sync_batch

字段说明：
  id：主键，自增
  sync_time：同步时间，datetime
  total_count：同步监管组织总数，int
  auto_match_count：自动匹配成功数，int
  mapped_count：命中历史映射数，int
  unmatched_count：未匹配数（管辖区域留空），int
  status：批次状态，varchar(20)，枚举值：completed（已完成）
  created_at：创建时间，datetime

表名：org_sync_item

字段说明：
  id：主键，自增
  batch_id：批次ID，bigint，关联org_sync_batch
  org_code：机构编码，varchar(50)
  org_name：监管组织名称，varchar(200)
  area_code：解析出的管辖区域编码，varchar(20)
  area_name：解析出的管辖区域名称，varchar(100)
  match_status：匹配状态，varchar(20)，枚举值：mapped（已映射）/auto_matched（自动匹配）/unmatched（未匹配）
  fail_reason：未匹配原因，varchar(500)
  mapping_id：使用的映射记录ID，bigint
  created_at：创建时间，datetime

### 5.4 匹配状态说明

状态定义：
  mapped（已映射）：命中历史映射表，直接采用历史映射结果
  auto_matched（自动匹配）：按规则引擎解析成功，自动填充管辖区域
  unmatched（未匹配）：规则引擎无法解析，管辖区域留空

状态说明：
  mapped和auto_matched均为终态，无需人工干预
  unmatched状态下监管组织正常入库，管辖区域字段为空
  后续在监管单位编辑页修正管辖区域后，自动写入映射表

### 5.5 权限配置时的管辖区域校验规则

校验触发场景：
  为监管组织添加用户时
  配置AI审核流程时
  其他需要按管辖区域过滤数据的业务场景

校验逻辑：
  查询该监管组织的管辖区域字段是否为空
  如为空，则阻断操作，提示"该监管组织管辖区域未配置，请先配置管辖区域"
  提示中包含"去配置"链接，点击后跳转至监管单位编辑页
  配置完成后返回原操作页面继续

如监管组织已配置管辖区域，则正常继续操作，不做任何提示。

### 5.6 页面权限规则

映射关系管理页的访问权限：
  仅系统管理员角色可见和操作
  普通运营人员不可访问

监管单位编辑页的管辖区域修正权限：
  系统管理员可修正所有监管组织的管辖区域
  运营人员如有监管单位编辑权限，也可修正

### 5.7 行政区划数据规则

行政区划树数据来源：
  系统现有的行政区划表（管理后台中管辖区域选择器已使用）
  无需新增行政区划数据

青浦区下辖11个街镇（基于产品上下文）：
  夏阳街道、盈浦街道、香花桥街道、朱家角镇、练塘镇、金泽镇、赵巷镇、徐泾镇、华新镇、重固镇、白鹤镇

虚拟街镇（基于用户提供信息）：
  西虹桥商务区、青浦工业园区

⚠️ 待确认：西虹桥商务区和青浦工业园区是否作为独立行政区划节点存在于系统的行政区划表中

| 5 | 已上线的监管单位中，当前有多少监管组织的管辖区域为空？是否需要补全历史数据？ | 历史数据补全方案 | 产品/技术 |
| 6 | 映射关系的维护权限是否仅限系统管理员，还是可以下放给运营人员？ | 权限设计 | 产品 |

## 八、决策记录

| 决策内容 | 选择方案 | 决策依据/原因 |
|---------|---------|-------------|
| 自动解析失败后如何处理 | 管辖区域留空，后续添加权限时再修正 | 用户明确要求：解析失败就空着，不需要阻塞同步流程 |
| 映射表采用精确匹配还是模糊匹配 | 采用模糊匹配（keyword包含即可命中） | 监管组织名称可能包含后缀变化（如"盈浦街道-城运中心"和"盈浦街道-城运中心（临时）"应归属同一区域），模糊匹配更灵活 |
| 同步后是否需要人工确认 | 不需要，自动填充后直接入库 | 用户明确要求不需要确认环节，解析失败留空即可 |
| 权限配置时是否校验管辖区域 | 校验，为空则阻断并提示配置 | 管辖区域为空会导致数据权限异常，在真正需要使用的时候再做校验 |
| 是否需要在同步接口中阻塞等待解析完成 | 异步处理 | 解析过程可能涉及大量数据，异步处理不影响同步接口响应速度 |
