## 设计范围

- 本文档仅讨论在方案A基础上新增「小IDI」入口的前端交互设计。
- 本文档不涉及铃铛保保险商城的内部流程设计，仅涉及从备案平台到商城的导流入口。
- 本文档不涉及保险公司接入方式变更，仍沿用方案A的H5内嵌方式。

## 背景与目标

### 现状

- 方案A已完成线上投保全流程设计：选保司 → 支付 → 回调 → 投保成功。
- 投保成功页面仅展示保单关联信息和「提交备案申请」按钮，无后续商业转化路径。
- 铃铛保保险商城定位为独立保险销售平台，需从备案流程中获取流量。

### 业务目标

- 在安责险投保完成后，向用户推荐商业补充险，实现从"强制险"到"自愿补充险"的导流。
- 类比车险「小IDI」模式，用户在完成基础投保后，自然进入附加险选购场景。
- 为铃铛保保险商城导入精准流量，提升商业险投保转化率。

## 功能概述

方案B在方案A的投保成功页面中新增「小IDI」入口。用户在完成安责险线上投保、看到"保单已关联"的成功状态后，页面下方展示补充险推荐卡片，引导用户进入铃铛保保险商城选购第三者责任险等商业补充险。

## 业务流程

### 流程图

```text
用户完成安责险线上投保（方案A流程）
  -> 进入投保成功页
  -> 查看保单已关联状态
  -> 看到「小IDI」补充险推荐入口
      -> 用户点击「小IDI」入口
          -> 跳转至铃铛保保险商城
          -> 用户浏览/选购商业补充险
          -> 完成支付
          -> 商业险保单同步至用户账户
      -> 用户不点击「小IDI」入口
          -> 直接点击「提交备案申请」
          -> 继续备案流程
```

### 与方案A的差异点

| 页面 | 方案A | 方案B |
|------|-------|-------|
| 投保成功页 | 仅展示保单信息 + 提交备案申请按钮 | 增加「小IDI」补充险推荐卡片 |
| 用户路径 | 投保成功 → 提交备案 | 投保成功 → [可选]小IDI → 提交备案 |
| 商业闭环 | 仅完成安责险强制投保 | 安责险 + 商业补充险双重转化 |

## 页面设计

### 投保成功页（方案B改版）

页面定位
投保子流程完成页，在告知用户安责险投保成功的同时，推荐商业补充险。

页面结构（在方案A基础上新增）

1. 成功状态区（方案A已有）
   - 保单已关联图标 + 文案
   - 已投保标签
   - 保单信息摘要

2. **「小IDI」补充险推荐区（新增）**
   - 位置：保单信息摘要下方、PDF下载区域上方或下方
   - 卡片样式：品牌色渐变边框，区别于普通卡片，吸引注意力
   - 标题：「推荐」标签 + "加一份保障，更安心"
   - 推荐险种：
     - 主推荐：第三者责任险（类比车险小IDI，受众最广）
     - 可选展示：其他商业补充险（如有）
   - 利益点文案：
     - "保障施工期间第三方人身伤亡和财产损失"
     - "保费低至XX元起"（如已知）
     - "线上投保，即刻生效"
   - 操作按钮：「去选购」或「了解更多」
   - 关闭/跳过选项：「暂不需要」文字按钮，允许用户跳过

3. 保单PDF下载（方案A已有）

4. 底部操作区（方案A已有）
   - [ 提交备案申请 ] 主按钮
   - [ 临时保存 ] 次按钮

业务规则
- 「小IDI」入口为推荐位，非强制，用户可自由选择是否点击。
- 用户点击「暂不需要」后，该推荐卡片收起或变为最小化状态。
- 点击「去选购」后跳转至铃铛保保险商城，跳转方式根据合规要求确定（外链/内嵌等）。
- 若用户未点击「小IDI」入口直接提交备案，不影响备案流程。
- 「小IDI」入口仅在线上投保成功后展示，线下上传保单成功不展示（因线下场景用户已完成自主投保决策）。
- 推荐险种由铃铛保保险商城后台配置，支持动态调整。

### 小IDI入口状态规则

| 用户行为 | 入口展示状态 | 后续处理 |
|----------|-------------|----------|
| 首次进入投保成功页 | 完整展示推荐卡片 | 记录展示埋点 |
| 点击「去选购」 | 跳转铃铛保商城 | 记录点击埋点 |
| 点击「暂不需要」 | 卡片收起/最小化 | 记录关闭埋点，本次会话不再展示 |
| 返回投保成功页（从商城返回） | 展示「已选购」状态或保持收起 | 根据商城回传数据更新 |
| 直接提交备案 | 忽略入口 | 正常走备案流程 |

## 与铃铛保保险商城的衔接

### 数据传递

从备案平台跳转至铃铛保商城时，需传递以下信息：
- 用户标识（脱敏）
- 当前备案工程ID（用于关联商业险保单）
- 工程基本信息（工程类型、地址、工期等，用于保费计算）
- 已购安责险信息（保险公司、保单号，用于风控服务关联）

### 返回机制

用户完成或不完成商业险选购后，支持返回备案平台：
- 完成选购：返回投保成功页，「小IDI」区域展示"已补充XX险"状态
- 未完成选购：返回投保成功页，「小IDI」区域保持原样或展示"您还未选购"
- 主动关闭商城：直接返回投保成功页

## 待确认事项

| 待确认事项 | 影响范围 |
|------------|----------|
| 「小IDI」入口的合规性：是否允许在政府采购平台中嵌入商业保险导流 | 入口形式、文案口径、是否需要审批 |
| 跳转方式：直接跳转铃铛保商城外链 / 内嵌H5 / 其他形式 | 技术实现、用户体验 |
| 推荐险种的配置方式：后台配置 / 固定展示 / 按用户画像智能推荐 | 产品形态、开发复杂度 |
| 铃铛保商城域名/入口形式是否已确定 | 跳转链接、品牌展示 |
| 「小IDI」名称是否最终确认，是否需要更中性的名称 | 品牌、合规 |
| 商业补充险保费是否支持动态报价 | 保费展示逻辑 |
| 从商城返回备案平台的数据同步机制 | 技术方案 |

## 决策记录

| 决策内容 | 备选方案 | 最终结论 | 决策依据 |
|----------|----------|----------|----------|
| 小IDI入口位置 | 投保成功页 / 备案回执页 / 工程详情页 | 投保成功页 | 用户在完成安责险投保后决策意愿最强，是最佳转化时机 |
| 入口强制程度 | 强制展示 / 可关闭 / 可跳过 | 可跳过，非强制 | 合规要求，避免"强制购买"感受 |
| 入口关闭后是否再次展示 | 每次展示 / 仅展示一次 / 按周期展示 | 本次会话不再展示 | 避免过度打扰用户 |
| 主推荐险种 | 第三者责任险 / 财产险 / 多险种并列 | 第三者责任险 | 受众最广、认知度最高、类比车险小IDI |
| 线下投保成功是否展示小IDI | 展示 / 不展示 | 不展示 | 线下场景用户已完成自主投保决策，不推荐打扰 |
