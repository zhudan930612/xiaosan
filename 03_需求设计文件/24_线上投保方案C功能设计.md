## 设计范围

- 本文档讨论保费定价模型从"固定两档"升级为"多档动态定价"的前端交互和业务规则设计。
- 本文档包含小IDI入口设计（与方案B一致）。
- 本文档不涉及保险公司侧保费精算模型，仅涉及平台侧保费展示和用户交互。

## 背景与目标

### 现状

- 当前安责险采用500元/1000元两档强制投保，保险公司每单净亏损933元。
- 财政兜底救助险2023年支出210万元，现行模式不可持续。
- 方案A/B仍沿用固定两档定价，未解决亏损根本问题。

### 业务目标

来源：[小散业务产品规划.md](../01_产品主文档/小散业务产品规划.md) 保险商业模式章节、2026-05-09经营决策会议

- 推动「两档变多档」改革，按工程风险等级实施差异化定价。
- 匹配保费与风险成本，提升保险公司盈利能力。
- 从"羊毛出在羊身上"转向"羊毛出在猪身上"的科学定价模式。
- 同步保留小IDI商业补充险入口，形成"安责险+商业险"的完整商业闭环。

### 战略定位

- 方案C是解决安责险亏损的唯一科学路径，需政策突破（朱区长支持）。
- 多档定价是平台长期可持续发展的基础，铃铛保商城是市场化转型的核心载体。

## 功能概述

方案C在方案B的基础上，将保费定价模型从"固定两档"升级为"多档动态定价"。系统根据工程类型、投资额、施工内容、风险因素等自动计算风险等级，匹配对应的保费档位。用户在选保司页面即可看到基于自身工程风险的动态保费报价。

## 多档定价模型

### 风险等级划分

| 风险等级 | 判定标准（待细化） | 保费区间（待确认） |
|----------|-------------------|-------------------|
| 低风险 | 投资额低、施工内容简单、无特种作业 | 300-500元 |
| 中低风险 | 投资额中等、常规施工、少量特种作业 | 500-800元 |
| 中风险 | 投资额较高、涉及高空/动火等作业 | 800-1200元 |
| 高风险 | 投资额高、施工复杂、多项特种作业 | 1200-2000元 |
| 极高风险 | 涉及深基坑、大型拆除等高危作业 | 2000元以上 |

### 风险因子

系统综合以下因子计算风险等级：

1. **工程基础因子**
   - 工程类型（小型工程/住宅装修/零星作业）
   - 投资额
   - 建筑面积/施工范围
   - 施工工期

2. **施工内容因子**
   - 是否涉及高处作业
   - 是否涉及动火作业
   - 是否涉及临时用电
   - 是否涉及拆除作业
   - 是否涉及深基坑/脚手架

3. **人员设备因子**
   - 特种作业人员数量及工种
   - 大型机械设备使用
   - 施工队伍规模

4. **环境因子**
   - 施工地点（闹市区/居民区/工业区）
   - 周边敏感设施（学校/医院/养老院等）
   - 气候季节因素

### 保费计算方式

```
基础保费 = 风险等级对应基础费率
调整系数 = 各风险因子的加权系数
最终保费 = 基础保费 × 调整系数 × 保险公司系数
```

- **保险公司系数**：不同保险公司可根据自身定价策略设置微调系数（如0.9-1.1）。
- **动态报价**：用户切换保险公司时，保费根据该保司系数实时更新。
- **保费展示**：最终保费向上取整到十位，如「¥860」。

## 业务流程

### 流程图

```text
用户进入匹配保单页
  -> 系统根据备案信息自动计算风险等级
  -> 展示风险等级标签 + 动态保费
  -> 用户选择保险公司
      -> 保费根据保司系数实时更新
      -> 用户查看保障详情
      -> 用户确认投保
  -> 进入支付页
      -> 展示最终保费金额
      -> 用户完成支付
  -> 进入投保成功页
      -> 展示保单已关联
      -> 展示「小IDI」补充险推荐入口
          -> [可选]用户进入铃铛保商城选购商业补充险
  -> 提交备案申请
```

### 与方案B的差异点

| 环节 | 方案B | 方案C |
|------|-------|-------|
| 保费定价 | 固定两档：500/1000元 | 多档动态：按风险等级计算 |
| 选保司页 | 展示固定保费 | 展示动态保费，带风险等级标签 |
| 支付页 | 固定金额 | 动态金额，与选保司页一致 |
| 支付成功页 | 固定保费展示 | 动态保费展示 |
| 小IDI入口 | 有 | 有（同方案B） |

## 页面设计

### 匹配保单页（方案C改版）

页面结构（在方案A基础上调整）

1. 政策说明卡片（方案A已有）

2. 投保方式区（方案A已有）

3. **风险等级评估区（新增）**
   - 位置：投保方式下方、保险公司选择上方
   - 展示内容：
     - 风险等级标签：如「中风险工程」
     - 等级颜色：低风险(绿色)、中低风险(浅绿)、中风险(黄色)、高风险(橙色)、极高风险(红色)
     - 评估依据摘要：「基于您的工程投资额、施工内容及作业类型综合评估」
     - 展开查看详情：点击后展示各风险因子的具体评分
   - 样式：信息卡片形式，与投保方式区视觉区分

4. **保险公司选择区（改版）**
   - 每个保险公司卡片展示：
     - 保险公司logo + 名称
     - **动态保费金额**：大字号突出显示，如「¥860」
     - 保费说明：「基于您的工程风险等级计算」
     - 保障期限
   - 选中态：保费金额变为品牌色高亮
   - 切换保险公司：保费金额实时变化（带数字动画过渡效果）

5. 卖点区域（方案A已有）

6. 关联线下保单（方案A已有）

业务规则
- 风险等级由系统根据备案信息自动计算，用户不可手动修改。
- 若用户认为风险等级评估有误，可点击「申请复核」，触发人工审核流程（待确认）。
- 保费金额在选保司页和支付页必须保持一致，避免用户困惑。
- 不同保险公司对同一风险等级的报价可能存在差异（保险公司系数）。
- 系统需预留保费计算接口，支持后续与保险公司系统对接实时报价。

### 支付页（方案C改版）

页面结构（在方案A基础上调整）

1. 支付金额
   - 展示最终保费金额（与选保司页一致）
   - 增加「保费说明」：「本保费根据您的工程风险等级动态计算」
   - 增加「价格明细」展开按钮（可选）：展示基础保费、调整系数、保司系数

2. 支付信息（方案A已有）

3. 支付方式（方案A已有）

### 投保成功页（方案C = 方案B）

与方案B完全一致，展示：
- 保单已关联状态
- 实际保费金额（动态计算结果）
- 「小IDI」补充险推荐入口
- 提交备案申请按钮

## 待确认事项

| 待确认事项 | 影响范围 |
|------------|----------|
| 风险等级具体划分标准和保费区间 | 定价模型、用户感知、保险公司接受度 |
| 各风险因子的权重和评分规则 | 算法设计、公平性 |
| 保险公司系数范围（0.9-1.1或其他） | 市场竞争、用户选择 |
| 高风险/极高风险工程是否强制要求更高保额 | 政策合规 |
| 风险等级评估结果是否允许用户申诉/复核 | 流程设计、客服成本 |
| 多档定价是否需要区建管委政策文件支持 | 政策突破、合规审批 |
| 保费计算在平台端还是保险公司端完成 | 技术架构、数据安全 |
| 动态报价是否需要考虑地区差异（不同街镇） | 定价策略 |
| 历史已投保工程是否追溯调整保费标准 | 数据迁移、用户沟通 |

## 决策记录

| 决策内容 | 备选方案 | 最终结论 | 决策依据 |
|----------|----------|----------|----------|
| 定价模型 | 固定两档 / 多档动态 / 完全自由定价 | 多档动态定价 | 解决亏损的唯一科学路径，需政策突破但已获朱区长支持 |
| 风险等级数量 | 3档 / 5档 / 更多 | 5档 | 粒度适中，既能区分风险，又不至于过于复杂 |
| 保费计算位置 | 平台端计算 / 保险公司端计算 / 双方协商 | 平台端初步计算，保司端确认（待确认） | 用户体验优先，需与保险公司对接确认 |
| 是否展示保费明细 | 展示 / 不展示 / 可选展开 | 可选展开 | 默认简洁展示，用户可查看详细计算过程 |
| 风险等级标签颜色 | 单色 / 多色渐变 / 五色区分 | 五色区分 | 与平台现有风险标签体系一致，用户认知成本低 |
| 保险公司系数范围 | 统一价格 / 0.9-1.1 / 0.8-1.2 | 待与保险公司协商 | 需在竞争力和盈利性之间平衡 |
| 小IDI入口是否保留 | 保留 / 移除 | 保留 | 商业闭环需要，多档定价解决安责险亏损，小IDI贡献商业险收入 |
