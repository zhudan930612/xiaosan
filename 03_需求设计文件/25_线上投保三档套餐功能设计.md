## 设计范围

- 本文档重构线上投保的产品形态，从"单一安责险产品"升级为"三档套餐选购"。
- 本文档覆盖选保司后的"保障方案确认页"完整交互设计。
- 本文档不涉及保险公司承保规则，仅涉及前端产品展示和用户交互逻辑。

## 产品概述

用户在备案流程中选择保险公司后，进入"保障方案确认页"。页面以三档套餐形式展示可选方案，用户选择一档后查看详情并支付。

**核心设计理念**：
- 不卖"保险"，卖"保障方案"
- 用套餐形式降低用户决策成本
- 用智能推荐提升补充险转化
- 所有补充险均为自愿勾选

## 三档套餐定义

### 基础版

**定位**：满足政府强制要求的最低保障

| 项目 | 内容 |
|------|------|
| 包含险种 | 安全生产责任保险（安责险） |
| 保障范围 | 从业人员伤亡 + 第三者伤亡和财产损失 + 事故救援费用 |
| 保额 | 法定标准（死亡伤残不低于40万/人） |
| 保费 | 按工程风险等级动态计算（如¥500-2000元） |
| 适用用户 | 只想完成备案强制要求的用户 |

### 进阶版

**定位**：安责险（法定保额）+ 全部补充险自由选择

| 项目 | 内容 |
|------|------|
| 包含险种 | 安责险（法定保额）+ 全部补充险可选 |
| 安责险保额 | 法定标准，与基础版一致，不可选择 |
| 补充险展示 | 全部补充险展示，根据工程信息智能排序，匹配度最高的标记"推荐" |
| 勾选规则 | 安责险始终勾选；切换到进阶版时，默认勾选匹配度最高的1款补充险（标记"推荐"），其余补充险不勾选；用户可自由取消所有补充险（即"只买安责险"也允许） |
| 价格展示 | 安责险价格 + 各补充险单独价格 + 勾选后的合计价格 |
| 适用用户 | 想获得更完整保障、但安责险保额无特别需求的用户 |

### 尊享版

**定位**：高保额安责险（多档可选）+ 全部补充险（至少选1个）

| 项目 | 内容 |
|------|------|
| 包含险种 | 多档可选的安责险 + 全部补充险 |
| 安责险保额 | **3档保额单选，必选其一**：低档（法定标准）/ 中档 / 高档 |
| 补充险展示 | 全部补充险展示，与进阶版相同的智能排序 |
| 勾选规则 | 安责险3档单选（默认选中"中档"，用户可切换）；补充险**至少勾选1个，可多选**；切换到尊享版时，默认勾选匹配度最高的1款补充险 |
| 价格展示 | 所选安责险档位价格 + 各补充险价格 + 合计 |
| 适用用户 | 高投资额工程、风险意识强、希望获得最高保障的用户 |

**尊享版安责险3档保额（示例）**：

| 档位 | 死亡伤残/人 | 医疗费用/人 | 第三者死亡伤残/人 | 第三者财产损失/次 | 保费 |
|------|------------|------------|------------------|------------------|------|
| 低档 | 40万（法定） | 5万 | 40万 | 20万 | ¥500 |
| **中档**（默认） | 60万 | 8万 | 60万 | 40万 | ¥800 |
| 高档 | 100万 | 10万 | 100万 | 80万 | ¥1200 |

**进阶版补充险池（示例）**：

| 补充险 | 保障内容 | 推荐场景 |
|--------|---------|---------|
| 工程一切险 | 施工材料、设备损失；工程本身损坏 | 所有工程 |
| 雇主责任险 | 施工人员更高额度保障 | 人员密集型工程 |
| 公众责任险 | 第三者更高额度保障 | 闹市区/居民区工程 |
| 停工损失险 | 恶劣天气/事故导致停工的误工损失 | 工期紧张/天气多变地区 |
| 装修品质险 | 装修后墙面开裂、水管渗漏等质量问题 | 住宅装修工程 |
| 高空作业险 | 高空坠物、坠落等专项保障 | 涉及高空作业工程 |
| 建筑工程团体意外险 | 施工人员意外伤亡的额外保障 | 人员较多、工期较长的工程 |
| 施工机具保险 | 施工机械、设备的损坏或被盗 | 使用大型/贵重设备的工程 |
| 质量保证保险 | 工程完工后一定期限内的质量缺陷修复 | 对质量要求高的工程 |
| 履约保证保险 | 施工方未能履约导致业主损失的赔偿 | 投资额较高的工程 |
| 建筑工人工资支付保证险 | 施工方拖欠工人工资的先行垫付 | 人员密集型工程 |
| 环境污染责任险 | 施工导致周边环境污染的赔偿 | 化工、拆除等可能污染环境的工程 |
| 设计责任保险 | 设计缺陷导致工程损失的赔偿 | 涉及设计变更的工程 |
| 特种设备安全责任险 | 特种设备（电梯、起重机等）事故的额外保障 | 使用特种设备的工程 |

### 尊享版

**定位**：高保额安责险（多档可选）+ 全部补充险（至少选1个）

详见上方表格定义。

## 补充险智能推荐逻辑

### 推荐因子

系统根据以下信息计算每款补充险与当前工程的匹配度：

| 因子 | 权重 | 说明 |
|------|------|------|
| 工程类型 | 高 | 住宅装修/小型工程/零星作业 |
| 施工内容 | 高 | 是否涉及高空、动火、拆除、深基坑等 |
| 投资额 | 中 | 高投资额工程推荐高保额补充险 |
| 地理位置 | 中 | 闹市区/居民区/学校医院周边 |
| 工期季节 | 中 | 台风季/雨季推荐停工损失险 |
| 历史数据 | 低 | 同类工程高频购买的补充险 |

### 排序规则

1. 计算每款补充险与当前工程的匹配度得分（0-100）
2. 按得分降序排列
3. 得分最高的标记"推荐"标签
4. 进阶版和尊享版均展示全部补充险，差异在于安责险保额和勾选规则

### 推荐示例

**场景一：住宅装修工程（投资额30万，无特种作业）**

| 排序 | 补充险 | 匹配度 | 标签 |
|------|--------|--------|------|
| 1 | 装修品质险 | 95 | 推荐 |
| 2 | 工程一切险 | 80 | - |
| 3 | 公众责任险 | 70 | - |
| 4 | 停工损失险 | 40 | - |
| 5 | 高空作业险 | 10 | - |

**场景二：拆除工程（投资额80万，涉及高空）**

| 排序 | 补充险 | 匹配度 | 标签 |
|------|--------|--------|------|
| 1 | 工程一切险 | 92 | 推荐 |
| 2 | 高空作业险 | 88 | - |
| 3 | 雇主责任险 | 75 | - |
| 4 | 公众责任险 | 60 | - |
| 5 | 停工损失险 | 45 | - |

## 页面流程

### 流程图

```text
匹配保单页
  → 选择保险公司
    → 弹窗确认"即将进入投保页面"
      → 【保障方案确认页】（三档套餐）
          → 用户选择基础/进阶/尊享
            → 基础版：展示安责险详情
              → 点击"确认投保"→ 支付
            → 进阶版：展示安责险 + 3款推荐补充险
              → 用户勾选/取消补充险
              → 实时计算总价
              → 点击"确认投保"→ 支付
            → 尊享版：展示安责险（高保额）+ 全部补充险
              → 用户勾选/取消补充险
              → 实时计算总价
              → 点击"确认投保"→ 支付
          → 支付成功 → 保单回调 → 投保成功页
```

### 三档切换交互

- 默认选中"进阶版"（既突出补充险，又不强制）
- 或默认选中"基础版"（最保守，用户主动升级）
- 切换档位时，补充险列表动态更新
- 已勾选的补充险在切换档位时保留选择状态

### 补充险卡片设计

每款补充险独立卡片，包含：

```
┌── 推荐  ─────────────────────┐
│   工程一切险      [勾选框]   │
│                       │ ← 匹配度最高时显示
│  保障施工材料、设备损失       │
│  以及工程本身的损坏          │
│                            │
│  保额：50万    保费：¥200   │
└─────────────────────────────┘
```

- 勾选框：方形，默认未选中
- "推荐"标签：橙黄色/品牌色，仅匹配度最高的一款显示
- 保障说明：2-3行简洁文案
- 保额和保费：清晰展示

### 价格汇总区

- 固定展示安责险价格（不可取消）
- 已勾选的补充险逐行列出价格
- 合计价格：大字号突出
- 可展开查看明细

## 业务规则

### 档位切换规则

| 操作 | 规则 |
|------|------|
| 从基础切到进阶 | 补充险列表出现（全部补充险），默认勾选匹配度最高的1款 |
| 从进阶切到尊享 | 安责险变为3档单选（默认中档），已勾选的补充险保留；若未勾选任何补充险，自动勾选匹配度最高的1款 |
| 从尊享切到进阶 | 安责险自动变为法定保额（无单选），已勾选的补充险保留 |
| 从任何档位切到基础 | 所有补充险勾选自动取消 |

### 勾选规则

| 档位 | 安责险勾选 | 补充险勾选 |
|------|-----------|-----------|
| 基础版 | 法定保额，固定勾选不可取消 | 不展示，无法勾选 |
| 进阶版 | 法定保额，固定勾选不可取消 | 默认勾选匹配度最高的1款，**用户可全部取消（即只买安责险）** |
| 尊享版 | 3档保额单选，默认选"中档"，**必选其一不可取消** | **必须至少勾选1款**，默认勾选匹配度最高的1款，可多选 |

### 校验规则

- **基础版**：用户点击"我要投保"，直接进入支付
- **进阶版**：用户点击"我要投保"，不论是否勾选补充险，均可进入支付
- **尊享版**：用户点击"我要投保"时校验：
  - 若未选中安责险档位 → 弹窗提示"请选择安责险保额档位"
  - 若未勾选任何补充险 → 弹窗提示"尊享版需至少选择1款补充险，您也可以切换至进阶版"

### 价格计算规则

```
合计保费 = 安责险保费（根据风险等级）
        + Σ（已勾选补充险保费）
        × 套餐优惠系数（如有）
```

- 安责险保费：按方案C的多档动态定价
- 补充险保费：固定价格（由保险公司或后台配置）
- 套餐优惠：三档套餐整体购买可享折扣（如进阶版整体9折）

### 支付规则

- 一次性支付所有选中的险种
- 支付完成后，各保单分别生成
- 回调时分别同步各保单信息

## 合规边界

### 必须遵守

| 规则 | 说明 |
|------|------|
| 默认选中基础版 | 或默认选中进阶版但补充险全部未勾选。不能默认选中带补充险的档位。 |
| 补充险默认不勾选 | 任何补充险的勾选框默认都是空的 |
| "推荐"标签仅1个 | 同一页面最多只有1款补充险显示"推荐"，避免全部标推荐等于没标 |
| 可自由降级 | 用户选了进阶版/尊享版后，随时可以切回基础版，补充险自动取消 |
| 安责险始终可单独购买 | 任何情况下，用户都可以只买安责险，完成备案 |

### 建议避免

| 风险行为 | 替代方案 |
|---------|---------|
| 用红色/警告色强调补充险 | 用品牌绿色或中性色 |
| "不买就要承担XX风险" | "了解可选保障，提升工程抗风险能力" |
| 弹窗阻断必须看补充险 | 平铺展示，用户一扫即过 |
| 隐藏基础版入口 | 三档Tab始终可见，基础版在第一位 |
| 限时倒计时制造焦虑 | "本月组合优惠"等真实活动 |

## 与小IDI入口的关系

**小IDI定位**：保险商城的通用入口，不仅限于施工相关补充险。

**核心原则**：
- 小IDI在投保成功页**始终展示**，不因用户是否购买补充险而隐藏
- 用户点击小IDI进入保险商城，可浏览/购买**任意险种**（包括但不限于施工补充险、家财险、意外险、宠物险等）
- 三档套餐内的补充险推荐是**施工场景内的精准转化**，小IDI是**通用保险商城的泛化入口**，两者互补

**文案差异化**：

| 场景 | 用户行为 | 小IDI文案 |
|------|---------|----------|
| 选了基础版 | 只买安责险 | "您还可以为工程补充保障，或进入保险商城浏览更多" |
| 选了进阶/尊享并买了补充险 | 已购买施工补充险 | "保障已升级！保险商城还有更多险种等您了解" |
| 选了进阶/尊享但没买补充险 | 看了但没勾选 | "刚才未选的补充险仍可购买，或看看商城其他推荐" |

**流量分配逻辑**：
- 三档套餐：承接施工场景内的高意向补充险转化（精准、时效性强）
- 小IDI：承接泛保险需求，为保险商城导流（长期、全品类）

## 决策记录

| 决策内容 | 备选方案 | 最终结论 | 决策依据 |
|----------|----------|----------|----------|
| 默认选中档位 | 基础版 / 进阶版（补充险未勾） | 基础版 | 最保守、最不打扰用户；进阶版需用户主动切换才会看到补充险，减少"被推销"的感受 |
| 进阶版展示几款补充险 | 1款 / 3款 / 不限 | 3款 | 平铺展示不折叠，3款刚好一屏，不产生选择困难 |
| 尊享版安责险保额 | 法定标准 / 更高保额 | 标注政策风险，倾向"法定+补充险叠加"实现高保障 | 安责险保额可能受政策限制 |
| 补充险推荐数量 | 1个"推荐" / 多个"推荐" | 仅1个"推荐" | 避免标签泛滥失去意义 |
| 档位切换时是否保留勾选 | 保留 / 不保留 | 保留（进阶↔尊享），取消（切到基础） | 减少用户重复操作，但切到基础时清空符合预期 |
| 套餐是否有价格优惠 | 无优惠 / 整体折扣 / 满减 | 整体折扣（如进阶版套餐9折） | 激励用户选择进阶版 |

## 待确认事项

| 待确认事项 | 影响范围 |
|------------|----------|
| 尊享版安责险更高保额是否政策允许 | 产品形态设计 |
| 补充险池具体包含哪些险种 | 推荐算法和页面展示 |
| 各补充险的保费定价 | 价格汇总计算 |
| 保险公司是否支持一单多险合并支付 | 技术实现和支付流程 |
| 套餐折扣力度（9折/85折/其他） | 商业策略 |
| 补充险推荐算法是否需要机器学习 | 开发成本 |
| 不同保险公司的补充险产品是否统一 | 产品标准化 |
| 历史工程数据是否可用于推荐模型训练 | 数据积累 |
