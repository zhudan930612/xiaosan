## 设计范围

- 本文档覆盖施工端"更多服务"页面中"保险服务"入口的设计，以及点击进入后的"我的保险"页面完整交互。
- 本文档涉及保单状态管理、工程维度聚合、补充险推荐和保险商城导流。
- 本文档不涉及保险公司承保规则、保单数据同步接口、理赔流程后端处理。

## 产品概述

施工用户在"更多服务"页面点击"保险服务"入口，进入"我的保险"页面。页面以**工程为维度**聚合展示该用户名下所有工程对应的保单信息，支持按状态筛选、搜索，并推荐可补充购买的险种。

**核心设计理念**：
- 以工程为维度聚合保单，符合用户认知（"我这个工程的保险"）
- 状态一目了然，用户一眼知道哪些工程有保障、哪些需要处理
- 管理（看已有保单）和转化（买补充险）在同一个页面完成

## 用户场景

| 场景 | 用户目标 | 页面提供 |
|------|---------|---------|
| 刚投保完，想确认保单生效 | 查看刚买的安责险是否已关联 | 工程卡片显示保单列表 |
| 有多个工程，想统一管理保单 | 查看所有工程的保险状态 | 工程列表 + 状态Tab筛选 |
| 工程快到期了 | 查看保障期限，决定续保 | 即将到期标签 + 续保入口 |
| 备案后忘记投保 | 确认哪些工程还没投保 | 未投保标签 + 去投保入口 |
| 之前买了基础版，想升级 | 加购补充险 | 推荐保障区 + 去商城 |
| 出险了需要理赔 | 找到对应保单 | 保单详情页 + 理赔入口 |

## 页面结构

### 页面层级关系

```
更多服务首页
  → [保险服务] 入口（概览）
      → 我的保险页面（工程列表 + 推荐保障）
          → 工程保单详情页（单工程下所有保单）
              → 电子保单查看 / 理赔申请 / 补充险购买
```

### 1. 更多服务首页 - 保险服务入口

**定位**：保险服务在"更多服务"页面中只占一个列表项，不展开详细内容。

**入口设计**：

```
┌──────────────────────────────────┐
│  [盾牌图标] 保险服务          [3]  │
│  3个工程保障中，1个即将到期       │
└──────────────────────────────────┘
```

**入口元素**：

| 元素 | 说明 |
|------|------|
| 图标 | 盾牌/保险相关图标 |
| 主标题 | "保险服务"或"我的保险" |
| 右侧角标 | 保障中的工程数量（无保单时不显示角标） |
| 副标题 | 动态文案，根据用户状态变化 |

**副标题文案规则**：

| 用户状态 | 副标题文案 |
|---------|-----------|
| 有保单且正常 | "X个工程保障中" / "X个工程保障中，Y个即将到期" |
| 有即将到期 | "X个工程保障中，Y个即将到期" |
| 有未投保工程 | "X个工程未投保" |
| 无保单 | "查看保险相关服务" |

**交互**：点击整个列表项，进入"我的保险"页面。

#### 2.1 状态统计区

**作用**：给用户一个全局概览，不切换Tab也能知道整体情况。

**展示内容**：

| 统计项 | 说明 |
|--------|------|
| 保障中 X | 当前有保障的工程数量 |
| 即将到期 X | 剩余有效期 ≤ 30天的工程数量 |
| 已过期 X | 保障期限已过的工程数量 |
| 未投保 X | 已备案但未匹配保单的工程数量 |

**展示规则**：
- 数量为 0 的项仍显示，但置灰或弱化
- 点击统计项可快速切换到对应 Tab

---

#### 2.2 状态 Tab 标签

**Tab 列表**：

| Tab | 说明 | 默认选中 |
|-----|------|---------|
| **全部** | 所有工程，无论保险状态 | ✅ 默认 |
| **保障中** | 至少有一份保单在有效期内 | - |
| **即将到期** | 剩余有效期 ≤ 30天 | - |
| **已过期** | 所有保单均已过期 | - |
| **未投保** | 工程已备案，但未匹配任何保单 | - |

**Tab 角标**：
- 每个 Tab 右侧显示该状态下的工程数量
- 数量为 0 时不显示角标

**切换交互**：
- 点击 Tab 切换内容区
- 切换时保留搜索关键词（如果有）
- 切换时滚动位置回到顶部

---

#### 2.3 搜索框

**位置**：Tab 标签下方。

**搜索范围**：
- 工程名称（备案时填写的名称）
- 工程地址

**筛选功能**：
- 搜索框右侧提供"筛选"按钮
- 可按工程类型筛选：全部 / 小型工程 / 住宅装修 / 零星作业
- 筛选和搜索可同时使用

**交互**：
- 实时搜索，输入即过滤当前 Tab 下的结果
- 支持清空按钮
- 无结果时显示空状态

#### 2.4 工程卡片列表

**核心设计原则：按工程聚合，而非按保单平铺。**

一个用户可能有多个工程，每个工程可能有多份保单（安责险 + 补充险）。用户记忆方式是"那个装修工程的保险"，不是"那份安责险保单"。

**工程卡片结构**：

```
┌──────────────────────────────┐
│  [住宅装修] 青浦区XX路XX号装修   │  ← 工程类型标签 + 工程名称
│  上海市青浦区XX路XX号XX室       │  ← 工程地址
│  施工状态：在建                │  ← 施工状态（在建/已完工/已销项）
│                              │
│  [保障中]                     │  ← 保险状态标签（颜色区分）
│                              │
│  已投保：                     │
│  · 安全生产责任保险            │
│  · 工程一切险                 │
│  · 装修品质险                 │
│                              │
│  保障期限：2026.5.1 - 2026.8.31 │
│                              │
│  [查看详情]    [加购保障]      │  ← 操作按钮
└──────────────────────────────┘
```

**工程类型标签**：

| 工程类型 | 标签文案 | 标签颜色 | 投保要求 |
|---------|---------|---------|---------|
| 限额以下建设工程 | [小型工程] | 蓝色 | **强制投保**安责险 |
| 住宅室内装修 | [住宅装修] | 绿色 | 鼓励投保 |
| 零星施工作业 | [零星作业] | 橙色 | 鼓励投保 |

**工程类型标签的作用**：
1. 帮助用户快速识别是哪个工程（用户可能同时有多个同地址的工程）
2. 区分投保要求的强弱（强制 vs 鼓励），影响未投保时的提示文案
3. 支持按工程类型筛选（搜索框旁可增加类型筛选）

**状态标签定义**：

| 状态 | 判定条件 | 标签颜色 | 说明 |
|------|---------|---------|------|
| **保障中** | 至少有一份保单在有效期内 | 绿色 | 正常状态 |
| **即将到期** | 最近一份保单剩余有效期 ≤ 30天 | 橙色 | 需要关注 |
| **已过期** | 所有保单均已过保障期限 | 红色 | 保障已断 |
| **未投保** | 工程已备案，但未关联任何保单 | 红色 | 必须投保才能施工（除住宅装修和零星作业外） |

**状态优先级**（一个工程同时满足多个条件时）：
- 未投保 > 已过期 > 即将到期 > 保障中
- 例：工程已过期且未重新投保，显示"未投保"

**操作按钮**：

| 状态 | 主按钮 | 次按钮 |
|------|--------|--------|
| 保障中 | [加购保障] | [查看详情] |
| 即将到期 | [立即续保] | [查看详情] |
| 已过期 | [重新投保] | [查看详情] |
| 未投保 | [去投保] | - |

**未投保状态的提示文案**（根据工程类型区分）：

| 工程类型 | 提示文案 | 提示样式 |
|---------|---------|---------|
| 限额以下建设工程（强制投保） | "必须投保安责险后才能施工" | 红色高亮，紧迫感强 |
| 住宅室内装修（鼓励投保） | "建议投保安责险，提升工程保障" | 普通提示，非强制 |
| 零星施工作业（鼓励投保） | "建议投保安责险，提升工程保障" | 普通提示，非强制 |

**按钮说明**：
- **查看详情**：进入该工程的保单详情页
- **加购保障**：跳转到保险商城，为该工程选购补充险
- **立即续保** / **重新投保** / **去投保**：跳转到投保流程，工程信息自动带入

#### 2.5 推荐保障区（商城推荐）

**定位**：在工程列表下方，根据用户已有保单推荐可补充购买的险种。

**展示条件**：
- 用户至少有一个"保障中"或"即将到期"的工程
- 该工程还有未购买的补充险

**推荐卡片**：

```
┌──────────────────────────────┐
│  【推荐保障】                 │
│                              │
│  根据您的工程推荐             │
│                              │
│  ┌─────────────────────┐    │
│  │ 工程一切险           │    │
│  │ 保障施工材料设备损失  │    │
│  │ ¥200/次    [去了解]  │    │
│  └─────────────────────┘    │
│                              │
│  [进入保险商城 >]            │
└──────────────────────────────┘
```

**推荐逻辑**：
- 优先推荐用户"保障中"工程中尚未购买的补充险
- 按匹配度排序（复用三档套餐的智能推荐逻辑）
- 最多展示 3 个推荐卡片

**点击交互**：
- 点击推荐卡片：进入保险商城，定位到该补充险的产品详情页，工程信息自动带入
- 点击"进入保险商城"：进入保险商城首页

**空状态**：
- 用户无保单：显示"您暂无保单，进入保险商城选购"+ [去商城] 按钮
- 用户已买全补充险：显示"您的保障已很完善"+ [去商城看看] 按钮
- 隐藏推荐区

```
┌─────────────────────────────┐
│  理赔服务  |  常见问题       │
└─────────────────────────────┘
```

**理赔服务**：
- 页面内容：理赔流程说明（文字+流程图）、理赔所需材料清单、保险公司报案电话
- 理赔申请：点击后跳转至保险公司理赔系统（或提供报案电话一键拨打）

- 页面内容：保险相关高频问题（10个以内）

**决策说明**：理赔服务和常见问题作为保险服务的配套支持功能，在本页面提供入口，但具体内容在独立页面展示，不占用"我的保险"主页面空间。

## 业务规则

### 保单状态流转

```
工程备案完成
    ↓
┌─────────────┐
│   未投保    │ ← 初始状态（住宅装修和零星作业可长期停留在此状态）
└──────┬──────┘
       │ 完成投保
       ▼
┌─────────────┐
│   保障中    │ ← 保单在有效期内
└──────┬──────┘
       │ 有效期剩余 ≤ 30天
       ▼
┌─────────────┐
│  即将到期   │
└──────┬──────┘
       │ 超过保障期限
       ▼
┌─────────────┐
│   已过期    │
└──────┬──────┘
       │ 续保/重新投保
       ▼
┌─────────────┐
│   保障中    │
└─────────────┘
```

### 工程卡片展示规则

1. **一个工程显示一张卡片**，不重复。
2. 一个工程有多份保单时，卡片中列出所有险种名称。
3. 工程状态按优先级判定，只显示一个状态标签。
4. 保障期限显示该工程下**最近到期**的保单日期。

### 投保引导规则

| 工程类型 | 投保要求 | 未投保时的处理 |
|---------|---------|--------------|
| 限额以下建设工程（非住宅装修/非零星作业） | **强制投保**安责险 | 未投保 Tab 中红色高亮，提示"必须投保才能施工" |
| 住宅室内装修 | 鼓励投保 | 未投保 Tab 中正常展示，提示"建议投保" |
| 零星施工作业 | 鼓励投保 | 未投保 Tab 中正常展示，提示"建议投保" |

### 数据同步规则

- 用户在备案流程中完成在线投保后，保单信息自动同步到"我的保险"页面。
- 用户通过小IDI入口在保险商城购买补充险后，保单信息自动同步。
- 保单状态（保障中/即将到期/已过期）由系统根据保障期限自动计算，无需用户操作。

## 决策记录

| 决策内容 | 备选方案 | 最终结论 | 决策依据 |
|----------|----------|----------|----------|
| 入口层级 | 更多服务首页展开保险内容 / 点击进入独立页面 | 点击进入独立页面 | 首页保持简洁，保险内容在独立页面完整展示 |
| 聚合维度 | 按保单平铺 / 按工程聚合 | 按工程聚合 | 用户认知模型是"工程的保险" |
| 状态标签 | 只分"有保障/无保障"两档 / 细分为4档 | 细分为4档（保障中/即将到期/已过期/未投保） | 满足不同场景的用户需求 |
| 默认Tab | 全部 / 保障中 | 全部 | 让用户一眼看到全貌 |
| 推荐区位置 | 顶部 / 中部 / 底部 | 工程列表下方（底部） | 先管理已有，再引导购买 |
| 推荐区展示条件 | 始终展示 / 有条件展示 | 有条件展示（有保单且未买全时） | 避免无意义推荐 |
