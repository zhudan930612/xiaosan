# 短信通知配置功能设计

---

## 一、一句话定义

为管理后台运营人员提供独立的短信通知配置页面，按用户-组织关联维度配置两类通知（备案系统通知、巡检系统通知）的接收开关，支持按角色设置默认值与手动覆盖。

---

## 二、触发条件

| 触发场景 | 说明 |
|---------|------|
| 运营人员主动进入 | 运营管理 → 短信通知配置 |
| 新增/导入用户组织关联后 | 系统自动按角色默认值初始化配置 |
| 角色默认值变更时 | 系统自动更新所有未手动修改过的配置 |
| 发送短信前 | 系统校验目标用户-组织关联的对应通知开关状态 |

---

## 三、页面位置

**页面路径**：运营管理 → 短信通知配置

**Tab切换**：
- 人员通知配置（默认）
- 默认规则配置

**入口**：左侧菜单栏「运营管理」下新增「短信通知配置」菜单项，位于「AI审核流程配置」下方

---

## 四、核心流程

### 4.1 人员通知配置流程

```
进入短信通知配置页面 → 人员通知配置Tab
    ↓
用户列表（一行一个用户，所属监管单位标签展示）
    ↓
搜索/筛选条件查询
    ↓
点击「配置」按钮 → 右侧抽屉展开
    ↓
抽屉内展示该用户的所有组织关联卡片
    ↓
每个卡片内独立控制：备案系统通知[开关]、巡检系统通知[开关]
    ↓
点击保存 → 更新配置，关闭抽屉
```

### 4.2 默认规则配置流程

```
进入短信通知配置页面 → 默认规则配置Tab
    ↓
角色默认配置列表
    ↓
编辑角色默认开关（备案系统通知、巡检系统通知）
    ↓
系统提示影响人数
    ↓
确认后批量更新所有 is_customized=false 的配置
```

### 4.3 短信发送前校验流程

```
触发短信发送事件（如备案提交待审核）
    ↓
确定通知类型（备案系统通知 / 巡检系统通知）
    ↓
确定目标用户及组织关联上下文
    ↓
查询 user_org_sms_config 表中对应记录的开关状态
    ↓
enabled=true → 执行发送
enabled=false → 跳过发送，记录日志
```

## 五、业务规则

### 5.1 角色默认配置规则

每个角色独立配置两类通知的默认开关，不采用「审核人/负责人」归类抽象。

| 角色 | 备案系统通知默认 | 巡检系统通知默认 |
|------|----------------|----------------|
| 普通审核员 | 开启 | 开启 |
| 街道巡查员 | 开启 | 开启 |
| 科室负责人 | 关闭 | 关闭 |
| 街道领导 | 关闭 | 关闭 |
| 区级领导 | 关闭 | 关闭 |

**规则说明**：
- 每个角色的默认开关独立配置，运营人员可随时调整
- 修改角色默认值后，所有未手动修改过的（is_customized=false）用户配置自动更新
- 已手动修改过的（is_customized=true）配置不受影响

### 5.2 开关控制规则

- 开关粒度：**组织关联级**（每个「用户+监管单位+角色」组合独立控制）
- 配置权限：**仅后台可配**（运营人员在管理后台配置，用户端不可见不可改）
- 默认值逻辑：新建/导入用户组织关联时，自动按角色默认配置生成初始开关状态
- 手动覆盖逻辑：运营人员手动修改后，标记 `is_customized=true`，后续角色默认值变更不再影响该配置
- 恢复默认：运营人员可主动将已手动修改的配置恢复为当前角色的默认值

### 5.3 通知类型定义

| 通知大类 | 包含场景 | 说明 |
|---------|---------|------|
| 备案系统通知 | 备案提交待审核、备案审核通过、备案审核驳回、备案分派、备案超时提醒 | 与备案业务流程相关的所有短信通知 |
| 巡检系统通知 | 巡查任务分配、巡查结果通知、巡查超时提醒、隐患整改通知 | 与巡查检查业务相关的所有短信通知 |

### 5.4 列表展示规则

- 列表以**用户为维度**，一行展示一个用户
- **所属监管单位**列：用标签展示该用户所有组织关联对应的监管单位名称
  - 3个及以下：全部展示，如「徐泾镇」「青浦工业园区08」「夏阳街道」
  - 超过3个：展示前2个 + 等N个，如「徐泾镇、青浦工业园区08等3个」
- **角色**列：用标签展示该用户所有组织关联对应的角色
- 当用户有多个组织关联时，**通知状态列**展示汇总状态：
  - 「全部开启」：该用户所有组织关联的该类型通知均为开启
  - 「部分开启」：该用户部分组织关联开启、部分关闭
  - 「全部关闭」：该用户所有组织关联的该类型通知均为关闭
  - 「未配置」：该用户没有任何短信配置记录（系统异常）

## 六、数据规则

### 6.1 数据模型

#### 角色默认配置表（role_sms_default）

| 字段 | 类型 | 说明 |
|------|------|------|
| role_id | varchar(64) | 角色ID，主键 |
| role_name | varchar(64) | 角色名称 |
| notify_record_default | tinyint(1) | 备案系统通知默认开关：0=关闭，1=开启 |
| notify_inspection_default | tinyint(1) | 巡检系统通知默认开关：0=关闭，1=开启 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

#### 用户组织关联短信配置表（user_org_sms_config）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 自增主键 |
| user_id | bigint | 用户ID |
| org_relation_id | bigint | 组织关联ID（关联用户权限配置中的组织关联）|
| supervise_org_id | bigint | 监管单位ID |
| supervise_org_name | varchar(128) | 监管单位名称（冗余）|
| role | varchar(64) | 角色（冗余）|
| notify_record | tinyint(1) | 备案系统通知开关：0=关闭，1=开启 |
| notify_inspection | tinyint(1) | 巡检系统通知开关：0=关闭，1=开启 |
| is_customized | tinyint(1) | 是否手动修改过：0=否（跟随角色默认），1=是 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

**唯一索引**：(user_id, org_relation_id)

### 6.2 数据初始化

- **历史数据迁移**：遍历所有现有用户组织关联记录，按角色默认配置生成初始配置，`is_customized=false`
- **新增用户组织关联**：在用户权限配置中新增/导入组织关联后，自动触发初始化，按角色默认配置生成开关状态
- **角色无默认配置处理**：如果角色未在 `role_sms_default` 中配置默认值，则该角色用户的组织关联默认两类通知均关闭

### 6.3 数据更新规则

| 场景 | 更新逻辑 |
|------|---------|
| 运营人员手动修改开关 | 更新对应字段，`is_customized=true` |
| 运营人员点击「恢复默认」 | 按当前角色默认配置重置开关，`is_customized=false` |
| 角色默认值变更 | 遍历该角色所有 `is_customized=false` 的记录，更新为新的默认值 |
| 组织关联删除 | 级联删除对应的 `user_org_sms_config` 记录 |
| 用户删除 | 级联删除该用户所有 `user_org_sms_config` 记录 |

## 七、边界情况

| 场景 | 处理方案 |
|------|---------|
| 用户无组织关联 | 列表中不显示，无短信配置记录 |
| 用户有多个组织关联，部分开启部分关闭 | 列表中显示「部分开启」状态标签 |
| 角色无默认配置 | 该角色用户的组织关联默认两类通知均关闭 |
| 角色默认值变更后用户手动恢复默认 | 恢复为新的角色默认值，`is_customized=false` |
| 监管单位被禁用 | 保留配置记录但暂停发送；单位恢复后继续按配置发送 |
| 短信发送时配置记录不存在 | 默认不发送，记录跳过日志 |
| 批量恢复默认时部分记录角色已变更 | 按当前角色默认值恢复 |
| 用户在同一个组织关联中被赋予多个角色 | 以组织关联维度存储配置，角色变更时重新按新角色初始化默认值（如已手动修改则保留） |

## 八、状态变化

### 8.1 列表汇总状态

| 状态 | 判定条件 | 视觉表现 |
|------|---------|---------|
| 全部开启 | 该用户所有组织关联的该类型通知均为开启 | 绿色开关图标 + 「全部开启」 |
| 部分开启 | 该用户部分组织关联开启、部分关闭 | 黄色开关图标 + 「部分开启」 |
| 全部关闭 | 该用户所有组织关联的该类型通知均为关闭 | 灰色开关图标 + 「全部关闭」 |
| 未配置 | 该用户无任何短信配置记录 | 灰色文字「未配置」 |

### 8.2 配置记录状态

| 状态 | 说明 |
|------|------|
| 跟随默认 | `is_customized=false`，开关值随角色默认值变更自动更新 |
| 自定义 | `is_customized=true`，开关值由运营人员手动设定，不受角色默认值变更影响 |
