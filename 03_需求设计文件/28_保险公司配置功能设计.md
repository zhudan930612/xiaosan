# 保险公司配置功能设计

---

## 一、一句话定义

为管理后台运营人员提供保险公司信息的配置管理功能，支持新增、编辑、上架/下架保险公司，并按自定义排序控制展示顺序。

---

## 二、触发条件

| 触发场景 | 说明 |
|---------|------|
| 运营人员主动进入 | 运营管理 → 保险公司配置 |
| 需要新增合作保险公司时 | 线上投保对接新保司，或线下保司列表需要补充 |
| 需要调整保司展示顺序时 | 运营人员修改排序值 |
| 需要上下架保司时 | 合作终止或恢复合作时 |

---

## 三、页面位置

**页面路径**：运营管理 → 保险公司配置

**入口**：左侧菜单栏「运营管理」下新增「保险公司配置」菜单项，位于「短信通知配置」下方

---

## 四、核心流程

### 4.1 列表查询流程

```
进入保险公司配置页面
    ↓
展示保险公司列表（按排序值升序排列）
    ↓
可按名称模糊搜索、类型筛选、状态筛选
```

### 4.2 新增保险公司流程

```
点击「新增」按钮 → 弹出新增弹窗
    ↓
填写保险公司名称、上传logo、选择类型、填写排序值
    ↓
点击保存 → 校验通过后保存，关闭弹窗，列表刷新
```

### 4.3 编辑保险公司流程

```
点击「编辑」按钮 → 弹出编辑弹窗
    ↓
修改对应字段
    ↓
点击保存 → 校验通过后更新，关闭弹窗，列表刷新
```

### 4.4 上架/下架流程

```
在列表中点击状态开关
    ↓
上架：该保险公司在前端可选列表中展示
    ↓
下架：该保险公司在前端不可见，但历史数据保留
```

---

## 五、业务规则

### 5.1 字段规则

| 字段 | 必填 | 规则说明 |
|------|------|---------|
| 保险公司名称 | 是 | 唯一，不允许重复，最多50个字符 |
| 保险公司logo | 是 | 支持jpg/png/gif格式，单张不超过2MB，建议尺寸200x200px |
| 类型 | 是 | 复选：线上 / 线下，可同时勾选，至少选择一项 |
| 排序 | 是（系统维护） | 正整数，值越小排序越靠前，由拖拽自动重排，弹窗中只读 |
| 状态 | 是 | 上架（开启）/ 下架（关闭），默认上架 |
| 创建时间 | 系统自动 | 记录创建时间 |
| 更新时间 | 系统自动 | 记录最后更新时间 |

### 5.2 类型定义

| 类型 | 说明 | 使用场景 |
|------|------|---------|
| 线上 | 与系统对接，支持线上投保流程 | 用户在线购买保险时可选 |
| 线下 | 未与系统对接，仅作为保司名称展示 | 用户上传保单时选择所属保司 |

**说明**：一个保险公司可以同时勾选线上和线下两种类型。同时勾选时，该保司既在在线投保列表中展示，也在上传保单的保司选择列表中展示。

### 5.3 排序规则

- 列表默认按「排序」字段升序排列（值越小越靠前）
- 排序值由系统自动维护，用户不可手动编辑
- 拖拽排序后，系统自动重新计算所有记录的排序值（1, 2, 3...）
- 新增时自动排在末尾（当前最大值+1）
- 弹窗中排序字段只读展示

### 5.4 状态规则

- **上架**：保险公司在前端展示，用户可选择
- **下架**：保险公司在前端隐藏，用户不可选择
- 下架不影响已产生的历史数据（如已投保记录中的保司信息仍正常显示）
- 至少保留1个上架的线上保司和1个上架的线下保司

## 六、数据规则

### 6.1 数据模型

#### 保险公司配置表（insurance_company）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 自增主键 |
| name | varchar(64) | 保险公司名称，唯一 |
| logo_url | varchar(256) | logo图片URL |
| is_online | tinyint(1) | 是否线上：0=否，1=是 |
| is_offline | tinyint(1) | 是否线下：0=否，1=是 |
| sort_order | int | 排序值，越小越靠前，自动维护 |
| status | tinyint(1) | 状态：0=下架，1=上架，默认1 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

**唯一索引**：name, sort_order

### 6.2 数据初始化

- 系统首次部署时，由运营人员手动录入首批保险公司数据
- 线上保司数量一般较少（对接的系统保司）
- 线下保司数量一般较多（仅用于展示选择）

### 6.3 数据更新规则

| 场景 | 更新逻辑 |
|------|---------|
| 新增保司 | 校验名称和排序唯一性后插入 |
| 编辑保司 | 校验名称和排序唯一性后更新（排除自身） |
| 切换上架/下架 | 更新status字段，实时生效 |
| 删除保司 | ⚠️ 需确认：是否允许删除？删除后历史数据如何处理？ |

## 七、边界情况

| 场景 | 处理方案 |
|------|---------|
| 名称重复 | 保存时校验，提示「保险公司名称已存在」 |
| 排序值重复 | 保存时校验，提示「排序值已被使用，请重新输入」 |
| 排序值超出范围 | 输入限制1~9999，超出时提示「排序值范围为1-9999」 |
| 最后一个线上保司被下架 | 提示「至少需要保留1个上架的线上保险公司」 |
| 最后一个线下保司被下架 | 提示「至少需要保留1个上架的线下保险公司」 |
| logo上传失败 | 提示上传失败原因（格式不符/大小超限/网络异常） |
| 下架保司在前端的历史展示 | 已产生的投保记录中的保司名称正常显示，不受影响 |
| 前端读取保司列表 | 仅返回status=1的保司，按sort_order升序排列 |

## 八、状态变化

### 8.1 保司状态

| 状态 | 说明 | 前端表现 |
|------|------|---------|
| 上架 | 保司正常展示 | 用户在投保/上传保单时可见可选 |
| 下架 | 保司隐藏 | 用户在前端不可见，不可选择 |

### 8.2 类型状态

| 类型 | 对接方式 | 数量特点 |
|------|---------|---------|
| 线上 | 系统对接 | 数量较少 |
| 线下 | 无系统对接 | 数量较多 |
