# 智慧大屏备案报表导出功能设计

---

**设计日期**：2026-06-01

**文档状态**：🟡 AI草稿，待人工确认

**文档定位**：定义智慧大屏场景下的“备案报表导出”能力，聚焦当前已上线的备案相关数据导出与汇报留档，不包含尚未上线的巡查数据。

**适用范围**：青浦区小型工程安全纳管系统 - 智慧大屏 / 备案数据

**设计原则**：
- 只基于当前已上线的备案能力设计，不把巡查、隐患闭环、整改率等未上线指标纳入一期范围。
- 导出能力优先服务管理层汇报、备案明细分析和数据留档。
- 与现有大屏数据导出规则保持一致：权限范围内导出、自动脱敏、支持 Excel / PDF。

## 一、背景与目标

### 1.1 背景

当前系统已上线备案相关模块，智慧大屏也已上线，但巡查相关能力尚未上线。现阶段如果直接做“全量大屏报表导出”，会遇到两个问题：

1. 导出范围不完整，巡查指标无法真实填充。
2. 大屏与报表口径容易混淆，用户会误以为巡查数据已上线。

因此，本期建议将功能收敛为“**备案报表导出**”，先把已上线备案数据的导出、汇总、留档能力做扎实。

### 1.2 目标

- 支持备案数据按条件导出，满足管理层日常汇报和人工分析。
- 支持从智慧大屏或备案列表进入导出流程，降低查找成本。
- 支持 Excel 明细导出和 PDF 汇总导出两种形态。
- 支持导出范围控制、脱敏、留痕和文件保存。

### 1.3 非目标

- 不在一期包含巡查统揽、隐患整改、巡查覆盖率等指标。
- 不在一期做复杂专题分析、自动生成 AI 报告、订阅推送。
- 不在一期做大屏全量指标导出，避免出现空指标或口径不一致。

## 二、功能定位

### 2.1 功能一句话定义

在智慧大屏和备案业务页面中，为已上线的备案数据提供按条件筛选后的 Excel / PDF 导出能力，支持管理层汇报、明细分析和留档。

### 2.2 面向用户

| 用户角色 | 使用目的 | 主要场景 |
|---|---|---|
| 区级领导 | 查看备案总体情况并形成汇报材料 | 大屏查看后导出汇总报表 |
| 街镇管理人员 | 查看本辖区备案进度与明细 | 本街镇备案列表导出 |
| 备案审核人员 | 快速提取待审、已审和驳回数据 | 备案审核列表导出 |
| 系统管理员 | 用于运维、统计和核查 | 按权限导出全区或本辖区数据 |

### 2.3 核心价值

- **对管理层**：快速输出备案概览和结构分析，减少手工整理。
- **对业务人员**：快速导出筛选结果，减少重复查询。
- **对系统**：为后续巡查模块上线预留报表框架和口径。

## 三、功能范围

### 3.1 一期范围

一期只做备案相关导出，建议覆盖以下内容：

1. 备案汇总报表导出
   - 全区 / 街镇维度备案总量
   - 审核状态分布
   - 工程类型分布
   - 时间趋势
2. 备案明细导出
   - 工程名称、地址、类型、街镇
   - 备案状态、审核状态
   - 提交时间、审核时间、通过/驳回时间
3. 审核工作导出
   - 待审、已通过、已驳回清单
   - 驳回原因摘要
4. 大屏汇总导出
   - 从智慧大屏当前筛选条件直接导出备案汇总数据

### 3.2 二期预留

二期在巡查模块上线后再扩展以下内容：

- 巡查覆盖率
- 隐患整改率
- 逾期整改率
- 巡查专题分析报表

### 3.3 明确不做

- 巡查数据导出
- 风险趋势综合分析
- AI 自动生成专题报告
- 订阅式定时推送

## 四、入口与页面

### 4.1 入口设计

| 入口位置 | 入口名称 | 说明 |
|---|---|---|
| 智慧大屏首页 | 导出备案报表 | 面向管理层的汇总导出入口 |
| 备案列表页 | 导出当前筛选结果 | 面向业务人员的明细导出入口 |
| 备案审核列表页 | 导出审核清单 | 面向审核人员的工作导出入口 |
| 备案详情页 | 导出单条备案详情 | 用于单条数据留档或沟通 |

### 4.2 交互入口建议

- 智慧大屏首页右上角增加“导出备案报表”按钮。
- 备案列表页保留“导出”按钮，名称改为“导出当前筛选结果”。
- 审核列表页增加“导出审核清单”。
- 单条详情页增加“导出详情 PDF”。

### 4.3 推荐优先级

1. 备案列表导出
2. 大屏备案汇总导出
3. 审核清单导出
4. 单条详情 PDF 导出

## 五、导出内容

### 5.1 备案汇总报表

适用于大屏和管理汇报场景。

**建议字段**
- 统计范围
- 导出时间
- 备案总量
- 待填报数量
- 待审核数量
- 已通过数量
- 已驳回数量
- 工程类型分布
- 街镇分布
- 最近 7 天 / 30 天变化趋势

**建议展示形式**
- PDF：适合汇报、打印、留档
- Excel：适合二次分析

### 5.2 备案明细报表

适用于业务分析和数据核查。

**建议字段**
- 工程名称
- 工程地址
- 街镇 / 社区
- 工程类型
- 备案状态
- 审核状态
- 提交时间
- 审核时间
- 通过时间
- 驳回原因

### 5.3 审核清单

适用于审核人员批量查看。

**建议字段**
- 工程名称
- 申请单位
- 当前状态
- 审核人
- 最新处理时间
- 驳回原因摘要

### 5.4 单条详情 PDF

适用于留档和外部传阅。

**建议内容**
- 工程基础信息
- 申报主体信息
- 审核记录摘要
- 备案结果
- 二维码或关联编号

## 六、核心流程

### 6.1 导出流程

```text
用户进入大屏/备案列表
  → 选择筛选条件
  → 点击导出
  → 系统校验权限与范围
  → 生成导出文件
  → 返回下载链接或文件预览
```

### 6.2 交互流程

1. 用户选择导出入口。
2. 系统展示可导出的报表类型。
3. 用户选择时间范围、街镇、工程类型、状态等筛选条件。
4. 系统校验用户权限范围。
5. 系统生成 Excel 或 PDF 文件。
6. 用户下载文件或在线预览。

### 6.3 大屏导出流程

1. 用户在智慧大屏首页点击“导出备案报表”。
2. 系统默认带入当前筛选条件。
3. 用户确认导出范围。
4. 系统生成备案汇总报表。
5. 用户下载 PDF 或 Excel。

## 七、业务规则

### 7.1 导出范围规则

- 用户只能导出自己权限范围内的数据。
- 区级用户可导出全区数据。
- 街镇用户仅可导出本街镇数据。
- 单条详情导出时也必须先校验权限。

### 7.2 口径规则

- 一期报表只统计备案业务数据。
- 不引用巡查、隐患、整改等未上线指标。
- 同一导出文件中的指标口径必须统一，不能混用备案列表口径和大屏口径。

### 7.3 脱敏规则

以下字段导出时默认脱敏：

| 字段 | 脱敏规则 | 示例 |
|---|---|---|
| 手机号 | 保留前 3 位和后 4 位 | 138****1234 |
| 身份证号 | 保留前 6 位和后 4 位 | 310101********1234 |
| 银行卡号 | 仅保留后 4 位 | ****1234 |
| 个人姓名 | 保留首字，其余用 * 替代 | 张* |

### 7.4 数据保留规则

- 导出文件支持下载后再次取回。
- 建议文件保存 7 天至 15 天，具体时长由后端存储策略决定。
- 超时后文件自动失效，避免长期占用存储空间。

### 7.5 操作留痕

每次导出都需要记录：

- 导出人
- 导出时间
- 导出入口
- 筛选条件
- 导出范围
- 导出文件类型

## 九、技术建议

### 9.1 生成方式

- 小文件量：同步生成，直接返回下载链接。
- 大文件量：异步生成，任务完成后通知用户下载。

### 9.2 数据来源

一期建议只接入以下数据源：

- 备案管理
- 备案审核
- 备案详情

### 9.3 接口建议

建议后端提供统一导出接口，按报表类型区分模板：

- `备案汇总报表`
- `备案明细报表`
- `审核清单`
- `单条详情PDF`

### 9.4 文件格式建议

| 类型 | 格式 | 用途 |
|---|---|---|
| 汇总报表 | PDF | 汇报留档 |
| 明细报表 | Excel | 二次分析 |
| 审核清单 | Excel | 审核工作 |
| 单条详情 | PDF | 存档传阅 |

## 十、验收标准

### 10.1 功能验收

- 能从智慧大屏导出备案汇总报表。
- 能从备案列表导出当前筛选结果。
- 能从审核列表导出审核清单。
- 能导出单条备案详情 PDF。
- 巡查相关字段不会出现在一期导出中。

### 10.2 规则验收

- 导出范围与用户权限一致。
- 导出数据自动脱敏。
- 导出文件带有时间、范围和导出人信息。
- 导出结果可下载、可留档。

### 10.3 体验验收

- 导出入口清晰，用户能快速找到。
- 导出过程有明确的进度反馈。
- 大文件导出失败时有重试或重新发起入口。
