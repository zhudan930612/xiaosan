# 智慧大屏备案报表导出功能设计

---

**设计日期**：2026-06-01

**文档状态**：AI 草稿，待确认

**目标模块**：智慧大屏 / 备案数据导出

**适用范围**：仅覆盖已上线的备案相关模块，不包含巡查相关数据与指标

---

## 一、背景与目标

当前系统已经上线备案管理、备案审核和智慧大屏模块，业务方希望在大屏场景下补充“报表导出”能力，用于日常汇报、留档归档和管理层查看。

本次设计从零开始，不复用那份 `28_智慧大屏备案报表导出功能设计` 的结构与结论，仅基于当前已上线能力重新定义范围：

- 只导出备案相关数据
- 只使用当前已经上线并可查询到的数据
- 不把巡查、隐患、整改等未上线能力纳入一期
- 以“管理层可直接使用的报表”作为核心目标，而不是单纯导出原始列表

## 二、设计原则

- 以现有能力为边界，先做能稳定落地的备案报表导出
- 导出结果要和当前大屏筛选条件一致，避免“看见一套、导出一套”
- 默认脱敏，敏感信息不在报表中裸露展示
- 同一套导出引擎同时服务大屏、备案列表和备案详情，避免后续重复建设
- 为后续巡查模块预留扩展位，但本期不展示、不接入

## 三、方案对比与推荐

### 方案 A：仅导出当前大屏截图 PDF

特点：
- 实现快
- 内容直观
- 适合临时汇报

问题：
- 只有图片，没有结构化数据
- 无法用于二次分析
- 不能支撑单条备案留档

### 方案 B：汇总报表 PDF + 明细 Excel + 单条详情 PDF

特点：
- 汇总报表适合汇报和留档
- 明细 Excel 适合二次分析和核对
- 单条详情 PDF 适合备案留存

优点：
- 同时满足管理层和业务人员
- 与现有 AI 报表模板能力一致
- 后续接巡查时只需增加报表类型

### 方案 C：开放自定义报表模板

特点：
- 灵活
- 可扩展

问题：
- 配置复杂
- 需求不稳定时容易反复调整
- 一期不适合直接上

### 推荐方案

推荐采用 **方案 B**。它是当前最稳妥的落地方式，也最符合“只基于已上线备案能力”的约束。

## 四、功能定义

### 4.1 一句话定义

在智慧大屏、备案列表和备案详情中，基于当前用户权限与当前筛选条件，导出备案汇总报表、备案明细表和单条备案详情 PDF，用于汇报、留档和二次分析。

### 4.2 用户角色

| 角色 | 使用目的 | 主要场景 |
|---|---|---|
| 区级管理人员 | 查看全区备案概况并形成汇报材料 | 大屏汇总导出 |
| 街镇管理人员 | 查看本辖区备案进展与明细 | 大屏/列表导出 |
| 备案审核人员 | 快速整理待审、已审、驳回记录 | 审核清单导出 |
| 系统管理员 | 运维、留档、核查 | 按权限导出 |

### 4.3 入口设计

建议设置 3 个导出入口：

1. 智慧大屏右上角“导出备案报表”
2. 备案列表页右上角“导出当前筛选结果”
3. 备案详情页右上角“导出详情 PDF”

### 4.4 核心流程

```text
用户进入大屏/列表/详情
  -> 选择当前筛选条件
  -> 点击导出
  -> 系统校验权限与数据范围
  -> 生成对应格式文件
  -> 返回下载链接或文件预览
```

## 五、详细设计

### 5.1 导出类型

#### 5.1.1 备案汇总报表

适用入口：
- 智慧大屏

输出格式：
- PDF
- Excel（可选）

内容定位：
- 面向管理层汇报
- 体现当前筛选范围内的备案总览
- 只展示备案数据，不带巡查指标

#### 5.1.2 备案明细报表

适用入口：
- 智慧大屏
- 备案列表

输出格式：
- Excel

内容定位：
- 面向业务核对和二次分析
- 展示当前筛选范围内的备案列表
- 默认脱敏

#### 5.1.3 单条备案详情 PDF

适用入口：
- 备案详情页

输出格式：
- PDF

内容定位：
- 面向留档和外部传阅
- 按单条备案完整输出
- 默认脱敏

### 5.2 大屏汇总报表内容

建议包含以下内容：

1. 报表标题
2. 生成时间
3. 筛选范围
4. 备案总量
5. 待填报、待审核、已通过、已驳回数量
6. 各街镇备案分布
7. 各工程类型分布
8. 近 7 天或当前周期趋势
9. 本期重点问题与建议

#### 建议字段

| 类别 | 字段 |
|---|---|
| 基础信息 | 报表名称、生成时间、统计范围、导出人 |
| 核心指标 | 备案总量、待填报数、待审核数、已通过数、已驳回数 |
| 结构分布 | 街镇分布、工程类型分布、状态分布 |
| 趋势信息 | 近 7 天新增备案、近 30 天新增备案、提交趋势 |
| 管理建议 | 重点街镇、待审堆积、临期风险、驳回原因摘要 |

### 5.3 明细报表内容

建议字段：

- 工程名称
- 工程地址
- 街镇
- 社区
- 工程类型
- 备案状态
- 审核状态
- 提交时间
- 审核时间
- 通过时间
- 驳回原因摘要
- 申报单位
- 联系人
- 联系电话

### 5.4 单条详情 PDF 内容

建议结构：

1. 基础信息
2. 申报主体信息
3. 审核记录摘要
4. 备案结果
5. 二维码或关联编号

### 5.5 筛选条件

导出时默认继承当前页面筛选条件，用户无需重新选择。

可继承的筛选条件建议包括：

- 时间范围
- 街镇
- 社区
- 工程类型
- 备案状态
- 审核状态
- 关键字

### 5.6 业务规则

#### 5.6.1 数据范围规则

- 仅导出当前用户有权限查看的数据
- 区级用户可导出授权范围内全部数据
- 街镇用户仅可导出本辖区数据
- 本期不包含巡查相关数据

#### 5.6.2 状态规则

导出数据按当前备案模块可见状态输出：

| 状态 | 含义 |
|---|---|
| 待填报 | 已创建草稿，未提交 |
| 待审核 | 已提交，等待审核 |
| 已通过 | 审核通过，进入留档 |
| 已驳回 | 审核不通过，需要修改后重提 |

#### 5.6.3 脱敏规则

默认脱敏字段：

| 字段 | 脱敏方式 | 示例 |
|---|---|---|
| 手机号 | 保留前 3 位和后 4 位 | 138****1234 |
| 身份证号 | 保留前 6 位和后 4 位 | 310101********1234 |
| 联系人姓名 | 保留首字，其余用 * 替代 | 张* |

说明：
- 汇总报表默认脱敏
- 明细报表默认脱敏
- 只有具备额外权限的管理员角色才可关闭脱敏

#### 5.6.4 文件命名规则

建议格式：

```text
智慧大屏备案汇总报表_20260601_全区_导出人张三.pdf
备案明细报表_20260601_夏阳街道_导出人张三.xlsx
备案详情_工程名称_20260601.pdf
```

#### 5.6.5 文件保存规则

- 文件生成后支持立即下载
- 建议后端保留 7 到 15 天
- 到期自动清理

#### 5.6.6 审计规则

每次导出都要记录：

- 导出人
- 导出时间
- 导出入口
- 导出范围
- 导出格式
- 导出结果

### 5.7 异常处理

| 场景 | 处理方式 |
|---|---|
| 没有可导出数据 | 仍可生成空报表，明确提示“暂无数据” |
| 权限不足 | 禁止导出，并提示当前仅可导出授权范围内数据 |
| 文件生成失败 | 提示重试，必要时异步生成 |
| 数据量过大 | 转异步任务，完成后通知用户下载 |
| 脱敏失败 | 默认阻断导出，不允许裸数据直接生成 |

### 5.8 大屏页面结构

#### 大屏导出弹窗

建议包含：

- 导出类型
- 时间范围
- 街镇范围
- 文件格式
- 是否脱敏
- 备注说明

#### 导出结果页

建议包含：

- 生成中
- 生成成功
- 生成失败
- 重新生成
- 直接下载

## 六、与现有模块的关系

### 6.1 与备案管理模块

- 明细导出主要复用备案管理的列表字段
- 单条详情 PDF 主要复用备案详情页字段

### 6.2 与备案审核模块

- 审核状态、审核时间、驳回原因可以直接进入报表
- 审核清单导出可作为独立入口，但仍属于备案数据

### 6.3 与智慧大屏模块

- 大屏汇总报表默认继承当前筛选条件
- 导出结果和大屏卡片口径保持一致
- 只导出备案，不纳入巡查

### 6.4 与后续巡查模块

- 本期不接入巡查指标
- 预留报表类型扩展位
- 后续巡查上线后，可复用同一导出引擎新增巡查报表

---

## 七、验收标准

1. 能从智慧大屏导出备案汇总报表
2. 能从备案列表导出当前筛选结果
3. 能从备案详情导出单条 PDF
4. 导出结果与当前用户权限范围一致
5. 导出文件默认脱敏
6. 导出文件带有时间、范围、导出人信息
7. 大数据量场景可转异步处理
8. 导出内容不包含巡查数据
9. 失败场景有明确提示
10. 导出操作有审计记录

## 八、待确认事项

以下事项建议在开发前确认：

1. 一期是否必须同时支持 PDF 和 Excel 汇总报表
2. 明细报表是否允许关闭脱敏
3. 文件保留期限最终定为 7 天还是 15 天
4. 导出是否需要支持历史筛选条件复用
5. 后续巡查上线后是否复用同一导出入口
