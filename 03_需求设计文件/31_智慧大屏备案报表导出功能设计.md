## 一、功能概述

### 1.1 背景与目标

智慧大屏目前已上线备案统揽、实时统揽、备案申请记录等模块，积累了完整的备案数据。当前系统支持数据可视化展示，但**缺少报表导出能力**，导致：

- 区级领导需要定期汇报时，只能手工截图或抄录数据
- 日常监管分析时，无法将数据导出到本地进行二次加工
- 街镇考核时，缺乏标准化的数据报表作为考核依据

本功能旨在为智慧大屏增加备案数据报表导出能力，支持**Excel（日常分析）**和**PDF（领导汇报）**两种格式，覆盖综合统计、区域对比、明细清单、驳回分析、趋势分析5类报表。

### 1.2 使用场景

| 场景 | 用户角色 | 使用频率 | 导出格式 | 典型用途 |
|------|---------|---------|---------|---------|
| 向区级领导定期汇报 | 区级管理人员 | 每周/每月 | PDF | 周/月总结汇报 |
| 街镇考核排名 | 区级管理人员 | 每月 | PDF/Excel | 考核通报 |
| 备案数据核查 | 街镇管理人员 | 按需 | Excel | 日常监管、数据核对 |
| 驳回原因分析 | 区级/街镇管理人员 | 每月 | Excel | 提升备案质量 |
| 趋势分析决策 | 区级管理人员 | 每季度 | PDF/Excel | 辅助决策 |

### 1.3 设计原则

1. **一键导出**：在首页备案统揽卡片上直接触发，减少操作路径
2. **灵活配置**：支持时间范围、街道、工程类型、格式多维度筛选
3. **权限隔离**：严格按用户数据权限范围导出，不可越权
4. **脱敏合规**：敏感字段自动脱敏，符合数据安全要求
5. **审计留痕**：每次导出记录操作日志，便于追溯

### 1.4 适用范围

- **使用端**：PC端智慧大屏
- **使用角色**：区级管理人员、街镇管理人员
- **数据范围**：仅备案相关数据（巡查模块未上线，本期不包含巡查数据）

## 三、详细功能设计

### 3.1 首页备案统揽卡片导出入口

#### 一句话定义
在智慧大屏主控制台的"备案统揽"卡片右上角增加导出图标按钮，作为备案报表导出的统一入口。

#### 触发条件
- 用户已登录且拥有智慧大屏访问权限
- 用户角色为区级管理人员或街镇管理人员
- 用户当前位于智慧大屏首页（主控制台）

#### 页面位置
- **主控制台 → 备案统揽卡片 → 右上角**
- 图标样式：下载/导出 SVG 图标（线性描边，20px，颜色 `#409EFF`）
- 图标位置：卡片标题行右侧，与"刷新数据"按钮并列或在其右侧

#### 核心流程

```
用户进入智慧大屏首页
  → 查看备案统揽卡片
  → 点击卡片右上角【导出】图标
  → 弹出"导出配置弹窗"
  → 配置导出参数
  → 点击【确认导出】
  → 系统校验权限和参数
  → 生成报表文件
  → 浏览器自动下载
  → 记录导出日志
```

#### 业务规则

1. **入口可见性规则**：
   - 仅"备案统揽"卡片显示导出入口，其他三张卡片（实时统揽、巡查统揽、组织统揽）不显示
   - 理由：备案数据是当前唯一有实际数据的模块，巡查数据均为0，导出无意义

2. **图标交互规则**：
   - 鼠标悬浮：显示 Tooltip "导出备案报表"
   - 点击：弹出导出配置弹窗
   - 禁用状态：数据加载中或用户无导出权限时，图标置灰不可点击

3. **权限校验规则**：
   - 区级管理人员：可见导出入口，可导出全区数据
   - 街镇管理人员：可见导出入口，仅能导出管辖区域内数据
   - 无大屏权限用户：不可见导出入口

#### 数据规则

1. **数据实时性**：导出数据以点击【确认导出】时的系统数据为准，非弹窗打开时的快照
2. **数据范围**：受当前页面筛选条件（工程类型、街道）的叠加影响
   - 若用户在首页已选择"小型工程"+"夏阳街道"，则导出数据自动叠加这两个筛选条件
   - 弹窗内可进一步修改筛选条件，覆盖首页的当前筛选

#### 边界情况

| 场景 | 处理方案 |
|------|---------|
| 备案数据为空（0条） | 弹窗内【确认导出】按钮置灰，提示"当前筛选条件下暂无备案数据" |
| 用户无导出权限 | 图标不显示或置灰，悬浮提示"无导出权限" |
| 数据量过大（>10000条） | 转为异步任务，提示"数据量较大，正在后台生成，请稍后到导出记录中下载" |
| 网络异常导致导出失败 | 提示"导出失败，请检查网络后重试"，记录失败日志 |
| 同时多次点击导出 | 防抖处理，2秒内重复点击无效，防止重复生成 |

#### 状态变化

| 状态 | 触发条件 | 表现 |
|------|---------|------|
| 默认态 | 页面加载完成 | 导出图标正常显示，可点击 |
| 悬浮态 | 鼠标hover图标 | Tooltip显示"导出备案报表"，图标颜色加深 |
| 加载态 | 点击导出后数据校验中 | 图标显示loading动画，不可重复点击 |
| 禁用态 | 无权限或数据为空 | 图标置灰，不可点击 |

### 3.2 导出配置弹窗

#### 一句话定义
点击首页备案统揽卡片的导出按钮后弹出的配置面板，用于选择报表类型、时间范围、街道、工程类型和导出格式。

#### 触发条件
- 用户点击首页备案统揽卡片的【导出】图标

#### 页面位置
- 屏幕中央弹窗，蒙层遮罩背景（半透明黑色 `rgba(0,0,0,0.5)`）
- 弹窗宽度：600px，最大高度：80vh，超出可滚动

#### 核心流程

```
弹窗打开
  → 默认填充当前首页筛选条件（工程类型、街道）
  → 用户选择/修改配置项
  → 实时校验配置合法性
  → 点击【确认导出】
  → 关闭弹窗，开始生成报表
  → 显示导出进度提示
  → 完成后自动下载
```

#### 业务规则

**配置项定义**：

| 配置项 | 类型 | 必填 | 默认值 | 选项说明 |
|--------|------|------|--------|---------|
| 报表类型 | 单选 | 是 | 综合统计报表 | 5个选项互斥 |
| 时间范围 | 组合 | 是 | 本周 | 快速选项或自定义日期 |
| 街道筛选 | 多选 | 是 | 当前选中街道 | 区级可多选，街镇锁定为管辖街道 |
| 工程类型 | 多选 | 是 | 当前选中类型 | 全部/小型工程/住宅装修/零星作业 |
| 导出格式 | 单选 | 是 | PDF | PDF/Excel/同时导出两种格式 |

**各报表类型支持的格式**：

| 报表类型 | 支持PDF | 支持Excel | 默认格式 |
|---------|---------|-----------|---------|
| 综合统计报表 | ✅ | ✅ | PDF |
| 区域对比报表 | ✅ | ✅ | PDF |
| 备案明细清单 | ❌ | ✅ | Excel |
| 驳回分析报表 | ❌ | ✅ | Excel |
| 趋势分析报表 | ✅ | ✅ | Excel |

**选择格式时的联动规则**：
- 当选中"备案明细清单"或"驳回分析报表"时，PDF选项自动禁用，仅允许选择Excel
- 当选中"综合统计报表"时，默认选中PDF，允许切换到Excel
- 选择"同时导出两种格式"时，若当前报表类型不支持PDF，则自动降级为仅Excel

**时间范围规则**：
- 快速选项：今日、本周、近七日、本月、本年
- 自定义：日期范围选择器，起止日期均必选，最大跨度365天
- 时间定义同大屏现有口径（见[智慧大屏模块.md](../02_已上线功能描述/智慧大屏模块.md)3.2.1节）

**街道筛选规则**：
- 区级管理人员：下拉多选框，可选全部街道或任意组合，默认"全部街道"
- 街镇管理人员：下拉框锁定为管辖街道，不可修改，显示提示"仅可导出管辖区域数据"

**工程类型筛选规则**：
- 复选框组：全部（全选）、小型工程、住宅装修、零星作业
- 与首页当前选中的工程类型联动：弹窗打开时默认与首页一致
- "全部"选中时，其他项自动全选；取消任一子项，"全部"自动取消

#### 数据规则

1. **配置项联动校验**：
   - 时间范围结束日期 ≥ 开始日期，否则提示"结束日期不能早于开始日期"
   - 自定义日期跨度 > 365天时，提示"时间范围最大支持365天"
   - 街道和工程类型至少各选一项，否则【确认导出】置灰

2. **数据量预估**：
   - 弹窗底部实时显示预估数据条数："预计导出约 N 条数据"
   - 预估逻辑：根据当前筛选条件查询符合条件的记录数（ COUNT 查询，不拉取明细）

#### 边界情况

| 场景 | 处理方案 |
|------|---------|
| 预估数据量为0 | 【确认导出】置灰，提示"当前筛选条件下暂无数据" |
| 预估数据量 > 5000 | 显示警告"数据量较大，导出可能需要较长时间" |
| 预估数据量 > 10000 | 提示"数据量过大，建议缩小时间范围或分批次导出"，但仍允许导出（走异步任务） |
| 用户快速切换报表类型 | 保留其他配置项不变，仅重置格式选项的可用状态 |
| 弹窗打开时首页正在加载数据 | 配置项默认值显示"加载中..."，数据加载完成后自动填充 |

#### 状态变化

| 状态 | 触发条件 | 表现 |
|------|---------|------|
| 初始态 | 弹窗打开 | 默认填充当前首页筛选条件，【确认导出】按钮根据数据量判断是否可用 |
| 编辑态 | 用户修改配置项 | 实时校验，预估数据量动态更新 |
| 校验失败态 | 配置项不合法 | 对应字段标红，显示错误提示，【确认导出】置灰 |
| 导出中态 | 点击【确认导出】 | 弹窗关闭，显示导出进度提示（Toast或进度条） |

### 3.3 备案综合统计报表导出

#### 一句话定义
以正式公文格式生成备案核心指标的汇总统计报表，面向区级领导汇报场景，包含总备案量、新增量、状态分布、街镇排名、工程类型分布等核心数据。

#### 触发条件
- 用户在导出配置弹窗中选择"综合统计报表"类型，确认导出

#### 页面位置
- 不单独页面，导出后生成文件直接下载

#### 核心流程

```
用户选择"综合统计报表" + 时间范围 + 筛选条件 + 格式
  → 系统按模板组装报表数据
  → 查询统计数据（备案总量、新增量、各状态数量、各街镇数量、各类型数量）
  → 按报表模板渲染
  → 生成PDF/Excel文件
  → 浏览器下载
```

#### 业务规则

**报表结构（PDF版）**：

| 区块 | 内容 | 说明 |
|------|------|------|
| 报表头 | 标题"青浦区小型工程备案统计报表"+统计周期+制表单位+制表日期 | 正式公文头部 |
| 概览区 | 5个核心指标卡片（横向排列） | 总备案数、本周新增、待审核数、已驳回数、已通过数 |
| 状态分布区 | 备案状态环形图/表格 | 待填报、待审核、待确认、已驳回、已通过 |
| 街镇排名区 | 各街镇备案数量排名表格 | 11个街镇按备案数倒序排列 |
| 工程类型分布区 | 工程类型占比表格 | 三大类及细分类型占比 |
| 风险类型分布区 | 风险类型占比表格 | 动火、拆改、高空、用电等 |
| 数据说明区 | 页脚注明数据来源、统计口径、制表人 | 增加报表权威性 |

**报表结构（Excel版）**：

| Sheet名称 | 内容 |
|-----------|------|
| 概览 | 核心指标汇总、状态分布 |
| 街镇排名 | 各街镇备案数据明细 |
| 工程类型 | 各类型备案数量及占比 |
| 风险类型 | 各风险类型数量及占比 |
| 数据说明 | 统计口径说明、制表信息 |

**核心指标计算口径**（同大屏现有口径）：

| 指标 | 计算口径 |
|------|---------|
| 总备案数量 | 历史累计已通过审核的备案总数（按最新有效备案去重） |
| 本周新增备案 | 本周内首次提交审核的工程数（含待审核、已通过、已驳回） |
| 待审核备案 | 当前状态为"待审核"的工程数 |
| 已驳回备案 | 当前状态为"已驳回"的工程数 |
| 已通过备案 | 历史累计已通过审核的备案数 |

#### 数据规则

1. **数据来源**：
   - 备案基础数据：备案信息表
   - 状态统计：按备案状态字段分组COUNT
   - 街镇数据：按所属街道字段分组COUNT
   - 工程类型：按工程类型字段分组COUNT
   - 风险类型：按作业类型字段映射到风险类型后分组COUNT

2. **时间过滤**：
   - "新增备案"按首次提交审核时间过滤
   - "已通过"按审核通过时间过滤
   - "待审核/待填报/已驳回"按当前状态统计（不受时间范围影响，显示当前在库数量）
   - ⚠️ 注意：状态分布中的"待填报/待审核/待确认/已驳回"统计的是当前在库数量，与"已通过"的时间口径不同

3. **排序规则**：
   - 街镇排名：按备案数量倒序，数量相同按街道名称拼音正序
   - 工程类型：按类型编码正序
   - 风险类型：按数量倒序，数量相同按风险类型名称正序

#### 边界情况

| 场景 | 处理方案 |
|------|---------|
| 某街道无备案数据 | 表格中该街道显示为0，仍保留在列表中 |
| 某工程类型无数据 | 表格中显示为0，仍保留在列表中 |
| 某风险类型无数据 | 不显示该风险类型行（避免报表冗长） |
| 时间范围内无新增备案 | 新增指标显示0，环比显示"--" |

#### 状态变化

报表文件本身无状态变化，生成后即下载。

### 3.4 区域备案对比报表导出

#### 一句话定义
以表格形式展示11个街镇的备案数据横向对比，支持区级领导进行街镇考核和区域差异分析。

#### 触发条件
- 用户在导出配置弹窗中选择"区域对比报表"类型，确认导出

#### 页面位置
- 不单独页面，导出后生成文件直接下载

#### 核心流程

```
用户选择"区域对比报表" + 时间范围 + 筛选条件 + 格式
  → 查询11个街镇的备案数据
  → 计算各街镇的备案量、驳回率、审核时长等指标
  → 按对比报表模板渲染
  → 生成PDF/Excel文件
```

#### 一句话定义
展示备案数量随时间变化的趋势数据，支持按日/周/月维度查看备案量变化，辅助管理层进行趋势判断和决策。

## 六、导出任务管理

### 6.1 同步导出 vs 异步导出

| 场景 | 导出方式 | 用户体验 |
|------|---------|---------|
| 数据量 ≤ 5000条 | 同步导出 | 点击后数秒内完成，直接下载 |
| 数据量 > 5000条 | 异步导出 | 提示"正在生成报表"，后台处理完成后通知用户下载 |

### 6.2 异步任务流程

```
点击【确认导出】
  → 判断数据量
  → 数据量 > 5000条
  → 弹窗提示"数据量较大，报表正在后台生成中..."
  → 创建异步导出任务
  → 用户可关闭提示，继续操作大屏
  → 后台生成报表文件
  → 生成完成后，页面右上角显示通知"报表已生成，点击下载"
  → 用户点击通知下载文件
  → 超过24小时未下载，自动清理文件
```

### 6.3 导出记录查询
