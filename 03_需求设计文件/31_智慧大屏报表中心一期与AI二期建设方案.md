## 一、方案概述

### 1.1 背景

当前系统已经具备智慧大屏展示能力，但报表相关文档存在三类不一致：

1. 同一批巡查指标在不同文档中的对外状态表述不一致。
2. 旧方案以“备案报表导出”为主，无法覆盖当前用户希望一期正式发布的全量大屏指标。
3. AI 报表设想较多，但当前 AI 算力与接口条件尚未稳定，不适合直接承担一期正式报表职责。

因此，本方案将报表能力拆为两期：

- **一期**：先做“正式报表底座”，确保数据口径、导出能力、打印模板、权限留痕稳定可发布。
- **二期**：在一期底座上叠加 AI 选报表、AI 写摘要、AI 专题分析等能力，负责提效和增值，不负责定义口径。

### 1.2 一句话定义

建设一个统一的“报表中心/导出中心”，把大屏现有正式指标沉淀为 `Excel 明细报表`、`Excel 汇总报表`、`A4 汇报报表` 三类稳定输出，并在二期扩展为可对话、可解读、可专题分析的 AI 报表助手。

### 1.3 核心价值

| 面向角色 | 一期价值 | 二期价值 |
|---|---|---|
| 区级管理人员 | 快速导出正式汇报材料，减少手工拼表 | 自动生成摘要、风险提示和专题简报 |
| 街镇管理人员 | 快速导出辖区明细、周报、月报 | 自动生成街镇执行总结和下周建议 |
| 业务审核/巡查人员 | 按筛选条件导出工作清单，便于核数与跟进 | 通过自然语言直接发起报表任务 |
| 产品/开发团队 | 固化指标口径与模板边界，减少反复返工 | 在稳定底座之上安全接入 AI 能力 |

## 二、一期与二期总体拆分

### 2.1 分期边界

| 阶段 | 定位 | 核心目标 | 不做内容 |
|---|---|---|---|
| 一期 | 正式报表底座 | 全量大屏正式指标可导出、可打印、可留痕 | AI 自动结论、智能问答、订阅推送 |
| 二期 | AI 报表助手 | 自动选模板、自动写摘要、自动专题分析 | 重新定义一期指标口径 |

### 2.2 一期建设顺序

1. **第 0 步：口径冻结**
   - 先统一“哪些指标允许进入正式报表”。
   - 输出《报表指标口径台账》。
   - 对文档冲突做显式登记，不把冲突口径直接写成事实。

2. **第 1 步：Excel 能力**
   - 汇总导出
   - 明细导出
   - 审核/巡查工作清单导出
   - 脱敏、权限、留痕、异步任务、失败重试

3. **第 2 步：A4 固定模板**
   - 区级周/月汇总
   - 街镇周/月执行
   - 专项简报

4. **第 3 步：历史管理**
   - 导出记录
   - 文件有效期
   - 再下载
   - 操作审计

### 2.3 二期建设顺序

1. 智能选报表
2. 智能写摘要
3. 智能专题分析
4. 历史报表复用
5. 自动推送与订阅

## 三、一期正式报表底座设计

### 3.1 功能单元一：报表中心入口与能力矩阵

**一句话定义**：在智慧大屏、统揽页和列表页提供统一的报表入口，让用户基于当前筛选条件进入同一套报表中心。

| 要素 | 设计结论 |
|---|---|
| 触发条件 | 用户位于智慧大屏首页、备案统揽页、巡查统揽页、记录列表页或详情页，并具备导出权限 |
| 页面位置 | 大屏右上角、统揽页工具区、列表页操作区、详情页页头操作区 |
| 核心流程 | 进入页面 → 点击“导出/生成报表” → 带入当前筛选条件 → 选择报表类型 → 生成文件/预览 |
| 业务规则 | 一期所有入口共用同一套报表类型、同一套权限校验、同一套导出任务机制 |
| 数据规则 | 入口只传筛选条件，不直接拼接指标结果，避免页面口径和导出口径分离 |
| 边界情况 | 无权限时入口隐藏或禁用；无数据时允许查看空态说明但不生成正式文件 |
| 状态变化 | 默认态 → 加载态 → 可生成态 / 无权限态 / 无数据态 |
| 设计依据 | 来源于智慧大屏统一导出诉求和一期“统一成导出中心”的规划要求 |

纯文本规则：
一期对外固定三类报表：Excel明细报表、Excel汇总报表、A4汇报报表。入口可以分散，但底层模板、任务、脱敏、留痕规则必须统一，不允许某个入口另起一套字段和按钮文案。

### 3.2 功能单元二：报表指标口径冻结机制

**一句话定义**：用一份台账先冻结大屏正式报表可用指标，确保页面展示、Excel 导出、A4 模板三者口径一致。

| 要素 | 设计结论 |
|---|---|
| 触发条件 | 进入一期开发前、每次新增报表指标前、每次修改核心统计口径前 |
| 页面位置 | 不单独做用户页面，作为报表中心后台配置与评审资料存在 |
| 核心流程 | 新增/修改指标 → 填写口径台账 → 评审确认 → 标记状态 → 才允许进入模板 |
| 业务规则 | 未进入台账或台账状态未冻结的指标，不允许进入正式报表模板 |
| 数据规则 | 台账必须包含定义、分子分母、时间口径、去重规则、页面/导出一致性、来源模块、责任人、验收人 |
| 边界情况 | 同名指标在不同页面口径不一致时，必须先拆分命名或统一定义，不能继续沿用模糊名称 |
| 状态变化 | 草稿 → 待确认 → 已冻结 → 需复核 |
| 设计依据 | 来源于当前文档冲突和一期“全部正式发布”的前置条件 |

纯文本规则：
当前已发现的文档冲突包括：产品规划文档中“巡查统揽指标”为“已开发”，而智慧大屏模块文档将相应页面作为“已上线”描述。一期报表建设不直接裁定哪份文档为真，而是在报表侧增加“口径状态”字段，以“是否允许进入正式模板”为唯一准入标准。

### 3.3 功能单元三：Excel 明细报表与 Excel 汇总报表

**一句话定义**：优先建设 Excel 导出能力，满足核数、再加工、留档和跨部门沟通。

| 要素 | 设计结论 |
|---|---|
| 触发条件 | 用户在报表中心选择 Excel 明细报表或 Excel 汇总报表 |
| 页面位置 | 报表中心弹窗/独立页中的“报表类型”选择区 |
| 核心流程 | 选择模板 → 选择周期/街道/工程类型/状态 → 预估数据量 → 生成任务 → 下载 |
| 业务规则 | 明细报表面向列表型数据；汇总报表面向指标、排名、趋势、状态分布 |
| 数据规则 | Excel 明细默认按时间倒序；汇总默认按指标维度成组输出；所有敏感字段走统一脱敏规则 |
| 边界情况 | 大文件走异步生成；失败可重试；超出上限时提示缩小范围 |
| 状态变化 | 待选择 → 参数已确认 → 生成中 → 生成成功 / 生成失败 / 文件过期 |
| 设计依据 | 来源于用户明确选择“Excel 优先” |

纯文本规则：
Excel明细报表优先承接备案记录、巡查记录、隐患整改记录、审核清单、导出历史等列表数据；Excel汇总报表优先承接核心指标、街镇排名、趋势分析、状态分布、TOP5分析等管理类数据。

### 3.4 功能单元四：A4 汇报报表

**一句话定义**：提供适合打印、适合会议汇报、结构稳定的固定版式报表，不依赖 AI 实时生成结论。

| 要素 | 设计结论 |
|---|---|
| 触发条件 | 用户在报表中心选择 A4 汇报报表 |
| 页面位置 | 报表中心“格式与模板”区域 |
| 核心流程 | 选择模板族 → 选择周期和范围 → 生成 A4 预览 → 下载 PDF / 打印 |
| 业务规则 | 一期只落 3 套模板族，不直接实装现有 8 套全部候选模板 |
| 数据规则 | A4 只展示正式冻结指标；页面摘要只写事实性标题和数据，不写 AI 结论 |
| 边界情况 | 无某模块数据时展示“本期无数据”说明，不自动删页导致版式跳动 |
| 状态变化 | 待选模板 → 预览中 → 可打印态 |
| 设计依据 | 来源于当前用户要求“适合打印的简约报表”和现有 A4 模板候选池 |

纯文本规则：
一期模板族与当前候选原型映射如下：
1. 区级周/月汇总：吸收模板02、模板05中的结构。
2. 街镇周/月执行：吸收模板01、模板06中的结构。
3. 专项简报：吸收模板03，并允许后续扩展到模板04、07、08中的局部模块。

### 3.5 功能单元五：导出历史管理与操作审计

**一句话定义**：让所有报表生成结果可追溯、可再下载、可审计。

| 要素 | 设计结论 |
|---|---|
| 触发条件 | 用户成功生成任一报表，或进入“导出历史”页面 |
| 页面位置 | 报表中心二级入口“历史记录” |
| 核心流程 | 生成报表 → 写入任务记录 → 支持查看、再次下载、查看失败原因 |
| 业务规则 | 历史记录只显示权限范围内的报表任务；敏感报表任务也必须保留审计记录 |
| 数据规则 | 保存导出时间、导出人、入口、筛选条件、报表类型、文件格式、结果状态、文件过期时间 |
| 边界情况 | 文件过期后仍保留任务记录，但下载按钮置灰并允许重新发起 |
| 状态变化 | 生成中 → 成功 / 失败 / 已过期 |
| 设计依据 | 来源于一期对“文件有效期、再下载、操作审计”的要求 |

纯文本规则：
历史管理不等于文件永久存储。推荐保存策略为：任务记录长期保留，文件实体短期保留。文件过期后不能继续下载旧文件，但可以基于原筛选条件重新发起一次新任务。

## 四、二期AI报表助手设计

### 4.1 功能单元六：智能选报表

**一句话定义**：用户通过自然语言描述需求，系统自动匹配报表模板、范围、周期和格式。

| 要素 | 设计结论 |
|---|---|
| 触发条件 | 用户进入 AI 助手或报表中心的“智能生成”入口 |
| 页面位置 | 大屏 AI 入口、移动端 AI 报表入口、报表中心内“智能生成” |
| 核心流程 | 输入自然语言 → 解析意图 → 回填模板与筛选参数 → 用户确认 → 调用一期报表生成任务 |
| 业务规则 | 二期 AI 只允许选择一期已存在模板，不能临时拼装一套未冻结口径的新报表 |
| 数据规则 | 意图识别结果应回填到报表任务参数中，不允许直接跳过参数层调用底层数据 |
| 边界情况 | 意图不明确时返回候选模板；字段缺失时要求用户补充 |
| 状态变化 | 待输入 → 已识别 → 待确认 → 已生成 |
| 设计依据 | 来源于 AI 报表对话式生成诉求 |

### 4.2 功能单元七：智能写摘要

**一句话定义**：基于已生成报表输出一句话总结、风险提示和下步建议。

| 要素 | 设计结论 |
|---|---|
| 触发条件 | 报表生成成功，且用户点击“生成摘要”或配置自动摘要 |
| 页面位置 | 报表详情页、AI 助手侧边栏 |
| 核心流程 | 读取报表结果 → 调用 AI 解读接口 → 输出摘要卡片 |
| 业务规则 | 摘要必须引用已存在指标；不得生成报表中不存在的数字或结论 |
| 数据规则 | 摘要的依据范围限定为单份报表 ID 对应的数据快照 |
| 边界情况 | 数据不足时明确提示“仅供参考”或“当前无法生成稳定结论” |
| 状态变化 | 未生成摘要 → 生成中 → 已生成 / 生成失败 |
| 设计依据 | 来源于 AI 报表“提效和增值”的定位 |

### 4.3 功能单元八：智能专题分析

**一句话定义**：对异常事件、街镇对比、隐患类型等场景生成专题报告。

| 要素 | 设计结论 |
|---|---|
| 触发条件 | 用户主动发起专题分析，或满足预设阈值触发专题任务 |
| 页面位置 | AI 报表助手中的“专题分析” |
| 核心流程 | 选择专题类型 → 选择时间与范围 → 生成结构化专题报告 |
| 业务规则 | 专题分析使用一期已冻结指标和历史报表快照，不直接绕开报表中心拉底层散数据 |
| 数据规则 | 每份专题报告必须显示引用的数据周期、模板、版本和生成时间 |
| 边界情况 | 指标缺失、数据量不足、跨模块口径未冻结时，禁止输出正式专题报告 |
| 状态变化 | 待配置 → 生成中 → 可预览 → 可导出 |
| 设计依据 | 来源于后续 AI 专题分析和自动化汇报需求 |

## 五、报表指标口径台账

> 说明：以下台账用于一期冻结正式报表口径。责任人和验收责任人如未在现有文档中明确，一律标注 `⚠️ 待确认`，不得推断。

| 指标名称 | 指标定义 | 分子 / 分母 | 时间口径 | 去重规则 | 页面/导出一致性 | 来源模块 | 责任人 | 验收责任人 | 当前状态 |
|---|---|---|---|---|---|---|---|---|---|
| 总备案数量 | 历史累计已通过审核且已出具回执的工程总数 | 非比率指标 | 历史累计 | 同一工程按最新有效备案去重 | 必须一致 | 备案管理、备案审核、智慧大屏 | ⚠️ 待确认 | ⚠️ 待确认 | 待冻结 |
| 本周新增备案 | 当前自然周首次提交审核的备案数量 | 非比率指标 | 本周 | 同一工程同一周期按首次提交计 1 次 | 必须一致 | 备案管理、智慧大屏 | ⚠️ 待确认 | ⚠️ 待确认 | 待冻结 |

## 六、原型对照清单

> 本清单是 `31_智慧大屏报表中心原型设计.html` 的实现依据。原型只能表达下列规则，不得擅自删改业务边界。

### 6.1 页面与模块

1. 原型必须至少包含 4 个页面或视图：
   - 报表中心首页
   - 导出配置弹窗/页
   - A4 模板预览
   - 历史记录页
2. 原型中需要预留二期 AI 入口，但必须标注“二期能力，非一期交付”。

### 6.2 一期范围表达

1. 首页必须体现三类一期正式报表：Excel 明细、Excel 汇总、A4 汇报。
2. 首页必须体现“口径台账已冻结/待冻结”的状态提示区。
3. 导出配置中必须支持：周期、街道、工程类型、状态、格式选择。
4. 导出历史中必须体现：导出时间、导出人、筛选条件、任务状态、文件有效期。

### 6.3 A4 模板表达

1. 原型中必须明确一期只落 3 套模板族，不得展示为全部 8 套都已进入开发。
2. A4 页面必须体现“适合打印、结构稳定、数据可核对”的简约风格。
3. 模板内容只允许使用正式冻结指标，不允许加入 AI 自由发挥结论。

### 6.4 二期 AI 预留表达

1. AI 入口可展示“智能选报表”“智能写摘要”“专题分析”三个能力卡片。
2. 必须明确这些能力依赖一期已生成报表，不直接拉底层数据。
3. 必须有“仅供参考/待 AI 能力接入”的提示样式。

## 七、验收标准

### 7.1 一期功能验收

1. 用户能从大屏或业务入口进入统一的报表中心。
2. 用户能生成 Excel 明细报表、Excel 汇总报表、A4 汇报报表三类结果。
3. 报表历史记录可查看、可识别成功/失败/过期状态。
4. A4 仅提供 3 套一期正式模板族，不误导用户为 8 套全量实装。

### 7.2 一期规则验收

1. 同一筛选条件下，大屏数字、Excel 汇总、A4 模板三者一致。
2. 区级/街镇/工程类型切换后，权限范围内结果一致。
3. 脱敏字段在 Excel 和 A4 中都生效。
4. 每次导出都有留痕，且记录入口、时间、筛选条件、结果状态。

### 7.3 二期 AI 验收

1. AI 生成的摘要只能引用一期已存在指标。
2. AI 输出与原报表数字不能矛盾。
3. 数据不足时必须提示“仅供参考”或“当前无法生成稳定结论”。

## 八、决策记录

| 决策内容 | 选择方案 | 决策依据 / 原因 |
|---|---|---|
| 一期是否依赖 AI | 不依赖 | 当前 AI 算力和接口条件未稳定，正式报表不应依赖 AI |
| 一期输出形态优先级 | Excel 优先，A4 同步简约版 | 满足核数、留档、再加工的高频需求 |
| 一期报表范围 | 覆盖全量大屏正式指标 | 与当前用户对一期“正式发布”的要求一致 |
| 报表能力组织方式 | 统一为报表中心 / 导出中心 | 避免各页面各做一套入口和口径 |
| A4 模板策略 | 先收敛为 3 套模板族 | 控制一期范围，避免 8 套全部实装导致排期失控 |
| 二期 AI 角色定位 | 解读层与提效层 | AI 不反向定义正式口径 |
