## 一、文档信息

| 字段 | 内容 |

## 二、业务场景

已备案通过并进入施工中的工程，可能因为天气、材料、现场条件、配合审批等原因无法按原计划完工，需要申请延期。

本功能主要覆盖以下场景：

1. 施工方主动申请延期，由巡检端审核确认。
2. 业主方主动申请延期，由巡检端审核确认。
3. 巡检人员在线下已确认需要延期时，可在巡检端直接延期。
4. 小型工程若延期后超出原保单保障周期，平台需要求补充覆盖完整延期周期的续保保单信息和材料。
5. 住宅装修、零星作业即使已购线上保单且延期后超期，也只提醒、不拦截。

## 三、设计目标

1. 为施工中工程提供统一的“申请延期”能力。
2. 将延期申请、保险补充、审核处理沉淀为同一张申请单，不拆独立保险流程。
3. 明确小型工程延期超保时的平台提醒和审核责任。
4. 在一期先落地“线下续保后回传保单”的方案，不做线上续保购买闭环。

## 四、范围说明

### 4.1 本期包含

1. 施工方发起延期申请。
2. 业主方发起延期申请。
3. 巡检端审核延期申请。
4. 巡检端直接延期。
5. 延期申请单内补充续保保单信息与附件。
6. 延期记录留痕。
7. 页面提醒与短信提醒。

### 4.2 本期不包含

1. 线上续保购买流程。
2. 保司自动报价、自动核保、自动回传保单。
3. OCR识别保单日期。
4. 独立的“保险续保订单”流程。

## 五、核心设计结论

1. 延期是一张申请单，不拆成“延期单”和“保险单”两个流程。
2. 发起方支持施工方和业主方，处理方为巡检端。
3. 巡检端支持两类动作：审核延期申请、直接延期。
4. 小型工程延期后如超出当前保单保障止期，申请单内必须补充续保保单信息，否则不能通过，也不能直接延期生效。
5. 住宅装修、零星作业即使存在保单超期，也只提醒，不因未补保险信息而拦截延期申请。
6. 保险公司字段复用后台已配置数据。
7. 险种字段使用固定选项，不做自定义自由填写。
8. 延期原因为手填，不做枚举。
9. 工程所属大类不在申请表中展示，只作为系统内部判断规则使用。

## 六、申请单设计

### 6.1 表单结构

延期申请单包含三部分：

1. 工程基本信息。
2. 延期信息。
3. 保险补充信息。

### 6.2 工程基本信息

| 字段 | 填写方式 | 说明 |
|------|----------|------|
| 工程名称 | 只读 | 带入当前工程 |
| 工程地址 | 只读 | 用于审核快速定位 |
| 具体工程类型 | 只读 | 展示备案时选择的具体类型 |
| 施工内容 | 只读 | 展示当前工程实际施工内容 |
| 原计划开工日期 | 只读 | 带入备案信息 |
| 原计划完工日期 | 只读 | 带入备案信息 |

### 6.3 延期信息

| 字段 | 必填 | 说明 |
|------|------|------|
| 延期后完工日期 | 是 | 延期后的新完工日期 |
| 延期原因 | 是 | 手工填写 |
| 是否涉及暂停施工 | 是 | 单选：是/否 |
| 暂停施工开始日期 | 条件必填 | 当“是否涉及暂停施工=是”时显示 |
| 预计恢复施工日期 | 条件必填 | 当“是否涉及暂停施工=是”时显示 |

### 6.4 保险补充信息

仅在“小型工程且延期后完工日期超过当前保单止期”时强制显示。

| 字段 | 必填 | 说明 |
|------|------|------|
| 保险公司 | 是 | 选择后台已配置保险公司 |
| 险种 | 是 | 固定选项 |
| 保单号 | 是 | 手工填写 |
| 保单起期 | 是 | 手工选择 |
| 保单止期 | 是 | 手工选择 |
| 保单附件 | 是 | 支持图片或PDF |

### 6.5 险种固定选项

1. 安全生产责任险
2. 建筑工程一切险
3. 建筑工程团体意外险
4. 雇主责任险
5. 工伤保险
6. 其他险种

## 七、业务规则

### 7.1 发起规则

1. 仅已备案通过、未完工核销的工程可申请延期。
2. 同一工程存在“待审核”的延期申请单时，不允许再次发起延期申请。
3. 巡检端直接延期与普通延期申请共用同一套表单结构，但处理结果不同。

### 7.2 日期规则

1. 延期后完工日期必须晚于原计划完工日期。
2. 若涉及暂停施工，预计恢复施工日期不得早于暂停施工开始日期。
3. 小型工程超保场景下，续保保单止期必须覆盖延期后完工日期。

### 7.3 保险联动规则

1. 小型工程属于强制投保场景。
2. 若小型工程延期后超出当前保单保障周期，则申请单中必须补齐续保保单信息和附件。
3. 平台一期不提供线上续保购买，仅支持用户线下办理后回传材料。
4. 续保保单需覆盖完整延期施工周期，页面需明确提示。
5. 住宅装修、零星作业不强制投保，若已有保单但延期超期，仅做提醒，不做拦截。

### 7.4 审核规则

1. 巡检端审核的是整张延期申请单，不拆分延期审核和保险审核。
2. 巡检端审核通过后，工程主数据中的完工日期更新为延期后完工日期。
3. 巡检端驳回后，申请单状态更新为“已驳回”，申请人可修改后重新提交。
4. 巡检端直接延期提交成功后立即生效，不再进入二次审核。

## 八、状态设计

### 8.1 延期申请单状态

| 状态 | 说明 |
|------|------|
| 待审核 | 申请已提交，等待巡检端处理 |
| 已通过 | 延期审核通过，工期已更新 |
| 已驳回 | 延期申请被退回，等待申请人修改重提 |

### 8.2 工程详情展示状态

| 场景 | 展示方式 |
|------|----------|
| 无延期申请 | 显示“申请延期”入口 |
| 存在待审核延期申请 | 显示“延期待审核”状态，不再显示新的申请入口 |
| 已有历史延期记录 | 显示“延期记录”入口 |

## 九、提醒与短信

### 9.1 页面提醒

1. 小型工程延期超保时，页面提示“请先线下办理续保，并上传覆盖完整延期施工周期的续保保单”。
2. 住宅装修、零星作业若保单超期，仅提示“当前保单可能已超期，请自行与保司确认是否需要续保”。

### 9.2 短信文案建议

| 场景 | 接收人 | 短信文案 |
|------|--------|----------|
| 提交延期申请成功 | 申请人 | 您提交的【{工程名称}】延期申请已提交，当前状态为“待审核”，请留意后续处理结果。 |
| 小型工程延期超保提醒 | 申请人 | 您提交的【{工程名称}】延期申请已超出当前保单保障周期，请尽快线下办理续保，并补充上传覆盖完整延期施工周期的保单材料。 |
| 有新的延期待审核 | 巡检人员 | 您有一条新的延期申请待处理。工程名称：{工程名称}，工程地址：{工程地址}，请及时登录巡检端处理。 |
| 延期申请审核通过 | 申请人 | 您提交的【{工程名称}】延期申请已审核通过，新的完工日期为 {延期后完工日期}。 |
| 延期申请已驳回 | 申请人 | 您提交的【{工程名称}】延期申请已被驳回，驳回原因：{驳回原因}，请修改后重新提交。 |
| 巡检端直接延期完成 | 施工方/业主方 | 【{工程名称}】已由巡检人员直接延期，新的完工日期为 {延期后完工日期}，请按最新工期安排施工。 |

## 十一、页面间交互逻辑

### 11.1 施工端 / 业主端交互流程

```text
首页（P1）
└── 点击“申请延期” ----> 延期申请工程列表页（P2）
    ├── 查看已逾期且可申请延期的工程列表
    ├── 点击列表项“申请延期” ----> 延期申请填写页（P3）
    │   ├── 查看工程基础信息（工程名称、工程地址、具体工程类型、施工内容、原计划工期）
    │   ├── 点击“查看完整工程详情” ----> 工程详情页（P4）
    │   │   └── 点击“延期申请记录” ----> 延期申请记录页（P5）
    │   ├── 返回延期申请填写页（P3）
    │   ├── 填写延期信息（延期后完工日期、延期原因、是否暂停施工）
    │   ├── 若“是否涉及暂停施工 = 是”
    │   │   └── 显示“暂停施工开始日期、预计恢复施工日期”
    │   ├── 若“小型工程且延期后超出当前保单止期”
    │   │   └── 自动展开保险补充区
    │   │       ├── 选择保险公司
    │   │       ├── 选择险种
    │   │       ├── 填写保单号
    │   │       ├── 选择保单起止时间
    │   │       └── 上传保单图片/PDF
    │   ├── 点击“提交申请”
    │   │   ├── 校验通过 ---> 弹窗“提交成功” ---> 返回延期申请工程列表页（P2）
    │   │   │   └── 当前申请状态显示为“待审核”
    │   │   └── 校验不通过 ---> 停留当前页并提示缺失项
    │
    └── 点击列表项“查看记录”或从工程详情进入 ----> 延期申请记录页（P5）
        ├── 查看历史延期申请记录时间线
        ├── 查看每次延期的审核状态
        └── 查看驳回原因或保险补充信息摘要
```

### 11.2 巡检端交互流程

```text
首页（P6）
└── 点击“延期申请” ----> 延期申请待审核列表页（P7）
    ├── 查看已提交延期申请的工程列表
    ├── 可按工程类型筛选
    ├── 可按工程名称或备案编号搜索
    └── 点击列表项 ----> 延期申请审核页（P8）
        ├── 查看工程基本信息
        ├── 查看延期申请信息
        ├── 查看保险补充信息与保单材料
        ├── 点击“通过”
        │   └── 弹窗“审核通过，延期已生效” ---> 返回待审核列表页（P7）
        └── 点击“驳回”
            ├── 填写驳回原因
            └── 弹窗“已驳回” ---> 返回待审核列表页（P7）
```

### 11.3 巡检端补充能力流程

```text
延迟申请审核页（P8）或巡检端业务入口
└── 点击“直接延期” ----> 直接延期页（P9）
    ├── 填写延期信息
    ├── 若“小型工程且延期后超出当前保单止期”
    │   └── 强制补齐保险公司、险种、保单号、保单起止时间、保单附件
    ├── 点击“提交并生效”
    │   ├── 校验通过 ---> 弹窗“延期已生效”
    │   └── 校验不通过 ---> 停留当前页并提示缺失项
```

## 十二、原型对照清单

| 编号 | 规则内容 | 原型必须体现的位置 | 验收标准 |
|------|----------|-------------------|---------|
| R01 | 延期是一张申请单，不拆独立保险流程 | P3、P8、P9 | 同页同时体现延期信息和保险补充信息 |
| R02 | 施工方和业主方都可从首页进入延期申请流程 | P1 | 两类角色都能看到“申请延期”入口 |
| R03 | 延期入口先进入逾期工程列表，而不是直接进入工程详情 | P2 | 页面先罗列可申请延期的逾期工程 |
| R04 | 延期申请页可跳转查看完整工程详情 | P3、P4 | P3存在“查看完整工程详情”入口，P4可承接跳转 |
