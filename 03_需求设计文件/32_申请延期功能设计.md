## 二、背景与目标

### 2.1 背景

当前系统已具备“备案申请 → 审核通过 → 施工监管 → 完工核销”的主流程，但尚未提供施工期间的工期延期能力。实际业务中存在以下场景：

1. 施工方或业主/建设单位因天气、现场条件、材料进场、审批配合等原因，需要申请延长工期。
2. 监管人员在巡查、走访或线下沟通中，可能直接确认该工程需要延期，不一定先由施工方发起申请。
3. 小型工程属于强制投保场景，延期后若超出原保单保障期，平台需要尽到提醒和控制义务，避免“延期已生效，但保险未覆盖完整施工周期”的责任灰区。

### 2.2 目标

1. 为备案通过后的工程提供标准化“延期申请单”能力，支持施工方、业主/建设单位发起延期。
2. 为监管端提供“审核延期申请”和“直接延期”能力。
3. 对小型工程建立延期与保险覆盖的联动校验：延期后的施工周期超出当前保单保障期时，必须在延期申请单中补充新的保险信息和保单材料。
4. 一期不做线上续保交易，只做延期申请、保险信息补充、人工审核、短信提醒与工程档案留痕。

### 2.3 非目标

1. 本期不建设线上续保、展期、批改、支付、保司接口直连。
2. 本期不自动识别保单图片或 PDF 中的起止日期，仍以用户手填 + 监管人工核对为准。
3. 本期不新增独立“保险补传流程”或“续保订单流程”，保险内容并入延期申请单处理。
4. 本期不改变完工核销、巡查整改、备案回执等既有主流程。

## 四、设计口径总览

### 4.1 一句话定义

为备案通过后的工程提供一张“延期申请单”，由施工方、业主/建设单位或监管填写延期内容；当小型工程延期后超出当前保单保障期时，必须在同一张延期申请单中补充新的保险信息与保单材料，审核通过后延期才正式生效。

### 4.2 核心设计结论

1. 延期是“一张申请单”，不拆成独立的延期流程和保险流程。
2. 施工方、业主/建设单位都可以发起延期申请；监管可以直接延期。
3. 小型工程延期后若超出当前保单保障期，不允许只做提醒后直接生效，必须在申请单内补齐新的保险信息和保单材料。
4. 住宅装修、零星作业不强制投保；如已有线上保单但延期后超保，只提醒，不拦截。
5. 一期不做线上续保入口，只支持线下办理后在平台上传新保单或相关材料。

## 五、核心流程

### 5.1 施工方/业主发起延期流程

```text
备案已通过工程
  → 进入工程详情
  → 点击“申请延期”
  → 填写延期申请单
      → 填延期后完工日期、延期原因、是否暂停施工
      → 如为小型工程且延期后超保，补填保险信息并上传保单材料
  → 提交申请
  → 系统校验
      → 校验通过：进入待审核
      → 校验失败：提示并留在编辑页
  → 监管审核
      → 通过：延期生效
      → 驳回：退回修改
```

### 5.2 监管直接延期流程

```text
监管进入工程详情
  → 点击“直接延期”
  → 填写延期申请单（监管视角）
      → 填延期后完工日期、延期原因、是否暂停施工
      → 如为小型工程且延期后超保，补填保险信息并上传保单材料
  → 提交
  → 系统校验
      → 校验通过：直接生效并留痕
      → 校验失败：提示并留在编辑页
```

### 5.3 保险联动判断流程

```text
用户填写延期后完工日期
  → 系统读取工程类型与当前保单止期
  → 判断是否为小型工程
      → 否：仅提示已有保单可能超期，不强制补传
      → 是：继续判断
  → 判断延期后完工日期是否大于当前保单止期
      → 否：不强制补传保险
      → 是：强制显示保险补充信息区，未补齐不可提交
```

## 六、功能单元

### 6.1 功能单元一：延期申请单发起

**一句话定义**

施工方、业主/建设单位在备案通过后的工程详情页发起延期申请，填写延期信息；如小型工程延期后超保，在同一申请单中同步补充新的保险信息与保单材料。

**页面位置**

施工端 / 备案项目详情页 / 工程详情页

**入口角色**

1. 施工方
2. 业主/建设单位

**入口条件**

1. 工程已备案审核通过
2. 工程未完工核销
3. 当前不存在待审核的延期申请单

**页面名称**

申请延期

**页面信息结构**

1. 工程基本信息（只读）
2. 延期信息
3. 保险补充信息（按条件显示）
4. 提交说明与按钮区

**字段清单**

| 分组 | 字段 | 必填 | 说明 |
|------|------|------|------|
| 工程基本信息 | 工程名称 | 只读 | 带入当前工程 |
| 工程基本信息 | 工程地址 | 只读 | 带入当前工程地址，便于审核定位 |
| 工程基本信息 | 具体工程类型 | 只读 | 展示备案时选择的具体工程类型，如园林绿化工程、住宅室内装饰装修工程、户外广告安装维护拆除作业等 |
| 工程基本信息 | 施工内容 | 只读 | 展示备案时填写的具体施工/作业内容 |
| 工程基本信息 | 原计划开工日期 | 只读 | 带入备案信息 |
| 工程基本信息 | 原计划完工日期 | 只读 | 带入备案信息 |
| 延期信息 | 延期后完工日期 | 是 | 新的完工日期 |
| 延期信息 | 延期原因 | 是 | 手填 |
| 延期信息 | 是否涉及暂停施工 | 是 | 是 / 否 |
| 延期信息 | 暂停施工开始日期 | 条件必填 | 选“是”后显示 |
| 延期信息 | 预计恢复施工日期 | 条件必填 | 选“是”后显示 |
| 保险补充信息 | 保险公司 | 条件必填 | 选择后台已配置保险公司 |
| 保险补充信息 | 险种 | 条件必填 | 固定选择项 |
| 保险补充信息 | 保单号 | 条件必填 | 文本输入 |
| 保险补充信息 | 保单起期 | 条件必填 | 日期选择 |
| 保险补充信息 | 保单止期 | 条件必填 | 日期选择 |
| 保险补充信息 | 保单材料 | 条件必填 | 图片 / PDF |

**险种固定选项**

1. 安全生产责任险
2. 建筑工程一切险
3. 建筑工程团体意外险
4. 雇主责任险
5. 工伤保险
6. 其他险种

**业务规则**

1. 延期申请单只允许针对已备案通过、未核销工程发起。
2. 施工方和业主/建设单位都可以发起延期申请，二者共用同一张申请单。
3. `申请延期后完工日期`是本功能的核心生效日期，后续监管、巡查、完工核销均以该日期作为新的工期终点。
4. 工程基本信息区需优先展示审核判断最常用的信息：工程名称、工程地址、具体工程类型、施工内容、原计划工期。
5. `工程所属大类`仅作为系统内部规则判断口径，用于触发是否强制补充保险信息；申请表和审核页不单独展示该字段。
6. `是否涉及暂停施工`用于表达天气、材料、现场条件等导致的中途暂停场景；若选择“是”，必须补充暂停与恢复时间。
7. 小型工程延期后若超出当前保单保障期，必须在申请单中补充新的保险公司、险种、保单号、保单起止时间和保单材料。
8. 住宅装修、零星作业延期后即使已有线上保单超期，也只提醒，不因未补传保险信息而阻断提交。
9. 平台一期不负责续保交易，仅负责记录和审核结果材料。

**字段规则**

1. `申请延期后完工日期`必须晚于原计划完工日期。
2. `暂停施工开始日期`、`预计恢复施工日期`只在“是否涉及暂停施工 = 是”时显示。
3. `预计恢复施工日期`必须晚于或等于`暂停施工开始日期`。
4. `保单材料`支持图片和 PDF；文件大小限制⚠️沿用现有上传规则，具体数值待统一。

**提交校验**

1. 工程类型为小型工程，且`申请延期后完工日期 > 当前保单止期`时：
   - 保险公司必填
   - 险种必填
   - 保单号必填
   - 保单起期必填
   - 保单止期必填
   - 保单材料必传
2. 小型工程超保场景下，`保单止期`必须大于或等于`申请延期后完工日期`，否则不允许提交。
3. 非小型工程，即使页面提醒已有保单超期，也不因保险信息缺失阻断提交。

**异常处理**

1. 当前已有待审核延期申请单：隐藏“申请延期”入口，展示“延期申请审核中”提示。
2. 当前工程已核销：不显示延期入口。
3. 上传材料失败：提示失败原因并允许重新上传。
4. 提交时网络中断：保留当前填写内容，重新进入页面后可继续编辑⚠️具体是否本地暂存待确认。

### 6.2 功能单元二：延期申请单审核

**一句话定义**

监管人员对施工方、业主/建设单位提交的延期申请单进行整体审核，同步审核延期信息和保险补充材料，决定通过或驳回。

**页面位置**

监管端 / 项目中心 / 延期申请待审核列表 / 延期申请详情

**使用角色**

街镇监管人员

**页面名称**

延期申请审核

**页面功能**

1. 查看工程基本信息
2. 查看延期申请信息
3. 查看保险补充信息和保单材料
4. 审核通过
5. 审核驳回
6. 查看历史延期记录

**工程基本信息展示重点**

1. 工程名称
2. 工程地址
3. 具体工程类型
4. 施工内容
5. 原计划开工日期
6. 原计划完工日期

**业务规则**

1. 监管审核的是一整张延期申请单，不拆成“延期审核”和“保险审核”两个动作。
2. 对于小型工程超保场景，保险补充信息是延期申请单的一部分，监管需一并核对。
3. 审核通过后，新的完工日期写回工程主数据，延期正式生效。
4. 审核驳回后，申请单状态变为已驳回，申请人可修改后重新提交。
5. 驳回原因应明确指出是延期信息问题、保险信息问题，还是材料问题。

**监管审核要点**

1. 延期后完工日期是否合理
2. 延期原因是否完整
3. 工程地址、具体工程类型、施工内容是否与本次延期场景匹配，避免审核人在仅看工程名称时判断不清
4. 暂停施工信息是否与延期说明匹配
5. 小型工程超保场景下，保险公司、险种、保单号、保单起止时间是否填写完整
6. 上传的保单材料是否与手填信息一致
7. 保单止期是否覆盖延期后的完整施工周期

**状态规则**

延期申请单仅保留以下状态：

1. 待审核
2. 已通过
3. 已驳回

**状态变化**

1. 申请人提交成功：状态 = 待审核
2. 监管审核通过：状态 = 已通过
3. 监管审核驳回：状态 = 已驳回
4. 驳回后重新提交：状态重新回到待审核

**驳回规则**

1. 可整体驳回，不要求拆分字段级驳回。
2. 驳回原因建议支持结构化记录，至少包含文字说明。
3. 若小型工程超保但保单材料与手填信息不一致，应驳回整张申请单。

**异常处理**

1. 审核时若工程已被核销⚠️待确认：默认不允许继续通过延期，提示工程状态已变化。
2. 审核时若同一工程存在更新后的保险材料但当前页面未刷新，提示刷新后再处理。
3. 审核通过写回失败时，不允许只改状态不改工期，必须整体回滚。

### 6.3 功能单元三：监管直接延期

**一句话定义**

监管人员可不等待施工方先发起申请，直接在监管端填写延期申请单并立即生效，用于线下已确认需要延期的场景。

**页面位置**

监管端 / 工程详情页 / 直接延期

**使用角色**

街镇监管人员

**页面名称**

直接延期

**页面结构**

1. 工程基本信息（只读）
2. 延期信息
3. 保险补充信息（按条件显示）
4. 提交按钮

**业务规则**

1. 监管直接延期与普通延期申请共用同一张表单结构，只是提交人不同。
2. 监管直接延期不需要再经过第二次人工审核，校验通过后直接生效并记录操作人。
3. 小型工程超保时，即使是监管直接延期，也必须在表单中补齐新的保险信息和保单材料，否则不能提交成功。
4. 监管直接延期后，施工方/业主应能在延期记录中看到该次延期来源为“监管直接延期”。

**权限规则**

1. 仅监管角色显示“直接延期”入口。
2. 施工方、业主/建设单位不可见该入口。

**留痕规则**

1. 记录直接延期操作人、操作时间、延期原因、延期前后完工日期。
2. 若补填了保险信息，同步记录保险补充字段和附件。
3. 监管直接延期页中的工程基本信息展示内容与延期申请审核页保持一致，至少展示工程名称、工程地址、具体工程类型、施工内容。

### 6.4 功能单元四：延期记录与工程档案

**一句话定义**

将每次延期申请、审核结果、保险补充信息和附件沉淀到工程档案中，便于后续巡查、核销和责任追溯。

**页面位置**

施工端 / 工程详情页 / 延期记录  
监管端 / 工程详情页 / 延期记录  
后台 / 工程档案

**记录字段**

1. 延期记录编号
2. 工程编号
3. 发起方式（施工方申请 / 业主申请 / 监管直接延期）
4. 发起人
5. 原计划完工日期
6. 申请延期后完工日期
7. 延期原因
8. 是否涉及暂停施工
9. 暂停施工开始日期
10. 预计恢复施工日期
11. 审核状态
12. 审核人
13. 审核时间
14. 驳回原因
15. 保险公司
16. 险种
17. 保单号
18. 保单起期
19. 保单止期
20. 保单材料

**业务规则**

1. 每次延期都沉淀为一条独立记录，不覆盖历史延期记录。
2. 工程当前有效完工日期取最后一次“已通过”或“监管直接延期成功”的记录。
3. 延期记录应纳入工程档案，与备案、巡查、整改、核销记录并列。
4. 后续如发生事故、争议或核销审核，应可回看延期申请时提交的保险材料。

## 七、短信与提醒规则

**信息来源说明**

提醒义务、一期先做短信和页面提醒、不做保司自动通知，来源于本轮用户确认。

**提醒对象**

1. 申请人
2. 监管人员

**提醒场景**

1. 提交延期申请成功
2. 小型工程延期后超保，页面展示提醒文案
3. 审核通过
4. 审核驳回

**提醒原则**

1. 一期不要求直接给保司发通知。
2. 一期可在页面展示保险办理提示与相关联系方式⚠️联系方式来源待确认。
3. 小型工程超保时，页面提示需明确：须先联系保险公司线下办理续保/展期，并在本申请单中补充新的保单信息，保障期需覆盖完整延期施工周期。

**短信文案建议**

1. 延期申请提交成功
   - 发送对象：申请人
   - 文案：您提交的【{工程名称}】延期申请已提交，当前状态为“待审核”，请留意后续审核结果通知。

2. 小型工程延期超保提醒
   - 发送对象：申请人
   - 触发条件：小型工程延期申请提交成功，且延期后完工日期超出当前保单保障期
   - 文案：您提交的【{工程名称}】延期申请已超出当前保单保障期，请尽快联系保险公司线下办理续保/展期，并在延期申请中补充新的保单信息，保障期需覆盖完整延期施工周期。

3. 延期申请待审核提醒
   - 发送对象：监管人员
   - 文案：您有一条新的延期申请待审核。工程名称：{工程名称}，工程地址：{工程地址}，请及时登录平台处理。

4. 延期申请审核通过
   - 发送对象：申请人
   - 文案：您提交的【{工程名称}】延期申请已审核通过，新的完工日期为 {延期后完工日期}，请按最新工期安排施工。

5. 延期申请审核驳回
   - 发送对象：申请人
   - 文案：您提交的【{工程名称}】延期申请已被驳回，驳回原因：{驳回原因}。请修改后重新提交。

6. 监管直接延期通知
   - 发送对象：施工方/业主申请联系人⚠️具体发送范围待确认
   - 文案：监管已对【{工程名称}】执行直接延期，新的完工日期为 {延期后完工日期}。请登录平台查看详情，并按最新工期安排施工。
