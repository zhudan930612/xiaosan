## 一、功能定位

### 1.1 与现有大屏的分工

大屏是**"看板"**（被动展示标准指标），AI助手是**"助手"**（主动提供个性化服务）。

| 维度 | 大屏（已有） | AI助手（补充） |
|-----|-------------|---------------|
| 核心交互 | 看图表 | 问问题 |
| 数据深度 | 标准指标展示 | 深层分析、根因分析 |
| 个性化 | 统一视图 | 千人千面（个人统计） |
| 主动性 | 被动展示 | 主动提醒待办 |
| 灵活性 | 固定图表、固定层级 | 自由钻取、灵活探索 |
| 输出能力 | 展示 | 生成报告、导出分享 |

### 1.2 AI助手不做的事情

以下能力大屏已经做得很好，AI助手**不重复建设**：

- ❌ 复杂的趋势图表展示（大屏已有趋势图）
- ❌ 工程状态分布可视化（大屏已有堆叠图）
- ❌ 隐患类别TOP5环形图（大屏已有）
- ❌ 地图点位可视化（大屏已有地图）

### 1.3 AI助手专注做的事情

以下是大屏做不了或做不好的，AI助手**重点提供**：

- ✅ **根因分析**: "为什么整改率下降？"
- ✅ **关联分析**: "备案量高的街道是否隐患率也高？"
- ✅ **个人视角**: "我本月巡查了多少次？"
- ✅ **待办提醒**: "我有哪些待处理事项？"
- ✅ **灵活钻取**: "这个工程历次巡查的详细记录"
- ✅ **报告生成**: "生成本周报告并导出PDF"

### 1.4 目标用户与场景

| 用户角色 | 使用端 | 大屏场景 | AI助手场景 |
|---------|-------|---------|-----------|
| 区级领导 | 大屏/PC | 看全区态势、对比各街道 | 问根因、生成报告、深度分析 |
| 街镇管理人员 | PC/H5 | 看本辖区指标 | 查待办、个人工作统计、工程详情 |
| 巡查人员 | H5 | 看地图点位 | 查个人统计、待整改提醒、快速查询 |

## 二、核心能力设计

### 能力1: 智能问答（与大屏差异化）

**设计理念**: 回答数字只是基础，解释原因和给出建议才是价值。

**与大屏的差异**:

| 查询类型 | 大屏展示 | AI助手回答 |
|---------|---------|-----------|
| 备案量 | 数字+趋势图 | 数字+环比解释+关联建议 |
| 整改率 | 百分比+对比 | 百分比+原因分析+改进建议 |

**交互示例**:

```
用户: 本月备案量

大屏展示: [数字卡片: 1,245] [趋势图]

AI助手回答:
"本月备案量1,245个，环比上月+15%。
 
 增长主要来自：
 • 夏阳街道：+45个（+30%）
 • 盈浦街道：+28个（+18%）
 
 可能原因：夏阳街道近期开展了备案宣传活动。
 
 [查看夏阳街道详情] [对比各街道]"
```

**支持的问题类型**:

| 类型 | 示例 | AI价值点 |
|-----|------|---------|
| 标准指标 | "本月备案量" | 数字+解释原因 |
| 根因分析 | "为什么夏阳街道整改率下降？" | 分析可能原因 |
| 关联分析 | "备案量与隐患率有关系吗？" | 相关性分析 |
| 排名对比 | "哪个街道表现最好？" | 排序+原因分析 |
| 明细钻取 | "查看这个工程的所有记录" | 跨表关联查询 |
| 预测问题 | "下周哪些隐患可能逾期？" | 趋势预测 |

---

### 能力2: 深度分析（大屏做不了）

**根因分析场景**:

| 场景 | 用户问题 | AI分析维度 |
|-----|---------|-----------|
| 整改率下降 | "为什么本月整改率下降了？" | 按街道分解、按隐患类型分解、按时间趋势分析 |
| 隐患高发 | "为什么这个工程隐患特别多？" | 工程类型、施工单位、施工阶段关联分析 |
| 巡查覆盖不足 | "为什么有些工程一直没巡查？" | 地理分布、工程类型、责任单位分析 |

**交互示例**:

```
用户: 为什么夏阳街道整改率低于平均？

AI:
"夏阳街道本月整改率为72%，低于全区平均（85%）。

可能原因分析：

1. 【隐患类型因素】该街道高处作业类隐患占比40%，
   此类隐患平均整改周期较长（5.2天 vs 平均3.1天）
   
2. 【施工单位因素】3家施工单位首次入驻本街道，
   整改响应经验不足，平均首次整改通过率仅45%
   
3. 【时间因素】本月新增隐患中，15%为本周新发现，
   尚在整改周期内

建议：
• 针对高处作业隐患，可推送标准整改模板
• 对新入驻施工单位开展整改流程培训
• [查看整改逾期工程清单]"
```

**个人统计**（巡查人员刚需，大屏不支持）:

```

AI:
"张三，本月工作统计：

巡查工作：
• 巡查次数：28次（目标30次，完成率93%）

待办事项：
• 🔴 3条隐患整改今日到期，待复核
• 🟡 2个工程超过7天未巡查

排名：
• 在本街道巡查员中排名第2（按巡查次数）

[查看待办] [查看明细]"
```

### 能力4: 报告生成（导出分享）

大屏展示但无法导出，AI助手提供生成和导出能力。

**报告类型**:

| 类型 | 内容 | 面向用户 |
|-----|------|---------|
| 周报 | 本周备案、巡查、整改总结+分析解读 | 街镇管理员 |
| 月报 | 月度全景+趋势分析+问题诊断 | 区级领导 |
| 专项报告 | 特定主题（如"住宅装修专项分析"） | 按需生成 |
| 个人报告 | 个人工作统计汇总 | 巡查人员 |

**交互示例**:

```
用户: 生成本周报告

AI:
"正在生成《青浦区小型工程监管周报（第13周）》...

报告包含：
✓ 核心指标摘要（含同比环比）
✓ 本周工作亮点
✓ 存在问题分析
✓ 下周工作建议

[在线预览] [下载PDF] [分享链接]"
```

## 四、功能点描述

### 4.1 入口与唤起

| 端 | 唤起方式 | 默认展示 |
|---|---------|---------|
| PC | 点击悬浮球 / Ctrl+K | 快捷入口菜单 |
| 大屏 | 语音"小安小安" / 点击图标 | 语音输入提示 |
| H5 | 点击底部"AI"Tab | 个人统计摘要 |

### 4.2 消息类型

| 类型 | 用途 | 示例 |
|-----|------|------|
| 文本 | 简单回复、解释说明 | "本月备案量1,245个，环比+15%" |
| 数据卡片 | 关键指标展示 | 数字+环比+简要解释 |
| 分析卡片 | 根因分析结果 | 分点展示分析结论 |
| 待办列表 | 待处理事项 | 分组展示紧急/重要/提示 |
| 报告卡片 | 报告生成结果 | 报告预览+下载按钮 |
| 快捷按钮 | 快速操作 | [查看详情] [立即处理] [下载] |

### 4.3 核心功能点

**功能点1: 根因分析**
- 入口: 输入框提问
- 触发词: "为什么..."、"原因..."、"分析..."
- 分析维度: 时间、空间、类型、责任主体
- 输出: 可能原因列表+置信度+建议

**功能点2: 个人统计**
- 入口: 快捷按钮"我的统计"或输入"我本月工作情况"
- 统计范围: 巡查次数、隐患发现、待办事项、排名
- 权限: 只能查看自己的数据

**功能点3: 待办提醒**
- 入口: 快捷按钮"待办"或输入"我有什么待处理"
- 分级: 🔴紧急（今日到期）、🟡重要（即将到期）、🟠提示（其他）
- 操作: 一键跳转处理页面

**功能点4: 报告生成**
- 入口: 快捷按钮"生成报告"
- 类型: 周报/月报/专项报告
- 输出: 在线预览 + PDF下载 + 分享链接

### 5.2 分析引擎设计

**根因分析算法**:
- 维度分解: 按街道、类型、时间、责任单位分解
- 对比分析: 与历史同期、与平均值、与标杆对比
- 相关性分析: 找出影响指标的关键因素

**示例**:
```
问题: 为什么整改率下降？

分析过程:
1. 按街道分解 → 发现主要是夏阳街道下降
2. 按类型分解 → 发现高处作业类隐患整改慢
3. 按时间分解 → 发现新入驻施工单位占比高
4. 关联分析 → 新单位首次整改通过率低

结论: 新入驻施工单位经验不足 + 高处作业隐患整改周期长
```

## 六、与大屏的协同设计

### 6.1 场景切换指南

| 用户场景 | 推荐入口 | 原因 |
|---------|---------|------|
| 看全区整体态势 | 大屏 | 图表直观、信息密度高 |
| 对比各街道数据 | 大屏 | 横向对比清晰 |
| 了解某个指标为什么异常 | AI助手 | 根因分析能力 |
| 查个人工作数据 | AI助手（H5） | 个人视角，大屏不支持 |
| 看待办事项 | AI助手 | 主动提醒，大屏不支持 |
| 生成报告导出 | AI助手 | 生成能力，大屏不支持 |
| 地图找工程 | 大屏 | 空间展示优势 |
| 查工程详细历史 | AI助手 | 灵活钻取对话 |

### 6.2 联动设计

```
大屏 → AI助手:
用户在大屏发现"夏阳街道整改率下降"
点击"分析原因" → 唤起AI助手并带入上下文
AI助手回答整改率下降的可能原因

AI助手 → 大屏:
用户在AI助手问"各街道备案量对比"
AI助手回答后可以"在地图查看分布"
跳转大屏并定位到相应区域
```

| 三期 | 智能化 | 预测分析 + 自然语言增强 | 4周 |

### 风险提示

1. **分析准确性**: 根因分析依赖数据质量和算法，初期需标注"仅供参考"
2. **权限控制**: 个人统计涉及敏感数据，务必做好权限隔离
3. **与大屏的边界**: 需清晰定义哪些能力在AI助手，避免功能蔓延
