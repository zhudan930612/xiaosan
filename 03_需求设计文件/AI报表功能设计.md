## 一、功能定位

**功能名称**: AI智能助手

**功能定义**: 跨端统一的AI数据分析助手，以对话式交互为核心，让用户通过自然语言快速获取监管数据洞察、接收异常提醒、生成数据报告，辅助监管决策。

**核心设计理念**:
- **一个入口，无限可能**: 统一的AI助手入口，不需要理解复杂的功能划分
- **对话即服务**: 像聊天一样获取数据和分析
- **多端一致体验**: H5、PC、大屏统一的交互范式

**目标用户与场景**:

| 用户角色 | 使用端 | 典型场景 |
|---------|--------|---------|
| 区级领导 | 大屏/PC | "全区本月备案情况如何？" |
| 街镇管理人员 | PC/H5 | "我辖区有哪些逾期隐患？" |
| 巡查人员 | H5 | "我本月巡查了多少次？" |

## 二、多端入口设计

### 入口位置

| 端 | 入口形态 | 位置 |
|---|---------|------|
| **PC审核端** | 悬浮球 | 页面右下角，可拖拽 |
| **智慧大屏** | 固定图标/语音唤醒 | 导航栏或侧边栏 |
| **巡检端H5** | 底部导航+悬浮球 | 底部导航"AI"入口，页面内悬浮快捷按钮 |

### 各端差异

| 特性 | PC端 | 大屏端 | H5端 |
|-----|------|--------|------|
| 交互方式 | 文字输入为主 | 语音+文字 | 文字输入为主 |
| 界面形态 | 侧边抽屉/弹窗 | 全屏/大面板 | 全屏页面 |
| 展示空间 | 较大，可展示复杂图表 | 最大，支持多图对比 | 较小，卡片式展示 |
| 特殊能力 | 支持快捷键唤起 | 支持语音唤醒 | 支持拍照识别 |

## 四、交互设计

### 核心交互范式

所有功能通过**对话**触发：

| 用户需求 | 对话方式 | AI响应 |
|---------|---------|--------|
| 查数据 | "本月备案量" / "各街道巡查对比" | 数据卡片+图表 |
| 看洞察 | "我有什么待处理事项" / "查看提醒" | 洞察卡片列表 |
| 生成报告 | "生成本周报告" / "导出月报" | 报告预览+下载 |
| 深度分析 | "为什么整改率下降了" | 根因分析+建议 |

### 快捷指令

底部输入框上方提供快捷指令栏（可配置）：

```
┌─────────────────────────────────────────────────────────────┐
│ [查数据] [看提醒] [生成报告] [我的统计] [帮助]              │
└─────────────────────────────────────────────────────────────┘
```

点击快捷指令等于发送预设问题：
- **查数据**: 展示常用数据查询问题列表
- **看提醒**: "我有哪些待处理事项"
- **生成报告**: 展示报告类型选择
- **我的统计**: "我的工作情况统计"

### 消息类型

对话界面支持的消息类型：

| 消息类型 | 用途 | 示例 |
|---------|------|------|
| 文字 | 简单回复、解释说明 | "本月备案量为1,245个" |
| 数据卡片 | 单指标展示 | 数字+环比变化+趋势小图 |
| 图表 | 多维度对比、趋势 | 柱状图、折线图、饼图 |
| 洞察卡片 | 异常提醒、待办事项 | 🔴紧急隐患提醒 |
| 报告卡片 | 报告生成结果 | 报告预览+下载按钮 |
| 列表 | 详细数据列表 | 工程列表、隐患列表 |
| 操作按钮 | 快速操作入口 | [查看详情] [确认已读] |

## 五、核心能力详解

### 能力1: 智能问答

**触发方式**: 直接输入问题或点击"查数据"快捷指令

**支持问题类型**:

| 类型 | 示例 | AI响应 |
|-----|------|--------|
| 单指标 | "本月备案量" | 数字卡片+环比 |
| 对比 | "各街道巡查次数对比" | 柱状图 |
| 趋势 | "近30天隐患趋势" | 折线图 |
| 排名 | "整改率最低的5个街道" | 排行榜 |
| 明细 | "夏阳街道未巡查工程" | 列表+筛选 |
| 综合 | "对比各街道巡查覆盖率和隐患整改率" | 双轴图 |

**交互示例**:

```
用户: 各街道本月巡查覆盖率

AI: [柱状图: 各街道巡查覆盖率对比]
    全区平均: 85%
    第1名 夏阳街道: 87.5%
    第2名 盈浦街道: 86.2%
    ...
    
    需要查看覆盖率较低的街道明细吗？
    [查看明细] [导出数据]
```

### 能力2: 洞察提醒

**触发方式**:
1. 用户主动询问: "我有什么待处理事项" / "查看提醒"
2. AI主动推送: 进入助手时展示未读提醒
3. 红点提醒: 入口图标显示未读洞察数量

**洞察分级展示**:

```
用户: 我有什么待处理事项

AI: 您有 3 项待处理事项：

    🔴 紧急（1）
    ┌────────────────────────────┐
    │ 您辖区有3条隐患今日到期      │
    │ 涉及: XX花园、XX大厦...      │
    │ [立即处理]                  │
    └────────────────────────────┘
    
    🟠 重要（1）
    ┌────────────────────────────┐
    │ 本周备案量环比下降30%        │
    │ [查看详情]                  │
    └────────────────────────────┘
    
    🟡 提示（1）
    ┌────────────────────────────┐
    │ 夏阳街道巡查覆盖率排名第1    │
    │ [查看详情]                  │
    └────────────────────────────┘
```

### 能力3: 报告生成

**触发方式**: "生成本周报告" / "导出月报" / 点击"生成报告"快捷指令

**交互流程**:

```
用户: 生成报告

AI: 请选择报告类型：
    [周报] [月报] [季报] [专项报告] [自定义]
    
用户: 月报

AI: 请选择时间范围：
    [本月] [上月] [自定义范围]
    
用户: 本月

AI: 正在生成《青浦区小型工程监管月报（2026年3月）》...
    
    [报告卡片预览]
    
    [在线查看] [下载PDF] [分享]
```

**报告内容**: 同原设计（摘要+备案分析+巡查分析+整改分析+工程状态+建议）

## 七、功能点描述

### 入口与唤起

| 端 | 唤起方式 | 说明 |
|---|---------|------|
| PC | 点击悬浮球 | 右下角固定位置，可拖拽，显示未读数量角标 |
| PC | 快捷键 | 支持 Cmd/Ctrl + K 快速唤起 |
| 大屏 | 点击图标 | 导航栏或侧边栏固定入口 |
| 大屏 | 语音唤醒 | 唤醒词: "小安小安" |
| H5 | 底部导航 | 底部Tab栏"AI"入口 |
| H5 | 悬浮按钮 | 特定页面显示悬浮快捷按钮 |

### 对话交互

入口路径: 各端AI助手入口

页面名称: AI智能助手

页面功能:
1. 消息列表（滚动查看历史对话）
2. 消息类型支持：文字、数据卡片、图表、洞察卡片、报告卡片、列表、操作按钮
3. 快捷指令栏（查数据/看提醒/生成报告/我的统计/帮助）
4. 输入框（文字输入+语音输入）
5. 未读提醒提示（进入时展示未读洞察）

业务规则:
1. 对话历史：保留最近30天对话记录
2. 上下文理解：支持多轮对话，AI理解上下文
3. 数据权限：用户只能查询权限范围内数据
4. 响应时间：简单查询<2秒，复杂查询<5秒
5. 离线提示：网络异常时提示"网络连接异常，请检查网络"

### 数据查询能力

支持查询的数据维度：
- 备案数据（新增、审核、分布、趋势）
- 巡查数据（巡查次数、覆盖率、隐患发现）
- 整改数据（整改率、逾期、闭环）
- 工程状态（正常、整改、停工、核销）
- 个人统计（巡查人员的个人工作统计）

### 洞察提醒能力

洞察类型：
- 🔴 紧急：隐患今日到期、整改逾期等需立即处理的事项
- 🟠 重要：数据大幅波动、执行偏差等需关注的事项
- 🟡 提示：趋势变化、排名情况等供参考的事项

推送规则：
- 紧急：即时推送，进入助手时优先展示
- 重要：每日汇总，首次进入助手时展示
- 提示：每周汇总，主动询问"看提醒"时展示

### 报告生成能力

报告类型：
- 周报（本周数据汇总）
- 月报（月度全景分析）
- 季报（季度综合评估）
- 专项报告（特定主题分析）
- 自定义报告（自选指标和时间）

## 十一、实施建议

### 阶段规划

| 阶段 | 目标 | 功能范围 | 建议工期 |
|-----|------|---------|---------|
| 一期 | MVP | PC端基础问答 + 5个核心洞察规则 + 周报 | 4周 |

### 风险提示

1. **大模型合规**: 政府项目需确认大模型使用合规性，建议优先私有化部署
2. **数据权限**: 务必做好数据权限隔离，避免跨街镇数据泄露
3. **性能瓶颈**: 全量数据实时查询可能存在性能问题，建议预计算热点指标
4. **语音识别**: 大屏端语音识别准确率需充分测试
