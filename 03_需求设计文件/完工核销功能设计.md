## 一、功能概述

完工核销功能实现工程完工后的闭环确认，覆盖以下四个功能单元：

1. 施工方完工核销申请
2. 巡查员核销审核
3. 巡检端直接核销
4. 核销记录与工程归档

核心目标：确保每个备案工程都有明确的结束节点，核销后工程归档，停止新增巡查，但允许核销前遗留整改在15天内继续完成。

## 三、业务规则

### 3.1 通用规则

1. 核销对象是备案已审核通过并取得回执的工程。
2. 施工方仅可在工程状态为"正常施工"时申请核销；"现场整改"或"停工整改"状态下，必须整改完成后才能申请核销。监管巡查人员（街道/社区）可直接核销除"已核销"外的任何状态工程。
3. 已核销工程为终态，不可再次申请核销或直接核销。
4. 一个工程同一时间只能存在一个待审核的核销申请。
5. 施工方申请核销待审核期间，巡检人员（街道/社区）不可对该工程新增巡查记录，但不影响整改记录的继续提交和审核。
6. 核销通过后，施工方可在15天内继续提交核销前遗留整改的整改记录，逾期后整改入口关闭。15天期限从核销通过时间开始计算。
7. 核销驳回后，工程恢复为申请前工程状态，可再次申请核销。
8. 核销审核期间，工程被登记为"拒不整改/移交执法"的，核销申请自动驳回，工程进入执法流程。

### 3.2 施工方核销申请规则

**触发条件**
- 工程未处于"已核销"状态
- 当前不存在待审核的核销申请
- 工程已通过备案审核并取得回执
- 当前用户为施工方/业主身份

**处理逻辑**
1. 用户进入工程详情页，系统判断工程是否满足核销条件。
2. 仅当工程状态为"正常施工"、不存在待审核核销申请且未核销时，显示"申请完工核销"按钮。工程处于"现场整改"或"停工整改"状态时，隐藏核销按钮，整改完成后方可申请。
3. 用户点击后进入核销申请页面，填写完工日期、上传现场照片、填写备注说明。
4. 提交后系统校验必填项（完工日期、至少1张现场照片）。
5. 校验通过后生成核销申请记录，记录申请前工程状态，工程状态变更为"核销待审核"。
6. 系统向该工程的巡查员推送核销申请通知。
7. 系统同步限制巡检端（街道/社区）对该项目新增巡查记录，但不影响整改记录继续提交和审核。
8. 核销通过后，15天内允许继续提交核销前遗留整改的整改记录。

**边界情况**
- 已存在待审核核销申请：提示"您已提交核销申请，正在审核中"。
- 已核销工程：不显示核销按钮，显示"已核销"状态标签。
- 工程存在未完成整改（处于"现场整改"或"停工整改"状态）：施工方不可发起核销申请，必须整改完成后方可申请。
- 网络中断：本地暂存，恢复后自动同步或提示用户重新提交。

### 3.3 巡查员核销审核规则

**触发条件**
- 巡查员收到核销申请通知（推送/消息）
- 或主动进入"核销待审核"列表查看

**处理逻辑**
1. 巡查员进入"核销待审核"列表，查看待审核的核销申请。
2. 点击某条申请进入审核详情页，查看工程信息、完工日期、现场照片、备注。
3. 根据实际情况做出审核决定：
   - 审核通过：点击"审核通过"→二次确认→核销通过→工程状态变更为"已核销"
   - 审核驳回：点击"驳回申请"→填写驳回原因→二次确认→核销驳回→工程状态恢复为申请前工程状态
4. 审核结果通知施工方。

**边界情况**
- 施工方提交核销申请后，在待审核期间，巡检人员（街道/社区）不可对该工程新增巡查记录。
- 核销审核期间，工程被登记为"拒不整改/移交执法"：核销申请自动驳回，工程进入执法流程。
- 核销通过后仍有未完成的整改（核销前遗留）：允许施工方在15天内继续上传整改记录，巡查员继续审核待审核中的整改记录，但工程状态保持"已核销"。
- 15天期限从核销通过时间开始计算，逾期后整改入口自动关闭，整改记录不可再提交。

### 3.4 巡检端直接核销规则

**触发条件**
- 当前用户为街道或社区巡检人员
- 巡检人员在巡查或走访中发现工程实际已完工
- 工程处于任一非"已核销"状态（包括"正常施工""现场整改""停工整改"等），且不存在待审核的施工方核销申请

**处理逻辑**
1. 巡检人员进入工程详情页。
2. 系统判断当前用户角色和工程状态，满足条件时显示"直接核销"按钮。
3. 巡检人员点击后进入直接核销页面。
4. 填写是否完工、上传现场图片，补充说明为选填。
5. 系统校验必填项，且仅当"是否完工"选择为"是"时允许提交。
6. 提交成功后工程状态直接变更为"已核销"。
7. 生成核销记录，标注为"巡检端直接核销"。
8. 通知施工方工程已被核销。

**边界情况**
- 工程存在未完成整改：允许直接核销，后续整改流程继续走完。
- 工程实际未完工：当"是否完工"选择为"否"时，不允许提交直接核销。
- 工程存在待审核核销申请：不显示"直接核销"按钮，引导巡检人员进入核销审核流程处理。

### 3.5 核销记录与归档规则

**核销记录字段**
- 核销编号（系统生成，唯一）
- 工程编号（关联工程）
- 核销方式（施工方申请核销/监管巡查核销/巡检端直接核销）
- 完工日期（施工方填写或系统默认）
- 申请时间（申请核销时记录）
- 审核时间（审核通过时记录）
- 核销照片（完工现场照片）
- 备注说明
- 审核人（巡查员）
- 驳回原因（驳回时记录）
- 申请前工程状态（驳回恢复时使用）

**归档规则**
- 核销完成后，工程档案完整归档，包含备案信息、巡查记录、整改记录、核销记录。
- 核销前遗留的整改记录在核销后15天内继续提交或审核完成的，继续沉淀到该工程档案中。
- 工程状态变更为"已核销"后，从在建工程列表移除。
- 可在"已核销工程"列表中查询历史核销记录。

## 四、功能点描述

### 4.1 施工方操作流程

#### 4.1.1 核销申请入口

**入口路径**
App → 工程列表 → 工程详情 → 申请完工核销

**页面名称**："工程详情"页面

**页面功能**

1. 核销入口显示逻辑：
   - 工程状态为"正常施工"且不存在待审核核销申请时，显示"申请完工核销"按钮。
   - 工程状态为"现场整改"或"停工整改"时，隐藏"申请完工核销"按钮，整改完成后方可申请。
   - 工程状态为"核销待审核"时，显示"核销审核中"状态标签，隐藏核销按钮。
   - 工程状态为"已核销"时，显示"已核销"状态标签。

2. 核销条件判断：
   - 前置条件：备案已审核通过。
   - 必要条件：工程未核销。
   - 互斥条件：不存在待审核的核销申请。

**业务规则**
- 已核销工程为终态，不可再次申请核销。
- 施工方仅可在工程状态为"正常施工"时发起核销申请。工程处于"现场整改"或"停工整改"状态时，必须整改完成后方可申请核销。
- 施工方申请核销待审核期间，巡检人员（街道/社区）不可新增巡查记录。
- 核销待审核期间，施工方仍可上传整改记录（核销前遗留的整改），审核人员可继续审核待审核中的整改记录。
- 核销通过后，15天内允许继续提交核销前遗留整改的整改记录，逾期后整改入口关闭。
- 同一工程同一时间只能存在一个待审核的核销申请。

**字段规则**
- 状态标签显示位置：工程名称下方或右侧，根据现有工程详情页布局确定。
- "申请完工核销"按钮位置：固定在页面底部，与"新增巡查记录"按钮并排（如存在整改则显示整改按钮）。

#### 4.1.2 核销申请页面

**入口路径**
工程详情 → 点击"申请完工核销"按钮

**页面名称**："完工核销申请"页面

**页面功能**

1. 工程信息展示区域（只读）：
   - 工程名称
   - 工程地址
   - 备案编号
   - 当前状态

2. 驳回原因回显区域（条件显示）：
   - 存在历史驳回记录时显示。
   - 显示内容：上次驳回时间（精确到分钟）、驳回原因（完整显示）、引导处理文案。
   - 默认展开，不可折叠。点击"我知道了"后高亮表单区域。

3. 完工信息填写区域：
   - 完工日期（必填，日期选择器）
     - 最早可选日期：施工开始日期
     - 最晚可选日期：当前日期
     - 首次进入默认为空，重新申请时回显上次日期
     - 未填提示"请选择日期"
     - 不能晚于当前日期

   - 完工现场照片（必填，图片上传）
     - 必传照片类型：完工全景照片至少1张、现场清理照片至少1张、安全设施拆除/关闭照片至少1张
     - 至少上传1张，最多9张
     - 单张不超过10MB
     - 支持jpg、png格式
     - 支持拍照上传和相册选择
     - 支持查看大图和删除（长按已上传图片可删除）

   - 备注说明（选填，多行文本）
     - 最多200字
     - 用于补充说明完工情况

4. 操作按钮：
   - 提交申请：校验必填项，通过后提交核销申请
   - 取消/返回：返回工程详情，不保存填写内容

**业务规则**
- 完工日期不能晚于当前日期。
- 现场照片至少上传1张，最多9张。
- 提交前进行完整校验，未填项高亮提示。
- 提交过程中显示loading，防止重复提交。
- 提交成功后工程状态变为"核销待审核"，toast提示"申请已提交"。
- 提交成功后返回工程详情，显示核销申请状态。
- 提交成功后系统记录申请前工程状态，用于驳回时恢复。

**待补充内容**
1. 现场照片的具体拍摄要求（如：需包含哪些场景？）
2. 照片压缩的具体参数
3. 核销申请提交后，巡查员接收通知的方式（推送/短信/站内信？）
4. 核销申请的审核时限要求

### 4.2 巡查员操作流程

#### 4.2.1 核销待审核列表

**入口路径**
App → 工作台 → 核销待审核（快捷入口）
或：App → 工程列表 → 筛选"核销待审核"

**页面名称**："核销待审核"页面

**页面功能**

1. 列表展示：
   - 按申请时间倒序排列。
   - 每条显示：工程名称、申请时间、申请人、工程地址、现场照片缩略图（最多3张）。
   - 点击卡片进入核销审核详情。

2. 空状态：
   - 无待审核核销申请时显示空状态页面。
   - 提示"暂无待审核的核销申请"。

3. 下拉刷新：
   - 支持下拉刷新列表。

**业务规则**
- 只显示当前巡查员管辖区域内的核销申请。
- 按申请时间倒序，先申请的先处理。
- 列表实时更新，避免已处理的申请仍显示。

**字段规则**
- 申请时间显示精确到日（格式：YYYY-MM-DD）。
- 申请人身份显示为"施工方"或"业主"。
- 现场照片缩略图最多显示3张，超过3张显示"+N"。

**待补充内容**

2. 是否支持按街镇/时间范围筛选？

#### 4.2.2 核销审核详情

**入口路径**
核销待审核列表 → 点击核销申请卡片

**页面名称**："核销审核详情"页面

**页面功能**

1. 工程信息区域（只读）：
   - 工程名称
   - 工程地址
   - 备案编号
   - 工程类型
   - 施工开始日期
   - 施工单位

2. 核销申请信息区域（只读）：
   - 完工日期
   - 申请时间（精确到秒）
   - 申请人/申请单位
   - 完工现场照片（可点击查看大图，支持左右滑动浏览）
   - 备注说明

3. 操作按钮区域：
   - 驳回申请：点击弹出驳回原因输入框
   - 审核通过：点击弹出二次确认弹窗

4. 驳回原因输入弹窗：
   - 多行文本输入框
   - 最多200字
   - 取消/确认按钮
