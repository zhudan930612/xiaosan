## 一、功能概述

在移动端内部政务平台中，将原本分散的"备案审核"和"安全巡检"两个独立入口整合为统一的"小型工程"入口。用户点击进入后，再按需选择"备案审核"或"安全巡检"功能模块。

**核心价值**：
- 对审核/巡查人员：减少政务平台首页入口数量，降低认知负担，统一业务查找路径
- 对平台管理：与现有产品架构对齐，备案和巡查同属"小型工程"业务域，统一入口更符合业务分类
- 对后续扩展：为未来可能新增的"完工核销"、"智慧大屏"等小型工程子模块预留统一的归集入口

**设计依据**：本产品备案管理和日常巡查两个模块均属于小型工程全生命周期管理（备案→施工监管→完工销项），在产品上下文中明确属于同一业务域。

## 二、业务流程

### 2.1 流程图

```
用户登录内部政务平台（移动端）
            │
            ▼
    在常用服务/工作台找到
    【小型工程】入口图标
            │
            ▼
    进入"小型工程"统一入口页
    ┌─────────────────────┐
    │                     │
    │   备案审核          │
    │   （点击跳转）       │
    │                     │
    │   安全巡检          │
    │   （点击跳转）       │
    │                     │
    └─────────────────────┘
            │
    ┌───────┴───────┐
    ▼               ▼
备案审核模块    安全巡检模块
（项目中心等）  （巡检首页等）
```

### 2.2 业务规则

1. 页面定位：本页面为纯跳转/导航页面，不承载任何业务数据展示和审批操作
2. 权限控制：
   - 若用户仅有备案审核权限，页面应仍可进入，但可隐藏"安全巡检"入口
   - 若用户仅有安全巡检权限，页面可隐藏"备案审核"入口
   - 若用户同时具有两项权限，两个入口均正常显示
3. 点击行为：点击入口卡片后，直接跳转至对应模块的首页
4. 无网络/异常处理：页面加载失败时显示标准网络异常提示，支持点击重试
5. 角色适配：审核人员和巡查人员可能使用同一账号，入口不做强制区分，由后端根据用户权限决定各入口的显示状态

## 四、功能点描述

### 4.1 入口图标（政务平台首页/工作台）

**入口路径**：内部政务平台（移动端）→ 常用服务/工作台 → "小型工程"图标

**图标设计建议**：
- 图标名称：小型工程
- 图标风格：与政务平台现有图标风格保持一致
- 建议图标元素：建筑/工程相关图标（如房屋+安全帽组合）

**业务规则**：
1. 该入口替代原有的"备案审核"和"安全巡检"两个独立图标
2. 原有两个入口的权限校验逻辑不变，仅前置到本统一入口的下一级

### 4.2 小型工程统一入口页

**页面名称**：小型工程

**页面结构**：

| 区域 | 元素 | 说明 |
|------|------|------|
| 顶部导航栏 | 返回按钮 + 页面标题"小型工程" | 标准移动端页面头部 |
| 内容区 | 备案审核入口卡片 | 大卡片，含图标、主标题、副标题 |
| 内容区 | 安全巡检入口卡片 | 大卡片，含图标、主标题、副标题 |

**备案审核卡片**：
- 图标：文件/审批类图标（建议使用蓝色系）
- 主标题：备案审核
- 副标题：工程备案资料审核与管理
- 点击行为：跳转至备案审核模块的首页（即当前的"项目中心"页面）

**安全巡检卡片**：
- 图标：盾牌/巡查类图标（建议使用绿色系）
- 主标题：安全巡检
- 副标题：工程巡查、隐患登记与整改跟踪
- 点击行为：跳转至安全巡检模块的首页（即设计中的"巡查概览/巡检首页"页面）

**交互规则**：
1. 卡片支持点击反馈（如点击时轻微缩放或背景色变化）
2. 页面支持下拉刷新（虽然本页无动态数据，但保持交互一致性）
3. 若用户无某模块权限，对应卡片不显示

**业务规则**：
1. 两个入口为平行关系，无主从之分
2. 备案审核入口的跳转目标为现有"项目中心"页面
3. 安全巡检入口的跳转目标为现有设计中"安全巡检首页（巡查概览）"页面
4. 各模块内部的权限控制、数据过滤、业务逻辑均保持原有逻辑，不因入口层级变化而调整

## 五、数据权限规则

### 5.1 入口可见性

| 权限情况 | 页面表现 |
|---------|---------|
| 仅有备案审核权限 | 显示"备案审核"卡片，隐藏"安全巡检"卡片 |
| 仅有安全巡检权限 | 显示"安全巡检"卡片，隐藏"备案审核"卡片 |
| 两项权限均有 | 两个卡片均正常显示 |
| 两项权限均无 | 统一入口图标不显示（或点击后提示无权限） |

### 5.2 跳转权限检查

1. 进入统一入口页时，后端返回当前用户可见的子模块列表
2. 前端根据返回列表渲染对应入口卡片
3. 点击入口后，各子模块内部仍执行原有的权限校验逻辑

## 六、待确认事项

| 序号 | 待确认内容 | 影响范围 | 建议 |
|------|-----------|---------|------|
| 1 | 无权限入口的展示方式（隐藏 vs 置灰提示） | 页面交互设计 | 建议隐藏，减少干扰 |
| 2 | 是否需要在入口页展示待办数量红点 | 页面复杂度、后端接口 | 首期建议不做，纯跳转页 |
| 3 | 原有首页的两个独立图标是否完全下线 | 产品迭代策略 | 建议完全下线，避免入口重复 |

---

## 七、验收标准

1. 用户在政务平台首页能看到"小型工程"入口图标，点击后进入统一入口页
2. 统一入口页包含"备案审核"和"安全巡检"两个可点击入口
3. 点击"备案审核"能正确跳转至备案管理的"项目中心"页面
4. 点击"安全巡检"能正确跳转至安全巡检的"巡检首页/巡查概览"页面
5. 用户仅有单项权限时，页面只展示有权限的入口
6. 页面样式与政务平台现有移动端风格保持一致
7. 返回按钮能正确返回政务平台首页/工作台
8. 页面加载异常时有标准的错误提示
9. 两个入口的点击区域足够大，适合移动端单手操作
10. 备案审核和安全巡检模块内部的原有功能不受入口层级调整的影响
