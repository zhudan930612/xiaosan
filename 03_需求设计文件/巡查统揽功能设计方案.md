## 一、模块概述

### 1.1 业务场景

巡查统揽是青浦区小型工程综合管理平台智慧大屏的核心监管模块，为区级和街镇管理人员提供巡查执行情况的全面可视化展示。通过巡查覆盖率、隐患整改率等核心指标，帮助管理人员掌握巡查工作执行力度和隐患闭环处理情况，支撑监管决策和考核评价。

### 1.2 核心价值

| 用户角色 | 核心价值 |
|---------|---------|
| 区级管理人员 | 全局巡查态势掌控，掌握各街镇巡查执行差异，发现巡查盲区 |
| 街镇管理人员 | 本辖区巡查数据查看，跟踪隐患整改闭环，提升监管效率 |

### 1.3 主要功能

| 功能类别 | 功能名称 | 状态 |
|---------|---------|------|
| 数据统揽 | 巡查统揽入口卡片 | ⏳ 待上线 |
| 数据统揽 | 巡查统揽专题页 | ⏳ 待上线 |
| 数据展示 | 巡查覆盖率统计 | ⏳ 待上线 |
| 数据展示 | 隐患整改率统计 | ⏳ 待上线 |
| 数据展示 | 巡查趋势分析 | ⏳ 待上线 |
| 数据展示 | 工程状态分布 | ⏳ 待上线 |
| 数据展示 | 隐患分布统计 | ⏳ 待上线 |
| 地图展示 | 巡查状态点位标识 | ⏳ 待上线 |
| 明细查询 | 巡查记录列表 | ⏳ 待上线 |
| 明细查询 | 隐患管理列表 | ⏳ 待上线 |
| 明细查询 | 工程巡查详情 | ⏳ 待上线 |

### 1.4 适用范围

- **使用端**：PC端大屏展示
- **使用场景**：区级/街镇监管中心大屏展示、领导视察、日常监管、巡查考核

### 1.5 受众角色

| 角色 | 使用场景 | 数据权限 |
|------|---------|---------|
| 区级管理人员 | 查看全区巡查态势、各街镇巡查执行对比、发现巡查盲区 | 全区所有巡查数据 |
| 街镇管理人员 | 查看本辖区巡查数据、跟踪隐患整改进度、管理巡查人员 | 管辖区域内巡查数据 |

### 1.6 业务价值

- **巡查执行可视**：一屏掌握全区巡查工作执行情况，发现未覆盖区域
- **隐患闭环跟踪**：从发现到整改的全流程跟踪，确保安全隐患得到处理
- **街镇对比分析**：通过排名和趋势对比，促进各街镇巡查工作平衡发展
- **考核数据支撑**：提供巡查覆盖率、整改率等考核所需的核心数据

## 二、基础信息

### 2.1 模块入口

**入口路径**：监管PC端 → 智慧大屏 → 点击"巡查统揽"卡片

```
智慧大屏首页

    → 巡查统揽专题页
      → 右侧弹层（街道/工程快览）
        → 工程详情页
```

### 2.2 全局筛选器

#### 2.2.1 时间维度切换

**位置**：巡查统揽专题页顶部Tab栏

**可选维度**：
- 今日
- 本周
- 近七日
- 本月

**默认选中**：本周

**交互规则**：
- 切换时间维度后，下方所有数据自动刷新
- 用户离开页面后重新进入，默认恢复"本周"维度

**业务规则**：
1. 时间维度定义：
   - 今日：当日0:00至当前时间
   - 本周：本周一0:00至当前时间
   - 近七日：前7天（含今天）
   - 本月：本月1日0:00至当前时间
2. 数据计算基准：
   - 巡查记录以提交时间为准
   - 隐患记录以发现时间为准
   - 整改记录以审核通过时间为准
3. 切换时间维度后，所有关联图表和表格联动更新
4. 首页入口卡片默认不跟随专题页时间切换，固定展示首页默认口径；专题页内部全部跟随当前选中时间维度

#### 2.2.2 街道筛选

**位置**：页面顶部右侧下拉框

**可选范围**：
- 全部街道
- 夏阳街道
- 盈浦街道
- 香花桥街道
- 朱家角镇
- 练塘镇
- 金泽镇
- 赵巷镇
- 徐泾镇
- 华新镇
- 重固镇
- 白鹤镇

**交互规则**：
- 区级管理人员：可选择全部街道或任意单个街道
- 街镇管理人员：根据管辖区域限制，只能查看管辖区域内的街道
- 切换街道后，数据自动刷新，地图聚焦到对应区域

**业务规则**：
1. 街道筛选与时间维度筛选是组合关系，数据需同时满足两个筛选条件
2. 选择"全部街道"时，展示用户权限范围内所有街道的数据汇总
3. 选择单条街道时，如该街道不在用户管辖范围内，提示"无权限查看"
4. 街道筛选状态在用户会话期间保持，关闭页面后重置

#### 2.2.3 组织维度筛选

**位置**：页面顶部筛选栏

**可选类型**：
- 全部
- 街道巡查
- 社区巡查
- ~~物业巡查~~（暂不支持，物业角色未接入系统）

**交互规则**：
- 切换组织维度后，巡查趋势图、巡查记录列表等联动刷新
- 仅支持街道巡查和社区巡查两类

**业务规则**：
1. 组织维度用于筛选巡查记录的发起方类型
2. 支持多选，多选时为"或"关系
3. 本期暂不支持物业巡查，未来物业角色接入后再扩展

### 2.3 工程状态定义

| 工程状态 | 定义 | 判定条件 |
|---------|------|---------|
| 正常施工 | 工程已备案通过，无未闭环整改事项 | 备案状态=已通过，且不存在状态为"待整改"或"整改中"的隐患 |
| 现场整改 | 工程存在现场整改级别隐患且未审核通过 | 存在隐患等级=一般，整改状态=待整改或整改中 |
| 停工整改 | 工程存在停工整改级别隐患且未审核通过 | 存在隐患等级=较大，整改状态=待整改或整改中 |
| 已核销 | 工程已完成完工核销 | 工程状态=已完工，且已完成核销手续 |

**优先级规则**：同一工程若同时满足多种状态，以优先级高的状态为准。
状态优先级：已核销 > 停工整改 > 现场整改 > 正常施工

## 三、详细功能描述

### 3.1 首页巡查统揽入口

**信息来源**：基于产品规划“可视化大屏 > 巡查统揽指标”、大屏巡查数据模块设计 3.1-3.3、安全巡检功能设计 2.1/2.5

**页面名称**：智慧大屏首页 - 巡查统揽卡片

**入口路径**：智慧大屏首页 → 四大统揽卡片区域

**位置**：页面顶部，横向排列4个卡片中的第3个

#### 3.1.1 卡片内容

**展示指标**：

| 指标名称 | 数值示例 | 说明 |
|---------|---------|------|
| 本周巡查覆盖率 | 86.5% | 本周有巡查记录的在建工程占比 |
| 本周隐患整改率 | 72.3% | 本周发现隐患中已完成审核通过整改的占比 |
| 本月巡查次数 | 1,258 | 当前月累计提交的巡查记录数 |
| 待整改隐患数 | 43 | 当前时点状态为待整改或整改中的隐患总数 |

**交互规则**：
- 点击卡片任意区域，进入"巡查统揽专题页"
- 鼠标悬停显示指标计算口径说明
- 首页卡片默认展示真实数据，不再显示"即将上线"占位符
- 点击进入专题页后，默认带入"本周 + 全部街道 + 全部组织"筛选条件

**业务规则**：
1. 数据计算口径：
   - 巡查覆盖率 = 本周有巡查记录的在建工程数 / 当前在建工程总数 × 100%
   - 本周隐患整改率 = 本周审核通过整改的隐患数 / 本周新发现隐患总数 × 100%
   - 本月巡查次数 = 本月成功提交的巡查记录数
   - 待整改隐患数 = 当前时点状态为"待整改"或"整改中"的隐患总数
2. 在建工程定义：状态为"正常施工"、"现场整改"、"停工整改"的工程
3. 首页卡片的时间口径固定，不跟随专题页内部时间切换：
   - 巡查覆盖率、本周隐患整改率固定取本周
   - 本月巡查次数固定取本月
   - 待整改隐患数固定取当前时点
4. 数据刷新频率：
   - 巡查覆盖率、本周隐患整改率、本月巡查次数：每小时自动刷新
   - 待整改隐患数：准实时刷新（5分钟延迟）
5. 异常处理：
   - 分母为0时展示"--"，不展示0%或100%
   - 本周未发现隐患时，本周隐患整改率展示"--"，提示"本周暂无新增隐患"
   - 待整改隐患数为0时直接展示0，不展示"--"

### 3.2 巡查统揽专题页

**信息来源**：基于产品规划“可视化大屏”章节、大屏巡查数据模块设计 3-7、安全巡检功能设计 2.1-2.5

**页面名称**："巡查统揽"专题页

**入口路径**：智慧大屏首页 → 点击"巡查统揽"卡片

**页面布局**：
- 顶部：返回按钮、页面标题、刷新数据按钮、时间维度切换Tab、街道筛选、组织维度筛选
- 第一排：6张核心指标卡
- 第二排：左侧街道排名 + 右侧趋势分析图
- 第三排：左侧地图巡查状态区 + 右侧工程状态分布/隐患分布区
- 底部：街道数据明细表

**补充说明**：

#### 3.2.1 核心指标卡片

**位置**：页面顶部Tab下方，横向排列6个指标卡片

**指标内容**：

| 指标名称 | 数值示例 | 环比变化 | 数据来源 |
|---------|---------|---------|---------|
| 巡查次数 | 1,258 | 较上周 ↑15.2% | 巡查记录表 |
| 巡查覆盖率 | 86.5% | 较上周 ↑5.3% | 巡查记录+工程表 |
| 隐患总数 | 227 | 较上周 ↑8.1% | 隐患记录表 |
| 累计隐患整改率 | 81.1% | 较上周 ↑3.2% | 隐患整改记录表 |
| 待整改隐患数 | 43 | 较上周 ↓12.5% | 隐患记录表 |
| 停工整改工程数 | 8 | 较上周 ↑2 | 工程状态表 |

**交互规则**：
- 点击指标卡片可下钻查看对应明细列表
- 鼠标悬停显示指标详细计算口径
- 环比变化增长显示绿色↑，下降显示红色↓，持平显示灰色-

**业务规则**：
1. 核心指标定义：
   - 巡查次数：当前时间维度内成功提交的巡查记录数
   - 巡查覆盖率：当前时间维度内有巡查记录的在建工程数 / 当前在建工程总数
   - 隐患总数：当前时间维度内新发现的隐患数
   - 累计隐患整改率：截至当前已整改的隐患数 / 截至当前发现的隐患总数
   - 待整改隐患数：当前时点状态为"待整改"或"整改中"的隐患总数
   - 停工整改工程数：当前时点状态为"停工整改"的工程数
2. 环比计算：与上一周期对比
   - 今日对比昨日
   - 本周对比上周
   - 近七日对比前七日
   - 本月对比上月
3. 数据更新频率：
   - 巡查次数、隐患数：准实时（5分钟延迟）
   - 覆盖率、整改率：每小时计算一次

## 四、数据权限规则

### 4.1 权限控制逻辑

**控制维度**：
1. **管辖区域**：根据用户所属组织的管辖区域限制可查看的街道
2. **工程类型**：根据用户权限设置可查看的工程类型

### 4.2 数据可见性判断

```
IF 用户角色 = "区级管理人员" THEN
    可见全部巡查数据
ELSE IF 用户角色 = "街镇管理人员" THEN
    IF 工程所属街道 IN 用户管辖街道列表 THEN
        IF 工程类型 IN 用户可查看工程类型列表 THEN
            可见该工程巡查数据
        ELSE
            不可见
        END IF
    ELSE
        不可见
    END IF
END IF
```

### 4.3 权限生效规则

**数据过滤逻辑**：
- 管辖区域：用户只能查看管辖区域内的工程巡查数据
- 工程类型：用户只能查看已授权的工程类型的巡查数据
- 组合条件：同时满足管辖区域和工程类型条件的数据才可见

**示例**：
- 某监管单位配置了"青浦区"管辖区域 + "小型工程、住宅装修"工程类型
- 该用户登录后：
  - 只能看到青浦区内的巡查数据
  - 只能看到小型工程和住宅装修类型的巡查数据
  - 零星作业类型的巡查数据不可见

## 五、业务规则详解

### 5.1 指标计算口径

#### 5.1.1 巡查覆盖率计算

**公式**：
```
巡查覆盖率 = 当前时间维度内有巡查记录的在建工程数 / 当前在建工程总数 × 100%
```

**计算规则**：
1. 分子：当前时间维度内至少有1条有效巡查记录的在建工程数
2. 分母：当前在建工程总数（状态=正常施工|现场整改|停工整改）
3. 专题页内巡查覆盖率跟随当前时间维度切换：今日/本周/近七日/本月
4. 首页入口卡片的巡查覆盖率固定展示本周口径，不跟随专题页切换
5. 时间筛选：如大屏统一时间筛选另有定义，以统一筛选为准

**异常处理**：
1. 分母为0时展示"--"，不展示0%或100%
2. 巡查频次配置未正式接入前，本期只判断"当前统计口径下是否被巡查过"，不判断"是否达到应巡查次数"

#### 5.1.2 本周隐患整改率计算

**公式**：
```
本周隐患整改率 = 本周审核通过整改的隐患数 / 本周新发现隐患总数 × 100%
```

**计算规则**：
1. 首页入口卡片固定展示本周口径
2. 分子：当前自然周内审核通过整改的隐患数
3. 分母：当前自然周内新发现的隐患数
4. 统计范围：当前用户权限范围内、且满足首页展示范围的隐患数据

**异常处理**：
1. 本周未发现隐患时展示"--"，提示"本周暂无新增隐患"
2. 本周发现隐患但尚无整改通过时展示0%

#### 5.1.3 累计隐患整改率计算

**公式**：
```
累计隐患整改率 = 截至当前已整改的隐患数 / 截至当前发现的隐患总数 × 100%
```

**计算规则**：
1. 分子：截至当前时点，整改状态为"已整改"的隐患总数（含历史上各时间发现的隐患）
2. 分母：截至当前时点，系统内登记过的隐患总数
3. 统计范围：当前用户权限范围内的所有隐患数据

**异常处理**：
1. 分母为0时展示"--"，提示"暂无隐患数据"
2. 分子可能小于、等于或大于当周期新整改数，但永远不会超过分母

#### 5.1.4 环比变化计算

**公式**：
```
环比变化率 = (本期数值 - 上期数值) / 上期数值 × 100%
```

**计算规则**：
1. 本期为今日，则上期为上一天（昨日）
2. 本期为本周，则上周为上一个自然周（周一到周日）
3. 本期为近七日，则上期为前七日
4. 本期为本月，则上月为上一个自然月
5. 上期数值为0时，环比显示"--"（避免除以0）
6. 增长时显示绿色↑，下降时显示红色↓，持平显示灰色-

### 5.2 状态定义与流转

#### 5.2.1 工程状态定义

| 工程状态 | 口径定义 | 信息来源 |
|---------|---------|---------|
| 正常施工 | 无未闭环整改事项 | 安全巡检功能设计 |
| 现场整改 | 存在现场整改且未审核通过 | 安全巡检功能设计 |
| 停工整改 | 存在停工整改且未审核通过 | 安全巡检功能设计 |
| 已核销 | 已完工核销的工程 | 安全巡检功能设计 |

**优先级规则**：同一工程若同时存在现场整改和停工整改记录，以"停工整改"计入统计。
已核销规则：已核销工程不再计入"在建工程总数"，但在工程状态分布图中保留独立维度。

#### 5.2.2 隐患等级定义

| 隐患等级 | 定义 | 判定标准 |
|---------|------|---------|
| 较大隐患 | 需停工整改的隐患 | 可能导致人员伤亡或重大财产损失 |
| 一般隐患 | 现场整改或不需要限时整改的隐患 | 一般性安全隐患，可现场纠正 |

#### 5.2.3 整改状态流转

| 状态 | 定义 |
|-----|------|
| 待整改 | 已下发整改要求，施工方尚未提交整改记录 |
| 整改中 | 施工方已提交整改记录，待审核 |
| 已整改 | 审核通过，整改完成 |

**流转规则**：
- 待整改 → 整改中（提交整改记录）
- 整改中 → 已整改（审核通过）
- 整改中 → 待整改（审核不通过，退回）

### 5.3 数据刷新规则

| 数据类型 | 刷新频率 | 说明 |
|---------|---------|------|
| 巡查记录 | 准实时 | 巡查提交后5分钟内更新 |
| 隐患记录/整改记录 | 准实时 | 隐患登记、整改提交或审核后5分钟内更新 |
| 首页入口卡片 | 每小时 | 覆盖率、整改率、巡查次数按小时刷新；待整改隐患数按准实时更新 |
| 专题页核心指标卡 | 每小时 | 覆盖率、累计整改率按小时刷新；待整改隐患数、停工整改工程数按准实时更新 |
| 趋势图表/分布图 | 每小时 | 采用定时任务汇总，避免大屏频繁抖动 |
| 街道排名/明细表 | 每小时 | 与专题页统计口径保持一致 |
| 地图点位状态 | 准实时 | 点位状态、弹层摘要按5分钟延迟更新 |

### 5.4 地图点位颜色规则

| 标识 | 含义 | 显示条件 |
|-----|------|---------|
| 🔴 红色闪烁 | 存在未闭环隐患 | 工程下存在"待整改"或"整改中"隐患 |
| 🟡 黄色圆点 | 整改中 | 工程下存在待审核整改记录，但无停工级待整改 |
| 🟢 绿色圆点 | 近7天已巡查 | 近7天有巡查记录，且无未闭环隐患 |
| ⚪ 灰色圆点 | 近7天未巡查 | 近7天无巡查记录，且无未闭环隐患 |

**优先级规则**：红色 > 黄色 > 绿色 > 灰色

### 5.5 隐患类别统计口径

**统计维度**：按"隐患类别"聚合，不按单条隐患描述聚合

**隐患类别清单**（基于安全隐患类型清单）：
- 信息登记
- 安全管理
- 高处作业
- 施工用电
- 消防安全
- 脚手架

**排序规则**：按隐患数量倒序取前5；数量相同时按最近一次发现时间倒序

**兜底规则**：未能映射到标准隐患类别的记录统一归入"其他/待映射"

## 七、决策记录

| 决策内容 | 选择方案 | 决策依据/原因 |
|---------|---------|--------------|
| 巡查统揽模块的入口形式 | 智慧大屏首页卡片 + 专题页 | 与现有实时统揽、备案统揽模块保持一致的交互模式，降低学习成本 |
| 首页卡片展示方式 | 4个真实指标 + 点击进入专题页 | 与产品规划及大屏巡查数据模块设计保持一致，避免首页长期停留在占位态 |
| 时间维度默认值 | 本周 | 巡查工作以周为单位进行考核，"本周"数据对管理人员最有参考价值 |
| 物业巡查是否纳入本期 | 暂不纳入 | 物业角色暂未接入系统，纳入后会导致数据不完整和交互逻辑悬空 |
| 工程状态优先级 | 已核销 > 停工整改 > 现场整改 > 正常施工 | 参照安全巡检功能设计中的状态定义，确保统计口径一致 |
| 巡查覆盖率计算方式 | 本周有被巡查过的在建工程占比 | 巡查频次配置未正式接入前，暂时无法判断"是否达到应巡查次数"，先以"是否被巡查过"为口径 |
| 地图点位颜色规则 | 红 > 黄 > 绿 > 灰 | 红色优先标识风险（未闭环隐患），绿色标识健康状态（近7天已巡查且无隐患），灰色标识盲区 |
| 专题页下钻形式 | 统一采用列表页/详情页链路，弹层仅承接快览 | 避免将全部明细塞入弹层，保证专题页、明细页、工程详情页链路清晰 |
