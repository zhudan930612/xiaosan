## 一、模块概述

### 1.1 业务场景

巡查统揽是青浦区小型工程安全纳管系统智慧大屏的核心监管模块，为区级和街镇管理人员提供巡查执行情况的全面可视化展示。通过巡查覆盖率、隐患整改率等核心指标，帮助管理人员掌握巡查工作执行力度和隐患闭环处理情况，支撑监管决策和考核评价。

### 1.2 核心价值

| 用户角色 | 核心价值 |
|---------|---------|
| 区级管理人员 | 全局巡查态势掌控，掌握各街镇巡查执行差异，发现巡查盲区 |
| 街镇管理人员 | 本辖区巡查数据查看，跟踪隐患整改闭环，提升监管效率 |

### 1.3 主要功能

| 功能类别 | 功能名称 | 状态 |
|---------|---------|------|
| 数据统揽 | 巡查统揽入口卡片 | ⏳ 待上线 |
| 数据统揽 | 巡查统揽专题页 | ⏳ 待上线 |
| 数据展示 | 巡查覆盖率统计 | ⏳ 待上线 |
| 数据展示 | 隐患整改率统计 | ⏳ 待上线 |
| 数据展示 | 巡查趋势分析 | ⏳ 待上线 |
| 数据展示 | 工程状态分布 | ⏳ 待上线 |
| 数据展示 | 隐患分布统计 | ⏳ 待上线 |
| 地图展示 | 巡查状态点位标识 | ⏳ 待上线 |
| 明细查询 | 巡查记录列表 | ⏳ 待上线 |
| 明细查询 | 隐患管理列表 | ⏳ 待上线 |
| 明细查询 | 工程巡查详情 | ⏳ 待上线 |

### 1.4 适用范围

- **使用端**：PC端大屏展示
- **使用场景**：区级/街镇监管中心大屏展示、领导视察、日常监管、巡查考核

### 1.5 受众角色

| 角色 | 使用场景 | 数据权限 |
|------|---------|---------|
| 区级管理人员 | 查看全区巡查态势、各街镇巡查执行对比、发现巡查盲区 | 全区所有巡查数据 |
| 街镇管理人员 | 查看本辖区巡查数据、跟踪隐患整改进度、管理巡查人员 | 管辖区域内巡查数据 |

### 1.6 业务价值

- **巡查执行可视**：一屏掌握全区巡查工作执行情况，发现未覆盖区域
- **隐患闭环跟踪**：从发现到整改的全流程跟踪，确保安全隐患得到处理
- **街镇对比分析**：通过排名和趋势对比，促进各街镇巡查工作平衡发展
- **考核数据支撑**：提供巡查覆盖率、整改率等考核所需的核心数据

## 二、基础信息

### 2.1 模块入口

**入口路径**：监管PC端 → 智慧大屏 → 点击"巡查统揽"卡片

**页面层级结构**：
```
智慧大屏首页
  → 巡查统揽入口卡片
    → 巡查统揽专题页
      → 右侧弹层（街道/工程快览）
        → 工程详情页
```

### 2.2 全局筛选器

#### 2.2.1 时间维度切换

**位置**：巡查统揽专题页顶部Tab栏

**可选维度**：
- 今日（当前选中）
- 本周
- 近七日
- 本月

**交互规则**：
- 切换时间维度后，下方所有数据自动刷新
- 默认显示"本周"数据

**业务规则**：
1. 时间维度定义：
   - 今日：当日0:00至当前时间
   - 本周：本周一0:00至当前时间
   - 近七日：前7天（含今天）
   - 本月：本月1日0:00至当前时间
2. 数据计算基准：
   - 巡查记录以提交时间为准
   - 隐患记录以发现时间为准
   - 整改记录以审核通过时间为准
3. 切换时间维度后，所有关联图表和表格联动更新
4. 用户离开页面后重新进入，默认恢复"本周"维度

#### 2.2.2 街道筛选

**位置**：页面顶部右侧下拉框

**可选范围**：
- 全部街道
- 夏阳街道
- 盈浦街道
- 香花桥街道
- 朱家角镇
- 练塘镇
- 金泽镇
- 赵巷镇
- 徐泾镇
- 华新镇
- 重固镇
- 白鹤镇

**交互规则**：
- 区级管理人员：可选择全部街道或任意单个街道
- 街镇管理人员：根据管辖区域限制，只能查看管辖区域内的街道
- 切换街道后，数据自动刷新，地图聚焦到对应区域

**业务规则**：
1. 街道筛选与时间维度筛选是组合关系，数据需同时满足两个筛选条件
2. 选择"全部街道"时，展示用户权限范围内所有街道的数据汇总
3. 选择单条街道时，如该街道不在用户管辖范围内，提示"无权限查看"
4. 街道筛选状态在用户会话期间保持，关闭页面后重置

#### 2.2.3 组织维度筛选

**位置**：页面顶部筛选栏

**可选类型**：
- 全部
- 街道巡查
- 社区巡查
- ~~物业巡查~~（暂不支持，物业角色未接入系统）

**交互规则**：
- 切换组织维度后，巡查趋势图、巡查记录列表等联动刷新
- 仅支持街道巡查和社区巡查两类

**业务规则**：
1. 组织维度用于筛选巡查记录的发起方类型
2. 支持多选，多选时为"或"关系
3. 本期暂不支持物业巡查，未来物业角色接入后再扩展

### 2.3 工程状态定义

| 工程状态 | 定义 | 判定条件 |
|---------|------|---------|
| 正常施工 | 工程已备案通过，无未闭环整改事项 | 备案状态=已通过，且不存在状态为"待整改"或"整改中"的隐患 |
| 现场整改 | 工程存在现场整改级别隐患且未审核通过 | 存在隐患等级=一般，整改状态=待整改或整改中 |
| 停工整改 | 工程存在停工整改级别隐患且未审核通过 | 存在隐患等级=较大，整改状态=待整改或整改中 |
| 已核销 | 工程已完成完工核销 | 工程状态=已完工，且已完成核销手续 |

**优先级规则**：同一工程若同时满足多种状态，以优先级高的状态为准。
状态优先级：已核销> 停工整改 > 现场整改 > 正常施工

## 三、详细功能描述

### 3.1 首页巡查统揽入口

**页面名称**：智慧大屏首页 - 巡查统揽卡片

**入口路径**：智慧大屏首页 → 四大统揽卡片区域

**位置**：页面顶部，横向排列4个卡片中的第3个

#### 3.1.1 卡片内容

**展示指标**：

| 指标名称 | 数值示例 | 说明 |
|---------|---------|------|
| 本周巡查覆盖率 | 86.5% | 本周有巡查记录的在建工程占比 |
| 本周隐患整改率 | 72.3% | 本周发现隐患的闭环完成比例 |

**交互规则**：
- 点击卡片任意区域，进入"巡查统揽专题页"
- 鼠标悬停显示指标计算口径说明
- 卡片显示"即将上线"占位符（当前状态），上线后显示真实数据

**业务规则**：
1. 数据计算口径：
   - 巡查覆盖率 = 本周有巡查记录的在建工程数 / 当前在建工程总数 × 100%
   - 隐患整改率 = 本周整改审核通过的隐患数 / 本周新发现隐患总数 × 100%
2. 在建工程定义：状态为"正常施工"、"现场整改"、"停工整改"的工程
3. 本周定义：自然周（周一至周日）
4. 数据刷新频率：每小时自动刷新
5. 异常处理：
   - 分母为0时展示"--"，不展示0%或100%
   - 本周未发现隐患时，整改率展示"--"，提示"本周暂无新增隐患"

### 3.2 巡查统揽专题页

**页面名称**："巡查统揽"专题页

**入口路径**：智慧大屏首页 → 点击"巡查统揽"卡片

**页面布局**：
- 顶部：返回按钮、页面标题、刷新数据按钮、时间维度切换Tab
- 第一排：6张核心指标卡
- 中部：左侧街道排名 + 右侧趋势分析图
- 底部：街道数据明细表

#### 3.2.1 核心指标卡片

**位置**：页面顶部Tab下方，横向排列6个指标卡片

**指标内容**：

| 指标名称 | 数值示例 | 环比变化 | 数据来源 |
|---------|---------|---------|---------|
| 巡查次数 | 1,258 | 较上周 ↑15.2% | 巡查记录表 |
| 巡查覆盖率 | 86.5% | 较上周 ↑5.3% | 巡查记录+工程表 |
| 隐患总数 | 227 | 较上周 ↑8.1% | 隐患记录表 |
| 隐患整改率 | 81.1% | 较上周 ↑3.2% | 隐患整改记录表 |
| 待整改隐患数 | 43 | 较上周 ↓12.5% | 隐患记录表 |
| 停工整改工程数 | 8 | 较上周 ↑2 | 工程状态表 |

**交互规则**：
- 点击指标卡片可下钻查看对应明细列表
- 鼠标悬停显示指标详细计算口径
- 环比变化增长显示绿色↑，下降显示红色↓，持平显示灰色-

**业务规则**：
1. 核心指标定义：
   - 巡查次数：当前时间维度内成功提交的巡查记录数
   - 巡查覆盖率：当前时间维度内有巡查记录的在建工程数 / 在建工程总数
   - 隐患总数：当前时间维度内新发现的隐患数
   - 隐患整改率：当前时间维度内已整改隐患数 / 发现隐患总数
   - 待整改隐患数：当前时点状态为"待整改"或"整改中"的隐患总数
   - 停工整改工程数：当前时点状态为"停工整改"的工程数
2. 环比计算：与上一周期对比（今日对比昨日，本周对比上周）
3. 数据更新频率：
   - 巡查次数、隐患数：准实时（5分钟延迟）
   - 覆盖率、整改率：每小时计算一次

#### 3.2.2 左侧街道排名

**位置**：页面左侧区域

**显示维度**：
- 下拉选择：巡查覆盖率、隐患整改率、巡查次数、隐患数、停工整改工程数

| 数值 | 对应指标的数值 |
| 趋势 | 环比变化趋势（柱状图展示） |

**示例数据**：

| 排名 | 街道 | 巡查覆盖率 | 趋势 |
|-----|------|-----------|------|
| 1 | 赵巷镇 | 95.2% | ████████ |
| 2 | 徐泾镇 | 92.8% | ███████░ |
| 3 | 华新镇 | 88.5% | ██████░░ |

**交互规则**：
- 点击排序维度下拉框切换排名依据
- 点击排名项可弹出右侧弹层，展示该街道详情
- 点击"查看全部街道"展开查看所有街道排名

**业务规则**：
1. 排名计算范围：用户权限范围内的所有街道
2. 数值相同时，按街道名称拼音排序
3. 巡查覆盖率相同时，按巡查次数倒序排列

#### 3.2.3 右侧趋势分析图

**位置**：页面中部右侧区域

**图表类型**：分组柱状图 / 堆叠柱状图

**Tab切换**：
- 巡查执行趋势（默认）
- 工程状态趋势

**巡查执行趋势图维度**：
- X轴：时间（今日按小时，本周/近七日按天，本月按天）
- Y轴：次数/数量
- 图例：街道巡查、社区巡查（本期暂不支持物业巡查）

**工程状态趋势图维度**：
- X轴：时间
- Y轴：工程数量
- 图例：正常施工、现场整改、停工整改、已核销（堆叠展示）

**交互规则**：
- 点击Tab切换趋势图类型
- 点击图例可切换显示/隐藏对应系列
- 点击柱状图可下钻查看对应时间段的明细数据

**业务规则**：
1. 本期仅支持街道巡查和社区巡查两类，物业巡查待后续物业角色接入后支持
2. 空数据处理：某时间点无数据时显示0，整段时间无数据时显示"暂无巡查数据"空态
3. 时间粒度：
   - 今日视图：按小时统计（0-23时）
   - 本周/近七日视图：按天统计
   - 本月视图：按天统计

## 四、数据权限规则

### 4.1 权限控制逻辑

**控制维度**：
1. **管辖区域**：根据用户所属组织的管辖区域限制可查看的街道
2. **工程类型**：根据用户权限设置可查看的工程类型

### 4.2 数据可见性判断

```
IF 用户角色 = "区级管理人员" THEN
    可见全部巡查数据
ELSE IF 用户角色 = "街镇管理人员" THEN
    IF 工程所属街道 IN 用户管辖街道列表 THEN
        IF 工程类型 IN 用户可查看工程类型列表 THEN
            可见该工程巡查数据
        ELSE
            不可见
        END IF
    ELSE
        不可见
    END IF
END IF
```

### 4.3 权限生效规则

**数据过滤逻辑**：
- 管辖区域：用户只能查看管辖区域内的工程巡查数据
- 工程类型：用户只能查看已授权的工程类型的巡查数据
- 组合条件：同时满足管辖区域和工程类型条件的数据才可见

**示例**：
- 某监管单位配置了"青浦区"管辖区域 + "小型工程、住宅装修"工程类型
- 该用户登录后：
  - 只能看到青浦区内的巡查数据
  - 只能看到小型工程和住宅装修类型的巡查数据
  - 零星作业类型的巡查数据不可见

## 五、业务规则详解

### 5.1 指标计算口径

#### 5.1.1 巡查覆盖率计算

**公式**：
```
巡查覆盖率 = 本周有巡查记录的在建工程数 / 当前在建工程总数 × 100%
```

**计算规则**：
1. 分子：本周至少有1条有效巡查记录的在建工程数
2. 分母：当前在建工程总数（状态=正常施工|现场整改|停工整改）
3. 统计周期：自然周（周一0:00至周日23:59:59）
4. 时间筛选：如大屏统一时间筛选另有定义，以统一筛选为准

**异常处理**：
1. 分母为0时展示"--"，不展示0%或100%
2. 巡查频次配置未正式接入前，本期只判断"本周是否被巡查过"，不判断"是否达到应巡查次数"

#### 5.1.2 隐患整改率计算

**公式**：
```
隐患整改率 = 本周整改审核通过的隐患数 / 本周新发现隐患总数 × 100%
```

**计算规则**：
1. 分子：统计周期内整改审核通过的隐患数
2. 分母：统计周期内新登记的隐患数
3. 统计范围：当前展示范围内，本周新发现的隐患

**异常处理**：
1. 本周未发现隐患时展示"--"，卡片下方提示"本周暂无新增隐患"
2. 分母为0时展示"--"

#### 5.1.3 环比变化计算

**公式**：
```
环比变化率 = (本期数值 - 上期数值) / 上期数值 × 100%
```

**计算规则**：
1. 本期为今日，则上期为上一天（昨日）
2. 本期为本周，则上周为上一个自然周（周一到周日）
3. 本期为本月，则上月为上一个自然月
4. 上期数值为0时，环比显示"--"（避免除以0）
5. 增长时显示绿色↑，下降时显示红色↓，持平显示灰色-

### 5.2 状态定义与流转

#### 5.2.1 工程状态定义

| 工程状态 | 口径定义 | 信息来源 |
|---------|---------|---------|
| 正常施工 | 无未闭环整改事项 | 安全巡检功能设计 |
| 现场整改 | 存在现场整改且未审核通过 | 安全巡检功能设计 |
| 停工整改 | 存在停工整改且未审核通过 | 安全巡检功能设计 |
| 已核销 | 已完工核销的工程 | 安全巡检功能设计 |

**优先级规则**：同一工程若同时存在现场整改和停工整改记录，以"停工整改"计入统计。
已核销规则：已核销工程不再计入"在建工程总数"，但在工程状态分布图中保留独立维度。

#### 5.2.2 隐患等级定义

| 隐患等级 | 定义 | 判定标准 |
|---------|------|---------|
| 较大隐患 | 需停工整改的隐患 | 可能导致人员伤亡或重大财产损失 |
| 一般隐患 | 现场整改或不需要限时整改的隐患 | 一般性安全隐患，可现场纠正 |

#### 5.2.3 整改状态流转

| 状态 | 定义 |
|-----|------|
| 待整改 | 已下发整改要求，施工方尚未提交整改记录 |
| 整改中 | 施工方已提交整改记录，待审核 |
| 已整改 | 审核通过，整改完成 |

**流转规则**：
- 待整改 → 整改中（提交整改记录）
- 整改中 → 已整改（审核通过）
- 整改中 → 待整改（审核不通过，退回）

### 5.3 数据刷新规则

| 数据类型 | 刷新频率 | 说明 |
|---------|---------|------|
| 巡查记录 | 准实时 | 巡查提交后5分钟内更新 |
| 隐患记录 | 准实时 | 隐患登记后5分钟内更新 |
| 核心指标卡 | 每小时 | 定时任务刷新 |
| 趋势图表 | 每小时 | 定时任务刷新 |
| 地图点位 | 每小时 | 定时任务刷新 |

### 5.4 地图点位颜色规则

| 标识 | 含义 | 显示条件 |
|-----|------|---------|
| 🔴 红色闪烁 | 存在未闭环隐患 | 工程下存在"待整改"或"整改中"隐患 |
| 🟡 黄色圆点 | 整改中 | 工程下存在待审核整改记录，但无停工级待整改 |
| 🟢 绿色圆点 | 近7天已巡查 | 近7天有巡查记录，且无未闭环隐患 |
| ⚪ 灰色圆点 | 近7天未巡查 | 近7天无巡查记录，且无未闭环隐患 |

**优先级规则**：红色 > 黄色 > 绿色 > 灰色

### 5.5 隐患类别统计口径

**统计维度**：按"隐患类别"聚合，不按单条隐患描述聚合

**隐患类别清单**（基于安全隐患类型清单）：
- 信息登记
- 安全管理
- 高处作业
- 施工用电
- 消防安全
- 脚手架

**排序规则**：按隐患数量倒序取前5；数量相同时按最近一次发现时间倒序

**兜底规则**：未能映射到标准隐患类别的记录统一归入"其他/待映射"
