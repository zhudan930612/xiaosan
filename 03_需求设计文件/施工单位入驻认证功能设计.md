## 一、功能背景

### 1.1 业务痛点

当前备案管理模块中,施工单位信息存在以下问题:

1. **企业信息可被随意使用**:任何人在备案时都可以新增施工单位,填写任意企业的营业执照和资质信息
2. **企业盗用风险高**:不法分子可能冒用正规企业资质进行备案,给企业带来法律风险
3. **企业无法管控自身信息**:正规企业无法管理和更新自己的资质信息
4. **数据质量参差不齐**:同一企业可能被多次录入,信息不一致

### 1.2 解决方案

通过"企业入驻认证"机制,实现:

1. **身份认证**:只有企业法人或授权人员才能申请企业入驻
2. **审核机制**:后台审核企业资质真实性,确保信息准确
3. **权限管控**:已认证企业的资料只能由认证人或法人修改
4. **成员管理**:认证人可邀请成员共同管理企业下的所有备案
5. **数据统一**:已认证企业数据替换未认证的同名企业数据

## 二、核心概念

### 2.1 施工单位认证状态

| 状态 | 说明 | 数据来源 | 权限 |
|------|------|----------|------|
| **未认证** | 备案时由用户临时新增的施工单位 | 备案申报时新增 | 所有人都可以补充资料 |
| **已认证** | 通过企业入驻审核的施工单位 | 个人中心申请入驻 | 只有认证人/法人/授权成员可以补充资料 |

### 2.2 角色定义

| 角色 | 说明 | 权限 |
|------|------|------|
| **认证申请人** | 提交企业入驻申请的人 | 管理企业资料、邀请成员、管理企业下所有备案 |
| **企业法人** | 企业营业执照上的法定代表人 | 管理企业资料、邀请成员、管理企业下所有备案 |
| **授权成员** | 被认证人邀请的成员 | 管理企业下所有备案(不能修改企业资料、不能邀请成员) |
| **普通用户** | 未加入该企业的用户 | 只能选择使用已认证企业进行备案,不能修改企业资料 |

## 三、完整业务流程

### 3.1 流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                     施工单位入驻认证完整流程                     │
└─────────────────────────────────────────────────────────────────┘

【阶段一:入驻申请】

施工方进入个人中心
      ↓
点击"企业入驻"入口
      ↓
选择入驻方式
  ├─ 法人入驻(我是法人)
  └─ 授权入驻(我是授权人员)
      ↓
填写企业基本信息
  • 上传营业执照(OCR识别)
  • 企业名称、统一信用代码、法人姓名
  • 资质证明、安全生产许可证
  • 法人身份证(正反面+手持)
  • 法人手机号
      ↓
判断入驻方式
  ├─ 法人入驻 ──→ 直接提交申请(使用法人信息作为申请人信息)
  └─ 授权入驻 ──→ 填写申请人信息
                    • 申请人身份证
                    • 申请人姓名、手机号
                    • 上传授权书
      ↓
提交入驻申请
      ↓

【阶段二:后台审核】

后台审核人员查看申请
      ↓
审核企业资质真实性
  • 营业执照真实性
  • 资质证明有效性
  • 法人身份真实性
  • 授权书真实性(授权入驻时)
      ↓
审核结果判断
  ├─ 审核通过 ──→ 进入阶段三
  └─ 审核不通过 ──→ 驳回并说明原因
                      ↓
                  申请人收到驳回通知
                      ↓
                  修改后重新提交 ──→ 返回阶段二

【阶段三:认证完成】

施工单位状态变更为"已认证"
      ↓
系统自动处理数据
  • 查找库中同名未认证企业
  • 将未认证企业的备案记录关联到已认证企业
  • 标记未认证企业为"已被认证企业替换"
      ↓
认证人收到审核通过通知
      ↓

【阶段四:成员管理】

认证人进入"我的企业"
      ↓
点击"成员管理"
      ↓
邀请成员
  • 输入成员手机号
  • 发送邀请
      ↓
成员收到邀请通知
      ↓
成员确认加入
      ↓
成员可管理该企业下的所有备案

【阶段五:日常使用】

场景1:已认证企业补充资料
  认证人/法人/授权成员 ──→ 可以补充和修改企业资料
  其他用户 ──→ 无权限修改,只能选择使用

场景2:备案时选择施工单位
  用户选择施工单位 ──→ 优先显示已认证企业
  ├─ 选择已认证企业 ──→ 直接使用,无需审核
  └─ 选择未认证企业或新增 ──→ 需要审核

场景3:未认证企业被替换
  用户选择未认证企业 ──→ 系统提示"该企业已完成认证,请选择已认证企业"
  ──→ 自动跳转到已认证企业
```

### 3.2 业务规则

**触发条件**
- 施工方需要进行企业入驻认证时,从个人中心进入"企业入驻"功能
- 一个企业只能被认证一次,已认证企业不能重复申请
- 申请人必须是企业法人或持有企业授权书的人员

**处理逻辑**
1. 用户选择入驻方式(法人入驻/授权入驻),填写企业信息和申请人信息
2. 提交后进入后台审核流程,审核人员核验资质真实性
3. 审核通过后,施工单位状态变更为"已认证",系统自动处理数据替换
4. 认证人可以邀请成员共同管理企业,成员确认后加入企业
5. 已认证企业的资料只能由认证人/法人/授权成员修改

**边界情况**
- 同名企业处理:如果库中已存在同名未认证企业,审核通过后自动替换,未认证企业的备案记录关联到已认证企业
- 重复认证:如果企业已被认证,再次申请时系统提示"该企业已完成认证"
- 授权书有效期:授权书需在有效期内,过期需重新上传
- 成员退出:成员可以主动退出企业,退出后无法再管理该企业的备案
- 认证人变更:如果需要变更认证人,需要法人重新授权
- ⚠ 待补充:如果法人身份证信息与营业执照不一致,如何处理?
- ⚠ 待补充:如果企业资质过期,已认证状态是否自动失效?

**页面名称**: "个人中心"页面

**页面功能**:

1. **企业入驻入口**:
   - 入口位置:个人中心页面的功能列表中
   - 入口名称:"企业入驻"
   - 入口图标:企业认证图标
   - 点击行为:进入"企业入驻申请"页面

   - 入口图标:企业管理图标
   - 点击行为:进入"我的企业"页面

- 用户未认证企业时,只显示"企业入驻"入口
- 用户已认证企业或已加入企业时,同时显示"企业入驻"和"我的企业"入口

### 5.3 企业入驻申请-填写企业信息

**触发操作**:
- 在"选择入驻方式"页面选择入驻方式后,点击"下一步"

**页面名称**: "企业入驻申请"页面 - 步骤2/3

**页面功能**:

**1. 企业基本信息**:

> 注:以下字段与现有"新增施工单位"功能保持一致

- **施工单位营业执照**(必填,文件上传,支持OCR识别)
  - 上传按钮:"点击上传"
  - 支持格式:图片(JPG、PNG)、PDF
  - OCR识别:上传后自动识别企业名称、统一信用代码、公司法人
  - 识别失败:允许手动输入

- **施工单位名称**(必填,文本输入)
  - 提示文字:"上传后自动识别"
  - 上传营业执照后自动识别,可手动修改

- **统一信用代码**(必填,文本输入)
  - 提示文字:"上传后自动识别"
  - 上传营业执照后自动识别,可手动修改

- **公司法人**(必填,文本输入)
  - 提示文字:"上传后自动识别"
  - 上传营业执照后自动识别,可手动修改

- **资质证明**(必填,文件上传)
  - 上传按钮:"点击上传"

### 5.10 数据替换-未认证企业自动替换

**触发条件**:
- 企业入驻审核通过后,系统自动执行

**系统行为**:

**1. 查找同名未认证企业**:
- 根据企业名称和统一信用代码,查找库中所有未认证的同名企业

**2. 数据替换**:
- 将未认证企业的所有备案记录关联到已认证企业
- 标记未认证企业为"已被认证企业替换"
- 保留未认证企业的历史记录,但不再显示在选择器中

**3. 通知相关用户**:
- 向使用过该未认证企业的用户发送通知:"您使用的施工单位[企业名称]已完成认证,后续备案请选择已认证企业"

**业务规则**:
- 数据替换自动执行,无需人工干预
- 未认证企业的备案记录不会丢失,会关联到已认证企业
- 用户后续选择施工单位时,只能看到已认证企业,看不到未认证企业
- 如果用户尝试新增同名未认证企业,系统提示"该企业已完成认证,请在列表中选择"

## 六、验收标准

> AI 草稿,PM 逐条确认后打勾

### 6.1 企业入驻申请流程

- [ ] 用户可以在个人中心看到"企业入驻"入口
- [ ] 点击"企业入驻"后进入入驻方式选择页面,可以选择"我是法人"或"我是授权人员"
- [ ] 选择入驻方式后进入企业信息填写页面,所有必填项有明确标识
- [ ] 上传营业执照后自动OCR识别企业名称、统一信用代码、法人姓名,识别失败时可手动输入
- [ ] 上传法人身份证后自动OCR识别法人姓名、身份证号,识别失败时可手动输入
- [ ] **点击"下一步"时,系统检查企业是否已被他人认证(根据企业名称+统一信用代码)**
- [ ] **如果企业已被认证,弹窗提示"该企业已完成认证",显示认证时间和认证人(脱敏),阻止继续提交**
- [ ] **弹窗提示用户"如需使用该企业进行备案,请联系认证人邀请您加入企业"**
- [ ] 选择"我是法人"时,填写完企业信息后直接提交申请,无需填写申请人信息(系统自动使用法人信息)
- [ ] 选择"我是授权人员"时,填写完企业信息后进入申请人信息页面,必须填写申请人信息和上传授权书
- [ ] 授权入驻时,申请人信息不能与法人信息完全一致
- [ ] 提交申请时校验所有必填项,未填写的字段标红提示
- [ ] 提交成功后进入"审核中"状态,用户可在"我的企业"中查看审核进度

### 6.2 后台审核流程

- [ ] 后台审核人员可以看到所有待审核的企业入驻申请
- [ ] **审核前系统自动检查企业是否已被他人认证,如已认证则自动驳回(兜底机制)**
- [ ] **自动驳回时,驳回原因包含:企业已认证信息、认证时间、认证人(脱敏)、引导用户联系认证人加入企业**
- [ ] 点击申请后进入审核详情页面,可以查看企业信息和申请人信息
- [ ] 审核人员可以点击"通过"按钮审核通过,企业状态变更为"已认证"
- [ ] 审核人员可以点击"驳回"按钮审核不通过,必须输入驳回原因(限200字)
- [ ] 审核通过后,系统自动处理数据替换,将未认证企业的备案记录关联到已认证企业
- [ ] 审核不通过后,申请人收到驳回通知,可查看驳回原因并修改后重新提交

### 6.3 成员管理流程

- [ ] 认证人和法人可以在"企业详情"页面看到"成员管理"按钮
- [ ] 点击"成员管理"后进入成员列表页面,可以看到所有成员
- [ ] 认证人和法人可以点击"邀请成员"按钮,输入成员手机号发送邀请
- [ ] 被邀请人收到邀请通知后可以选择"接受"或"拒绝"
- [ ] 被邀请人接受后自动加入企业,角色为"成员",可以查看和管理该企业下的所有备案
- [ ] 认证人和法人可以移除成员,但不能移除认证人和法人
- [ ] 成员可以主动退出企业,退出后无法再管理该企业的备案

### 6.4 权限控制

- [ ] 已认证企业的资料只能由认证人和法人补充和修改,其他用户无权限修改
- [ ] 未认证企业的资料所有人都可以补充和修改
- [ ] 成员可以查看和管理该企业下的所有备案,但不能修改企业资料和邀请成员
- [ ] 普通用户只能选择使用已认证企业进行备案,不能修改企业资料

### 6.5 备案时选择施工单位

- [ ] 备案时选择施工单位,已认证企业优先显示在列表顶部,带"已认证"标签
- [ ] 选择已认证企业时,直接使用,无需审核
- [ ] 选择未认证企业时,需要审核(与现有流程一致)
- [ ] 如果用户选择的未认证企业已被认证,系统提示"该企业已完成认证,请选择已认证企业",并自动跳转到已认证企业
- [ ] 新增施工单位时,如果企业已被认证,系统提示"该企业已完成认证,请在列表中选择"
- [ ] 选择施工单位时,如果统一信用代码不满足18位数,弹窗提示"统一信用代码必须为18位,请补充完整信息"
- [ ] 选择施工单位时,如果资质有效期过期,弹窗提示"该施工单位资质有效期已过期,请更新资质证明"
- [ ] 选择施工单位时,如果没有营业执照、资质证明或安全许可证,弹窗提示"该施工单位缺少必要资质材料,请补充"
- [ ] 选择施工单位时,如果没有法人身份证,弹窗提示"该施工单位缺少法人身份证信息,请补充"
- [ ] 后台禁用的施工单位在小程序选择列表中不可选,显示为灰色并标注"已禁用"标签
- [ ] 点击已禁用的施工单位时,弹窗提示"该施工单位已被禁用,无法选择"
- [ ] 已禁用的施工单位不会出现在搜索结果中

### 6.6 数据替换

- [ ] 企业入驻审核通过后,系统自动查找库中同名未认证企业
- [ ] 系统自动将未认证企业的所有备案记录关联到已认证企业
- [ ] 系统自动标记未认证企业为"已被认证企业替换",不再显示在选择器中
- [ ] 系统自动向使用过该未认证企业的用户发送通知
- [ ] 用户后续选择施工单位时,只能看到已认证企业,看不到未认证企业

## 八、待确认问题

1. **法人身份证信息与营业执照不一致时如何处理?**
   - 建议:审核不通过,要求申请人重新上传正确的资料

2. **企业资质过期后,已认证状态是否自动失效?**
   - 建议:资质过期后,企业状态变更为"资质过期",需要补充新的资质证明后才能继续使用

3. **一个用户可以认证多少个企业?**
   - 建议:不限制数量,但需要每个企业都通过审核

4. **一个企业可以有多少个成员?**
   - 建议:不限制数量,由认证人和法人自行管理

5. **成员退出企业后,是否可以重新加入?**
   - 建议:可以重新加入,需要认证人或法人重新邀请

6. **认证人可以转让认证人身份吗?**
   - 建议:不可以直接转让,如需变更认证人,需要法人重新授权

7. **如果法人变更,如何处理?**
   - 建议:需要提供法人变更证明,重新提交企业入驻申请

8. **授权书的有效期如何判断?**
   - 建议:授权书上需明确标注有效期,审核人员人工核验
