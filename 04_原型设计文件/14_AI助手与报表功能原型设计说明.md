## 一、原型概述

### 1.1 设计范围

本原型覆盖AI助手在三端的交互设计：
- **PC端**（监管审核端）：悬浮球入口+侧边栏对话
- **智慧大屏**：语音唤醒+固定图标入口
- **H5移动端**（巡检端）：底部导航入口+个人工作台

### 1.2 交互原则

| 原则 | 说明 |
|-----|------|
| 一键唤起 | 所有端都支持一键进入AI助手，无需多级导航 |
| 对话优先 | 以自然语言对话为核心交互方式 |
| 快捷辅助 | 提供快捷按钮降低用户输入成本 |
| 多端一致 | 核心交互范式保持一致，降低学习成本 |

## 二、视觉风格指南

### 2.1 设计理念

AI助手界面采用**现代轻盈**的视觉风格，营造智能、友好、专业的视觉感受，让数据分析和交互体验更加愉悦。

### 2.2 毛玻璃（玻璃拟态）效果

**效果说明**:
所有功能模块采用半透明的毛玻璃质感，让界面看起来轻盈通透，同时与背景区分开，营造层次感和空间感。

**应用场景**:
- AI助手侧边栏背景
- 悬浮球背景
- 消息气泡背景
- 快捷按钮背景
- 报表卡片背景

**实现参数**:
```css
.glass-effect {
    background: rgba(255, 255, 255, 0.7);
    backdrop-filter: blur(20px);
    -webkit-backdrop-filter: blur(20px);
    border: 1px solid rgba(255, 255, 255, 0.3);
}

/* 深色模式下的毛玻璃 */
.glass-effect-dark {
    background: rgba(30, 30, 50, 0.6);
    backdrop-filter: blur(20px);
    -webkit-backdrop-filter: blur(20px);
    border: 1px solid rgba(255, 255, 255, 0.1);
}
```

**设计效果**:
| 元素 | 背景透明度 | 模糊度 | 边框 |
|-----|-----------|--------|------|
| 侧边栏 | 0.85 | 20px | 1px rgba(255,255,255,0.3) |
| 消息气泡 | 0.9 | 15px | 无 |
| 悬浮球 | 1.0 | 无 | 无（渐变代替） |
| 快捷按钮 | 0.8 | 10px | 1px rgba(0,0,0,0.05) |

### 2.3 3D立体呈现

**效果说明**:
界面元素带有微妙的立体效果，像是浮现在背景之上，按钮和图标都有细微的阴影和高光，增强真实感和触感。

**应用场景**:
- 悬浮球按钮
- AI消息气泡
- 快捷操作按钮
- 数据卡片
- 报表预览卡片

**阴影层次**:
```css
/* 一级阴影 - 轻微浮起 */
.shadow-1 {
    box-shadow: 0 2px 8px rgba(0, 0, 0, 0.06);
}

/* 二级阴影 - 明显浮起 */
.shadow-2 {
    box-shadow: 0 4px 15px rgba(102, 126, 234, 0.2);
}

/* 三级阴影 - 强调浮起 */
.shadow-3 {
    box-shadow: 
        0 8px 25px rgba(102, 126, 234, 0.25),
        0 2px 8px rgba(0, 0, 0, 0.1);
}

/* 悬浮球专用阴影 */
.shadow-float {
    box-shadow: 
        0 6px 20px rgba(102, 126, 234, 0.4),
        0 2px 8px rgba(0, 0, 0, 0.1),
        inset 0 1px 0 rgba(255, 255, 255, 0.3);

```

**高光效果**:
- 卡片顶部添加1px白色半透明边框，模拟高光
- 渐变按钮使用inset阴影增加内部光泽
- 悬浮球Hover时顶部出现白色高光条

### 2.5 低饱和度色调

**设计理念**:
整体色彩饱和度较低，不会过于刺眼，长时间观看也不会产生视觉疲劳，非常适合监管类、数据类应用场景。

**饱和度控制**:
- 主色调饱和度：60%-70%
- 辅助色饱和度：40%-50%
- 背景色饱和度：10%-20%
- 文字色饱和度：0%（黑白灰为主）

**色彩应用比例**:

| 色彩类型 | 比例 | 用途 |
|---------|------|------|
| 中性色（白/灰/黑） | 70% | 背景、文字、边框 |
| 主品牌色（蓝紫渐变） | 15% | 强调、按钮、悬浮球 |
| 功能色（绿/橙/红） | 10% | 状态标识、提示信息 |
| 装饰色（淡彩） | 5% | 图表、图标、点缀 |

**文字色彩体系**:

| 用途 | 颜色值 | 不透明度 | 适用场景 |
|-----|--------|---------|---------|

| 正文 | #4b5563 | 100% | 普通文本、说明文字 |
| 次要文字 | #6b7280 | 100% | 辅助说明、时间戳 |
| 占位文字 | #9ca3af | 100% | placeholder、禁用状态 |
| 浅色背景文字 | #ffffff | 90% | 深色背景上的文字 |

**背景色彩体系**:

### 2.2 悬浮球设计

**位置和尺寸**:
- 位置：页面右下角，距离右边缘40px，距离底部40px
- 尺寸：60px × 60px
- 形状：圆形

**视觉样式**:
- 背景：渐变 `linear-gradient(135deg, #667eea 0%, #764ba2 100%)`
- 图标：[AI图标] 机器人SVG图标，白色，24px
- 阴影：`0 4px 15px rgba(102, 126, 234, 0.4)`

**交互状态**:

| 状态 | 效果 |
|-----|------|
| 默认 | 静止显示，有呼吸动画（scale 1.0） |
| Hover | 放大1.1倍，阴影加深 |
| 点击 | 瞬间缩小至0.9再弹回，唤出侧边栏 |
| 有新消息 | 显示红点角标，红点有脉冲动画 |

**红点角标**:
- 位置：右上角偏移（-5px, -5px）
- 尺寸：22px × 22px
- 背景：#f5222d
- 文字：白色，未读数量，12px

**拖拽功能**:
- 支持拖拽调整位置
- 拖拽时透明度降低至0.8
- 释放后弹性归位或停留在新位置

### 2.5 交互流程

**流程1: 唤起AI助手**
```
用户点击悬浮球
    ↓
悬浮球缩小动画（0.9x）
    ↓
遮罩层淡入（opacity 0 → 0.5）
    ↓
侧边栏滑入（right: -420px → 0）
    ↓
输入框获得焦点（可选）
```

**流程2: 发送消息**
```
用户输入文字 / 点击快捷按钮
    ↓
用户消息气泡出现在对话区（动画：fadeIn + slideUp）
    ↓
显示"AI思考中..."（可选）
    ↓
AI回复消息出现（动画：fadeIn）
    ↓
如果是报告/分析类回复，展示卡片组件
    ↓
对话区自动滚动到底部
```

**流程3: 生成报告**
```
用户发送"生成本周报告"
    ↓
AI回复："正在生成报告，预计需要10秒"
    ↓
显示进度条或loading动画
    ↓
报告预览卡片展示
    ↓
用户可选择：在线预览 / 下载PDF / 导出Excel / 分享
```

**流程4: 快捷键唤起**
```
用户按下 Ctrl+K
    ↓
判断侧边栏状态
    ↓
如果关闭 → 执行唤起流程
如果打开 → 执行关闭流程
```

## 七、动画规范

### 6.1 缓动函数

| 名称 | 值 | 用途 |
|-----|---|------|
| ease-out | cubic-bezier(0, 0, 0.2, 1) | 进入动画 |
| ease-in | cubic-bezier(0.4, 0, 1, 1) | 退出动画 |
| ease-in-out | cubic-bezier(0.4, 0, 0.2, 1) | 对称动画 |
| spring | cubic-bezier(0.34, 1.56, 0.64, 1) | 弹性效果 |

### 6.2 动画时长

| 类型 | 时长 | 说明 |
|-----|------|------|
| 微交互 | 100-200ms | 按钮点击、状态切换 |
| 标准 | 300ms | 弹窗、侧边栏 |
| 复杂 | 500ms | 页面切换、复杂动画 |

### 6.3 常用动画

**fadeIn**:
```css
@keyframes fadeIn {
    from { opacity: 0; transform: translateY(10px); }
    to { opacity: 1; transform: translateY(0); }
}
```

**slideInRight**:
```css
@keyframes slideInRight {
    from { transform: translateX(100%); }
    to { transform: translateX(0); }
}
```

**pulse**（红点提醒）:
```css
@keyframes pulse {
    0%, 100% { transform: scale(1); }
    50% { transform: scale(1.1); }
}
```

## 八、响应式适配

### 7.1 PC端响应式

| 屏幕宽度 | 适配规则 |
|---------|---------|
| ≥1440px | 标准显示 |

- 16:9 标准比例

### 7.3 H5端响应式

| 设备类型 | 适配规则 |
|---------|---------|
| iPhone SE/小屏 | 缩小间距，字体微调 |
| iPhone 标准屏 | 标准设计稿尺寸 |
| iPhone Plus/Max | 增大间距，内容居中 |
| 安卓全面屏 | 适配刘海屏、水滴屏安全区域 |

## 九、开发注意事项

### 8.1 性能优化

- 动画使用 transform 和 opacity，避免触发重排
- 图片懒加载
- 长列表虚拟滚动（对话消息多的时候）
- 防抖节流处理（拖拽、输入等频繁触发事件）

### 8.2 兼容性

- 渐变文字使用 background-clip: text，注意浏览器前缀
- 语音功能需要处理浏览器兼容
- 移动端注意 iOS 和 Android 的差异

### 8.3 可访问性

- 合适的颜色对比度（符合 WCAG 标准）
- 支持键盘操作
- 语音功能提供文字替代
