业务流程图

一、发起转派流程

1
审核员进入备案审核详情页
在项目中心待审核列表中点击备案卡片进入
2
发现备案所属街道不一致
审核员发现该备案实际不归本街道管辖
3
点击底部"转派"按钮
转派按钮常驻在审核详情页底部，与驳回/通过并列
4
进入选择目标街道页面
从11个青浦区街镇列表中选择目标街道，支持搜索筛选
5
填写转派原因并确认
转派原因必填，限200字。系统校验目标街道≠当前街道
6
转派成功
备案状态变更为"转派中"，推送通知给目标街道审核员

二、确认接收流程

1
目标街道审核员进入项目中心
Tab切换到"待确认"状态，查看待确认备案列表
2
点击待确认备案进入详情
查看备案基本信息及转派来源信息
3
判断是否为本街道管辖
是 → 点击"确认接收" → 状态变"待审核" → 进入本街道待审核列表
否 → 点击【退回】→ 填写退回原因 → 备案退回至发起方待审核列表，标签显示【转派退回】

三、审核流程（确认接收后）

1
进入待审核列表
已确认接收的备案显示在待审核列表中
2
点击备案进入审核详情
页面显示转派来源信息条
3
点击"驳回"或"通过"
审核流程与正常备案完全一致
4
完成审核
生成备案回执，归属街道以备案信息中的所属街镇字段为准

页面1：审核详情页 — 需求说明

页面状态规则

  • 备案状态为"待审核"时显示此页面
  • 底部常驻驳回/通过/转派三个操作按钮
  • 转派按钮与驳回/通过并列，不根据归属权动态显示/隐藏
  • 审核员可随时发起转派，无需先判断归属权

数据规则

  • 备案基本信息从备案记录表读取
  • 状态标签根据备案当前状态动态显示颜色
  • 施工信息展示完整的施工内容、面积、周期、工人数量

交互分支规则

  • 点击"驳回" → 进入驳回操作页，需填写驳回原因
  • 点击"通过" → 直接审核通过，进入通过结果页
  • 点击"转派" → 进入选择目标街道页

边界情况

  • 转派中的备案，发起方不可再进行驳回或通过操作
  • 同一条备案在"转派中"状态下，不可再次发起转派

页面2：选择目标街道页 — 需求说明

页面状态规则

  • 从审核详情页点击"转派"后进入
  • 页面顶部显示当前所属街道（不可选）
  • 街道列表默认不预选任何选项
  • 选中街道后显示选中样式

数据规则

  • 街道列表数据来源：青浦区11个实体街镇 + 虚拟园区
  • 排序：按街道名称字母顺序排序
  • 搜索：支持按街道名称实时筛选
  • 当前街道标记为不可选，但仍在列表中展示（仅置灰提示）

字段规则

  • 目标街道：单选，必填，不能与当前街道相同
  • 目标居委会：根据备案的工程类型和场所选项动态决定是否展示，具体规则见下方"居委会选择规则"
  • 转派原因：文本，必填，限制200字
  • 转派原因仅对审核端可见，申报人不可见

居委会选择规则

  • 需要选择【街镇+村居委会】的情况：工程类型为"9.依法获得批准建设的农民自建或翻建低层住宅"
  • 需要选择【街镇+居委会】的情况：
    • 工程类型为第2项，且场所选项为"住宅小区内部道路"
    • 工程类型为第5、7、8、12项，且场所选项为"住宅小区公共区域内场所"
  • 不需要选择居委会的情况：不符合上述条件的备案，仅选择街镇即可
  • 居委会列表数据来源：系统根据所选街镇加载对应的居委会/村居委会列表
  • 居委会排序：按名称字母顺序排序
  • 居委会搜索：支持按居委会名称实时筛选

交互分支规则

  • 未选择目标街道 → 点击"确认转派"按钮置灰不可用
  • 需要选择居委会的场景，未选择居委会 → 点击"确认转派"按钮置灰不可用
  • 未填写转派原因 → 点击"确认转派"按钮置灰不可用
  • 选择目标街道=当前街道 → 提示"不能转派至当前街道"
  • 校验通过 → 备案状态变更为"转派中" → 生成转派记录 → 推送通知 → 显示成功页

边界情况

  • V1版本暂不支持批量转派
  • 一次只能选择一个目标街道
  • 转派后备案从发起方列表中移除，不可撤回
  • 若备案信息中工程类型或场所选项为空/异常，按仅需选择街镇处理

页面3：转派成功确认页 — 需求说明

页面状态规则

  • 转派操作成功后显示此结果页
  • 展示转派后的关键信息摘要
  • 状态显示为"转派中"

数据规则

  • 展示备案编号、目标街道、转派时间、当前状态
  • 数据来源于刚生成的转派记录

交互分支规则

  • 点击"返回审核列表" → 回到审核列表，该备案已从列表中移除

边界情况

  • 转派成功后，系统自动向目标街道所有有审核权限的人员推送待确认通知
  • 该备案从发起方待审核列表中移除
  • 发起方可在"转派记录-我发起的"中查看此记录

页面4：项目中心-待审核 — 需求说明

页面状态规则

  • 此为已上线的项目中心页面，状态标签保持原有：待填报、待审核、已驳回、已通过
  • 新增状态标签：待确认（展示与转派相关的备案，详见页面4-2和4-3）
  • "待审核"标签下只显示两种卡片标签：待审核、转派退回
  • 工程类型Tab保持不变：小型工程、住宅装修、零星作业

数据规则

  • 卡片标签"待审核"：正常提交后等待审核的备案
  • 卡片标签"转派退回"：我转派出去、对方点击"退回"后退回给我的备案
  • 排序：按提交/退回时间倒序排列（最新的在前）
  • 各状态标签显示对应状态的数量

交互分支规则

  • 点击"待审核"列表项 → 进入正常审核详情页
  • 点击"转派退回"列表项 → 重新选择目标街道继续转派
  • 点击"待确认"状态标签 → 进入待确认列表（分"我接收的"和"我发起的"两个子Tab）

边界情况

  • 转派退回的备案回到发起方待审核列表，发起方可重新转派
  • 转派过程在审批记录中留痕，不单独建设转派记录页面

页面4-2：项目中心-待确认（我接收的） — 需求说明

页面状态规则

  • 点击"待确认"状态标签后，默认显示"我接收的"子Tab
  • "我接收的"展示别人转派给我、我需要确认的备案
  • 子Tab：我接收的（当前）、我发起的

数据规则

  • 筛选条件：目标街道=当前登录人员所在街道 AND 状态=转派中
  • 排序：按转派时间倒序排列（最新的在前）
  • 列表项显示：工程名称、备案编号、来源街道、地址、转派时间

交互分支规则

  • 点击"我发起的"子Tab → 切换到"我发起的"列表
  • 点击列表项 → 进入确认接收详情页，底部显示"确认接收"和"退回"两个按钮

边界情况

  • 多个有权限人员同时查看，列表内容一致
  • 第一个确认接收的人员即为该备案的确认人
  • 已被其他同事确认接收的备案从列表中移除

页面4-3：项目中心-待确认（我发起的） — 需求说明

页面状态规则

  • "我发起的"展示我转派出去、对方尚未确认或退回的备案
  • 子Tab：我接收的、我发起的（当前）

数据规则

  • 筛选条件：转派人=当前登录人员 AND 状态=转派中
  • 排序：按转派时间倒序排列（最新的在前）
  • 列表项显示：工程名称、备案编号、目标街道、地址、转派时间

交互分支规则

  • 点击"我接收的"子Tab → 切换到"我接收的"列表
  • 列表项为只读展示，不需要主动追踪

边界情况

  • 对方确认接收后，该备案从列表中移除，进入对方待审核列表
  • 对方点击"退回"后，该备案退回到我的待审核列表（标记为"转派退回"）

页面5：确认接收详情页 — 需求说明

页面状态规则

  • 点击待确认列表项后进入
  • 页面顶部显示转派来源信息条（绿色渐变背景）
  • 备案状态显示为"转派中"
  • 底部显示"确认接收"和"退回"两个按钮

数据规则

  • 展示完整的备案基本信息和施工信息
  • 额外显示转派来源信息：来源街道、转派时间、转派原因
  • 转派原因仅对审核端可见

交互分支规则

  • 点击"确认接收" → 状态变更为"待审核" → 生成确认接收记录 → 进入本街道待审核列表 → 显示确认成功页
  • 点击"退回" → 弹出退回原因输入页 → 填写退回原因（必填，限200字）→ 备案退回至发起方待审核列表 → 发起方列表项显示【转派退回】标签

边界情况

  • 发起方在待审核列表中点击【转派退回】标签的备案 → 进入选择目标街道页 → 重新选择目标街道继续转派
  • 确认接收不限定由确认人本人审核，街道内任何有权限人员均可审核
  • 有确认权限即等同于有审核权限

页面6：确认接收成功页 — 需求说明

页面状态规则

  • 确认接收操作成功后显示此结果页
  • 展示状态流转时间线

数据规则

  • 时间线展示：发起转派 → 确认接收 → 等待审核
  • 每个节点显示操作内容和操作时间

交互分支规则

  • 点击"去审核" → 进入已确认审核详情页

边界情况

  • 确认接收后该备案进入本街道待审核列表
  • 本街道下所有有审核权限的人员均可在待审核列表中看到此备案

页面7：已确认审核详情页 — 需求说明

页面状态规则

  • 确认接收后进入正常审核流程
  • 页面顶部显示转派来源信息条
  • 备案状态显示为"待审核"
  • 底部常驻驳回/通过/转派三按钮

数据规则

  • 审核详情页需显示转派来源信息（来源街道、转派时间）
  • 所属街镇显示为确认后的街道（如：盈浦街道（已确认））
  • 其余信息与正常备案完全一致

交互分支规则

  • 点击"驳回" → 进入驳回操作页
  • 点击"通过" → 直接审核通过
  • 点击"转派" → 可再次转派至其他街道

边界情况

  • 审核通过后生成备案回执，归属街道以备案信息中的"所属街镇"字段为准
  • 转派记录永久保留，可在转派记录页查看

页面8：驳回操作页 — 需求说明

页面状态规则

  • 从审核详情页点击"驳回"后进入
  • 顶部展示备案信息摘要（编号+名称）
  • 下方为驳回原因输入区

字段规则

  • 驳回原因：文本，必填，限制500字
  • 提供快捷标签供选择：材料不齐全、信息填写有误、不符合施工条件、地址信息有误
  • 快捷标签可多选，选中后自动拼接至文本域

交互分支规则

  • 未填写驳回原因 → "确认驳回"按钮置灰不可用
  • 点击快捷标签 → 自动追加到文本域内容末尾
  • 点击"确认驳回" → 状态变更为"已驳回" → 通知申报人（施工方）

边界情况

  • 驳回后申报人（施工方）会收到驳回通知
  • 施工方修改后可重新提交，备案回到"待审核"状态

页面9：审核通过结果页 — 需求说明

页面状态规则

  • 审核通过操作成功后显示此结果页
  • 状态标签显示"已通过"（绿色）

数据规则

  • 展示备案编号、审核结果、审核时间、审核人、所属街镇
  • 生成备案回执二维码

交互分支规则

  • 点击"返回审核列表" → 回到待审核列表

边界情况

  • 审核通过后，备案状态不可逆
  • 转派记录中的状态同步更新为"已通过"
