页面1：匹配保单 — 需求说明
页面状态规则
  • 小型工程（限额以下建设工程）：强制投保，未投保则「提交备案申请」按钮置灰不可点
  • 住宅装修/零星作业：不强制投保，但未投保提交时会弹窗提示
  • 线上投保为默认选中方式，显示「推荐」标签
  • 选择保险公司后，按钮变为可点击状态
数据规则
  • 保险公司列表由后台配置管理，当前展示6家（平安、国任、人保、人寿、太平洋、阳光）
  • 备案基础信息（工程名称、地址、施工单位等）自动同步至保险公司，用户无需重复填写
  • 当前保费为固定两档：500元/1000元（方案A）
交互分支规则
  • 点击保险公司 → 选中状态 → 提交按钮可用
  • 点击「提交备案申请」→ 弹出确认框 → 进入保险收银台
  • 点击「关联线下已购保单」→ 跳转至保单上传页面（本原型不展示）
边界情况
  • 未选保险公司时点击提交：按钮置灰，无响应
  • 保险公司系统故障：显示「该保险公司暂不可用，请选择其他」
  • 网络异常：提示检查网络后重试
页面2：H5内嵌保险收银台 — 需求说明
页面状态规则
  • H5页面由保险公司提供，平台通过WebView内嵌展示
  • 收银台标题显示「{保险公司名称}保险收银台」
  • 默认选中微信支付，用户可切换至支付宝
数据规则
  • 保险单号：由保险公司系统生成，格式为数字串
  • 客户名称：取自备案信息中的施工单位名称
  • 商品名称：固定为「安全生产责任保险」
  • 保费金额：当前固定500元（方案A为两档定价）
交互分支规则
  • 点击「确认支付」→ 调起微信/支付宝支付SDK
  • 支付成功 → 自动返回平台 → 进入回调中页面
  • 支付失败/取消 → 返回收银台，可重新选择支付方式
  • 点击返回按钮 → 回到匹配保单页，保留已选保险公司
边界情况
  • 支付超时（5分钟）：订单自动取消，提示「支付超时」并返回
  • 余额不足：支付SDK提示，用户可更换支付方式
  • 网络中断：支付结果未知，需主动查询订单状态
页面3：保单回调中 — 需求说明
页面状态规则
  • 支付完成后，平台自动轮询保险公司接口查询保单状态
  • 显示loading动画，文案「保单数据更新中」
  • 此状态下「提交备案申请」按钮置灰
数据规则
  • 轮询间隔：每2秒查询一次，最多轮询30秒
  • 同步字段：保单号、保险公司、保险期限、保费金额、保单PDF链接
  • 同步成功后，保单信息写入工程档案-保险信息库
交互分支规则
  • 保单同步成功 → 自动跳转至投保成功页
  • 保单同步超时 → 提示「同步超时，请稍后刷新查看」
  • 保单同步失败 → 提示「保单同步异常，请联系客服」
边界情况
  • 用户在回调中退出应用：再次进入时自动继续查询
  • 轮询期间网络断开：恢复后重新发起轮询
  • 保险公司回调延迟超过30秒：提示手动刷新
页面4：线上投保成功 — 需求说明
页面状态规则
  • 保单同步成功后，匹配保单页变为「已投保」状态
  • 展示保单关键信息：保险公司、保单号、保险期限、保费金额
  • 显示「已投保」标签，「保单已关联」成功提示
  • 「提交备案申请」按钮变为可用状态
数据规则
  • 保险期限：根据工程工期自动计算，默认覆盖完整施工周期
  • 保单号：由保险公司系统生成，唯一标识
  • 保单PDF：由保险公司生成，平台提供下载入口
小IDI入口规则（方案B新增）
  • 仅在安责险线上投保成功后展示，线下上传保单成功不展示
  • 推荐险种：第三者责任险（类比车险小IDI模式）
  • 利益点文案：保障第三方人身伤亡、保障第三方财产损失、线上投保即刻生效
  • 用户点击「去选购」→ 跳转铃铛保保险商城选购商业补充险
  • 用户点击「暂不需要」→ 卡片收起，展示「已跳过」状态，本次会话可重新查看
  • 「小IDI」入口为推荐位，非强制，不影响备案流程
交互分支规则
  • 点击「提交备案申请」→ 进入备案提交流程
  • 点击「去选购」→ 跳转铃铛保保险商城
  • 点击「暂不需要」→ 收起推荐卡片，展示已跳过状态
  • 点击「重新查看」→ 恢复展示推荐卡片
  • 点击保单PDF → 调起浏览器/系统预览下载PDF
  • 点击「临时保存」→ 保存当前备案草稿
边界情况
  • 保单PDF下载失败：提示「文件加载失败，请重试」
  • 用户更换保险公司：需重新走投保流程，原保单自动解绑
  • 从商城返回后：根据回传数据更新「小IDI」展示状态（已选购/未选购）
  • 同一用户多次投保：每次线上投保成功均展示「小IDI」入口
页面5：支付失败 — 需求说明
页面状态规则
  • 支付失败场景：超时取消、余额不足、用户主动取消、网络异常
  • 根据不同失败原因展示对应提示文案
  • 提供「重新支付」和「返回修改」两个操作入口
数据规则
  • 失败订单状态：在后台标记为「支付失败」
  • 用户点击「重新支付」：复用原订单号重新调起支付
  • 用户点击「返回修改」：回到匹配保单页，保留已选保险公司
交互分支规则
  • 点击「重新支付」→ 回到保险收银台，保留原订单信息
  • 点击「返回修改」→ 回到匹配保单页，可更换保险公司
边界情况
  • 重复支付：同一订单只允许一笔成功支付，重复支付自动原路退回
  • 订单已过期：点击重新支付时提示「订单已过期，请重新选择保险公司」
页面6：中断后重新进入 — 需求说明
页面状态规则
  • 用户在支付流程中退出应用/切换页面，再次进入匹配保单页时检测未完成的投保
  • 顶部显示中断提示横幅：展示上次选择的保险公司和支付状态
  • 保险公司列表中保留上次选择状态
数据规则
  • 中断状态保存：本地缓存 + 服务端订单查询双重校验
  • 缓存有效期：30分钟，超时后清除中断状态
  • 如服务端订单已超时/取消，提示「上次投保已过期，请重新选择」
交互分支规则
  • 用户可直接点击「提交备案申请」→ 沿用上次选择继续投保流程
  • 用户可更换保险公司 → 取消上次订单，重新发起新投保
  • 用户可切换至「关联线下已购保单」→ 取消线上投保流程
边界情况
  • 同一设备切换账号：不展示其他账号的中断状态
  • 中断超过30分钟：清除状态，恢复正常初始页面
  • 服务端订单已支付成功但客户端未收到回调：自动跳转成功页
