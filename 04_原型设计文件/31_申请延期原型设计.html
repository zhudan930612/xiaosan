原型说明
右侧说明随页面切换同步展示。施工端首页按你提供的现有首页样式搭建，只在原业务区新增“申请延期”；施工端与巡检端的延期主流程仍保持为 P2-P9。
P1 首页

页面状态规则

  • 首页沿用现有施工端首页骨架：顶部欢迎区、四个业务卡、搜索区、三类工程 Tab、工程列表、底部导航。
  • “申请延期”作为新增快捷入口加在原业务区内，不替换现有四个入口。

交互规则

  • 点击“申请延期”后进入 P2 延期申请工程列表页。
  • 首页工程列表仍展示项目中心原有工程数据，不直接承接延期办理。
P2 延期申请工程列表页

页面状态规则

  • 顶部先按三大类工程切换，再按“已逾期 / 待审核 / 已驳回 / 已通过”分层。
  • “已逾期”仅展示已经逾期但还没有申请延期的项目。

字段规则

  • 列表卡片至少展示：工程名称、工程地址、计划施工时间、逾期天数、当前延期状态。
  • 状态不同，卡片主按钮分别为“申请延期 / 审核中 / 重新申请 / 查看记录”。
P3 延期申请填写页

字段规则

  • 工程基本信息只读展示工程名称、工程地址、具体工程类型、施工内容、原计划施工时间。
  • 延期原因手填，不做枚举。

业务规则

  • 延期与保险补充信息属于同一张申请单，不拆独立续保流程。
  • 小型工程延期超保时必须补齐保险公司、险种、保单号、起止日期和附件。
P4 工程详情页

页面结构规则

  • 沿用工程详情页骨架，不再做独立简化表单详情页。
  • 详情页内补“延期申请记录”入口和延期状态摘要。
P5 延期申请记录页

数据规则

  • 页面展示当前有效完工日期和历史延期申请时间线。
  • 驳回记录必须展示驳回原因。
P6 巡检端首页

页面状态规则

  • 巡检端首页按工作台待办结构组织，延期审核入口挂在待办区。
  • 延期属于备案完成后的在建过程事项，不放在备案审核端处理。
P7 延期申请待审核列表页

页面状态规则

  • 巡检端同样使用三大类工程 Tab，并区分“待审核 / 已驳回 / 已通过 / 全部”。
  • 默认进入“待审核”。
P8 延期申请审核页

业务规则

  • 审核对象是整张延期申请单，不拆成延期审核和保险审核两次动作。
  • 小型工程超保时，续保保单信息和附件都必须核对。
P9 直接延期页

业务规则

  • 直接延期为巡检端补充能力，不替代首页进入待审核列表的主流程。
  • 表单结构与申请填写页保持基本一致。
