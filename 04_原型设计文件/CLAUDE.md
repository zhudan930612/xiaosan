# 原型设计文件目录规则

本目录存放 HTML 原型、流程图、系统结构图等可视化文档。

## 前置要求（强制）

开始任何原型设计前，必须先读取 `01_产品主文档/*原型设计标准文档.md`，严格遵循其中的色彩规范、布局规范、组件规范、标注规范和 HTML 输出规范。

## 输出规范

- 输出格式：HTML
- 文件名：`NN_[功能名]原型设计.html` 或 `NN_[功能名]流程图.html`
- 需包含基础交互能力（点击跳转、状态切换）

## 原型类型

开始输出前，必须先确认原型类型。提供以下选项：
1. 原型可编辑，标注可编辑
2. 文案可编辑，标注可编辑
3. 不可编辑，有交互
4. 不可编辑，有交互且有标注
5. 其他自定义组合

**移动端默认**：移动端（小程序/APP）原型默认采用"不可编辑，有交互，有标注"。

## 移动端原型布局

- 左侧：手机原型区域
- 右侧：需求说明/标注区域，与左侧并排，独立滚动
- 导航按钮：浮动显示，按 F8 切换显示/隐藏

## 标注规则（如选择"有标注"）

每个页面的标注必须系统性地覆盖以下 5 类规则：
1. **页面状态规则**：不同业务状态下，页面元素的显示/隐藏、按钮的可用/禁用、文案的变化
2. **数据规则**：列表的排序字段与方向、筛选条件、数据范围、分页/加载/刷新机制
3. **字段规则**：每个字段的必填/选填、格式限制、默认值、取值范围、校验规则、错误提示
4. **交互分支规则**：点击/提交后的完整分支逻辑（成功、失败、无权限、网络异常、重复提交等）
5. **边界情况规则**：特殊场景的处理逻辑（如逾期、中断、并发、异常流程）

## 与功能设计的关联

- 功能设计：`03_需求设计文件/NN_[功能名]功能设计.md`
- 原型设计：`04_原型设计文件/NN_[功能名]原型设计.html`
- 编号必须一致，修改时必须同步处理

## 标注与功能设计的互补

- 若原型"有标注"，功能设计正文聚焦业务流程，原型标注补充页面级规则
- 若原型"无标注"，功能设计必须在"功能点描述"或"业务规则"章节中逐页补充完整规则
- 功能设计文档中禁止用 ASCII 图、文字画等形式"假装"做原型
