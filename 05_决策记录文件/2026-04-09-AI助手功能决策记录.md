## 一、问题定义

### 1.1 原始需求

为监管端和大屏设计AI报表功能，提升数据分析和决策支持能力。

### 1.2 关键发现（与现有大屏对比后）

经查阅 [大屏巡查数据模块设计.md](../03_需求设计文件/大屏巡查数据模块设计.md)，发现大屏已覆盖以下数据展示：

| 大屏已有功能 | 展示内容 | 大屏定位 |
|-------------|---------|---------|
| 巡查统揽指标卡 | 本周巡查覆盖率、本周隐患整改率、本月巡查次数、待整改隐患 | 宏观态势 |
| 巡查趋势图 | 近7天/30天/本年度趋势，分街道/社区/物业 | 趋势变化 |
| 工程状态分布 | 正常施工/现场整改/停工整改/已核销占比 | 状态分布 |
| 隐患分布统计 | 隐患等级、隐患类别TOP5、整改状态 | 风险分布 |
| 地图巡查状态 | 工程点位颜色标识、弹窗信息 | 空间分布 |

**核心问题**: 大屏已经展示了标准指标和固定看板，AI助手如果再重复这些，价值有限。

### 1.3 重新定义AI助手价值

大屏是**"看板"**（被动展示标准指标），AI助手应该是**"助手"**（主动提供个性化服务）。

| 维度 | 大屏（已有） | AI助手（应补充） |
|-----|-------------|-----------------|
| 交互方式 | 看 | 问 |
| 数据深度 | 标准指标 | 深层分析、根因分析 |
| 个性化 | 统一视图 | 千人千面 |
| 主动性 | 被动展示 | 主动提醒待办 |
| 灵活性 | 固定图表 | 灵活钻取、自由探索 |
| 输出能力 | 展示 | 生成报告、导出分享 |

## 二、方案对比

### 2.1 方案A：大屏补充者（调整后思路）

**设计描述**:
- AI助手不做大屏已有的标准指标展示
- 专注于大屏做不了的事情：深层分析、个性化查询、主动提醒、灵活探索

**AI助手核心能力**:

| 能力 | 与大屏的差异 | 示例 |
|-----|-------------|------|
| **根因分析** | 大屏只展示"整改率下降"，AI分析"为什么下降" | "为什么夏阳街道整改率低于平均？" |
| **关联分析** | 大屏展示单一维度，AI做多维度关联 | "备案量高的街道是否隐患率也高？" |
| **个性化查询** | 大屏统一视图，AI支持个人视角 | "我本月巡查了多少次？有哪些待整改？" |
| **灵活钻取** | 大屏固定层级，AI支持自由下钻 | "查看这个工程的历次巡查记录和整改详情" |
| **预测预警** | 大屏展示现状，AI预测趋势 | "下周哪些工程可能逾期整改？" |
| **报告生成** | 大屏展示，AI生成可导出报告 | "生成本周报告并导出PDF" |

**优点**:
- 与大屏形成互补，不重复建设
- 用户有明确的使用场景区分：大屏看概览，AI查细节
- 突出AI价值：分析、推理、生成

**缺点**:
- 需要用户理解大屏和AI的分工边界
- 技术实现上需要更强的语义理解和分析能力

---

### 2.2 方案对比总结

| 评估维度 | 方案A：大屏补充者 | 决策倾向 |
|---------|-----------------|---------|
| 与现有系统关系 | 互补而非重复 | **方案A** |
| AI价值体现 | 高（分析/推理/生成） | **方案A** |
| 用户学习成本 | 中（需理解分工） | - |
| 技术实现难度 | 高（需要推理能力） | - |
| 长期价值 | 高（越用越智能） | **方案A** |

## 三、决策过程

### 3.1 关键决策点

**决策点1：AI助手是否展示大屏已有的标准指标？**

- **产品经理立场**: 用户问"本月备案量"，AI应该回答吗？
- **AI助手建议**:
  - 应该回答，但**形式不同**
  - 大屏：展示数字和趋势图
  - AI：回答数字+解释原因+关联建议
  - 示例:
    ```
    用户: 本月备案量
    AI: 本月备案量1,245个，环比+15%。
        主要增长来自夏阳街道（+30%），可能原因是该街道近期开展了备案宣传。
        需要查看夏阳街道的备案详情吗？
    ```
- **最终决策**: **AI助手可以查询标准指标，但重点在分析和建议，而非单纯展示数字**

---

**决策点2：AI助手的核心差异化能力是什么？**

讨论过程:
- 如果只做自然语言查数据，和大屏点筛选器没区别
- AI的价值应该在于：**理解问题 → 分析数据 → 给出洞察**

**最终决策**: AI助手的三大核心差异化能力：

1. **深度分析能力**（Why）
   - 根因分析："为什么整改率下降？"
   - 关联分析："备案量与隐患率的关系"
   - 对比分析："哪个街道表现最好，为什么？"

2. **个性化服务能力**（Who/What）
   - 个人工作统计："我本月的工作情况"
   - 待办提醒："我有哪些事项需要处理"
   - 工程详情："这个工程的全生命周期"

3. **主动输出能力**（How）
   - 报告生成："生成本周报告"
   - 预警提醒："即将逾期的隐患"
   - 建议输出："如何提升巡查覆盖率"

---

**决策点3：多端入口形态是否调整？**

**决策**: 保持原方案，但各端侧重点不同：

| 端 | 入口形态 | 使用侧重 |
|---|---------|---------|
| PC审核端 | 悬浮球 | 深度分析、报告生成 |
| 智慧大屏 | 语音唤醒+图标 | 语音查询、快速下钻 |
| 巡检端H5 | 底部导航 | 个人统计、待办提醒 |

## 四、已确认结论

### 4.1 AI助手与大屏的分工

| 场景 | 大屏 | AI助手 |
|-----|------|--------|
| 看全区态势 | ✅ 首选 | 可查询，但侧重分析原因 |
| 查具体数字 | ✅ 标准指标展示 | 回答数字+解释+建议 |
| 根因分析 | ❌ 不支持 | ✅ AI核心能力 |
| 个人工作统计 | ❌ 不支持 | ✅ 巡查人员刚需 |
| 待办提醒 | ❌ 不支持 | ✅ 主动推送 |
| 灵活钻取 | ⚠️ 固定层级 | ✅ 自由对话下钻 |
| 报告导出 | ❌ 不支持 | ✅ 生成PDF |

### 4.2 AI助手核心能力（调整后）

**第一层：数据查询**（基础，大屏可替代）
- 查询备案量、巡查次数、隐患数等标准指标
- 形式：数字+简要解释（不做复杂图表，大屏已展示）

**第二层：深度分析**（核心，大屏不可替代）
- 根因分析："为什么整改率下降？"
- 关联分析："备案量与隐患率的关系"
- 异常解释："为什么这个工程隐患特别多？"

**第三层：个性化服务**（核心，大屏不可替代）
- 个人统计："我本月巡查了多少次"
- 待办提醒："我有哪些待处理事项"
- 工程详情："这个工程的全生命周期记录"

**第四层：生成与导出**（核心，大屏不可替代）
- 报告生成：周报/月报/专项报告
- 导出分享：PDF、Excel

### 4.3 一期MVP范围（调整后）

基于"大屏补充者"定位，一期聚焦大屏做不了的高价值场景：

| 能力 | 一期范围 | 不做（大屏已有） |
|-----|---------|-----------------|
| 数据查询 | 简单数字+简要解释 | 复杂图表展示 |
| 深度分析 | 2个根因分析场景 | - |
| 个性化 | 个人统计+待办提醒 | - |
| 报告生成 | 周报 | - |
