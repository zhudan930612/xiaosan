# 管理后台操作手册：组织与用户权限配置

> 本文档用于指导运营人员/系统管理员在青浦区小散工程安全监管平台管理后台中完成监管组织的创建和用户权限的分配。

---

## 一、前置准备

### 1.1 权限要求

执行本手册操作需要具备**系统管理员**或**运营人员**角色权限。

### 1.2 选择操作方式

根据数据量选择合适的方式：

| 数据量 | 推荐方式 | 是否需要准备 Excel |
|--------|---------|-------------------|
| 1~2 条 | 后台界面手动新增 | 不需要 |
| 3 条及以上 | 建议 Excel 批量导入 | 需要按模板整理 |

**批量导入时才需准备**：
- **组织数据**：参照《合并后组织导入数据.xlsx》格式
- **用户数据**：参照《合并后用户导入数据.xlsx》格式
- **填写模板**：参照《导入模板_最终版2.xlsx》

## 二、添加监管组织

### 2.1 方式一：后台界面手动新增

**适用场景**：单个组织新增，或临时补充组织

**操作步骤**：

1. 登录管理后台
2. 进入 **企业中心 → 监管单位**
3. 点击页面右上角 **【新增】** 按钮
4. 在弹窗中填写以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 组织名称 | 是 | 建议格式：`街镇名-科室/部门名`，如"白鹤镇-规保办" |
| 机构编码 | 是 | 组织的唯一编码，系统自动生成或按规则填写 |
| 管辖区域 | 是 | 下拉选择，如"上海市/青浦区/青浦区/白鹤镇" |

5. 点击 **【保存】**

**注意事项**：
- 组织名称建议统一采用"街镇-科室"格式，便于识别和管理
- 虚拟园区（如青浦工业园区、西虹桥商务区）在管辖区域中作为普通区域选择

---

### 2.2 方式二：Excel 批量导入

**适用场景**：批量初始化或大规模更新组织数据

**操作步骤**：

1. 下载或打开组织导入模板
2. 按模板要求填写数据
3. 进入 **企业中心 → 监管单位**
4. 点击页面右上角 **【导入】** 按钮（如后台支持）
5. 上传填写好的 Excel 文件
6. 确认导入结果

**模板字段说明**：

| 字段 | 必填 | 填写规则 |
|------|------|---------|
| 组织名称 | 是 | 建议格式：`街镇名-科室/部门名` |
| 管辖区域 | 是 | 与后台区域管理模块中的区域名称保持一致 |
| 小型工程类型 | 否 | 多选时以中文逗号（，）分隔。可填"全部"表示全部选中。以"非"开头表示排除，如"非交通工程，非水务工程" |
| 零星作业类型 | 否 | 同上 |
| 住宅装修类型 | 否 | 同上 |
| 小型工程作业场所 | 否 | 同上 |
| 零星作业作业场所 | 否 | 同上 |

**可选工程类型清单**：

- **小型工程类型**（12类）：新改扩房屋建设工程、交通工程、水务工程、园林绿化工程、既有建筑修缮工程、建筑装饰装修工程、既有建筑加装电梯工程、临时建设工程、农民自建房工程、农业设施建设工程、农村地区综合性建设工程、人防工程
- **住宅装修类型**（1类）：竣工验收合格后的住宅室内装饰装修
- **零星作业类型**（10类）：空调/太阳能/雨棚/防盗网/管道/工业环保设备设施等非主体工程配套设备设施的安装维护拆除作业、建筑外墙清洗修补屋面检修等各类建筑外墙零星高处作业、户外小型广告及门楣招牌的安装维护拆除作业、新能源汽车电动自行车充电设施的安装维护拆除作业、临时搭建舞台看台展台布景等搭设拆除作业、锅炉压力容器压力管道电梯起重机械等特种设备的安装改造修理、住宅区公共场所绿化枝叶修剪、道路绿化枝叶修剪、供水排水通信电力燃气等管线设施维护维修工程、其他存在事故风险的零星作业活动

**可选作业场所清单**（12项）：

商业综合体/大型商场/超市/商品交易市场等商贸场所、工业企业内场所、医疗卫生机构内场所、学校/幼儿园等教育机构内场所、文化场馆/旅游景区场所、体育场场所、党政机关集中办公区内场所、养老机构/福利院等民政服务机构内场所、区属国有企业经营管理场所内场所、住宅小区公共区域内场所、区管交通设施（如客运站/停车场）范围内场所、涉及公共绿地/公园/环卫设施等场所

## 三、添加用户并分配权限

### 3.1 用户类型说明

后台用户分为三类，本手册主要针对 **监管方** 用户：

| 用户类型 | 管理入口 | 说明 |
|---------|---------|------|
| 物业 | 运营管理 → 用户管理 → 物业 Tab | 物业公司管理人员 |
| 监管方 | 运营管理 → 用户管理 → 监管方 Tab | 街镇/区级监管部门人员 |
| 施工方 | 运营管理 → 用户管理 → 施工方 Tab | 施工单位人员 |

---

### 3.2 方式一：后台界面手动新增

**适用场景**：单个用户新增

**操作步骤**：

1. 登录管理后台
2. 进入 **运营管理 → 用户管理**
3. 切换到 **【监管方】** Tab
4. 点击页面右上角 **【新增监管用户】** 按钮
5. 在弹窗中填写以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 手机号 | 是 | 用户登录账号 |
| 用户名 | 是 | 用户真实姓名 |
| 所属监管单位 | 是 | 下拉选择已创建的组织 |
| 角色 | 是 | 单选：普通审核员 / 科室负责人 / 街道领导 / 区级领导 |
| 权限模式 | 是 | 单选：继承 / 自定义 |

6. 如权限模式选择 **【自定义】**，需额外勾选：
   - 可审核的小型工程类型
   - 可审核的住宅装修类型
   - 可审核的零星作业类型
   - 可管理的小型工程作业场所
   - 可管理的零星作业作业场所
7. 点击 **【保存】**

---

### 3.3 方式二：Excel 批量导入

**适用场景**：批量初始化用户权限

**操作步骤**：

1. 下载或打开人员导入模板
2. 按模板要求填写数据
3. 进入 **运营管理 → 用户管理 → 监管方 Tab**
4. 点击页面右上角 **【导入】** 按钮
5. 上传填写好的 Excel 文件
6. 确认导入结果

**模板字段说明**：

| 字段 | 必填 | 填写规则 |
|------|------|---------|
| 姓名 | 是 | 用户真实姓名 |
| 手机号 | 是 | 用户手机号，用于登录 |
| 所属组织名称 | 是 | 必须与后台已创建的组织名称完全一致 |
| 角色 | 是 | 单选：普通审核员 / 科室负责人 / 街道领导 / 区级领导 |
| 权限模式 | 是 | 单选：继承 / 自定义 |
| 自定义小型工程类型 | 否 | 权限模式为"自定义"时填写，填写规则同组织导入 |
| 自定义住宅装修类型 | 否 | 同上 |
| 自定义小型工程作业场所 | 否 | 同上 |
| 自定义零星作业场所 | 否 | 同上 |

## 四、角色与权限说明

### 4.1 四种角色

| 角色 | 数据可见范围 | 权限特点 |
|------|-------------|---------|
| 普通审核员 | 仅能看到分配给自己的备案 + 组织内未分配的备案 | 按工程类型和作业场所过滤 |
| 科室负责人 | 能看到本科室（组织）下的全部备案 | 不受工程类型限制 |
| 街道领导 | 能看到本街镇/园区管辖区域内的全部备案 | 挂靠镇政府/园区管委会组织 |
| 区级领导 | 能看到全区所有备案 | 挂靠区建管委组织 |

### 4.2 权限模式说明

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **继承** | 用户权限自动跟随所属组织的配置变更而更新 | 绝大多数情况，同一科室人员权限一致 |
| **自定义** | 用户拥有独立的权限配置，不受组织模板变更影响 | 个别人员职责与科室整体不同 |

**建议**：
- 90% 以上的用户应采用"继承"模式
- 仅当某个人员的职责明显不同于其所在科室的常规职责时，才使用"自定义"模式

## 五、常见操作流程

### 5.1 新街镇入驻（从零开始配置）

**步骤顺序**：

1. 先导入组织 → 再导入人员
2. 组织导入前，确认所有组织的管辖区域已在区域管理模块中配置
3. 人员导入前，确认所有所属组织名称与组织导入数据中的名称完全一致

### 5.2 人员调岗/权限调整

**操作方式**：

1. 进入 **运营管理 → 用户管理 → 监管方 Tab**
2. 搜索目标用户（按手机号或姓名）
3. 点击该用户所在行的 **【编辑】** 按钮
4. 修改所属组织、角色或权限模式
5. 保存

### 5.3 组织权限模板变更后同步给人员

**操作方式**：

1. 进入 **企业中心 → 监管单位**
2. 找到目标组织，点击 **【编辑】** 修改工程类型或作业场所
3. 保存后，可选择 **【同步权限给人员】**
4. 选择同步范围：
   - **仅同步继承模式的人员**（推荐）：只更新权限模式为"继承"的用户
   - **覆盖全部（含自定义）**：强制将所有用户的权限重置为组织模板

## 六、常见问题与检查清单

### 6.1 导入失败排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 组织导入失败 | 管辖区域名称与系统不一致 | 检查区域管理模块中的区域名称，确保完全一致 |
| 人员导入失败 | 所属组织名称不存在 | 先导入组织，再导入人员；检查组织名称拼写 |
| 人员导入失败 | 手机号重复 | 检查是否已存在相同手机号的用户 |
| 权限不生效 | 工程类型名称填写错误 | 对照"可选工程类型清单"检查名称拼写 |

### 6.2 操作前自检

- [ ] 所有组织数据已整理完毕
- [ ] 所有人员数据已整理完毕
- [ ] 组织名称、管辖区域名称与系统现有数据一致
- [ ] 工程类型名称、作业场所名称与标准名称一致
- [ ] 人员所属组织名称与组织导入数据中的名称完全一致

### 6.3 操作后验证

- [ ] 导入完成后，检查组织列表是否全部显示
- [ ] 检查用户列表是否全部显示
- [ ] 抽查几个用户，确认所属组织、角色、权限模式正确
- [ ] 如有自定义权限的用户，抽查确认自定义工程类型正确
