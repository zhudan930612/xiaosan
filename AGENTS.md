# Agent 工作入口

本文件是 Codex/Agent 进入项目后的入口说明，用于快速识别项目结构、技能触发和完成标准。项目级详细规则见 [CLAUDE.md](CLAUDE.md)，具体文档体系说明见 `00_项目说明/`。

## 适用范围

- 本文件适用于整个项目根目录。
- 如子目录后续出现独立的 `AGENTS.md`，以更靠近当前任务目录的说明为准。
- 本项目默认使用中文沟通，输出面向产品经理和开发人员，要求清晰、准确、无歧义。

## 默认工作约定

- 开始任何项目任务前，先阅读并遵守 `CLAUDE.md` 中的项目规则。
- 涉及文档、功能设计、原型、同步、对比或规划建议时，必须优先判断并使用下方对应 Skill。
- 产出型任务前必须读取相关参考文档，不得只凭记忆或经验生成内容。
- 不随意新增一级目录，不随意改变既有编号体系，不移动历史文档，除非用户明确要求。
- 无法从现有文档明确得出的业务规则、字段含义、数值、流程分支或文档冲突，必须按 `CLAUDE.md` 的要求标注或询问。
- 修改完成后说明处理内容、涉及文件、验证结果，以及是否存在待确认事项。

## 核心工作流技能

| 技能 | 触发词 | 说明 |
|------|--------|------|
| write-feature-doc | 编写功能文档、记录功能、文档化功能 | 编写已上线功能描述，输出到 `02_已上线功能描述/` |
| feature-design | 我要做XX功能、优化XX功能、设计XX功能 | 编写功能设计方案，输出到 `03_需求设计文件/` |
| prototype-design | 生成原型、做原型设计、页面原型 | 生成 HTML 原型，输出到 `04_原型设计文件/` |
| doc-sync | /sync、同步文档 | 同步更新产品上下文、产品规划和相关文档 |
| planning-advice | 下一步做什么、功能优先级、规划建议 | 基于产品上下文、规划和已上线功能输出优先级建议 |

## 辅助技能

| 技能 | 触发词 | 说明 |
|------|--------|------|
| diff-analysis | 对比、区别、差异、核对、清单对照 | 严格逐条对比两份文档或清单的差异 |
| doc-maintenance | 文档约束、格式规范、文档规则 | 查询和执行文档维护、格式、约束规则 |
| experience-triage | 经验分诊、该写到哪、新流程放哪 | 判断经验应沉淀到项目、Skill、Agent 规则或其他层级 |

## 技能触发顺序

- 新功能从想法到落地：`feature-design` -> `prototype-design` -> `doc-sync`
- 已上线功能补充说明：`write-feature-doc` -> `doc-sync`
- 原型生成或优化：先读取对应功能设计，再使用 `prototype-design`
- 差异核对、清单对照、文档比较：优先使用 `diff-analysis`
- 文档结构、命名、格式、归档规则疑问：优先使用 `doc-maintenance`
- 功能优先级、下一步规划：优先使用 `planning-advice`
- 经验沉淀位置不明确：优先使用 `experience-triage`

## 快速参考

| 类型 | 位置 |

| 产品上下文 | `01_产品主文档/*产品上下文.md` |

### 文件编号

- `03_需求设计文件/` 和 `04_原型设计文件/` 下使用两位数字前缀 `NN_`。
- 功能设计与原型设计编号必须一致。
- 新增编号前先检查目标目录最大编号，按 `最大编号 + 1` 递增。

## 产出前置检查

- 功能设计任务：读取相关 `02_已上线功能描述/*模块.md`，涉及备案时同时读取 `06_附件资料/备案资料清单.md`。
- 原型设计任务：读取 `01_产品主文档/*原型设计标准文档.md`、对应功能设计文档和《原型对照清单》。
- 编写功能描述任务：读取产品上下文、产品规划和对应已上线功能描述。
- 规划建议任务：读取产品规划、产品上下文和已上线功能描述。
- 表格/数据处理任务：遵守 `CLAUDE.md` 中的字段映射、筛选、空值和输出格式确认规则。

## 完成标准

- 文档路径、文件名、编号、标题层级符合项目规范。
- 功能设计中的待确认事项、设计依据、决策记录清晰可追溯。
- 原型设计覆盖对应功能设计和《原型对照清单》中的核心规则。
- 对比类任务必须逐条列出差异，不得只给概括结论。
- 如果改动影响产品上下文、产品规划、已上线功能描述或功能状态，完成后应执行或建议执行 `doc-sync`。
- 输出最终结果时列出已完成内容、涉及文件、验证方式和剩余风险。

## 文件编码

- Markdown 文件统一使用 UTF-8 编码保存。
- 避免因终端编码导致中文乱码；如读取异常，优先使用 UTF-8 方式重新读取。
