# 文档同步技能

## 技能目标
在功能文档完成后,自动同步更新产品上下文和产品规划,确保文档体系的一致性。

## 执行步骤

### 1. 确认触发条件
- 功能文档已完成(位于`功能描述文档/`目录)
- 用户明确要求同步文档

### 2. 定位并读取相关文档
按顺序执行:
1. 刚完成的功能文档(从用户消息或上下文获取路径)
2. 使用Glob搜索 `*产品上下文.md` 定位产品上下文文档
3. 使用Glob搜索 `*产品规划.md` 定位产品规划文档
4. 读取这三个文档

### 3. 更新产品上下文
**更新内容**:
- 在"已实现功能"章节添加新功能
- 格式: `- **[功能名称]**: [一句话描述]`
- 保持按模块分类的结构

**更新原则**:
- 只添加核心功能点,不重复已有内容
- 描述简洁,突出功能价值
- 保持与现有条目的风格一致

### 4. 更新产品规划
**更新内容**:
- 将已完成的功能从"待开发"移到"已完成"
- 更新功能状态和完成时间
- 调整优先级(如果需要)

**更新原则**:
- 保持规划的完整性和可追溯性
- 标注完成时间: `[已完成 YYYY-MM-DD]`
- 如果功能拆分或合并,在备注中说明

### 5. 确认更新
向用户展示:
1. 产品上下文的更新内容
2. 产品规划的更新内容
3. 询问是否需要调整

### 6. 执行更新
用户确认后,使用Edit工具更新两个文档。

## 输出格式

```markdown
## 文档同步完成

### 产品上下文更新
- 在"[模块名]"章节添加: [功能描述]

### 产品规划更新
- 将"[功能名]"标记为已完成
- 完成时间: [日期]

所有文档已同步更新。
```

## 注意事项
- 不要删除或修改已有内容,只添加新内容
- 保持文档的结构和格式一致性
- 如果不确定应该放在哪个模块,询问用户
- 同步完成后,建议用户提交git commit
