## 一、系统整体流程架构图

```mermaid
flowchart TB
    subgraph 备案阶段["📋 备案管理阶段"]
        A[业主/施工方<br/>在线申报] --> B[AI审核]
        B --> C[人工审核]
        C -->|通过| D[生成备案回执<br/>工程纳入监管]
        C -->|驳回| A
    end

    subgraph 施工监管阶段["🏗️ 施工监管阶段"]
        D --> E[工程状态: 正常施工]
        E --> F{巡查人员<br/>定期巡查}

        subgraph 巡查流程["巡查流程"]
            F -->|未发现违规| G[记录巡查结果<br/>保持正常施工]
            F -->|发现违规| H[违规登记]
            H --> I{隐患等级}
            I -->|较大隐患| J[停工整改]
            I -->|一般隐患| K[现场整改]
            I -->|轻微违规| L[记录在案<br/>不触发整改]
        end

        subgraph 整改流程["整改流程"]
            J --> M[施工方<br/>提交整改记录]
            K --> M
            M --> N[同组织审核员<br/>审核整改]
            N -->|通过| O[工程恢复<br/>正常施工]
            N -->|驳回| M
        end
    end

    subgraph 完工核销阶段["✅ 完工销项阶段"]
        O --> P{完工核销}
        E --> P

        subgraph 核销路径["核销路径"]
            P -->|路径1| Q[施工方<br/>申请核销]
            P -->|路径2| R[巡查员<br/>直接核销]
            Q --> S[巡查员审核]
            S -->|通过| T[工程状态: 已核销]
            S -->|驳回| Q
            R --> T
        end
    end

    subgraph 档案归档["📁 工程归档"]
        T --> U[工程档案归档]
        U --> V[备案信息]
        U --> W[巡查记录]
        U --> X[整改记录]
        U --> Y[核销记录]
    end

    %% 样式定义
    style 备案阶段 fill:#e6f7ff,stroke:#1890ff
    style 施工监管阶段 fill:#f6ffed,stroke:#52c41a
    style 完工核销阶段 fill:#fff7e6,stroke:#fa8c16
    style 档案归档 fill:#f9f0ff,stroke:#722ed1
    style 巡查流程 fill:#f0f5ff,stroke:#2f54eb
    style 整改流程 fill:#fff2f0,stroke:#eb2f96
    style 核销路径 fill:#e6fffb,stroke:#13c2c2
```

## 三、巡查与整改闭环流程图

```mermaid
flowchart TB
    subgraph 巡查发起["1️⃣ 巡查发起"]
        A[工程详情页] --> B[点击发起巡查]
        B --> C[填写巡查信息]
        C --> D{巡查结果}
        D -->|未发现违规| E[提交巡查记录<br/>流程结束]
        D -->|发现违规| F[进入违规登记]
    end

    subgraph 违规登记["2️⃣ 违规登记"]
        F --> G[选择隐患类型<br/>从标准隐患清单]
        G --> H[上传现场照片]
        H --> I[填写违规描述]
        I --> J[设置整改要求]
        J --> K{隐患等级}
        K -->|较大隐患| L[处置方式: 停工整改<br/>强制不可修改]
        K -->|一般隐患| M[选择处置方式<br/>现场整改/不需要限时整改]
        L --> N[设置整改期限]
        M -->|需要限时整改| N
        M -->|不需要限时整改| O[记录在案<br/>不改变工程状态]
        N --> P[提交违规登记]
    end

    subgraph 工程状态变更["3️⃣ 工程状态变更"]
        P --> Q{同次巡查<br/>多条违规}
        Q -->|含停工整改| R[工程状态: 停工整改]
        Q -->|仅现场整改| S[工程状态: 现场整改]
        Q -->|全部不需要限时整改| T[工程状态: 正常施工]
    end

    subgraph 整改流程["4️⃣ 整改流程"]
        R --> U[通知施工方<br/>提交整改记录]
        S --> U
        U --> V[施工方上传<br/>整改照片+说明]
        V --> W[提交整改记录<br/>状态: 待审核]
        W --> X[同组织审核员<br/>审核整改]
        X -->|整体通过| Y[上传验收照片<br/>工程恢复正常施工]
        X -->|整体驳回| Z[填写驳回原因<br/>返回待整改]
        Z --> V
    end

    style 巡查发起 fill:#e6f7ff,stroke:#1890ff
    style 违规登记 fill:#fff7e6,stroke:#fa8c16
    style 工程状态变更 fill:#fff1f0,stroke:#f5222d
    style 整改流程 fill:#f6ffed,stroke:#52c41a
```

## 四、工程状态流转图

```mermaid
stateDiagram-v2
    [*] --> 正常施工: 备案审核通过

    正常施工 --> 正常施工: 巡查无违规<br/>整改审核通过
    正常施工 --> 现场整改: 巡查发现一般隐患<br/>需现场整改
    正常施工 --> 停工整改: 巡查发现较大隐患<br/>需停工整改
    正常施工 --> 核销待审核: 发起核销申请
    正常施工 --> 已核销: 巡查员直接核销

    现场整改 --> 正常施工: 整改审核通过
    现场整改 --> 停工整改: 发现更严重违规<br/>升级为停工整改
    现场整改 --> 核销待审核: 发起核销申请
    现场整改 --> 已核销: 巡查员直接核销

    停工整改 --> 正常施工: 整改审核通过
    停工整改 --> 核销待审核: 发起核销申请
    停工整改 --> 已核销: 巡查员直接核销

    核销待审核 --> 已核销: 审核通过
    核销待审核 --> 正常施工: 审核驳回<br/>恢复申请前状态
    核销待审核 --> 现场整改: 审核驳回<br/>恢复申请前状态
    核销待审核 --> 停工整改: 审核驳回<br/>恢复申请前状态

    已核销 --> [*]: 工程归档

    note right of 正常施工
        可发起巡查
        可申请/直接核销
    end note

    note right of 现场整改
        可继续巡查
        需提交整改记录
    end note

    note right of 停工整改
        强制停工
        整改完成前不可复工
    end note

    note right of 已核销
        终态，不可撤销
        15天内可继续完成
        遗留整改流程
    end note
```

## 五、完工核销流程图

```mermaid
flowchart TB
    subgraph 触发条件["触发条件判断"]
        A[工程状态] --> B{是否已核销?}
        B -->|是| C[显示已核销标签<br/>流程结束]
        B -->|否| D{是否存在待审核<br/>核销申请?}
        D -->|是| E[显示审核中状态<br/>等待审核]
        D -->|否| F[显示核销入口]
    end

    subgraph 路径一["路径一: 施工方申请核销"]
        F --> G[施工方填写<br/>完工日期/现场照片/备注]
        G --> H[提交核销申请]
        H --> I[记录申请前工程状态]
        I --> J[工程状态: 核销待审核]
        J --> K[锁定新增巡查入口]
        K --> L[巡查员审核]
        L -->|通过| M[工程状态: 已核销]
        L -->|驳回| N[恢复申请前状态]
        N --> G
    end

    subgraph 路径二["路径二: 巡检端直接核销"]
        F --> O[巡查员确认是否完工]
        O -->|是| P[上传现场图片]
        P --> Q[工程状态: 已核销]
        O -->|否| R[提示: 未完工不可核销]
    end

    subgraph 核销后处理["核销后处理"]
        M --> S[生成核销记录]
        Q --> S
        S --> T[15天内允许继续<br/>提交遗留整改]
        T --> U[工程归档<br/>查询历史记录]
    end

    style 触发条件 fill:#e6f7ff,stroke:#1890ff
    style 路径一 fill:#f6ffed,stroke:#52c41a
    style 路径二 fill:#fff7e6,stroke:#fa8c16
    style 核销后处理 fill:#f9f0ff,stroke:#722ed1
```

## 六、三方巡查协同流程图（时序图）

```mermaid
sequenceDiagram
    participant 街道 as 🏛️ 街道巡查员
    participant 社区 as 🏘️ 社区巡查员
    participant 物业 as 🏢 物业巡查员
    participant 施工方 as 👷 施工方
    participant 系统 as 💻 系统

    rect rgb(230, 247, 255)
        Note over 街道,系统: 备案审核通过后工程纳入监管
        系统->>街道: 分配工程至各组织
        系统->>社区: 分配工程至各组织
        系统->>物业: 分配工程至各组织
    end

    rect rgb(246, 255, 237)
        Note over 街道,系统: 各方独立开展巡查
        par 街道巡查
            街道->>系统: 发起巡查
            系统-->>街道: 记录巡查结果
        and 社区巡查
            社区->>系统: 发起巡查
            系统-->>社区: 记录巡查结果
        and 物业巡查
            物业->>系统: 发起巡查
            系统-->>物业: 记录巡查结果
        end
    end

    rect rgb(255, 247, 230)
        Note over 街道,系统: 发现违规 → 独立整改流程
        街道->>系统: 登记违规(较大隐患)
        系统->>施工方: 通知停工整改
        施工方->>系统: 提交整改记录
        系统->>街道: 街道审核员审核
        街道-->>系统: 审核通过
        系统->>施工方: 通知整改通过
    end

    rect rgb(255, 240, 240)
        Note over 街道,系统: 跨组织可见整改状态
        社区->>系统: 查看工程详情
        系统-->>社区: 显示街道巡查记录<br/>显示整改状态
        物业->>系统: 查看工程详情
        系统-->>物业: 显示各方巡查统计<br/>不显示整改操作入口
    end

    rect rgb(240, 255, 250)
        Note over 街道,系统: 完工核销
        alt 施工方申请核销
            施工方->>系统: 提交核销申请
            系统->>街道: 通知审核
            街道->>系统: 审核通过
        else 巡查员直接核销
            街道->>系统: 直接核销
        end
        系统-->>系统: 工程状态: 已核销
        系统->>施工方: 通知核销完成
        系统->>社区: 同步核销状态
        系统->>物业: 同步核销状态
    end
```
