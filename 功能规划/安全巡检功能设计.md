## 一、功能背景

### 1.1 业务痛点

当前小型工程施工期间的安全监管存在以下问题:

1. **巡查记录不规范**:纸质记录易丢失,信息不完整,无法追溯
2. **隐患整改难跟踪**:发现隐患后缺少闭环管理,整改情况难以跟踪
3. **违规处置不及时**:发现违规行为后处置流程不清晰,容易推诿扯皮
4. **数据统计困难**:巡查数据分散,无法统计分析,难以支撑决策

### 1.2 解决方案

通过"安全巡检"功能,实现:

1. **规范巡查流程**:标准化巡查记录,现场拍照取证,数据实时上传
2. **闭环整改管理**:隐患登记→整改通知→整改提交→验收销号,全流程跟踪
3. **分级处置机制**:根据违规严重程度,自动触发不同处置流程
4. **数据可视化**:巡查数据实时统计,支持导出和分析

## 二、核心概念

### 2.1 工程状态定义

工程共有四种状态，状态流转关系如下：

```
正常施工 ──→ 现场整改 ──→ 正常施工
         └─→ 停工整改 ──→ 正常施工
         └─→ 已核销（终态）
```

---

#### 正常施工

**定义**：工程正常进行，无未处理的安全隐患。

**进入条件**：
- 工程初始状态（开工时默认为正常施工）
- 现场整改或停工整改的整改记录审核通过后，恢复为正常施工

**该状态下可执行的操作**：

| 角色 | 可执行操作 |
|------|-----------|
| 巡查人员 | 发起巡查、登记违规 |
| 监管人员 | 提交完工记录（触发核销） |
| 施工方 | 申请核销 |

**状态流转**：
- → **现场整改**：巡查登记违规，处置方式选择"现场整改"
- → **停工整改**：巡查登记违规，处置方式选择"停工整改"
- → **已核销**：监管提交完工记录，或施工方申请核销且监管审核通过

---

#### 现场整改

**定义**：巡查发现需要限时整改的违规，工程暂停施工，等待整改完成并通过审核。

**进入条件**：巡查登记违规时，"是否需要限时整改"选择"是"，处置方式选择"现场整改"。

**该状态下可执行的操作**：

| 角色 | 可执行操作 |
|------|-----------|
| 巡查人员 | 可继续对该工程发起新的巡查 |
| 施工方 | 提交整改记录（上传整改照片和整改说明） |
| 审核员 | 审核整改记录（通过/驳回） |
| 监管人员 | 查看整改进度、催办整改 |

**状态流转**：
- → **正常施工**：整改记录审核通过
- → **停工整改**：巡查发现更严重违规，处置方式升级为"停工整改"（新违规记录覆盖工程状态）

---

#### 停工整改

**定义**：巡查发现严重违规行为，工程强制停工，必须完成整改并通过审核后方可复工。

**进入条件**：巡查登记违规时，"是否需要限时整改"选择"是"，处置方式选择"停工整改"。

**该状态下可执行的操作**：

| 角色 | 可执行操作 |
|------|-----------|
| 巡查人员 | 可继续对该工程发起新的巡查 |
| 施工方 | 提交整改记录（上传整改照片和整改说明） |
| 审核员 | 审核整改记录（通过/驳回） |
| 监管人员 | 查看整改进度、催办整改、移交执法 |

**状态流转**：
- → **正常施工**：整改记录审核通过

---

#### 已核销

**定义**：工程已完工，核销审核通过后进入该状态。已核销为终态，不可撤销，不可流转至其他状态。

**进入条件**（二选一）：
1. **监管主动核销**：监管人员提交完工记录，直接核销
2. **施工方申请核销**：施工方发起核销申请，监管人员审核通过后核销

**该状态下可执行的操作**：

| 角色 | 可执行操作 | 说明 |
|------|-----------|------|
| 所有角色 | 查看历史巡查记录 | 只读，不可修改 |
| 施工方 | 继续提交未完成的整改记录 | 用于恢复因违规被扣的积分 |
| 巡查人员 | **不可**发起新的巡查 | — |
| 监管人员 | **不可**撤销核销状态 | — |

> **注意**：已核销工程仍可继续走完未结束的整改流程（提交整改记录→审核通过→积分恢复），但不产生新的巡查记录。

### 2.2 违规行为分类

根据是否需要限时整改,违规行为分为两类:

**不需要限时整改的违规**:
- 轻微违规,不影响施工安全
- 勾选后提交审核通过即可加分
- 不提交审核不加分
- 工程状态不改变

**需要限时整改的违规**:
- 存在安全隐患,需要整改后才能继续施工
- 必须提交整改记录并通过审核
- 工程状态变更为"现场整改"或"停工整改"
- 整改通过后恢复"正常施工"状态

### 2.3 角色与权限

| 角色 | 说明 | 核心操作 |
|------|------|----------|
| 巡查人员 | 街道/社区/物业的巡查工作人员 | 发起巡查、登记违规 |
| 审核员 | 与提交整改记录的施工方**同组织**的成员（街道/社区/物业各自内部审核） | 审核整改记录（通过/驳回） |
| 监管人员 | 上级监管部门工作人员 | 查看整改进度、催办整改、移交执法 |
| 施工方 | 施工单位负责人或相关人员 | 查看违规记录、提交整改记录、申请核销 |

**审核规则**：整改审核由**同组织内部**相互审核，即：
- 街道巡查员发起的违规 → 街道审核员审核整改
- 社区巡查员发起的违规 → 社区审核员审核整改
- 物业巡查员发起的违规 → 物业审核员审核整改

### 2.4 巡查频次规则

系统根据工程风险等级和类型，计算每个工程的"应巡查频次"，超出频次要求的工程在**待巡查列表**中提醒，不强制阻断操作。

**频次规则**（参考深圳标准）：

**高风险工程**：

| 组织 | 频次要求 |
|------|----------|
| 街道 | 每10天（含）内至少1条巡查记录 |
| 社区 | 每14天（含）内至少1条巡查记录 |
| 物业 | 每2天（含）内至少1条巡查记录 |

**非高风险工程 · 家装工程**（工程类型为"限额以下竣工后验收合格的室内住宅装修类型"）：

| 组织 | 频次要求 |
|------|----------|
| 街道 | 每30天（含）内至少1条巡查记录 |
| 社区 | 每21天（含）内至少1条巡查记录 |
| 物业 | 每7天（含）内至少1条巡查记录 |

**非高风险工程 · 非家装工程**：

| 组织 | 频次要求 |
|------|----------|
| 街道 | 每21天（含）内至少1条巡查记录 |
| 社区 | 每14天（含）内至少1条巡查记录 |
| 物业 | 每3天（含）内至少1条巡查记录 |

**系统行为**：
- 超出频次要求的工程显示在"待巡查"列表，并标注超期天数
- 仅作提醒，不阻断其他操作
- 待巡查数量在首页概览中统计展示

### 2.5 隐患分级与处置方式映射规则

巡查使用标准化《隐患清单》进行现场核查，每个检查项已预设隐患等级，等级直接映射处置方式：

| 隐患等级 | 定义 | 处置方式 | 对工程状态的影响 |
|---------|------|---------|----------------|
| **较大隐患** | 存在重大安全风险，需立即停工 | 停工整改（强制，不可修改） | 工程状态变为"停工整改" |
| **一般隐患** | 存在一般安全问题 | 由巡查员判断：现场整改 / 不需要限时整改 | 变为"现场整改"，或保持不变 |

**业务规则**：
- 巡查员选择**较大隐患**检查项后，处置方式自动锁定为"停工整改"，不可更改
- 巡查员选择**一般隐患**检查项后，可自行选择"现场整改"或"不需要限时整改"
- 同一次巡查登记多条隐患时，工程状态取最严级别（有一条较大隐患即停工整改）

> 完整隐患清单详见文末附录。

## 三、整体页面结构

```
首页（巡查概览）
  ├─ 统计区：小型工程 / 住宅装修 / 零星作业 各类型在建总数
  └─ 待巡查数量（超出频次要求的工程数）

功能区
  ├─ 待巡查列表 → 工程列表（需尽快巡查）
  └─ 全部工程列表
        ├─ 搜索框
        ├─ 类型Tab：小型工程 / 住宅装修 / 零星作业
        ├─ 状态Tab：正常施工 / 现场整改 / 停工整改 / 已核销
        └─ 工程卡片 → 工程详情页
                          ├─ 简化版备案信息（工程名、地址、施工单位、施工时间、负责人）
                          │     └─ "查看完整备案" 按钮 → 完整备案详情页（只读）
                          ├─ 地图（显示工程位置，方便巡查员导航）
                          ├─ 巡查记录列表（多条，倒序）
                          │     └─ 巡查详情页
                          ├─ 违规整改记录列表（按违规条目展示，多条）
                          │     └─ 整改详情页（含逐条整改进度）
                          └─ 操作按钮（根据角色和工程状态显示）
                                ├─ 发起巡查（巡查人员）
                                ├─ 提交整改记录（施工方，有待整改违规时）
                                ├─ 审核整改（审核员，有待审核整改时）
                                ├─ 申请核销（施工方，正常施工状态）
                                └─ 提交完工核销（监管人员，正常施工状态）
```

### 4.4 发起巡查

#### 4.4.1 页面结构

**页面名称**："发起巡查"页面

**触发条件**：在工程详情页点击"发起巡查"按钮

**页面功能**：

1. **工程信息**（只读，来自工程详情页）
   - 工程名称、工程地址（顶部展示条，不可修改）

2. **巡查信息**
   - 巡查时间（默认当前时间，可修改）
   - 巡查人员（默认当前登录用户，只读）
   - 是否施工中（单选：是 / 否）
   - 现场施工人员（可选择，支持多选）

3. **巡查结果选择**
   - 未发现违法违规（单选）
   - 发现违法违规行为（单选）

#### 4.4.2 交互说明

- 选择"未发现违法违规" → 直接提交，不改变工程状态
- 选择"发现违法违规行为" → 进入违规登记页面

---

### 4.5 违规登记

#### 4.5.1 页面结构

**页面名称**："违规登记"页面

**设计原则**：**每条违规独立登记**，一次巡查可添加多条违规记录，每条违规独立设置整改要求、期限和处置方式，施工方后续也按每条违规独立提交整改记录、逐条审核。

**页面功能**：

**顶部工程信息条**（只读）
- 工程名称

**违规记录卡片列表**（可添加多条，每条为一个独立卡片）

每条违规卡片包含：

1. **隐患选择**（必填，从标准隐患清单中选择）
   - 按类别展示检查项（信息登记管理 / 安全管理 / 高处作业 / 施工用电 / 施工机具 / 消防安全 / 物体打击 / 文明施工 / 拆除作业 / 脚手架 / 吊篮作业 / 起重吊装作业 / 有限空间作业 / 物料提升机）
   - 选中后自动带出"隐患等级"标签（较大隐患 / 一般隐患）
   - 支持搜索关键字快速定位检查项
   - 可选"其他"并手动输入描述（手动输入默认为一般隐患）

2. **违规详情**
   - 违规描述（文本输入，限500字，选填，用于补充说明）
   - 现场照片（必传，最多9张）

3. **整改要求**
   - **较大隐患**：处置方式自动锁定为"停工整改"，不可更改；整改期限必填
   - **一般隐患**：需选择处置方式（单选）
     - 不需要限时整改 → 此条隐患不影响工程状态，审核通过加分
     - 现场整改 → 工程状态变为"现场整改"，整改期限必填
   - 整改要求说明（文本输入，限500字）

**底部操作**
- "＋ 继续添加违规" 按钮（添加下一条违规卡片）
- "提交违规登记" 按钮

#### 4.5.2 工程状态变更规则

同一次巡查中登记多条违规时，工程状态取**最严级别**：
- 有任意一条选择"停工整改" → 工程状态变为"停工整改"
- 全部为"现场整改" → 工程状态变为"现场整改"
- 全部为"不需要限时整改" → 工程状态不变

#### 4.5.3 交互说明

- 每条违规卡片可独立展开/折叠
- 提交后，每条违规生成一条独立的整改记录，施工方收到通知后逐条处理

### 4.6 整改提交（施工方）

#### 4.6.1 页面结构

**页面名称**："提交整改记录"页面

**触发条件**：施工方在工程详情页点击"提交整改记录"按钮，或收到整改通知后进入

**页面功能**：

**待整改违规列表**（每条违规独立展示）

每条违规整改卡片包含：

1. **违规信息**（只读）
   - 违规类型
   - 违规描述
   - 整改要求说明
   - 整改期限（标注剩余天数，逾期时标红）
   - 巡查人员 / 登记时间
   - 违规现场照片（可查看）

2. **整改记录提交**
   - 整改说明（文本输入，限500字，必填）
   - 整改照片（必传，最多9张）

3. **该条违规操作按钮**
   - "提交此条整改" 按钮

#### 4.6.2 交互说明

- 每条违规独立提交，互不影响
- 某条违规提交整改后进入"待审核"状态，该卡片变为只读
- 所有需要限时整改的违规均审核通过后，工程状态恢复"正常施工"
- 驳回的违规可重新上传整改记录再次提交

### 4.7 整改审核（审核员）

#### 4.7.1 页面结构

**页面名称**："整改审核"页面

**页面功能**：

   - 整改要求、整改期限
   - 违规现场照片

2. **施工方整改记录**（只读）
   - 整改说明
   - 整改照片
   - 提交时间

3. **审核操作**
   - 通过
     - 上传验收照片（必传，最多9张）
     - 验收说明（选填，限200字）
   - 驳回
     - 驳回原因（必填，限200字）

#### 4.7.2 交互说明

- 每条违规独立审核，互不影响
- 审核通过 → 该条违规整改完成，施工方积分恢复；所有违规均通过后，工程状态恢复"正常施工"

## 五、业务流程

### 5.1 完整流程图

```
发起巡查（从工程详情页进入，工程已确定）
  │
  ├─ 未发现违法违规 → 提交巡查记录 → 不改变工程状态 → 结束
  │
  └─ 发现违法违规行为 → 逐条登记违规（每条独立设置整改要求）
         │
         ├─ 违规A：不需要限时整改
         │    └─ 提交 → 同组织审核员审核
         │          ├─ 通过 → 施工方加分 → 结束
         │          └─ 驳回 → 结束，不加分
         │
         ├─ 违规B：需要限时整改 · 现场整改
         │    └─ 工程状态变更为"现场整改"（若无更严重违规）
         │         └─ 施工方针对此条提交整改记录
         │              └─ 同组织审核员审核
         │                   ├─ 通过（上传验收照片）→ 施工方加分
         │                   │    └─ 所有限时整改违规均通过 → 工程恢复"正常施工"
         │                   └─ 驳回 → 通知施工方重新提交（循环）
         │
         └─ 违规C：需要限时整改 · 停工整改
              └─ 工程状态变更为"停工整改"（覆盖现场整改）
                   └─ 施工方针对此条提交整改记录
                        └─ 同组织审核员审核
                             ├─ 通过（上传验收照片）→ 施工方加分
                             │    └─ 所有限时整改违规均通过 → 工程恢复"正常施工"
                             └─ 驳回 → 通知施工方重新提交（循环）
```

### 5.2 整改逾期处理流程

```
每条违规整改期限到期（逐条判断）
  │
  ├─ 施工方已提交整改记录 → 等待同组织审核员审核
  │
  └─ 施工方未提交整改记录
       │
       ├─ 逾期1-3天 → 系统自动发送催办通知给施工方
       │
       ├─ 逾期3-7天 → 监管人员电话催办
       │
       └─ 逾期超过7天 → 可移交执法部门
```

### 5.3 工程核销流程图

```
工程正常施工状态（无未处理整改）
  │
  ├─ 路径一：监管主动核销
  │    └─ 监管人员提交完工记录 → 工程状态变更为"已核销"→ 结束
  │
  └─ 路径二：施工方申请核销
       └─ 施工方发起核销申请
            └─ 监管人员审核
                 ├─ 审核通过 → 工程状态变更为"已核销"→ 结束
                 └─ 审核驳回 → 通知施工方 → 结束

注：
- 已核销为终态，不可撤销
- 已核销工程存在未完成整改记录时，仍可继续提交整改→审核通过→积分恢复
- 已核销工程不可发起新的巡查
```

## 七、技术实现要点

### 7.1 工程状态管理

- 工程状态字段：status（正常施工/现场整改/停工整改/已核销）
- 状态变更规则：取同一次巡查中所有违规的最严处置方式
- 状态变更记录：记录状态变更时间、变更原因、操作人
- 状态联动：工程状态变更后，自动通知相关方
- 已核销为终态：数据库层面限制已核销工程的状态字段不可被修改

### 7.2 违规整改逐条管理

- 每条违规作为独立记录（violation），含：违规类型、描述、照片、整改要求、期限、处置方式
- 每条违规有独立整改记录（rectification），含：施工方提交内容、审核结果、审核照片
- 工程状态由所有**未完成的限时整改违规**中最严级别决定
- 所有限时整改违规均审核通过 → 工程状态恢复正常施工

### 7.3 整改期限提醒

- 整改期限到期前1天:发送提醒通知给施工方
- 整改期限到期:标记为逾期,发送催办通知
- 逾期超过设定天数:自动触发移交执法流程

### 7.4 照片上传与存储

- 支持拍照和相册选择
- 照片压缩后上传,减少流量消耗
- 照片存储在云端,支持查看和下载

### 7.5 消息通知

- 整改通知（逐条）：每条违规登记后分别发送给施工方
- 审核结果通知（逐条）：每条整改审核完成后发送给施工方
- 催办通知：发送给施工方
- 逾期预警：发送给监管人员
- 待巡查提醒：发送给对应组织的巡查人员

### 7.6 审核权限控制

- 审核员只能审核**本组织提交整改**记录的违规
- 同一人不能既提交整改又审核同一条违规
- 权限与备案模块的组织归属字段联动
