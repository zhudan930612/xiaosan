## 一、功能背景

### 1.1 业务痛点

当前小型工程施工期间的安全监管存在以下问题:

1. **巡查记录不规范**:纸质记录易丢失,信息不完整,无法追溯
2. **隐患整改难跟踪**:发现隐患后缺少闭环管理,整改情况难以跟踪
3. **违规处置不及时**:发现违规行为后处置流程不清晰,容易推诿扯皮
4. **数据统计困难**:巡查数据分散,无法统计分析,难以支撑决策

### 1.2 解决方案

通过"安全巡检"功能,实现:

1. **规范巡查流程**:标准化巡查记录,现场拍照取证,数据实时上传
2. **闭环整改管理**:隐患登记→整改通知→整改提交→验收销号,全流程跟踪
3. **分级处置机制**:根据违规严重程度,自动触发不同处置流程
4. **数据可视化**:巡查数据实时统计,支持导出和分析

## 二、核心概念

### 2.1 工程状态定义

工程在施工期间有以下状态:

- **正常施工**:工程按规范施工,无安全隐患
- **现场整改**:发现需要限时整改的违规行为,工程暂停施工,等待整改完成
- **停工整改**:发现严重违规行为,工程强制停工,必须整改后才能复工

### 2.2 违规行为分类

根据是否需要限时整改,违规行为分为两类:

**不需要限时整改的违规**:
- 轻微违规,不影响施工安全
- 勾选后提交审核通过即可加分
- 不提交审核不加分
- 工程状态不改变

**需要限时整改的违规**:
- 存在安全隐患,需要整改后才能继续施工
- 必须提交整改记录并通过审核
- 工程状态变更为"现场整改"或"停工整改"
- 整改通过后恢复"正常施工"状态

### 2.3 审核角色

- **施工方**:提交整改记录,上传整改照片
- **物业/街道/社区审核员**:审核整改记录,判断是否通过
- **监管人员**:查看巡查记录,监督整改进度

## 三、功能详细设计

### 3.1 发起巡查

#### 3.1.1 页面结构

**页面名称**:"发起巡查"页面

**页面功能**:

1. **选择工程**
   - 工程列表(显示正在施工的工程)
   - 工程名称、工程地址、施工单位
   - 支持搜索工程名称或地址

2. **巡查信息**
   - 巡查时间(默认当前时间,可修改)
   - 巡查人员(默认当前登录用户,只读)
   - 巡查类型(日常巡查/专项检查/物业巡查)

3. **巡查结果选择**
   - 未发现违法违规(单选)
   - 发现违法违规行为(单选)

#### 3.1.2 交互说明

- 选择"未发现违法违规"→ 直接提交,不改变工程状态
- 选择"发现违法违规行为"→ 进入违规登记页面

---

### 3.2 违规登记

#### 3.2.1 页面结构

**页面名称**:"违规登记"页面

**页面功能**:

1. **违规类型选择**(多选)
   - 用电作业违规
   - 高空作业违规
   - 动火作业违规
   - 有限空间作业违规
   - 动土作业违规
   - 拆除作业违规
   - 未佩戴安全帽
   - 未设置安全警示标志
   - 施工现场脏乱差
   - 其他违规(需填写具体内容)

2. **违规详情**
   - 违规描述(文本输入,限500字)
   - 现场照片(必传,最多9张)
   - 违规位置(地图标注,可选)

3. **整改要求**
   - 是否需要限时整改(单选)
     - 不需要限时整改
     - 需要限时整改
   - 整改期限(需要限时整改时必填,日期选择)
   - 整改要求(文本输入,限500字)

4. **处置方式**(需要限时整改时显示)
   - 现场整改(工程状态变更为"现场整改")
   - 停工整改(工程状态变更为"停工整改")

#### 3.2.2 交互说明

- 选择"不需要限时整改"→ 提交后进入审核流程,审核通过后加分,工程状态不变
- 选择"需要限时整改"→ 必须选择处置方式,提交后工程状态变更,通知施工方整改

---

### 3.3 整改提交(施工方)

#### 3.3.1 页面结构

**页面名称**:"整改提交"页面

**触发条件**:施工方收到整改通知后

**页面功能**:

1. **违规信息展示**(只读)
   - 违规类型
   - 违规描述
   - 现场照片
   - 整改要求
   - 整改期限

2. **整改记录提交**
   - 整改说明(文本输入,限500字)
   - 整改照片(必传,最多9张)
   - 提交时间(自动记录)

3. **操作按钮**
   - 提交整改记录(绿色按钮)

#### 3.3.2 交互说明

- 点击"提交整改记录"→ 提交后进入审核流程
- 审核通过→ 工程状态恢复为"正常施工"
- 审核驳回→ 需要重新提交整改记录

---

### 3.4 整改审核(物业/街道/社区)

#### 3.4.1 页面结构

**页面名称**:"整改审核"页面

**页面功能**:

1. **待审核列表**
   - 工程名称
   - 违规类型
   - 提交时间
   - 整改期限
   - 状态标签(待审核/已通过/已驳回)

2. **审核详情**
   - 违规信息(只读)
   - 整改记录(只读)
   - 整改前后对比照片

3. **审核操作**
   - 通过(需要拍照上传存档片)(加分)
     - 上传验收照片(必传,最多9张)
     - 验收说明(文本输入,限200字)
   - 驳回
     - 驳回原因(文本输入,限200字,必填)

#### 3.4.2 交互说明

- 点击"通过"→ 上传验收照片→ 工程状态恢复为"正常施工"→ 施工方加分
- 点击"驳回"→ 输入驳回原因→ 工程状态还是"现场整改"或"停工整改"→ 通知施工方重新提交

---

**页面名称**:"巡查记录"页面

- 点击"查看详情"→ 进入巡查详情页面

**页面名称**:"整改跟踪"页面

## 四、业务流程

### 4.1 完整流程图

```
发起巡查
  │
  ├─ 未发现违法违规 → 提交巡查记录 → 不改变工程状态 → 结束
  │
  └─ 发现违法违规行为
       │
       ├─ 不需要限时整改
       │    └─ 提交违规记录 → 审核
       │          ├─ 通过 → 加分 → 结束
       │          └─ 驳回 → 不加分 → 结束
       │
       └─ 需要限时整改
            │
            ├─ 现场整改
            │    └─ 工程状态变更为"现场整改"
            │         └─ 施工方提交整改记录
            │              └─ 物业/街道/社区审核
            │                   ├─ 通过(需拍照上传存档片)(加分)
            │                   │    └─ 工程状态恢复为"正常施工"
            │                   └─ 驳回
            │                        └─ 工程状态还是"现场整改"
            │                             └─ 施工方重新提交整改记录
            │
            └─ 停工整改
                 └─ 工程状态变更为"停工整改"
                      └─ 施工方提交整改记录
                           └─ 物业/街道/社区审核
                                ├─ 通过(需拍照上传存档片)(加分)
                                │    └─ 工程状态恢复为"正常施工"
                                └─ 驳回
                                     └─ 工程状态还是"停工整改"
                                          └─ 施工方重新提交整改记录
```

### 4.2 逾期处理流程

```
整改期限到期
  │
  ├─ 施工方已提交整改记录 → 等待审核
  │
  └─ 施工方未提交整改记录
       │
       ├─ 逾期1-3天 → 系统自动发送催办通知
       │
       ├─ 逾期3-7天 → 监管人员电话催办
       │
       └─ 逾期超过7天 → 移交执法部门
```

## 六、技术实现要点

### 6.1 工程状态管理

- 工程状态字段:status(正常施工/现场整改/停工整改)
- 状态变更记录:记录状态变更时间、变更原因、操作人
- 状态联动:工程状态变更后,自动通知相关方

### 6.2 整改期限提醒

- 整改期限到期前1天:发送提醒通知给施工方
- 整改期限到期:标记为逾期,发送催办通知
- 逾期超过设定天数:自动触发移交执法流程

### 6.3 照片上传与存储

- 支持拍照和相册选择
- 照片压缩后上传,减少流量消耗
- 照片存储在云端,支持查看和下载

### 6.4 消息通知

- 整改通知:发送给施工方
- 审核结果通知:发送给施工方
- 催办通知:发送给施工方
- 逾期预警:发送给监管人员
