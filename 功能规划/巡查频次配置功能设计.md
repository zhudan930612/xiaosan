## 设计依据说明

**基于产品上下文**
- 工程分类枚举（术语表）：小型工程包含三大类
  - 限额以下建设工程（11类）
  - 住宅室内装饰装修工程
  - 零星施工作业（9类）
- 三方巡查模型（日常巡查模块）：街道、社区、物业三类组织各自独立开展巡查
- 高风险工程定义：符合建设行政管理部门制定的高风险判定标准的小型工程

## 功能单元

### 功能概述

巡查频次配置是一个后台管理功能，允许系统管理员为不同**工程类型**配置不同**巡查组织**的巡查频次要求。采用**三维矩阵**配置模式：

1. **第一维 - 工程风险等级**：高风险 / 非高风险
2. **第二维 - 工程类型**：小型工程 / 住宅装修 / 零星作业
3. **第三维 - 巡查组织**：街道 / 社区 / 物业

组合后可配置如：
- 街道对高风险住宅装修的巡查频次
- 物业对非高风险零星作业的巡查频次
- 社区对非高风险小型工程的巡查频次

### 业务规则

**规则1：配置维度**
- 配置采用【风险等级】×【工程类型】×【巡查组织】的三维矩阵
- 2种风险等级 × 3种工程类型 × 3种巡查组织 = 18种配置组合
- 每种组合独立配置，互不影响

**规则2：频次设置**
- 巡查频次格式：每 [N] 天至少巡查 [M] 次
- N：周期天数，取值范围 1-365
- M：巡查次数，取值范围 1-99，默认1次

**规则3：启用控制**
- 每个组织的巡查要求可独立启用/禁用
- 禁用时，该组织对该类型工程不产生巡查频次要求
- 启用时，系统按配置计算逾期状态并提醒

**规则4：生效规则**
- 配置保存后立即生效
- 对存量工程：按新配置重新计算逾期状态
- 对新增工程：按当前生效配置执行

**规则5：默认值策略**

| 风险等级 | 工程类型 | 街道 | 社区 | 物业 |
|---------|---------|------|------|------|
| 高风险 | 小型工程 | 每10天1次 | 每14天1次 | 每2天1次 |
| 高风险 | 住宅装修 | 每10天1次 | 每14天1次 | 每2天1次 |
| 高风险 | 零星作业 | 每7天1次 | 每10天1次 | 每1天1次 |
| 非高风险 | 小型工程 | 每21天1次 | 每14天1次 | 每3天1次 |
| 非高风险 | 住宅装修 | 每30天1次 | 每21天1次 | 每7天1次 |
| 非高风险 | 零星作业 | 每14天1次 | 每10天1次 | 每2天1次 |

**规则6：逾期计算**
- 系统记录每个工程的上次巡查时间
- 若当前时间 - 上次巡查时间 > 配置周期天数，则标记为"逾期"
- 逾期工程在巡查首页显示"待巡查"提醒，并标红提示

**规则7：历史版本**
- 每次保存生成一个配置版本
- 保留最近10次配置修改记录
- 支持查看历史版本，不支持回滚（避免数据混乱）

## 功能点描述

### 功能点1：巡查频次配置页面

**入口路径**
- 后台管理 → 运营管理 → 巡查频次配置

**页面名称**
- 巡查频次配置

**页面功能**
1. **风险等级切换**
   - 提供两个选项：高风险、非高风险
   - 单选模式，切换后下方配置区刷新为对应风险等级的配置
   - 默认选中"非高风险"

2. **工程类型切换**
   - 提供三个选项：小型工程、住宅装修、零星作业
   - 使用Tab卡片形式展示
   - 切换后显示该工程类型在各组织下的巡查频次配置

3. **组织巡查配置卡片**
   - 每个组织（街道、社区、物业）一个配置卡片
   - 卡片包含：
     - 组织图标和名称
     - 启用开关：控制该组织的巡查要求是否生效
     - 周期天数输入框：数字输入，范围1-365，单位"天"
     - 巡查次数输入框：数字输入，范围1-99，默认1次
   - 禁用时，输入框置灰不可编辑

4. **操作按钮**
   - 重置默认：点击后弹出确认框，确认后将当前风险等级×工程类型的配置恢复为系统默认值
   - 保存配置：保存当前所有配置，保存后立即生效

**业务规则**
- 输入校验：天数和次数必须为整数，超出范围提示错误
- 保存校验：至少启用一个组织的巡查要求，否则提示"至少需要启用一个巡查组织"
- 并发控制：多人同时编辑时，后保存者覆盖前者，并提示"配置已被更新"

## 验收标准

### 功能验收

1. **配置维度完整性**
   - [ ] 支持配置2种风险等级（高风险/非高风险）
   - [ ] 支持配置3种工程类型（小型工程/住宅装修/零星作业）
   - [ ] 支持配置3种巡查组织（街道/社区/物业）
   - [ ] 18种组合配置互不影响

2. **频次设置功能**
   - [ ] 可设置周期天数（1-365天）
   - [ ] 可设置巡查次数（1-99次）
   - [ ] 可独立启用/禁用各组织的巡查要求
   - [ ] 禁用时输入框置灰不可编辑

3. **默认配置加载**
   - [ ] 首次进入页面显示默认配置
   - [ ] 默认配置符合业务规则（见上文默认值策略表）
   - [ ] 重置默认功能可将当前配置恢复为默认值

4. **保存与生效**
   - [ ] 保存成功后显示成功提示
   - [ ] 配置保存后立即生效
   - [ ] 系统重新计算所有工程的逾期状态

5. **输入校验**
   - [ ] 天数和次数必须为整数
   - [ ] 超出范围时提示错误并不允许保存
   - [ ] 至少启用一个组织才能保存

### 兼容性验收

6. **与现有功能兼容**
   - [ ] 巡查首页的"待巡查"提醒基于新配置计算
   - [ ] 工程详情的巡查统计展示基于新配置判断逾期
   - [ ] 不影响已完成的巡查记录

7. **数据一致性**
   - [ ] 配置变更后，逾期工程列表正确更新
   - [ ] 配置变更后，正常工程不会误标为逾期

### 性能验收

8. **配置保存性能**
   - [ ] 配置保存响应时间 < 1秒
   - [ ] 配置生效（重新计算逾期）< 5秒

### 安全验收

9. **权限控制**
   - [ ] 仅系统管理员可访问配置页面
   - [ ] 操作需记录日志

   - 预警触发逻辑需基于新配置
