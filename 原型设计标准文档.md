# 原型设计标准文档

> 小型工程安全纳管系统 UI 设计规范

---

## 1. 设计原则

- **iOS原生风格**：毛玻璃、大圆角、线性图标
- **清晰层级**：字体大小、字重、色彩区分信息层级
- **呼吸感留白**：20px页面边距，16px卡片间距
- **精致细节**：12px卡片圆角，10px按钮圆角，**头像圆形**
- **线性图标**：使用SVG线性图标（stroke-width: 1.5-2px），不使用emoji

**应用场景**：备案申请（小程序）、巡查检查（APP）、管理后台（Web）

#### 状态栏（Status Bar）
- **高度**：34px

- **文字颜色**：白色
- **字体**：12px，字重800

**状态图标样式**：
- 信号图标：4条竖线，高度递增（8px、11px、14px、17px），圆角
- WiFi图标：三层弧线 + 中心圆点
- 电池图标：圆角矩形边框 + 内部填充

**元素规范**：

  - 位置：左侧
  - 尺寸：34px × 34px

## 6. 原型输出规范

### 6.1 标注规范（强制）

**标注样式**：
- 背景：`#FFFBE6`（浅黄色）
- 边框：1px dashed `#FFC53D`
- 圆角：4px
- 标题：📌 需求说明，14px 加粗
- 正文：13px 常规

**标注内容性质**：**需求逻辑规则说明**，不是交互说明

**标注内容应包含**：
| 类型 | 说明 | 示例 |
|------|------|------|
| **数据规则** | 计算口径、数据来源、更新频率 | 巡查覆盖率 = 已巡查工程数 / 总工程数 |
| **业务规则** | 状态流转、权限判断、异常处理 | 隐患整改期限：一般3天，较大7天 |
| **字段规则** | 字段定义、校验逻辑、枚举值 | 风险等级：高风险/一般风险/低风险 |
| **需求说明** | 功能点描述、逻辑说明 | 本周指自然周（周一至周日） |

**标注内容不应包含**：
- ❌ 不写交互操作（点击哪里、怎么跳转）
- ❌ 不写页面流程（进入哪个页面）
- ❌ 不写显而易见的操作说明

**标注内容示例**：
```markdown
📌 巡查覆盖率计算规则
- 分子：本周有巡查记录的在建工程数
- 分母：当前在建工程总数（状态=正常施工|现场整改|停工整改）
- 计算公式：分子 / 分母 × 100%
- 数据来源：巡查记录表、工程状态表
- 更新频率：每日凌晨 2:00 自动更新
- 口径说明：暂按"有巡查即覆盖"，后续按频次要求

📌 隐患等级判定规则
- 较大隐患：需停工整改的隐患（如高处坠落风险）
- 一般隐患：现场整改或不限时整改的隐患
- 判定依据：标准化隐患清单 V2.0

📌 整改状态流转
- 待整改 → 整改中（提交整改记录）
- 整改中 → 已整改（审核通过）
- 整改中 → 待整改（审核不通过，退回）
```

**大屏/管理后台标注位置（强制抽屉式）**：
- 使用抽屉式标注，默认收起
- 固定在页面右侧，垂直居中
- 点击"标注"按钮展开/收起

### 6.2 HTML原型规范

**高度自适应**（强制）：
- `.phone-frame` 使用 `height: auto`，禁止固定高度
- `.phone-screen` 使用 `height: auto`
- 禁止 `<div style="height: 100px;">` 占位元素

**定位约束**（强制）：
- 底部按钮使用相对定位或 flex 布局
- 禁止使用 `position: fixed`

**弹窗默认隐藏**（强制）：
- 默认 `display: none`
- 通过 JS 控制显示/隐藏

**可编辑原则**：
- 文案使用 `contenteditable="true"`
- hover 显示绿色边框，focus 显示绿色描边
