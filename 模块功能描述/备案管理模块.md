## 模块概述

### 业务场景

青浦区小散工程安全监管平台的备案管理模块，旨在规范和简化小型工程、住宅装修、零星作业等小散工程的备案申报流程。通过线上化、智能化的方式，实现从申报、审核到回执生成的全流程数字化管理，提升监管效率，保障施工安全。

**核心价值**：
- **施工方/业主**：简化备案流程，线上提交材料，实时查看审核进度，快速获取备案回执
- **街镇监管部门**：提高审核效率，AI辅助审核减少人工工作量，统一管理辖区内所有备案项目
- **安全监管**：建立完整的工程档案，记录施工人员资质，为后续巡查和监管提供数据支撑

**适用范围**：
- 小型工程，其类型包括：
  （一）工程投资额在 100 万元（不含 100 万元）以下的新建、改建、扩建房屋建筑（及其附属设施）工程；
（二）工程投资额在 100 万元以下（不含 100 万元）的交通（及其附属设施）工程；
（三）工程投资额在100万元（不含100万元）以下的新（改）建供水、排水项目，以及不纳入区水务工程质量安全监督管理的河道、泵（闸）、农污、厂站等设施维修类水务工程；
（四）工程投资额在 100 万元（不含 100 万元）以下的园林绿化工程；
（五）工程投资额在 100 万元（不含 100 万元）以下的既有建筑修缮工程；
（六）工程投资额在 100 万元（不含 100 万元）以下并且建筑面积在 300 平方米（不含 300 平方米）以下的公共建筑、商铺、办公楼、厂房等各类非住宅既有建筑装饰装修工程（含原装饰装修内容拆除），对不涉及用途转换的商务楼宇简易装修工程,其建筑面积可放宽至 500 平方米以下；
（七）工程投资额在 100 万元以下的既有建筑加装电梯工程；
（八）已批准的建设工程基地范围内的临时设施施工及已取得《临时建设工程规划许可证》的临时建设工程；
（九）依法获得批准建设的农民自建或翻建低层住宅；
（十）农业基础性、生产性以及附属性设施等建设工程；
（十一）农村地区基础设施、公共服务场所、绿化美化美丽庭院（小三园）、标识标牌等综合性建设工程；
- 住宅室内装饰装修工程
- 零星施工作业，其类型包括：
（一）空调、太阳能、雨棚、防盗网、管道、工业环保设备设施等非主体工程配套设备设施的安装、维护、拆除作业；
（二）建筑外墙清洗、修补、屋面检修等各类建筑外墙零星高处作业；
（三）户外小型广告及门楣招牌的安装、维护、拆除作业；
（四）新能源汽车、电动自行车充电设施的安装、维护、拆除作业；
（五）临时搭建舞台、看台、展台、布景等搭设、拆除作业；
（六）锅炉、压力容器（含气瓶）、压力管道、电梯、起重机械、等特种设备的安装、改造、修理；
（七）住宅区、公共场所绿化枝叶修剪；
（八）道路绿化枝叶修剪
（九）供水、排水、通信、电力、燃气等管线设施维护、维修工程
（十）其他由存在坍塌、物体打击、高处坠落、消防安全、机械伤害等事故风险的零星作业活动

### 受众角色

#### 1. 施工方/业主（施工端）
**角色定义**：需要进行工程备案的施工单位或业主

**使用场景**：
- 发起新的备案申报
- 填写工程基础信息、施工信息、安全信息
- 上传相关证明材料和施工方案
- 管理施工人员信息
- 查看审核进度和结果
- 下载备案回执

**典型用户**：
- 小型施工企业负责人
- 住宅装修业主或装修公司
- 零星作业承包方

**核心诉求**：
- 操作简单，快速完成备案
- 材料要求清晰，避免反复修改
- 审核进度透明，及时获取反馈
- 快速获取备案回执，开始施工

#### 2. 街镇审核人员（监管端）
**角色定义**：负责审核备案申请的街镇监管部门工作人员

**使用场景**：
- 查看待审核的备案列表
- 审核备案材料的完整性和合规性
- 对不符合要求的字段进行驳回并说明原因
- 审核通过后生成备案回执
- 查看辖区内所有备案项目的状态

**典型用户**：
- 街镇建设管理部门工作人员
- 安全监管人员

**核心诉求**：
- 快速筛选待审核项目
- 清晰查看所有备案材料
- 便捷标注问题字段
- 高效完成审核工作

#### 3. AI审核系统（系统角色）
**角色定义**：自动审核备案材料的智能系统

**功能职责**：
- 对提交的备案信息进行多维度一致性校验
- 检查不同资料之间的信息一致性（如身份证信息与填写信息的匹配）
- 校验材料的完整性和逻辑合理性
- 检查字段之间的关联关系是否符合业务规则
- 初步筛选明显不符合要求的申请
- 减轻人工审核压力，提高审核效率

**审核范围**：
- 证件信息与填写信息的一致性（姓名、身份证号、手机号等）
- 上传文件与对应字段信息的��配性
- 必填字段的完整性
- 数值范围的合理性（工程总额、建筑面积等）
- 业务逻辑的合理性（如施工时间、作业类型与施工人员资质的匹配等）

**说明**：OCR识别功能在文件上传时自动触发，不属于AI审核环节。AI审核是在用户提交备案申请后，对所有已填写和上传的资料进行综合性的一致性检查。

#### 4. 系统管理员（后台管理）
**角色定义**：负责系统配置和维护的技术人员

**使用场景**：
- 配置审核流程（是否需要人工审核）
- 维护基础数据（街镇、社区、物业、施工单位等）
- 配置工程分类和规模限制
- 管理用户权限
- 监控系统运行状态

**核心诉求**：
- 灵活配置业务规则
- 便捷维护基础数据
- 及时发现和处理系统问题

### 业务价值

**对施工方/业主**：
- 减少线下跑腿次数，全程线上办理
- 缩短备案周期，从数天缩短至数小时
- 材料要求清晰，减少反复修改
- 实时查看进度，心中有数

**对监管部门**：
- AI辅助审核，减少60%以上的人工审核工作量
- 统一管理辖区内所有备案项目
- 建立完整的工程档案，便于后续监管
- 数据化管理，支持统计分析和决策

**对安全监管**：
- 施工人员资质可追溯
- 施工方案留档备查
- 风险作业类型明确标识
- 为日常巡查提供数据支撑

## 通用规则说明

### 1. 文件上传规则
- **支持格式**：图片主流格式（JPG、PNG）及PDF格式
- **文件大小限制**：待确认
- **多文件上传**：部分字段支持上传多个文件
- **OCR识别**：营业执照、身份证等证件类文件支持OCR自动识别，识别失败时允许手动输入

### 2. 字段校验规则
- **手机号**：必须符合手机号格式（11位数字）
- **身份证号**：必须符合身份证号格式（18位）
- **数字输入**：必须为数字，且>0
- **必填项**：不能为空
- **文件上传**：必须上传至少1个文件
- **日期选择**：结束日期必须大于开始日期

### 3. 页面交互规则
- **页面头部标准结构**:返回按钮、页面标题、操作按钮(更多/刷新/关闭)
- **选择器交互**:支持搜索、列表展示和确认操作
- **脱敏显示**:手机号(137****355)、身份证号(610************013)

### 4. 审核流程规则
- **审核配置**：后台可配置是否启用AI审核；不启用AI审核时，提交后直接进入[人工审核(街道级审核)](#人工审核街道级审核)
- **AI审核配置**：启用AI审核后，可进一步配置AI审核通过后是否需要人工审核
- **审核页面标准结构**：页面标题、顶部操作、流程进度卡片、Tab栏（基础信息、工程信息、安全信息、匹配保单）、内容展示、底部操作按钮
- **审核详情页状态**：待审核、审核通过、审核不通过、AI审核中（此状态用于审核详情页面的流程进度显示）
- **列表页状态筛选**：待填报、待审核、已驳回、已通过（此状态用于"我的项目"列表页和"项目中心"列表页的状态筛选标签）
- **驳回处理**：显示驳回字段、原填写信息、驳回原因；用户可选择"提交人工审核"或"修改驳回事项"

### 5. 动态表单规则
- **单选联动**：根据单选项显示不同字段组（如建设单位身份）
- **开关联动**：开启开关后显示详细信息区域（如施工作业风险类别）
- **卡片列表**：支持添加多个相同结构的卡片（如工人工种、施工人员）

### 6. 特种作业证要求
- **必须上传特种作业证的工种**：电工、电焊工、高空作业工
- **无需上传特种作业证的工种**：木工、架子工、钢筋工、抹灰工、油漆工、混凝土工、防水工、起重机驾驶员、挖掘机驾驶员、中小型建筑机器操作工、其他
- **说明**：在新增施工工人时，如果选择的工种属于"必须上传"类别，则特种作业证字段为必填；如果属于"无需上传"类别，则特种作业证字段为选填

## 完整业务流程

### 流程概述

备案管理模块涉及两个主要角色:用户(施工方/业主)和审核人员(街镇人员)。

#### 一、用户备案申报流程

用户从发起申报到最终拿到回执的完整流程,包括:
- **申报环节**:选择工程类型、填写备案信息(基础/工程/安全信息)、匹配保单
- **审核环节**:AI审核、人工审核、查看回执

**关键分支**:
- 提交备案 → 后台配置判断 → 启用AI审核/不启用AI审核
- AI审核通过 → 后台配置判断 → 需要人工审核/无需人工审核
- AI审核不通过 → 提交人工审核/修改驳回事项
- 人工审核通过 → 备案完成,进入日常巡查模块
- 人工审核不通过 → 施工方修改后重新提交

#### 二、审核人员审核管理流程

审核人员(街镇人员)的审核管理流程:
- 查看备案列表(筛选待审核/已审核等状态)
- 人工审核(查看备案信息、审核通过/驳回)

#### 进入备案系统并选择工程类型

**入口路径**：
- 随申办App/小程序 → 青浦区旗舰店 → 常用服务 → "小型工程备案"图标

**页面名称**："我的项目"列表页

**页面功能**：

1. **工程类型分类（Tab切换）**：
   - 小型工程（显示该类型的项目数量，如：42）
   - 住宅装修（显示该类型的项目数量，如：12）
   - 零星作业（显示该类型的项目数量，如：52）

2. **状态筛选（横向标签）**：
   - 待填报（数量）- 草稿状态，已保存但未提交
   - 待审核（数量）- 已提交，等待街镇审核
   - 已驳回（数量）- 审核不通过，需修改后重新提交
   - 已通过 - 审核通过，已拿到回执

3. **搜索功能**：
   - 支持按"工程名称或地址"搜索历史项目

4. **项目列表展示**：
   - 显示用户历史提交的所有备案记录
   - 列表项信息：工程名称、工程地址（根据截图显示的内容）
   - 点击行为：
     - 待填报状态：点击进入编辑页面，继续填写
     - 待审核/已驳回/已通过状态：点击查看详情页面

5. **新增入口**：
   - 右下角"+"浮动按钮
   - 点击后进入选择工程类型页面

**业务规则**：
- 用户只能查看自己提交的项目，无法查看其他人的项目
- 不同工程类型的项目分别统计和展示
- 待填报状态的项目可以继续编辑
- 已驳回的项目可以修改后重新提交

   - 零星作业备案（配有施工图标）

   - 点击任一工程类型卡片，进入对应的申报表单页面
   - 每个卡片右侧有箭头图标，提示可点击

**业务规则**：
- 三种工程类型对应不同的申报表单和材料要求
- 用户必须先选择工程类型，才能进入申报流程

### 一、所有备案类型共有的字段

**1. 项目信息（可编辑）**：

  - **工程规模选择页面**：

- **所属物业**（选填，点击后弹出物业选择器）

## 特殊场景

### 三类工程的差异

青浦区小散工程安全监管平台支持三类工程的备案申报:

1. **小型工程备案**(限额以下建设工程)
2. **住宅装修备案**(住宅室内装修)
3. **零星作业备案**(零星施工作业)

**差异说明**:

三类工程在备案流程中的差异主要体现在**第二步:填写基础信息**的字段上,其他流程步骤(工程信息、安全信息、匹配保单、审核流程等)完全一致。

**具体差异**:

| 字段 | 小型工程备案 | 住宅装修备案 | 零星作业备案 |
|------|-------------|-------------|-------------|
| **项目名称字段** | 工程项目（必填） | 工程项目（必填） | 作业项目（必填） |
| **规模/类别字段** | 工程规模（必填，11类选项） | 工程规模（必填，1类选项） | 作业类别（必填，10类选项） |
| **地址字段** | 工程地址（必填，地图选点） | 工程地址门牌号（室）（必填，文本输入） | 作业地址（必填，地图选点） |
| **街道社区** | 必填，手动选择 | 必填，手动选择 | 必填，手动选择 |
| **工程总额** | 必填，数字输入 | 选填，数字输入 | 选填，数字输入 |
| **建筑面积** | 必填，数字输入 | 选填，数字输入 | 选填，数字输入 |
| **建设单位区域标题** | 建设单位信息 | 装修人信息（业主或经业主授权的使用人） | 作业单位信息 |
| **身份字段标题** | 建设单位身份 | 装修人身份 | 作业单位身份 |
| **身份类型选项** | 单位 / 个人 | 单位 / 个人 | 单位 / 个人 |
| **承租人附加材料** | 所有权证明+租赁合同+授权委托书 | 所有权证明+租赁合同+授权委托书 | 所有权证明+租赁合同+授权委托书 |
| **施工图纸及技术资料** | 必填 | 选填 | 必填 |
| **相关项目批复/认定手续** | 必填 | 选填 | 必填 |
| **装修装饰管理协议** | 不显示 | 选填 | 不显示 |

**业务规则**:
- 小型工程和零星作业:通过地图选点确定工程地址,手动选择街道社区(必填)
- 住宅装修:工程地址门牌号（室）直接文本输入,街道社区手动选择
- 零星作业中"工程项目"改为"作业项目"、"工程规模"改为"作业类别"、"工程地址"改为"作业地址"、"建设单位身份"改为"作业单位身份"
- 住宅装修中"建设单位信息"改为"装修人信息","建设单位身份"改为"装修人身份"
- 工程规模/作业类别选项根据备案类型动态加载,详见产品上下文中的"工程分类枚举"

### 高风险工程 vs 普通工程

> **⚠️ 重要说明**：当前高风险工程定义为**内部口径**，基于现有备案字段自动判定。如后续区级正式标准下发，需同步调整规则与校验逻辑。

**高风险工程定义标准**：

系统根据备案信息自动判定，命中以下任一规则即判定为高风险工程：

| 工程类型 | 判定规则 | 命中原因 |
|---------|---------|---------|
| **小型工程** | 工程类型 = ⑦既有建筑加装电梯工程 | 既有建筑加装电梯 |
| | 作业场所 = 重点场所（商业综合体、学校、医院等） | 重点场所施工 |
| | 涉及消防设施变动 = 是 | 涉及消防设施变动 |
| | 涉及变动建筑主体结构和承重结构 = 是 | 涉及主体结构或承重结构 |
| | 涉及房屋外立面改动 = 是 | 涉及房屋外立面改动 |
| | 动火作业 = 开 | 涉及动火作业 |
| | 动土作业 = 开 | 涉及动土作业 |
| | 拆改作业 = 开 | 涉及拆改作业 |
| | 其他特种作业勾选任一选项 | 涉及特种作业 |
| **住宅装修** | S2 高空作业 = 开 | 涉及高空作业 |
| | S3 动火作业 = 开 | 涉及动火作业 |
| | S5 拆改作业 = 开 | 涉及拆改作业 |
| | S6 有限空间作业 = 开 | 涉及有限空间作业 |
| | S10 涉及落地式钢管脚手架 = 开 | 涉及脚手架作业 |
| | S11 涉及起重吊装 = 开 | 涉及起重吊装 |
| | S12 涉及其他高风险工程 = 开 | 涉及其他高风险工程 |
| | S13 涉及装修后使用功能与原产权证不符 = 开 | 涉及使用功能改变 |
| | S14 涉及消防设施变动 = 开 | 涉及消防设施变动 |
| | S15 涉及房屋外立面改动 = 开 | 涉及房屋外立面改动 |
| **零星作业** | 作业类型属于高处作业/户外广告牌/临时搭设/特种设备/公共管线/其他高风险 | 作业类型本身高风险 |
| | S1-S7 任一风险类别开关 = 开 | 涉及对应风险作业 |

**系统判定逻辑**：
- 判定时机：提交备案、重新提交、编辑备案信息时自动重算
- 判定结果：输出"是否高风险工程"（是/否）、命中规则编号、命中原因
- 多规则命中：允许同时命中多条规则，全部记录
- 人工干预：**不支持**人工上调或下调风险等级

**高风险工程标识展示**：
- 备案列表、工程列表、巡查列表需展示"高风险"标签
- 支持按"是否高风险"筛选

**当前边界**：
- 本版本仅按现有字段自动判定，未覆盖场景系统无法识别
- 判定结果不等同于政策文件中的高风险定义
- 待区级正式标准下发后，优先调整字段与规则，再同步更新本定义
