# 禅道导入工具使用说明（分发版）

这份目录适合同组同事直接使用，原理还是一样：

- 维护 `工作量.xlsx`
- 维护 `人员映射.xlsx`
- 双击 `导入禅道.bat`

和内部版的区别是：

- 包里不再放任何个人禅道账号密码
- 同事第一次运行时，会提示输入自己的禅道账号和密码
- 如果输入 `y`，工具会把个人登录信息保存到当前目录的 `config.local.json`
- 后续再次运行，就会直接使用自己的账号

## 先说最重要的 3 件事

1. 谁登录，禅道里的创建人就是谁
2. 这份分发版只适合同样环境的同事使用
3. 不要把 `config.local.json` 再分享给别人

## 适用范围

当前分发版按下面环境整理：

- Windows
- 禅道开源版 `21.3`
- 禅道地址：`http://120.46.219.85:8004/zentao`
- 电脑已安装 Node.js `18+`

如果不是这个环境，可能无法直接使用。

## 目录里的主要文件

- `导入禅道.bat`
  - 双击入口
- `config.json`
  - 公共配置，不含个人账号密码
- `config.local.json`
  - 首次运行后可自动生成，保存当前使用人的账号密码
- `工作量.xlsx`
  - 需求工时表
- `人员映射.xlsx`
  - 人员、角色、禅道显示名映射
- `templates\禅道导入模板.xlsx`
  - 备用模板
- `output\preview.txt`
  - 本次预览结果
- `output\import-result.txt`
  - 本次导入摘要
- `output\import-result.json`
  - 本次导入详细结果

## 第一次使用

1. 打开 `人员映射.xlsx`
2. 确认 `人员姓名`、`角色`、`禅道显示名`、`是否启用`
3. 打开 `工作量.xlsx`
4. 填好需求和人员工时
5. 双击 `导入禅道.bat`
6. 首次运行时输入自己的：
   - 禅道账号
   - 禅道密码
7. 如果提示是否保存到 `config.local.json`
   - 输入 `y` 表示保存，后面不用再输

## Excel 规则

### 工作量表

- 第一列必须是 `需求名称`
- 后面的每一列必须是人员姓名
- 单元格里填工时
- 空白表示这个人不参与该需求
- 工具会自动计算：需求工时 = 该行所有人员工时之和
- 工具会自动生成任务名：`【角色】需求名称`

### 人员映射表

- `人员姓名`
  - 必须和 `工作量.xlsx` 的列头一致
- `角色`
  - 例如：后端1、前端2、测试
- `禅道显示名`
  - 用来匹配禅道负责人
- `是否启用`
  - 填 `是` 的人才会被导入

## 导入模式

启动后会看到两个选项：

- `1`
  - 适合禅道里还没建需求的情况
  - 结果：先建需求，再建任务
- `2`
  - 适合禅道里需求已经建好，只差任务的情况
  - 结果：不建需求，只补任务

## 创建人是谁

工具不会伪造创建人。

- 如果同事用自己的账号登录
  - 创建人就是他自己
- 如果同事用了别人的账号
  - 创建人就会记到那个账号名下

所以每个人都应该使用自己的禅道账号。

## 分发建议

发给同事时，建议只发这份“分发版”目录，不要发内部版。

尤其注意：

- `config.json` 可以发
- `config.local.json` 不要发
- `output` 里的历史结果建议清空后再发

## 常见问题

### 1. 提示未检测到 Node.js

说明电脑没安装 Node.js，先安装再运行。

### 2. 提示未找到项目

检查输入的项目名称是否和禅道里完全一致。

### 3. 提示未找到负责人账号

检查 `人员映射.xlsx` 里的 `禅道显示名` 是否和禅道里真实显示名一致。

### 4. 只导任务时提示未找到已存在的需求

说明 `工作量.xlsx` 里的 `需求名称` 和禅道里需求标题不完全一致。

### 5. 想换成自己的账号

有两种方式：

- 删除当前目录的 `config.local.json`，下次运行重新输入
- 直接手工修改 `config.local.json`
