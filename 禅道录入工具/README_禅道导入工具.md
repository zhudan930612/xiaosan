# 禅道导入工具使用说明

这个工具的目标是：你只维护两份 Excel，然后双击 `导入禅道.bat`，就能先看到预览，再决定是否正式导入到禅道。

## 你会用到的文件

- `导入禅道.bat`
  - 双击运行入口
- `config.json`
  - 禅道地址、账号、密码配置
- `工作量.xlsx`
  - 你平时填写的需求工时表
- `人员映射.xlsx`
  - 人员姓名、角色、禅道显示名映射表
- `templates\禅道导入模板.xlsx`
  - 备用模板
- `output\preview.txt`
  - 每次预览结果
- `output\import-result.txt`
  - 正式导入后的结果摘要
- `output\import-result.json`
  - 正式导入后的详细结果

## 第一次使用前要做什么

1. 打开 `人员映射.xlsx`
2. 检查每个人的 `人员姓名`、`角色`、`禅道显示名`、`是否启用`
3. 打开 `工作量.xlsx`
4. 从第二列开始，列头必须和 `人员映射.xlsx` 里的 `人员姓名` 一致
5. 第一列必须是 `需求名称`

## 当前禅道版本说明

- 这套工具已经按你的环境适配到 `禅道开源版 21.3`
- `config.json` 里的禅道地址建议写完整入口：
  - `http://120.46.219.85:8004/zentao`
- 如果你只写到站点根地址，例如 `http://120.46.219.85:8004`
  - 工具也会自动补成 `/zentao`
- Scrum 项目在页面上看是“项目”，但工具内部会自动找到它对应的执行
  - 例如：`风险科技-Sprint178 -> execution 1286`

## 工作量表填写规则

- 一行代表一条需求
- 第一列 `需求名称` 填需求标题
- 其他列按人填写工时
- 空白表示这个人不参与该需求
- 工具会自动计算：需求工时 = 该行所有人员工时之和
- 工具会自动生成任务标题：`【角色】需求名称`

## 人员映射表规则

- `人员姓名`
  - 必须和 `工作量.xlsx` 的列头一致
- `角色`
  - 例如：后端1、前端2、测试
- `禅道显示名`
  - 用来在禅道里匹配负责人
  - 如果和人员姓名一致，就直接填同样的名字
- `是否启用`
  - 填 `是` 才会参与导入匹配

## 最简单的使用方式

1. 填好 `工作量.xlsx`
2. 确认 `人员映射.xlsx` 没问题
3. 双击 `导入禅道.bat`
4. 按提示输入项目名称
   - 例如：`风险科技-Sprint178`
5. 工具会先生成预览
6. 如果确认无误，输入 `y` 开始正式导入

## 只看预览，不真正导入

```powershell
node src/cli.js --project "风险科技-Sprint178" --dry-run
```

这会只生成：

- `output\preview.txt`

不会真正创建需求和任务。

## 正式导入：需求和任务一起导

```powershell
node src/cli.js --project "风险科技-Sprint178"
```

如果你已经确认过，不想再输入 `y`，可以用：

```powershell
node src/cli.js --project "风险科技-Sprint178" --yes
```

## 正式导入：只导任务

如果你的需求已经提前在禅道里建好了，只剩任务没补，可以用：

```powershell
node src/cli.js --project "风险科技-Sprint178" --tasks-only
```

如果你不想再手工确认，可以用：

```powershell
node src/cli.js --project "风险科技-Sprint178" --tasks-only --yes
```

这个模式下工具会：

- 不新建需求
- 按 `工作量.xlsx` 里的 `需求名称` 去当前 Scrum 对应执行里查找同名需求
- 找到后只创建任务
- 找不到的需求会记录到失败结果里，不会乱建新需求

所以这个模式最适合：

- 需求已经建好了
- 只剩任务没补
- 需要避免重复创建需求

## 常见问题

### 1. 提示“人员列 xxx 未在人员映射表中启用”

说明 `工作量.xlsx` 里的列头，和 `人员映射.xlsx` 里的 `人员姓名` 对不上，或者该成员 `是否启用` 不是 `是`。

### 2. 提示“发现重复的需求名称”

说明工作量表里有两行需求名称相同，工具会阻止导入，避免重复创建。

### 3. 提示“未找到负责人账号”

说明 `人员映射.xlsx` 里的 `禅道显示名` 和禅道里的人员显示名不一致，需要改成禅道里真实显示的名字。

### 4. 提示“无法自动确定需求所属产品”

说明工具没法自动判断这个执行对应哪个产品，需要在 `config.json` 里填写 `productId`。

### 5. 只导任务时提示“未找到已存在的需求，无法只导任务”

说明 `工作量.xlsx` 里的 `需求名称`，和禅道里已经创建的需求标题不完全一致。这个模式按需求标题精确匹配，所以要确保名字一模一样。
